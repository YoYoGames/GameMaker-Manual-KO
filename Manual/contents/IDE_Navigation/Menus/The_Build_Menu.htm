빌드 메뉴

빌드 메뉴에서는 테스트, 디버깅 또는 최종 실행 파일로 프로젝트를 빌드하도록 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 실행 - 테스트를 위해 선택한 대상 플랫폼에서 현재 프로젝트를 실행합니다. 컴파일할 때 사용할 수 있는 다양한 컴파일 옵션은 컴파일 섹션을 참조하세요. 이 명령은 실행 버튼을 누르는 것과 동일합니다. IDE에서.
  • 디버그 - 자세한 성능 정보를 확인하고 발생할 수 있는 오류 또는 문제를 디버그할 수 있도록 디버거로 테스트하기 위해 현재 프로젝트를 실행합니다. 디버깅에 대한 자세한 내용은 디버깅 섹션을 참조하세요. 이 명령은 디버그 버튼을 누르는 것과 동일합니다. IDE에서.
  • 정리 - GameMaker 는 테스트 시 컴파일 시간을 단축하기 위해 파일 캐시 를 유지합니다. 그러나 때때로 "부실" 캐시는 게임에서 이상하고 예기치 않은 오류를 유발할 수 있습니다. 이 경우 가장 먼저 해야 할 일은 여기에서 캐시를 정리하고 다시 테스트하는 것입니다. 대상 플랫폼에 대한 최종 실행 파일을 만들기 전에 캐시를 정리하는 것도 좋은 생각입니다. 이 명령은 캐시 정리 버튼을 누르는 것과 같습니다. IDE에서.
  • 실행 파일 만들기 - 현재 프로젝트를 컴파일하고 대상 플랫폼에 대한 실행 가능한 패키지 를 만듭니다. 먼저 파일 탐색기를 열어 프로젝트를 저장할 위치를 선택한 다음 게임을 컴파일할 수 있습니다. 컴파일 대상 설정에 대한 자세한 내용은 컴파일 섹션을 참조하십시오. 이 명령은 Create Executable 버튼을 누르는 것과 같습니다. IDE에서.
