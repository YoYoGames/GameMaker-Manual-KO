마켓플레이스 튜토리얼 만들기

튜토리얼을 만드는 것은 일반 Marketplace 자산 패키지를 만드는 것과 비슷합니다. 이제 특정 포함 파일을 추가하고 초기 튜토리얼 페이지(및 선택적 비디오 페이지)를 포함하면 됩니다. 자습서 작업 공간은 다음과 같습니다. 일반 Marketplace 자산에 대해 위에서 설명한 것처럼 게시자를 선택한 다음 자습서에 이름과 고유 ID(역방향 URL 형식)를 제공한 다음 "유형" 드롭다운에서 자습서를 선택해야 합니다. 그러면 튜토리얼을 시작할 기본 HTML 파일과 (선택 사항) 비디오 파일 및 URL을 추가할 수 있는 추가 옵션이 제공됩니다. 비디오 파일은 여기 에서 얻을 수 있는 나머지 튜토리얼과 마찬가지로 Marketplace에서 제공한 템플릿을 사용해야 하며, 사용자가 튜토리얼을 로드할 때 비디오는 IDE 내의 별도 프레임에 로드됩니다. 시작 페이지 및 비디오 페이지에 대해 제공하는 페이지는 포함된 파일 내의 상대 경로이며 GameMaker 가 필드를 채우는 기본 항목 이외의 것을 선택하기 위해 탐색기를 열 수 있습니다.

실제로 튜토리얼을 생성하려면 튜토리얼 프로젝트에 포함된 파일 을 추가해야 하며, IDE가 튜토리얼에 사용할 파일을 제대로 감지하려면 이러한 파일을 매우 구체적인 방식으로 추가해야 합니다. 기본 폴더 구조는 아래 이미지와 같습니다.

따라서 두 개의 기본 폴더가 있습니다.

  • TutorialContent: 모든 튜토리얼 HTML 파일, CSS 스타일 지정을 위해 필요한 이미지(리소스에 필요한 이미지가 아니라 튜토리얼에 표시되는 이미지)와 비디오 템플릿(필요한 경우)은 여기로 이동하십시오. 예를 들어 튜토리얼을 챕터로 분할하려는 경우 하위 폴더도 가질 수 있지만 최소한 하나의 기본이 있어야 합니다. HTML 튜토리얼의 시작 페이지로 사용할 파일입니다.
  • TutorialResources: 이 폴더에는 이미지 및 사운드 자산에 대한 두 개의 하위 폴더가 더 있습니다. 예를 들어 튜토리얼에서 사용자가 스프라이트를 추가해야 하고 튜토리얼에 포함하려면 이미지 리소스를 이미지 폴더에 배치합니다. 이 폴더는 튜토리얼 프로젝트로 가져오고 사용자가 스프라이트 편집기를 열고 이미지를 추가하려고 하면 탐색기 창의 기본 경로가 이 위치가 되어 사용자가 필요한 자산을 더 쉽게 찾을 수 있습니다.

튜토리얼이 IDE 내에서 작동 하려면 이 폴더 구조와 기본 폴더가 있어야 합니다. HTML 시작 페이지 CSS 스타일링을 위한 파일입니다. 비디오의 일부로 건너뛰거나 특정 섹션 또는 기능에 대한 매뉴얼을 여는 튜토리얼 프로젝트 내에서 사용할 수 있는 많은 특수 링크가 있으며, 이들은 모두 Marketplace의 공식 튜토리얼 템플릿에 제공됩니다.

중요한! 자체 자습서를 만들기 전에 Marketplace에서 자습서 템플릿을 다운로드하는 것이 좋습니다. 여기에는 HTML의 모든 중요한 부분에 대한 예제와 함께 튜토리얼 내에서 버튼과 메뉴를 재생성하기 위한 CSS 및 이미지가 포함되어 있으므로 표시 방법보다는 내용에 대해 자유롭게 걱정할 수 있습니다.

포함된 파일을 설정하고 필요한 정보를 제공한 후 확인 을 클릭하면 튜토리얼이 마켓플레이스에 업로드되고 완료되면 자산 목록에서 작업 공간을 열어 편집하고 아이콘, 텍스트, 스크린샷 등을 추가할 수 있습니다.... 튜토리얼 자산이 게시되면 사용자는 다운로드 및 설치가 가능한 시작 페이지 의 튜토리얼 섹션에서 찾을 수 있습니다.
