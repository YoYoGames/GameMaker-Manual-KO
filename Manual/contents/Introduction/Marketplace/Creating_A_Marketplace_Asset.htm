마켓플레이스 자산 생성

IDE Marketplace 메뉴 에서 패키지 만들기 옵션을 선택하면 다음 작업 공간으로 이동합니다. 여기 에서 먼저 자산 패키지를 게시할 게시자 ID( 게시자 대시보드 에서 설정해야 함)를 선택한 다음 표시 이름 과 버전 번호 를 제공해야 합니다. 그런 다음 사용된 표시 이름과 게시자 계정을 기반으로 패키지 ID가 생성되지만 역 URL 형식을 유지하는 한 원하는 경우 이를 다른 것으로 편집할 수 있습니다. 그런 다음 만들 자산 패키지 유형을 선택해야 합니다.

  • Asset: 마켓플레이스에서 판매하고자 하는 자산(스프라이트, 스크립트, 개체 등)의 모음입니다.
  • 데모: 이것은 몇 가지 개념이나 게임 아이디어를 시연하기 위한 데모입니다.
  • 튜토리얼: 업로드되는 자산은 GameMaker IDE 내에서 실행되도록 설계된 튜토리얼입니다(자세한 내용은 이 페이지 참조).

완료하면 오른쪽 섹션을 사용하여 자산 패키지에 추가할 프로젝트의 자산을 선택할 수 있습니다. 모든 자산을 추가하거나 일부만 추가할 수 있을 뿐만 아니라 실수로 추가한 자산을 제거할 수도 있습니다. 이전에 GameMaker 에서 업로드할 자산을 생성했으며 현재 프로젝트에 업로드에 필요한 모든 것이 포함되어 있다고 가정합니다.

기억하다! 자산 패키지를 생성하거나 사용하려면 이전에 사용자 프로필 인증서를 생성해야 합니다. 여기 에 설명된 Marketplace 기본 설정에서 이 작업을 수행할 수 있습니다.

자산 선택에 만족하고 패키지를 마켓플레이스에 업로드하려면 하단의 확인 버튼을 클릭하기만 하면 자산 패키지가 생성되어 마켓플레이스에 업로드됩니다. 업로드가 완료되면 편집할 자산 페이지의 브라우저 보기와 함께 새 작업 공간이 열립니다. 설정 방법에 대한 자세한 내용은 다음 페이지를 참조하십시오.

  • 자산 나열

Marketplace 자산을 업데이트해야 하는 경우 원본 패키지를 만들 때와 거의 동일하게 수행합니다. 이제 사용 가능한 패키지의 드롭다운 목록에서 업데이트할 패키지를 지정하기만 하면 됩니다.

업데이트할 때 패키지 ID를 변경할 수 없으며 표시된 버전 번호 가 현재 버전 번호임을 알고 있어야 하며 적절하게 이를 증가시켜야 합니다. 그렇지 않으면 업데이트가 실패합니다(두 개의 자산을 가질 수 없기 때문에 동일한 버전 번호의 패키지). 또한 자산에 대한 변경 로그에 대한 텍스트를 추가할 수 있는 필드가 있으며 이는 자산과 함께 업로드되고 Marketplace 페이지에 표시됩니다.

중요한! GameMaker 버전 2.3 이상으로 만든 자산은 버전 2.3 이상을 사용하는 사용자만 사용할 수 있습니다. 이전 버전에서 자산을 업데이트하는 경우 해당 자산은 업데이트되지 않은 사용자가 계속 사용할 수 있으며 업데이트된 2.3+ 버전과 공존합니다.

웹 페이지를 통한 업로드

IDE가 아닌 실제 Marketplace 웹 페이지를 통해 자산을 업로드할 수 있습니다. 이는 IDE를 통한 업로드가 어떤 이유로든 실패한 경우 필요할 수 있습니다. 먼저 패키지를 생성하고 위에 표시된 대로 IDE를 통해 업로드를 시도합니다. 실패하면 출력 창 으로 이동하여 생성 중인 패키지가 저장된 위치를 보여주는 행에 대한 로그를 확인합니다. 업로드를 시도하기 전에. 예를 들어:

Windows:

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMaker\GM_USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

macOS:

Created package file:
/Users/MACUSERNAME/.config/GameMaker/GM_USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

해당 파일 위치로 이동하여 복사해야 합니다. *.YYMP 마켓플레이스에 업로드할 파일이므로 안전한 위치에 파일을 저장합니다.

이 시점에서 웹 브라우저에서 마켓플레이스를 열고 자산에 대한 자산 목록으로 이동한 다음 버전 섹션으로 이동한 다음 아래 이미지와 같이 버전 업로드 버튼을 클릭해야 합니다.

그러면 필요한 경우 업로드되는 버전에 대한 정보를 추가할 수 있는 새 버전의 업로드 페이지로 이동하고 파일 선택 버튼을 클릭하고 이전에 생성된 파일을 선택하여 파일을 제공할 수도 있습니다. *.YYMP 파일:

파일을 추가하면 버전이 업로드되고 "진행률" 표시줄이 표시되며, 업로드가 완료되고 세부 정보에 만족하면 저장 버튼을 클릭하여 자산을 목록에 저장할 수 있습니다. 자산은 게시되기 전에 몇 분 동안 처리 단계에 있습니다.
