작업 공간

로그인하고 새 프로젝트를 시작한 후 GameMaker 는 IDE 에 도킹된 몇 가지 기본 창이 있는 초기 작업 공간 으로 이동합니다. 일반적으로 작업 공간은 작업하는 동안 게임에 대한 다양한 자산을 구성할 수 있는 단순히 영역입니다.

보시다시피 초기 작업 공간은 화면 상단의 탭에 있지만(탭을 두 번 클릭하여 이름을 바꿀 수 있음), 단일 프로젝트에 대해 여러 가능한 작업 공간을 제공합니다. 예를 들어 플레이어와 여러 적 개체 간의 상호 작용을 작업 중일 수 있으므로 플레이어는 자체 작업 공간에, 적 개체는 다른 작업 영역에 두거나 다른 작업 영역에 둘 다 사용하는 스크립트만 표시할 수도 있습니다.

작업 공간의 또 다른 중요한 기능은 탭에서 - 여전히 마우스 버튼을 누르고 있습니다. 아래로 - 예를 들어 여러 디스플레이를 사용하는 경우 항목을 매우 쉽게 구성할 수 있도록 기본 IDE 창에서 자체 개별 창으로 드래그합니다. 또한 탭을 첫 번째 창으로 다시 끌어 이러한 보조 작업 영역 창을 기본 창으로 다시 병합합니다. 이 작업을 수행할 때 실행 중인 IDE 인스턴스가 두 개 있는 것처럼 보이지만 둘 다 동일한 프로젝트 에 대한 것이며 GameMaker 의 두 인스턴스를 특별히 열지 않는 한 한 프로젝트에 다른 프로젝트를 가질 수 없습니다.

GameMaker 를 처음 시작할 때 작업 공간은 이미 IDE에 " 도킹 "되는 몇 개의 창으로 채워져 있습니다. 아래에 간략하게 설명되어 있습니다.

  • 출력 - 출력 창 위의 이미지에서도 볼 수 있습니다. 이 창에는 Source Control, Searching 및 Debugging 과 관련된 여러 하위 탭이 있으며 초기 탭은 콘솔/컴파일러 출력용이며 테스트를 위해 게임을 컴파일하거나 생성할 때 어떤 일이 일어나는지 보여줍니다. 배포를 위한 최종 패키지. 이것은 또한 런타임에 프로젝트에서 보내기로 선택한 디버그 메시지를 표시하며 일반 기본 설정 에서 다양한 정보를 표시하도록 구성할 수 있습니다. 출력 창을 닫고 복구하려면 Windows 메뉴 를 사용할 수 있습니다.
  • 자산 - 화면 오른쪽에서 자산 브라우저 를 찾을 수 있습니다. 여기에서 게임에서 사용하는 다양한 자산을 만들고 편집할 수 있을 뿐만 아니라 구성, 게임 옵션, 방 순서 및 기타 항목을 생성 및 변경할 수 있습니다. 자산은 마우스 오른쪽 버튼을 클릭하여 생성됩니다. 자산 폴더(또는 기본 자산 브라우저 보기의 아무 곳에서나)에서 생성 을 선택하거나 상단에 있는 더하기 아이콘을 클릭하고 자산 생성 창에서 자산을 선택하여(이 방법을 사용하면 동일한 자산을 여러 개 생성할 수 있습니다. 하단에 값을 설정하여 한 번에 입력):

    이러한 옵션 중 하나를 선택하면 지정된 유형의 비어 있는 새 리소스가 생성되고 이름 필드가 강조 표시되어 즉시 이름을 지정할 수 있습니다. 자산 브라우저는 기본적으로 IDE에 도킹되어 있지만 상단의 "자산 브라우저" 텍스트에서 드래그합니다. IDE의 측면이나 하단으로 드래그하여 언제든지 다시 도킹할 수 있습니다. 자산 브라우저 창을 닫고 복구하려는 경우 Windows 메뉴 를 사용할 수 있습니다. 사용 가능한 자산에 대한 자세한 내용은 편집기 섹션을 참조하십시오.

또한 특정 항목이 작업 공간에 표시되는 방식과 관련된 기본 설정 집합이 있으므로 일반 워크플로의 특정 측면뿐만 아니라 모양을 사용자 지정할 수 있습니다. 다음 페이지에서 이러한 기본 설정을 찾을 수 있습니다.

작업 공간 RMB 메뉴

작업 공간 내에서 마우스 오른쪽 버튼을 클릭하여 언제든지 오른쪽 버튼 메뉴를 불러올 수 있습니다. 다음 메뉴가 열립니다.

자산 을 클릭하면 생성할 수 있는 자산 브라우저 의 모든 자산 목록이 열리고 그 중 하나를 선택하면 해당 자산 리소스가 생성되고 작업 공간이 해당 리소스의 편집기 창에 집중됩니다. Windows 옵션을 클릭하면 작업 공간 내에서 현재 열려 있는 모든 창의 목록이 표시되며 그 중 하나를 선택하여 작업 공간이 해당 창에 초점을 맞추도록 할 수 있습니다. 마지막으로 다음 창을 여는 이동 옵션이 있습니다.

이 창에서는 입력한 검색어에 대해 프로젝트의 모든 자산, 기능, 게임 옵션 및 기본 설정 을 검색할 수 있습니다. 예를 들어, 검색어로 "icon"을 입력하면 이름 어딘가에 "icon"이 있는 프로젝트의 모든 항목이 표시됩니다. 예: 스프라이트가 다음과 같은 경우 spr_Icon_Play, 그러면 여기에 표시되며 개체도 " obj_Iconoclastde"> ", 그리고 " Icon for hdpi "와 같은 다양한 아이콘에 대한 게임 옵션도 볼 수 있습니다. 필터링된 목록에 표시된 항목을 클릭하면 현재 작업 공간에서 열리거나 현재 작업 공간에 초점이 맞춰져 있는 경우 해당 작업 공간에 초점을 맞춥니다. 이미 열려 있습니다.

키보드 단축키를 사용할 수도 있습니다. / + T 를 눌러 이동 창을 열고 나열된 항목 수가 (기본적으로) 최대 16개로 설정되어 있는지 확인합니다. 일반 기본 설정 의 이동 섹션에서 다른 검색 옵션과 함께 이 최대 제한을 변경할 수 있습니다.

작업 공간 탐색에 대한 자세한 내용은 IDE 입력 및 탐색 섹션을 참조하십시오.

도킹

위에서 현재 작업 공간 창에 많은 창을 도킹할 수 있고 이미 도킹된 창을 도킹 해제할 수 있다고 위에서 간략하게 언급했습니다. 클릭하면 예를 들어 상단의 자산 브라우저 탭에서 현재 작업 공간의 왼쪽으로 드래그하면 자유 부동 창이 되는 것을 볼 수 있습니다.

이 반대는 사실이며 특정 창을 작업 공간의 측면(또는 하단)으로 드래그하면 도킹됩니다. 즉, 본질적으로 IDE 창 오버레이의 일부가 되고 더 이상 독립적이지 않습니다. 이 작업을 수행할 때 창을 도킹하려는 영역으로 드래그하면 해당 영역에 창을 도킹할 수 있음을 보여주기 위해 강조 표시됩니다.

이러한 방식으로 작업 흐름과 작업 방식에 맞는 개인화된 IDE 환경을 만들 수 있습니다. 도킹된 항목 옆에 있는 버튼을 클릭하여 도킹된 모든 창을 개별적으로 숨기거나 숨김 해제할 수 있습니다(이 페이지 상단의 이미지에서 강조 표시됨). 단추 IDE 상단에 있고 IDE는 세션 사이에 도킹된 창을 기억합니다.

참고: 룸 편집기에서와 같이 편집기를 변경할 때 일부 창은 독에서 사라집니다. 그러나 문제의 편집기로 돌아가면 독에서 다시 열립니다.
