자산 브라우저

게임의 핵심은 기본적으로 IDE 의 오른쪽에 있는 자산 브라우저 에 추가된 자산에서 생성됩니다. 여기에서 게임룸, 스프라이트, 개체, 경로 및 기타 여러 항목을 포함하여 게임을 실행하는 데 필요한 모든 것을 추가할 수 있습니다. GameMaker 의 기본 게임은 실행할 방이 필요하며(새 프로젝트는 항상 이미 생성된 방 자산으로 생성됨) 일반적으로 적어도 하나의 개체와 스프라이트가 필요하지만 더 많이 사용할 것입니다!

자산 브라우저는 게임 자산을 만들고 구성할 뿐만 아니라 플랫폼별 설정 및 구성과 같은 특정 측면을 수정하는 곳입니다. 아래에서 각 섹션에 대한 설명을 찾을 수 있습니다.

검색 창

검색 표시줄에서 텍스트를 입력하고 이를 사용하여 아래의 기본 자산 목록에 표시되는 내용을 필터링할 수 있습니다. 예를 들어 "room"을 입력하면 이름에 "room"이 포함된 모든 자산이 표시됩니다. 자산 목록을 다시 기본 보기로 재설정하려면 검색 창에서 텍스트를 삭제하기만 하면 됩니다.

자산 추가

자산 추가 버튼을 사용하여 자산 목록에 새 자산을 추가하도록 선택할 수 있습니다. 클릭하면 다음 창이 열립니다.

위의 두 가지 보기 중 하나를 클릭하여 사용할 보기를 선택할 수 있습니다. 목록 아이콘 또는 그리드 아이콘을 클릭할 수도 있습니다. 다른 프로젝트에서 이미 존재하는 자산을 가져오려면 가져오기 아이콘을 클릭합니다. 아무 것도 가져오지 않으려면 생성할 자산을 선택하고 하단의 생성 버튼을 클릭하면 됩니다. 창 하단에서 생성할 수를 설정한 다음 생성을 클릭하면 선택한 자산의 해당 번호가 자산 브라우저에 자동으로 추가되므로 여기에 단일 자산만 추가하는 것으로 제한되지 않습니다. 이 방법으로 생성된 모든 자산은 자산 목록의 맨 아래 폴더 외부에 추가되며 편집할 준비가 된 현재 작업 공간에서도 열립니다.

여기에 추가할 수 있는 자산은 매뉴얼의 이 섹션 아래에 자세히 나열되어 있습니다.

필터

자산 필터 버튼을 클릭하면 다음 창이 열립니다.

여기에서 자산 브라우저에 나열된 자산을 필터링하고 표시하는 방법을 선택할 수 있습니다. A - Z 또는 Z - A에서 주문할 수 있을 뿐만 아니라 그룹을 먼저 주문한 다음 그룹 내 자산을 주문하거나 모든 것을 주문하도록 선택할 수 있습니다. 사용자 지정 순서 필터를 선택하면 하나 이상의 자산을 선택하고 자산 브라우저 트리를 위나 아래로 끌어 필요에 따라 배치할 수 있습니다("그룹 우선" 옵션은 비활성화됨).

또한 하나 이상의 자산 유형을 선택하여 필터링할 수 있으며 해당 유형의 프로젝트에 있는 자산만 자산 목록에 표시됩니다. "모든 유형"을 선택하면 필터가 재설정됩니다. 할당한 태그를 사용하여 필터링할 수도 있습니다(태그는 매뉴얼의 이 섹션에서 자세히 설명함). "모든 유형" 이외의 필터를 선택한 경우 필터 버튼의 색상이 변경되어 필터링이 활성화되었음을 시각적으로 표시합니다.

엑스트라

추가 메뉴를 사용하면 다음 창을 열 수 있습니다(각 창은 링크된 페이지에서 별도로 설명됨).

  • 룸매니저
  • 구성 편집기
  • 게임 옵션
  • 포함된 파일

빠른 접근

자산 브라우저의 이 섹션은 중요하거나 자주 사용하는 자산에 빠르게 액세스할 수 있는 다양한 방법을 제공하는 데 사용되며 다음 섹션으로 나뉩니다.

  • 최근: 가장 최근에 사용한 자산 목록을 볼 수 있습니다. 기본적으로 최대 10개의 자산이 여기에 나열되지만 자산 브라우저 기본 설정 에서 이를 변경할 수 있습니다.
  • 즐겨찾기: 자산 목록에 일부 자산을 추가한 후에는 특정 시간에 자주 액세스하는 자산이 있을 수 있으므로 작업을 더 쉽게 하기 위해 이러한 자산을 "즐겨찾기"로 플래그를 지정할 수 있습니다. 그러면 해당 자산이 이 목록에 추가되고 당신은 빨리 그들을 찾을 수 있습니다. 자산을 즐겨찾기 로 플래그 지정하려면 마우스 오른쪽 버튼을 사용하십시오. 를 선택하고 팝업 메뉴에서 즐겨찾기를 선택합니다. 마우스 오른쪽 버튼을 사용할 수 있습니다. 자산을 즐겨찾기 해제 하려면 다시 한 번 누르고 기본 즐겨찾기 폴더의 버튼을 사용하면 모두 지우기를 선택하여 모든 항목을 제거할 수 있습니다.
  • 방 순서: 여기에서 방이 실행되는 순서를 변경할 수 있습니다. 모든 프로젝트에는 실행하려면 적어도 하나의 방이 필요하며 목록 맨 위에 있는 방은 "집" 방이며 게임이 처음 시작됩니다("홈" 아이콘으로 표시됨 ). 객실 순서를 빠르게 변경하려면 방을 목록 위나 아래로 끈 다음 마우스 버튼을 놓아 새 위치에 배치합니다. "홈" 아이콘을 클릭하면 룸매니저 가 열리며 마우스 오른쪽 버튼을 사용하면 방에 어린이 방을 만들 수 있습니다. 객실 주문 및 어린이 객실에 대한 자세한 내용은 객실 관리자 페이지를 참조하십시오.
  • 저장된 필터: 필터링 옵션을 사용할 때 나중에 참조할 수 있도록 선택한 항목을 저장할 수 있습니다. 그렇게 하려면 이 페이지의 추가 설명에 따라 필터를 설정한 다음 마우스 오른쪽 버튼을 클릭합니다. 이 옵션에서 필터 저장 을 선택합니다. 필터가 이 목록에 저장되며 필요한 경우 이름을 바꿀 수 있습니다. 필터를 사용하려면 두 번 클릭하기만 하면 됩니다. 그것에 또는 마우스 오른쪽 버튼을 사용 메뉴 옵션 필터 적용. RMB에서 저장된 필터를 삭제 하고 이름 을 변경할 수도 있습니다. 메뉴.
  • 태그: 자산 브라우저의 모든 자산에는 사용자 정의 태그 를 할당할 수 있습니다. 그런 다음 여기에 나열되며 이를 사용하여 자산 목록에 표시된 자산을 필터링할 수 있으며 두 번 클릭할 수 있습니다. 모든 태그(또는 마우스 오른쪽 버튼 사용 태그별 필터 ) 옵션을 사용하여 해당 태그를 자산 목록 필터에 적용합니다. 적용된 모든 태그는 정보 섹션의 자산 브라우저 하단에 표시됩니다. 태그에 대한 자세한 내용은 이 페이지의 아래쪽을 참조하십시오.
  • 게임 옵션: 여기에서 프로젝트에 대한 다양한 게임 옵션 에 빠르게 액세스할 수 있습니다. 이는 다양한 대상 플랫폼에 대한 설정 및 컴파일과 관련된 프로젝트의 다양한 측면을 제어합니다(사용 가능한 게임 옵션은 보유한 라이선스 유형에 따라 다름). 게임 옵션 섹션에서 나열된 각 옵션에 대한 자세한 정보를 찾을 수 있습니다.

자산 목록

자산 브라우저의 이 섹션에는 게임에서 사용 중인 다양한 자산이 모두 나열됩니다. 기본적으로 새 프로젝트를 만들 때 추가할 수 있는 다른 종류의 자산으로 레이블이 지정된 수많은 폴더 그룹으로 채워지지만 이름을 바꾸거나 제거할 수도 있으며 동일한 자산을 저장할 필요가 없습니다. 각 그룹에 친절합니다. 예를 들어, 각 자산 유형보다 게임의 각 레벨에 대한 자산 그룹을 생성하는 것이 더 유용할 수 있으므로 "레벨 1", "레벨 2" 등의 폴더 그룹을 생성할 수 있습니다. 동일한 폴더에 각 레벨에 대한 모든 다른 자산을 저장합니다. 당신이 물건을 정리하는 방법은 당신에게 달려 있습니다!

목록에 새 자산을 추가하려면 자산 브라우저 상단의 자산 추가 버튼을 사용하거나 마우스 오른쪽 버튼을 열 수 있습니다. 자산 목록의 아무 곳에서나 메뉴를 선택한 다음 생성할 리소스를 선택합니다.

자산이 생성되면 작업 공간에서 열리고 게임에 적합한 이름으로 이름을 바꿀 수 있지만 자산 이름은 영숫자만 가능하고 숫자로 시작할 수 없으며 추가 문자로 "_"에 밑줄을 긋습니다. 또한 오른쪽 버튼 메뉴를 사용하여 그룹 만들기 옵션을 사용하여 자산을 포함할 새 폴더 그룹을 만들 수 있으며 원하는 이름을 지정할 수 있습니다. 자산 또는 폴더 그룹의 이름을 바꾸려면 느린 두 번 클릭을 수행할 수 있습니다. 그들에 또는 RMB를 사용 메뉴 옵션 이름 바꾸기.

자산 브라우저의 중요한 기능은 자산에 색상을 지정하여 목록 내에서 자산을 즉시 인식할 수 있다는 것입니다. 마우스 오른쪽 버튼을 클릭하면이 작업을 수행 할 수 있습니다 왼쪽의 자산 목록 노드에서 를 클릭한 다음 팝업되는 색상 선택기 팔레트에서 색상을 선택합니다. 폴더에서 이 작업을 수행하면 폴더에 포함된 모든 자산의 색상도 지정됩니다.

이와 같은 노드에서 색상 선택기 팔레트를 열 때 사용할 사용자 정의 색상을 생성하려면 색상 견본 중 하나에서

개별 자산은 RMB를 사용하여 그룹 색상과 독립적으로 색상을 지정할 수도 있습니다. 자산 메모 또는 RMB 하단의 팔레트를 사용하여 메뉴( 자산 브라우저 기본 설정 에서 비활성화할 수 있음)를 사용하고 다음을 사용할 수 있습니다. 또는 / 마우스 왼쪽 버튼과 함께 여러 자산을 선택하고 모든 자산에 한 번에 색상을 지정하려면:

색상을 지정하는 방법은 귀하에게 달려 있습니다. 자산 유형, 용도 또는 기타 기준에 따라 달라질 수 있습니다!

자산 목록에서 마지막으로 주의해야 할 사항은 폴더와 자산을 선택한 다음 드래그하여 자산 목록의 다른 폴더 그룹 안팎으로 이동합니다. 필요하지 않은 경우 폴더 그룹의 일부 또는 전체를 삭제할 수도 있으므로 자산 목록을 깔끔하게 유지하고 필요한 요소만 표시할 수 있습니다. 프로젝트를 위해.

정보

자산 브라우저 하단의 정보 표시줄은 현재 선택된 자산의 수뿐만 아니라 자산 목록에 있는 총 자산 수를 보여줍니다. 이 섹션에는 자산 목록을 필터링하는 데 사용된 모든 태그도 표시되며(태그에 대한 자세한 내용은 아래 참조) 필요한 경우 해당 태그를 클릭하여 태그 필터를 제거할 수 있습니다.

줌 컨트롤

확대/축소 컨트롤을 사용하여 GameMaker IDE의 현재 DPI의 최소 50%에서 최대 250%로 자산 브라우저의 콘텐츠를 더 크거나 작게 만들 수 있습니다. 기본값인 100%로 재설정하려면 간단히 두 번 클릭하십시오. 슬라이더 왼쪽의 백분율 값에서 Asset Browser Preferences 에서 작동 방식을 처리하는 데 사용할 수 있는 몇 가지 기본 설정이 있습니다.

태그

위의 다양한 섹션에서 언급했듯이 자산 브라우저는 개별 자산과 그룹 폴더에 태그를 추가할 수 있는 기능을 제공합니다. 태그는 자산 브라우저에 나열된 자산을 구성하고 필터링할 뿐만 아니라 코드에서 할당된 태그를 사용할 수 있으므로 일반적으로 게임을 프로그래밍할 때도 매우 강력한 도구입니다.

태그를 만들려면 RMB를 사용해야 합니다. 자산에서 메뉴를 클릭한 다음 태그 편집 옵션을 선택합니다. 사용하려는 태그를 입력할 수 있는 태그 편집기 가 표시되고 다음 중 하나를 누릅니다. 또는 쉼표를 입력하여 적용하십시오.

단일 항목에 여러 태그를 할당할 수 있으며 다음을 사용할 수도 있습니다. 또는 / 마우스 왼쪽 버튼과 함께 여러 자산을 선택하고 한 번에 모든 자산에 태그를 할당합니다. 추가되면 모든 태그가 빠른 액세스 메뉴와 필터 옵션(위 섹션에서 설명한 대로)에서 볼 수 있으므로 자산 목록을 빠르게 필터링하고 선택한 태그가 있는 항목만 표시하는 데 사용할 수 있습니다..

폴더 그룹에 태그를 할당하면 그 안에 있는 자산에 자동으로 할당 되지 않습니다.

태그가 활성 필터의 일부인 동안 생성된 자산에는 해당 태그가 자동으로 부여됩니다. 즉, 예를 들어 "레벨 1" 태그 아래에 있는 모든 자산을 보고 있는 경우 이 필터를 활성화하여 생성된 모든 새 자산에는 생성 시 "레벨 1" 태그도 추가됩니다.

마지막으로, 프로젝트 코드에서 태그를 사용하여 자산을 식별한 다음 해당 작업을 수행할 수 있으므로 태그 또는 태그 그룹을 기반으로 동작을 코딩할 수 있으므로 매우 강력한 도구가 됩니다. 예를 들어 "Die On Touch " 태그. 프로젝트를 프로그래밍할 때 이 태그와 적절한 코드 기능을 확인할 수 있으며, 나중에 아티스트와 레벨 디자이너는 모든 유형의 새 자산을 만들고 "Die On Touch " 태그는 레벨을 디자인할 때 자산을 사용합니다. 이 태그를 사용하도록 프로그래밍된 모든 코드에는 이러한 새 자산이 자동으로 포함됩니다.

프로젝트 코드에서 태그를 사용하는 방법에 대한 자세한 내용은 자산 및 태그 섹션을 참조하십시오.

위안화 메뉴

자산 브라우저로 작업할 때 마우스 오른쪽 버튼을 사용할 수 있습니다. 리소스 또는 폴더에서 옵션 메뉴를 열려면 다음을 수행하십시오.

사용할 수 있는 정확한 옵션은 클릭한 컨텍스트에 따라 다르며 자산 목록의 자산 또는 폴더 그룹에서 메뉴를 열면 아래에 내용이 표시됩니다(다른 컨텍스트에는 전체가 아닌 일부만 표시됨).

  • 만들기 - 새 자산을 만듭니다. 이것을 선택하면 하나를 선택하기 위해 생성할 수 있는 사용 가능한 자산 유형 목록이 표시됩니다. 특정 자산에서 이 메뉴를 열면 목록에서 자산 유형을 찾을 필요가 없도록 바로 아래에 동일한 유형이 나열된 경우 리소스를 생성하는 추가 옵션이 있습니다.
  • 편집(모두) - 선택한 자산 또는 폴더 그룹에 대한 리소스 속성 편집기를 엽니다. 단일 자산에서도 편집을 위해 열 수 있음).
  • 태그 편집 - 선택한 자산 또는 폴더 그룹에 태그를 추가하거나 제거할 수 있는 태그 편집기가 열립니다.
  • 이름 바꾸기 - 자산 또는 폴더 그룹의 이름을 바꿉니다.
  • 복제 - 선택한 자산의 복제본을 생성합니다.
  • 즐겨찾기 - 자산 브라우저 상단의 빠른 액세스 섹션에 추가하여 자산을 즐겨찾기로 플래그 지정합니다.
  • 기존 자산 추가 - 다른 GameMaker 프로젝트 폴더에서 기존 자산을 추가합니다. 2.3.0 이전 버전을 사용하여 생성된 자산은 2.3.0 업데이트로 프로젝트 파일 형식이 변경되었으므로 변환이 필요합니다. 즉, 자산을 다른 프로젝트로 자주 가져오는 경우 전체 프로젝트를 가져와 모든 것을 변환한 다음 새 위치에 저장하는 것이 바람직할 수 있습니다.
  • 내 라이브러리에서 기존 항목 추가 - Marketplace 라이브러리의 항목에서 자산 목록에 자산을 추가합니다. 2.3.0 이전 버전을 사용하여 만든 Marketplace 자산은 2.3.0 업데이트로 프로젝트 파일 형식이 변경되었으므로 변환이 필요합니다.
  • 삭제 - 프로젝트에서 선택한 자산을 삭제합니다. 이 취소 할 수 없습니다!
  • 그룹 생성 - 자산 목록에 새 폴더 그룹을 추가합니다.
  • 하위 확장 - 현재 선택한 폴더 그룹을 포함할 수 있는 모든 하위 폴더와 함께 확장합니다.
  • 모두 확장 - 자산 목록의 모든 폴더 그룹을 확장합니다.
  • 모두 계약 - 자산 목록의 모든 폴더 그룹을 계약합니다.
  • 새 자산 브라우저에서 열기 - 이 옵션은 폴더 그룹에만 사용할 수 있으며 옵션이 선택된 상위 그룹의 내용만 포함하는 새 자산 브라우저 창에서 그룹을 엽니다.
  • 탐색기에서 열기 - 프로젝트 폴더의 리소스에 대해 저장된 파일 위치에서 파일 탐색기가 열립니다.
  • 색상 견본 - 여기에서 사용할 자산의 새 색상을 선택할 수 있습니다. 폴더 그룹에서 사용하면 폴더 내의 모든 자산이 선택한 색상을 갖게 되며, 하나 이상의 자산에서 사용되는 경우 폴더 그룹에 대해 설정된 모든 색상을 재정의합니다( 자산 브라우저 기본 설정 에서 이 옵션을 재정의하지 않는 한 ).

자산 유형

이제 자산 브라우저의 작동 방식을 알았으므로 추가할 수 있는 다양한 유형의 자산을 알아야 합니다. 아래에서는 프로젝트에서 사용할 수 있는 다양한 자산 유형에 대한 개요를 제공합니다.

애니메이션 곡선

애니메이션 곡선은 시간이 지남에 따라 값이 어떻게 변하는지를 나타내는 하나 이상의 곡선을 포함하는 리소스이며 선형 보간 또는 catmull-rom 보간 을 사용하여 곡선의 값 사이를 이동할 수 있습니다. 설정한 값은 세로 축에서 -1과 1 사이일 수 있고 가로 축을 따라 지속 시간은 항상 0에서 1 형식으로 정규화되므로 게임 코드에서 승수를 사용하여 다른 시간 범위를 쉽게 대상으로 지정할 수 있습니다. 애니메이션 곡선은 "채널"로 구성되며 각 채널은 고유한 곡선 설정을 가질 수 있습니다. 이를 통해 예를 들어 x/y 위치에 대해 두 개의 채널이 있는 공간 위치를 설명하거나 색상 형식. 애니메이션 곡선 자산은 시퀀스 작업 시 사용할 수 있으며 코드를 사용하여 액세스할 수도 있으므로 게임을 만들 때 강력한 도구가 됩니다.

애니메이션 곡선 섹션에서 애니메이션 곡선 추가에 대해 자세히 알아볼 수 있습니다.

확장

확장은 GameMaker 기능을 확장하고 마켓플레이스 에서 게임에 자산을 추가하기 위해 추가할 수 있는 추가 파일 및 코드입니다. 확장의 형식은 최종 대상 플랫폼에 따라 다르지만 일반적으로 C++, C# 또는 JavaScript로 작성됩니다.

확장 편집기 섹션에서 확장 추가에 대해 자세히 알아볼 수 있습니다.

글꼴

게임에서 텍스트를 그리려는 경우 이 텍스트는 기본적으로 표준 Arial 12포인트 글꼴로 그려지지만 더 흥미롭거나 독특한 모양의 텍스트를 만들려면 다른 글꼴을 사용하고 싶을 것입니다. 글꼴 편집기 가 사용되는 곳입니다. 여기에서 컴퓨터에 있는 글꼴을 선택하고 게임에서 사용하기 위해 GameMaker 로 가져와 크기, 무게 또는 스타일과 같은 다양한 속성을 설정할 수 있습니다. 리소스로 가져와 추가한 후에는 적절한 코드나 작업을 사용하여 그리기용으로 설정할 수 있습니다.

참고: 게임에 글꼴을 추가하는 경우 공개 도메인 또는 저작권이 없는 경우가 아니면 재배포에 필요한 라이선스가 있는지 확인하십시오.

위안화 글꼴 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(또한 이 페이지의 맨 위에 나열된 옵션도 포함됨).

  • 텍스처 그룹 할당 - 이 옵션을 선택하면 사용 가능한 텍스처 그룹 목록이 표시되며 필요한 경우 하나를 선택하여 글꼴을 추가하거나 새로 만들 수 있습니다. 자산 브라우저에서 여러 글꼴을 선택하거나 글꼴 그룹 폴더를 선택한 경우 모든 글꼴이 선택한 텍스처 그룹에 추가됩니다. 텍스처 그룹에 대한 자세한 내용은 여기 를 참조하십시오.
  • 글꼴 재생성 - 이 옵션을 사용하면 GameMaker 가 글꼴 캐시를 재생성하도록 할 수 있습니다. 이것은 일반적으로 글꼴이 손상되거나 시스템에서 변경된 경우에만 필요합니다.

글꼴 편집기 섹션에서 글꼴 추가에 대해 자세히 알아볼 수 있습니다.

메모

메모는 무엇이든 작성할 수 있는 단순히 코드 편집기 창입니다. 코드 조각, 게임 정보, 할 일 목록, 팀 커뮤니케이션 등을 보관하는 장소로 설계되었습니다.

메모 편집기 섹션에서 메모 추가에 대해 자세히 알아볼 수 있습니다.

사물

개체는 게임의 측면을 제어하고 특정 작업을 수행하는 데 사용하는 특별한 리소스입니다. 대부분의 경우 게임 룸에서 볼 수 있도록 연결된 스프라이트 가 있지만 때로는 사용자 또는 타이밍 등과 관련된 작업을 수행하기 위해 "뒤에서" 컨트롤러로 사용됩니다. 행동이 주어지고 특정 이벤트에 반응할 수도 있고 서로 반응할 수도 있습니다. 게임에서 보는 대부분의 사물은 개체와 개체의 상호 작용을 기반으로 합니다. 실제로 개체를 게임 룸에 직접 배치하는 것이 아니라 기본적으로 개체 리소스의 복사본(또는 원하는 경우 복제)인 이러한 개체의 인스턴스 를 배치하기 때문에 "기반"이라고 말합니다. 이것은 인스턴스와 객체가 같은 것이 아니며 각각에 영향을 줄 수 있는 고유한 기능 집합을 가지고 있기 때문에 기억해야 하는 매우 중요한 사항입니다.

모든 객체에는 물리 를 사용하는지 여부 또는 다른 객체의 "자식" 객체인지 여부와 같이 스프라이트와 같이 객체 편집기에서 지정할 수 있는 속성 집합이 있습니다. 또한 액션, 스크립트 및 코드에서 사용할 수 있는 일련의 내장 변수가 있습니다. 이러한 내장 변수는 게임 룸에 배치될 때 인스턴스의 위치, 애니메이션 속도, 이동 방향 및 기타 여러 가지를 정의하는 데 사용됩니다. 시간이 지남에 따라 개체가 동작하는 방식을 제어하기 위해 각 게임 프레임이 일련의 이벤트로 분할되므로 특정 이벤트에 코드 또는 작업을 추가할 수 있으며 이벤트가 트리거될 때만 실행되며 다음과 같은 이벤트에 의해 이벤트가 트리거될 수 있습니다. 마우스 버튼을 누르거나 다른 인스턴스와 충돌합니다.

개체는 매우 실제적인 의미에서 게임의 빌딩 블록이며, 방과 함께 당신이 만드는 모든 프로젝트의 핵심을 형성합니다.

위안화 개체 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(또한 이 페이지의 위쪽에 나열된 옵션도 포함됨).

  • 모든 이벤트 스크립트 열기 - 이 옵션을 선택하면 새 작업 공간의 개체에 대해 할당된 모든 이벤트가 스크립트 탭으로 열립니다.

개체 편집기 의 섹션에서 개체 추가에 대해 자세히 알아볼 수 있습니다.

경로

게임을 만드는 동안 어느 시점에서 레벨을 통과하는 경로를 따라가기 위해 개체 중 하나의 인스턴스가 필요할 수 있습니다. 이제 이것은 방 안에 위치 배열을 생성한 다음 인스턴스가 위치 사이를 이동하게 하여 수행할 수 있지만 설정하기가 번거롭고 다양한 항목에 대해 변경, 테스트 및 적응하기 어렵습니다. 경로 리소스를 만들고 싶을 때입니다.

경로 리소스의 기본 아이디어는 다소 간단합니다. 경로 편집기에서 경로를 그려 경로를 정의한 다음 개체의 이벤트에 작업(또는 코드)을 배치하여 개체가 특정 경로를 따르도록 할 수 있습니다. 게임방. 경로를 따라가는 속도와 공간 내 경로의 위치 및 방향과 관련된 여러 다른 작업도 설정할 수 있습니다.

경로 편집기 의 섹션에서 경로 추가에 대해 자세히 알아볼 수 있습니다.

객실

GameMaker 에서 만드는 모든 게임은 실행할 방이 하나 이상 필요하며(하지만 더 많이 가질 수 있음) 방은 기본적으로 게임을 구성하는 개체의 인스턴스를 배치하고 게임의 동작이 실행되는 공간입니다. 일어날 것이다. 그러나 Room Editor는 개체의 인스턴스를 추가할 수 있을 뿐만 아니라 게임 룸에 "보기"를 설정하고, 그려지는 배경과 타일을 추가할 수 있도록 하기 때문에 사용할 수 있는 가장 강력한 리소스 중 하나입니다. 게임 세계를 만들고, 개체/인스턴스 시스템과 독립적인 추가 특수 효과를 만들고, 개체와 독립적으로 코드를 실행합니다. 방은 또한 다른 방의 속성을 상속 하도록 설정할 수 있습니다. 즉, 예를 들어 타일이 많은 단일 방을 만든 다음 이러한 타일을 상속하는 다른 방을 만들 수 있으므로 다시 만들 필요가 없습니다. 그들을 다시.

위안화 객실 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(이 페이지 위쪽에 나열된 옵션 포함):

  • 자식 방 만들기 - 선택한 방의 자식으로 새 방을 만듭니다. 하위 방은 상위 방의 속성, 레이어 및 레이어 콘텐츠를 상속하는 새로운 방 자산이며 방 관리자 에서 상위 방에 링크되어 표시됩니다.

방 추가에 대한 자세한 내용은 방 편집기 섹션에서 확인할 수 있습니다.

스크립트

스크립트 리소스는 단순히 작성한 하나 이상의 기능을 위한 컨테이너입니다. 예를 들어, 모든 적 개체가 죽을 때 여러 가지 다른 인스턴스를 생성하도록 하고 싶다고 가정해 봅시다. 금, 폭발, 피와 같은 사운드를 재생하고 재생합니다. 이제 필요한 모든 코드를 필요한 각 개체에 추가할 수 있지만 일반적으로 스크립트 리소스에서 단일 함수를 만든 다음 호출하는 것이 좋습니다. 즉, 이 새로운 스크립트 함수를 호출하기 위해 개체에 한 줄의 코드만 추가하면 되며, 변경하려는 경우 스크립트에서 한 번만 변경하면 모든 개체가 자동으로 새 코드를 실행합니다. 모든 개체를 살펴보고 여러 위치에서 동일한 것을 변경해야 하는 것보다.

위안화 스크립트 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(또한 이 페이지의 위쪽에 나열된 옵션도 포함됨).

  • GML Visual / GML 로 변환 - 선택한 스크립트를 GML Visual 또는 GML로 변환합니다. 스크립트가 GML인 경우 코드 실행 작업 내에 배치되고 GML Visual은 GML 코드로 직접 변환됩니다( GML Visual을 코드로 변경 섹션 참조).

스크립트 편집기 섹션에서 스크립트 추가에 대해 자세히 알아볼 수 있습니다.

시퀀스

시퀀스 자산은 기본적으로 시간이 지남에 따라 동적 애니메이션을 수행하는 다른 리소스의 모음입니다. 여기에는 스프라이트, 인스턴스, 사운드 및 기타 항목이 포함될 수 있으며, 각각은 시퀀스 내의 특정 트랙 에 자산을 할당하여 시퀀스 자체 내에서 시간이 지남에 따라 이동하거나 색상을 변경하거나 시작/중지하도록 설정할 수 있습니다. 이러한 트랙은 정적이거나 시간이 지남에 따라 변경될 수 있는 다른 속성을 적용할 수 있습니다. 시퀀스의 모든 편집은 다음으로 구성된 시퀀스 편집기 에서 이루어집니다.

  • 캔버스 - 시퀀스를 구성하는 자산을 배치하는 위치
  • 트랙 패널 - 자산을 배치하여 자산 트랙을 생성할 뿐만 아니라 시간 경과에 따라 자산을 조작하기 위한 매개변수 트랙 생성
  • 도프 시트 - 각 트랙의 키프레임을 추가/제거/편집하고 시간이 지남에 따라 동작하는 방식을 제어하는 곳입니다.

시퀀스 내에 시퀀스를 사용하여 복잡한 애니메이션과 효과를 만들 수 있으며, 코드를 통해 시퀀스에 액세스하고 변경할 수도 있으므로 게임에서 시퀀스가 표시되고 사용되는 방식을 최대한 제어할 수 있습니다.

시퀀스 추가에 대한 자세한 내용은 다음 섹션에서 확인할 수 있습니다. 시퀀스 편집기.

셰이더

셰이더는 게임이 화면에 렌더링하는 그래픽을 조작하는 데 사용할 수 있는 매우 강력한 도구로, 예를 들어 스프라이트에 미묘한 색조를 추가하는 것부터 전체 화면 왜곡 효과에 이르기까지 다양한 범위의 엄청나게 빠른 효과를 허용합니다.. 기본적으로 그래픽 카드 자체에서 직접 실행되는 두 부분으로 구성된 프로그램으로, GPU가 모든 작업을 수행하고 게임 코드에 대한 CPU 주기를 확보하기 때문에 속도가 매우 빨라집니다. 전체 셰이더는 정점 셰이더 프로그램과 프래그먼트 셰이더 프로그램(픽셀 셰이더라고도 함)으로 구성됩니다. 이 두 개의 작은 프로그램은 그래픽 카드가 화면에 렌더링하는 것을 조작하기 위해 함께 작동합니다. 그러면 실제로 디스플레이 버퍼에 렌더링되는 위치, 색상 및 알파 값을 실시간으로 조작할 수 있습니다.

GameMaker 는 다음 셰이더 언어를 지원합니다.

셰이더 언어 대상 플랫폼
GLSL ES 모든 대상 플랫폼
GLSL 맥과 우분투(리눅스)
HLSL11 윈도우, UWP, 엑스박스원
PSSL 플레이 스테이션 4


위안화 셰이더 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(또한 이 페이지 위쪽에 나열된 옵션도 포함).

  • 셰이더 유형 - 여기에서 셰이더 리소스에 사용되는 셰이더 언어 유형을 설정할 수 있습니다.

소리

사운드는 게임 플레이에 깊이를 더하고 음향 효과의 형태로 피드백을 제공하고 음악의 형태로 분위기를 추가하기 위해 모든 게임에서 중요한 부분입니다. GameMaker 수락 WAV, MP3 그리고 OGG 형식 파일.

일반적으로, WAV 파일은 일반적으로 더 큰 파일이지만 재생을 위해 어떤 유형의 디코딩도 필요하지 않기 때문에 즉시 재생되므로 짧은 음향 효과에 사용됩니다. MP3 그리고 OGG 파일은 배경 음악이나 더 오랜 시간 동안 재생되거나 파일 크기가 다소 큰 효과에 사용해야 합니다. 이러한 파일은 WAV 파일보다 훨씬 작지만 재생하기 전에 디코딩해야 하므로 관련 CPU 오버헤드 가 있습니다.

위안화 사운드 메뉴에는 다음과 같은 옵션이 포함되어 있습니다(또한 이 페이지의 위쪽에 나열된 옵션도 포함).

  • 사운드 믹서에서 열기 - 사운드 믹서 에서 선택한 모든 사운드를 엽니다.
  • 오디오 그룹 할당 - 이 옵션을 선택하면 사용 가능한 오디오 그룹 목록이 표시되며 필요한 경우 하나를 선택하여 사운드를 추가하거나 새 그룹을 만들 수 있습니다. 자산 브라우저에서 여러 사운드를 선택하거나 사운드 그룹 폴더를 선택한 경우 모든 사운드가 선택한 오디오 그룹에 추가됩니다. 오디오 그룹에 대한 자세한 내용은 여기 를 참조하십시오.

스프라이트

스프라이트는 일반적으로 여러분이 만드는 게임 내의 객체를 시각적으로 표현한 것입니다. 따라서 스프라이트는 GameMaker Image Editor 를 사용하여 그리거나 외부 그리기 프로그램으로 만들어 GameMaker 로 가져온 이미지입니다. 사용된 이미지는 하나의 정적 이미지일 필요도 없습니다... "스트립" 이미지도 될 수 있습니다. 즉, 단일 파일에 있는 여러 이미지를 애니메이션 스프라이트 하나로 만드는 데 사용할 수 있습니다. 예를 들어 다음 10개의 이미지는 스카이다이빙 중 팔을 흔드는 캐릭터의 스프라이트를 형성합니다.
그리고 실제 완성된 애니메이션은 게임에 배치될 때 다음과 같이 보일 것입니다.

이와 같은 스프라이트는 일반적으로 PNG 형식 파일이지만 GameMaker 는 벡터 스프라이트도 허용합니다. SWF 형식 및 Spine 스프라이트 JSON 형식( 수반되는 텍스처와 함께 atlas 파일).

  • 이미지에서 스프라이트 생성 - 이 옵션을 선택하면 스프라이트를 생성할 하나 이상의 이미지를 선택할 수 있는 파일 탐색기가 열립니다. 여러 이미지를 선택하면 선택한 각 이미지에 대해 하나씩 여러 프레임으로 스프라이트가 생성됩니다.
  • 이미지 편집기에서 열기 - 먼저 스프라이트 편집기를 열지 않고 편집을 위해 이미지 편집기에서 스프라이트를 엽니다.

타일셋

타일 세트는 스프라이트 리소스에서 가져 오지만 GameMaker 가 게임을 생성할 때 이를 다르게 처리하므로 별도의 리소스로 분류됩니다. 기본적으로 타일 세트는 GameMaker 가 다양한 설정에 대해 제공한 값을 기반으로 별도의 섹션으로 나눌 단일 이미지입니다. 그런 다음 룸 편집기에서(또는 코드를 통해 절차적으로) 이를 사용하여 룸에서 타일 맵을 생성할 수 있습니다. 지형, 벽, 배경 등과 같은 방의 정적 항목을 디자인하는 데 유용합니다. 개체와 같은 오버헤드가 없기 때문입니다.

위는 벽을 만들기 위한 타일 세트로 사용되는 예제 스프라이트입니다. 보시다시피, 64x64픽셀 청크로 "분할" 수 있으며, 그런 다음 룸 편집기에 배치됩니다.

참고 타일은 항상 정사각형이므로 그 외에 다른 것이 필요한 경우 룸 편집기에서 스프라이트 또는 자산 레이어가 있는 인스턴스를 사용해야 합니다.

위안화 타일셋 메뉴에는 다음 옵션과 이 페이지 아래에 나열된 옵션이 포함되어 있습니다.

  • 텍스처 그룹 할당 - 이 옵션을 선택하면 사용 가능한 텍스처 그룹 목록이 표시되며 필요한 경우 타일셋을 추가하거나 새로 생성할 텍스처 그룹을 선택할 수 있습니다. 자산 브라우저에서 여러 타일셋을 선택하거나 타일셋의 그룹 폴더를 선택한 경우 모든 타일셋이 선택한 텍스처 그룹에 추가됩니다. 텍스처 그룹에 대한 자세한 내용은 여기 를 참조하십시오.

타일 세트 편집기 섹션에서 타일 세트 추가에 대해 자세히 알아볼 수 있습니다.

타임라인

대부분의 게임에서 특정 순간에 특정 일이 일어나기를 원할 것입니다. 이제 인스턴스에서 알람 이벤트 를 사용하여 이를 달성할 수 있지만, 상황이 너무 복잡해지면 특히 12개의 알람으로 제한되기 때문에 더 이상 작동하지 않습니다. 이것이 바로 타임라인 리소스가 있는 이유입니다. 타임라인에서 게임 시간의 특정 순간에 발생해야 하는 작업을 지정하고 코드뿐만 아니라 다양한 이벤트에서 개체에 사용할 수 있는 모든 작업을 사용할 수 있습니다. 타임라인을 생성하면 객체에 할당할 수 있으며 해당 객체의 인스턴스는 방 내에 배치되거나 생성될 때 지정된 시간에 작업과 코드를 실행합니다.
