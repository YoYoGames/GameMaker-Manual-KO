스크립트 편집기

이 섹션에서는 게임 실행에 필요한 모든 코드를 입력하는 스크립트 편집기 ( 텍스트 편집기 라고도 함)를 다룹니다. 이 편집기는 사용자 정의 스크립트 자산을 생성하고 객체 이벤트 를 코딩하고 회의실 생성 코드 를 추가하는 등의 여러 작업에 사용됩니다.

이 섹션에서는 스크립트 자산을 만드는 방법이나 GML 코드 또는 GML Visual 을 사용하여 스크립트를 만드는 방법이 아닌 스크립트 편집기 의 작동 방식에 대해 간단히 설명합니다. 이를 위해 다음 섹션을 참조하십시오.

  • GML 코드 스크립트
  • GML 비주얼 스크립트

스크립트 자산을 생성할 때 GML Visual 과 GML Code 중에서 선택하라는 메시지가 표시될 수 있습니다. 자세한 내용은 GameMaker 언어 를 참조하십시오.

스크립트 편집기는 GameMaker가 사용하는 내장 프로그래밍 언어인 GameMaker 언어 코드를 작성하는 곳입니다(자세한 내용은 GameMaker 언어 섹션 참조). GameMaker 에 더 익숙해지고 최대한 활용하려면 이 언어를 사용하는 법을 배우는 것이 좋습니다. 이 언어는 게임을 만들 때 가능성을 크게 확장하기 때문입니다.

스크립트 자산은 마우스 오른쪽 버튼을 클릭하여 이름을 바꿀 수 있습니다. 자산 브라우저 에서 이름 바꾸기 를 선택하고 스크립트 이름은 자산에 대한 스크립팅 규칙을 따라야 하므로 영숫자만 사용할 수 있고 문자로 시작해야 하며 포함할 수 있는 유일한 기호는 " _" 언더바 기호. 다른 위치에 있는 스크립트 편집기는 이름을 변경할 수 없으며 예를 들어 개체 이벤트와 같이 편집 중인 항목과 관련된 이름을 갖습니다.

스크립트 편집기 창을 열면 다음과 같은 옵션과 레이아웃이 표시됩니다.

스크립트/이벤트 탭

스크립트 편집기는 한 창에 여러 스크립트 또는 이벤트를 포함할 수 있도록 상단에 탭이 있는 창에서 열립니다(이 동작은 환경 설정 에서 변경하여 각 스크립트 또는 이벤트에 새 창을 제공할 수 있음). 클릭할 수 있습니다. 탭에서 끌어서 탭 순서를 변경하거나 - 원하는 경우 - 현재 창에서 끌어 작업 공간에 배치하여 해당 스크립트에 대한 새 창을 생성하거나 다른 창에 추가할 수 있습니다. ), 스크립트 편집기를 최대화하여 새 작업 공간도 만들 수 있습니다. IDE 창 밖으로 스크립트 탭을 드래그하면 이 스크립트 리소스를 보유하기 위해 새 IDE가 생성되고 기본 창에서처럼 사용할 수 있습니다.

스크립트 편집기에서 개체 이벤트 의 코드를 편집 중이고 스크립트 편집기를 최대화했거나 별도의 창이나 작업 공간에 있는 경우 마우스 오른쪽 버튼을 클릭하면 편집기 메뉴에는 몇 가지 추가 옵션이 있습니다.

  • 작업 공간으로 복원: 스크립트 편집기를 최대화/작업 공간 상태에서 벗어나 작업 공간의 개체에 다시 연결합니다.
  • 개체로 이동: 코드가 있는 개체가 있는 작업 공간으로 이동하고 개체에 초점을 맞춥니다.
  • 이벤트 추가/열기: 현재 스크립트가 속한 개체에 새 이벤트를 추가할 수 있으며 추가된 이벤트에 대해 스크립트 편집기에서 새 코드 탭이 열립니다. 선택한 이벤트에 이미 코드가 있는 경우 새 탭에서 열립니다.

나머지 오른쪽 클릭에 대한 정보는 메뉴 옵션은 아래의 편집 섹션을 참조하십시오.

홈통

거터는 코드의 줄 번호를 표시하고 코드 상태에 대한 특정 정보를 전달하는 데 사용됩니다. 가장 중요한 것은 오타 를 냈거나 코드를 잘못 구성한 경우 GameMaker IDE 에서 코드 행에 빨간색 느낌표를 표시하여 문제를 알려줍니다. 구문 오류를 표시합니다. 그런 다음 기호 위로 마우스를 가져가면 문제에 대한 간략한 설명을 볼 수 있습니다.

다음 섹션에서 구문 오류(및 코드를 테스트하거나 컴파일할 때 발생하는 일반적인 오류)에 대한 자세한 정보를 찾을 수 있습니다.

  • 오류 보고

여백은 중단점이 추가된 스크립트 행도 표시합니다. 중단점은 단순히 게임 실행에 도달했을 때 디버거 가 게임 실행을 일시 중지하도록 하는 스크립트의 위치입니다. "f9 " 또는 오른쪽 클릭을 사용하여 메뉴. 중단점을 추가하면 거터에 다음과 같이 표시됩니다.

마지막으로 거터는 북마크된 항목도 표시합니다. 코드 줄을 북마크하려면 / + + Number (0에서 9까지), 이렇게 하면 단순히 다음을 사용하여 IDE의 어느 곳에서나 이 코드 줄로 다시 건너뛸 수 있습니다. / + Number (0에서 9까지). 책갈피를 삭제하려면 코드 줄에 책갈피를 추가하는 데 사용한 것과 동일한 작업을 수행하기만 하면 됩니다. 북마크가 추가되면 다음과 같이 표시됩니다.

검색/바꾸기

스크립트 편집기에서 작업하는 동안 다음을 누를 수 있습니다. / + "F " 검색창을 열거나 / + "H " 검색 및 바꾸기 열기:

여기에서 입력한 키워드에 대한 로컬 검색을 수행할 수 있으며 검색어를 입력한 후 검색 창의 오른쪽 상단에 있는 화살표를 사용하여 스크립트에서 찾은 하나의 용어에서 다음 용어로 건너뛸 수 있습니다. 다음 버튼을 토글하여 검색 작업이 수행되는 방식을 변경할 수 있습니다.

이것을 토글하면 검색 기능은 전체 입력 문자열과 일치하는 문자열만 강조 표시합니다. 예를 들어, " random "을 검색하지 않으면 이 문자열을 포함하는 모든 단어가 표시됩니다. irandom(), 또는 randomise() - 켜짐으로 전환하는 동안 기능만 표시됩니다. random().
이 옵션이 토글되면 검색 문자열의 내용뿐만 아니라 대소문자도 확인하도록 GameMaker 에 지시합니다. 예를 들어 "spr_Dog " 이 토글 을 끈 상태 에서 " dog "를 검색하면 스프라이트 문자열이 강조 표시되지만 토글이 켜져 있으면 " Dog "가 더 이상 " dog "와 같은 것으로 간주되지 않기 때문에 강조 표시되지 않습니다.

다음을 사용하여 검색 창을 연 경우 / + "F "를 입력하면 바꾸기 기능이 표시되지 않으므로 바꾸기 버튼을 클릭해야 합니다. 그것을 열려면. 열리면 다음 버튼을 사용하여 주어진 검색 문자열을 대체하는 데 사용할 문자열을 입력하여 작업을 수행할 수 있습니다.

이것을 클릭하면 스크립트에서 찾은 다음 검색 문자열이 주어진 바꾸기 문자열로 바뀝니다. "다음" 용어는 현재 커서 위치 다음으로 간주되며 검색 창의 오른쪽 상단에 있는 화살표 버튼을 사용하여 다른 용어로 건너뛸 수 있습니다.
이것을 클릭하면 주어진 대체 문자열을 사용하여 스크립트 내에서 검색 문자열의 모든 예가 대체됩니다.

전체 검색(즉, 현재 스크립트 또는 이벤트가 아닌 전체 프로젝트 검색)을 수행하려는 경우 다음을 누를 수 있습니다. / + + "F".
