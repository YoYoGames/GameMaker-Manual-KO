방송 메시지

시퀀스 와 스프라이트 는 길이에 따라 어느 지점에서나 브로드캐스트 메시지 라고 하는 것을 생성할 수 있습니다. 이 메시지는 애니메이션 타임라인을 따라 특정 프레임에 추가되는 단순한 문자열이며, 타임라인의 해당 지점에 도달하면 문자열을 수신하는 모든 인스턴스에 문자열이 브로드캐스트됩니다. 브로드캐스트 메시지 이벤트 (" 기타 " 이벤트 카테고리에 있음)가 있는 모든 인스턴스는 타임라인에 도달하면 메시지 문자열을 수신하며, 이 시점에서 이를 구문 분석하고 그에 따라 반응할 일부 코드를 갖는 것은 사용자에게 달려 있습니다. 시퀀스는 브로드캐스트 메시지를 수신할 수도 있지만 이를 위해서는 먼저 코드를 사용하여 수신기 메서드를 설정해야 합니다(자세한 내용은 시퀀스 이벤트 및 순간 의 GML 섹션 참조).

참고 인스턴스의 image_index 를 프레임으로 직접 설정하면 개체 스프라이트의 해당 프레임에 있을 수 있는 브로드캐스트 메시지가 트리거 되지 않습니다.

시퀀스 또는 스프라이트 타임라인에 브로드캐스트 메시지를 추가하려면 간단히 단추. 이렇게 하면 브로드캐스트할 메시지를 추가할 수 있는 대화 상자가 열립니다.

추가되면 메시지가 도프 시트(또는 스프라이트 프레임 보기)에 표시됩니다. 편집하려면 마우스 오른쪽 버튼을 클릭해야 합니다. 메시지 아이콘을 클릭하여 프레임과 겹치는 모든 메시지 목록을 엽니다(프레임에는 하나의 메시지만 있을 수 있지만 연속 프레임에 여러 메시지를 배치하면 시각적으로 겹치게 되어 원하는 메시지를 선택하기 더 어려워집니다. 그리고 같은 프레임에 메시지와 순간 이 있을 수도 있습니다.) 그런 다음 이 목록에서 하나를 선택하면 편집하거나 제거할 수 있는 메시지 대화 상자가 열립니다.

브로드캐스트 메시지 이벤트가 트리거되면 특수 DS 맵 이 생성되어 내장 변수에 저장됩니다. event_data. 이 변수 ID는 내장 전역 범위 변수이지만, 이를 트리거한 이벤트(이 경우에는 브로드캐스트 메시지 이벤트)의 DS 맵 데이터만 포함하고 다른 시간에는 -1을 반환합니다. 이벤트에 포함되는 키는 다음과 같습니다.

메시지 이벤트 구조체
변하기 쉬운 유형 설명
event_type 끈 이것은 문자열 "sequence event " 시퀀스에서 보낸 메시지의 경우 "sprite event "는 Sprite에서 보낸 메시지입니다.
message 끈 여기에는 수신된 메시지 문자열이 포함됩니다.
element_id 레이어 요소 ID 여기에는 메시지를 발생시킨 레이어 요소(시퀀스/스프라이트/인스턴스)의 ID가 포함됩니다. 그런 다음 이 ID 값을 사용하여 함수를 사용하여 메시지를 생성한 요소의 종류를 찾을 수 있습니다. layer_get_element_type() 필요에 따라 다른 관련 기능을 사용하여 대상을 지정하거나 사운드 재생과 같은 작업을 수행합니다.

인스턴스의 브로드캐스트 메시지 이벤트는 브로드캐스트 메시지가 전송될 때마다 여러 문자열을 수신합니다. 그들 모두에 대해 조치를 취할 필요는 없으며 다른 브로드캐스트 문자열을 수신하는 다른 인스턴스를 가질 수 있습니다. 그러나 동시에 수신될 수 있는 메시지를 수신하는 인스턴스가 있을 수 있습니다. 이런 일이 발생하면 이벤트가 여러 번 트리거 됩니다(각 메시지 문자열에 대해 한 번).

다음은 이 이벤트를 확인하는 방법에 대한 간단한 예입니다.

if event_data[? "event_type"] == "sequence event" // or you can check "sprite event"
{
    switch (event_data[? "message"])
    {
        case "hit":
            audio_play_sound(snd_hit, 0, false);
        break;

        case "destroy":
            sequence_destroy(event_data[? "element_id"]);
        break;
    }
}
