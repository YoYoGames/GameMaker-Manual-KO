비동기 이벤트

비동기식 이벤트는 GameMaker 가 웹이나 게임을 실행하는 기기에서 외부 소스로부터 콜백 을 수신할 때 시작되는 이벤트입니다. 기본적으로 GameMaker 에게 이미지 로드와 같은 작업을 수행하도록 지시하면 이 작업을 시작하지만 작동하는 동안 해야 할 다른 작업은 계속 수행합니다. 그런 다음 요청이 완료되면 GameMaker Studio 2로 콜백이 전송되고 해당 유형의 콜백에 대해 정의된 모든 비동기 이벤트가 시작됩니다.

비동기 이벤트는 키보드 이벤트와 마찬가지로 이벤트가 있는 모든 인스턴스에 대해 발생하므로 다음을 수행할 수 있습니다. http_get() 한 인스턴스에서 호출하지만 콜백을 처리하기 위해 다른 인스턴스에 비동기 HTTP 이벤트 가 있습니다.

참고: 브라우저의 XSS 보호로 인해 도메인 전체에서 리소스를 로드하려는 요청 및 시도가 차단되고 다음 이벤트를 사용할 때 빈 결과를 반환하는 것으로 나타날 수 있습니다.

비동기식 이벤트 범주와 관련된 다양한 유형의 이벤트가 있으며 모두 아래 섹션에 설명되어 있습니다.

참고: 다음 섹션에서 언급되는 async_load 변수 는 이 이벤트에서만 유효합니다. 이 변수가 가리키는 DS 맵은 이벤트 시작 시 생성되고 종료 시 다시 삭제되며 이 변수는 다음으로 재설정됩니다. 다른 모든 시간에는 -1 값입니다.

  • 오디오 재생
  • 오디오 녹음
  • 구름
  • 대화
  • HTTP
  • 인앱 구매
  • 이미지 로드됨
  • 네트워킹
  • 푸시 알림
  • 저장/불러오기
  • 사회의
  • 증기
  • 체계
