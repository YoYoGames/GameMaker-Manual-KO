이벤트 주문

GameMaker 에서 이벤트를 고려할 때 각 단계에서 모든 이벤트가 발생하는 정확한 순서는 GameMaker 의 내부 작동에 의존하고 소프트웨어 개발에 따라 변경될 수 있기 때문에 명확하게 말할 수 없다는 점에 유의해야 합니다.. 그러나 항상 같은 순서로 실행되는 특정 이벤트가 있습니다.

항상 같은 방식으로 발생하는 첫 번째 이벤트 집합은 방에 처음 들어갈 때 발생하는 이벤트입니다. 다른 이벤트가 실행되는 순서는 다음과 같습니다.

  • 개체 변수/인스턴스 변수가 초기화됩니다 (이는 개체 편집기 또는 룸 편집기 의 IDE 에서 정의할 수 있는 변수입니다). 변수가 연결된 개체가 있는 경우 다른 개체보다 먼저 생성되고 변수가 연결된 모든 인스턴스가 이후에 정의되므로 룸 편집기에서 개체 변수를 인스턴스별 변수로 덮어쓸 수 있습니다.
  • Create Event 다음에 각 인스턴스의 Instance Creation Code - 각 인스턴스가 생성될 때 생성할 다음 인스턴스로 이동하기 전에 먼저 Create 이벤트를 실행한 다음 Instance Creation Code를 실행합니다.
  • 게임 시작 이벤트 - 이 이벤트는 방 편집기에서 방에 배치된 모든 인스턴스에 대해 게임의 맨 처음 방에서 한 번 트리거됩니다(game_restart()를 호출하면 이 이벤트가 다시 트리거됩니다.
  • 방 생성 코드 - 방이 처음 들어갈 때 방 편집기에 작성된 일회용 코드입니다.
  • 모든 인스턴스의 룸 시작 이벤트 - 이벤트의 "기타" 범주 중 하나이며 모든 인스턴스에 대해 지속적이든 아니든 실행됩니다.

또한 특정 인스턴스를 Instance Layer Properties 창의 목록 위 또는 아래로 이동하여 룸 편집기 자체 내에서 특정 인스턴스가 생성되는 순서를 설정할 수 있다는 점도 주목할 가치가 있습니다.

이러한 특정 이벤트를 제외하고 3 단계 이벤트 및 다른 추첨 이벤트에 속하는 것과 상관없이 항상 동일한 방식으로 발생하는 유일한 알려진 순서입니다. 이것들은 항상 일관성을 유지하므로 게임의 각 단계에서 특정 타이밍에 의존하는 코드가 있는 경우 다음을 사용해야 합니다.

  • 시작 단계 이벤트
  • 단계 이벤트 (단계 이벤트는 인스턴스가 새 위치에 놓이기 직전에 실행됩니다)
  • 종료 단계 이벤트

그리기를 위한 모든 이벤트는 다음과 같이 항상 동일한 순서로 처리됩니다(다르게 트리거되는 창 크기 조정 이벤트 제외).

  • 사전 추첨 이벤트
  • 추첨 시작 이벤트
  • 추첨 이벤트
  • 추첨 종료 이벤트
  • 추첨 후 이벤트
  • GUI 시작 이벤트 그리기
  • GUI 이벤트 그리기
  • GUI 종료 이벤트 그리기
