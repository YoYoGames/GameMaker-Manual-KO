HTTP

HTTP 이벤트는 다음 중 하나의 콜백 에 의해 트리거되는 이벤트입니다. http_*()기능, 같은 http_post_string(). 실제로 이 이벤트에 독점적이고 특수 변수에 저장되는 DS Map 을 생성합니다. async_load (이 이벤트의 사용을 더 자세히 설명하는 코드 예제는 개별 함수를 참조하십시오). 이 DS 맵의 일반 구조는 다음과 같습니다.

  • "id ": 명령에서 반환된 id입니다. 일련의 발사를 하면 http_ 그런 다음 응답을 받는 대상을 알아야 하므로 이 값을 사용하여 원래 요청을 보냈을 때 저장한 값과 비교하여 올바른 것을 찾을 수 있습니다.
  • "status ": 오류는 0, 성공은 0, 다운로드 중이면 1보다 작은 값을 반환합니다.
  • "result ": 수신된 데이터(문자열만) 또는 다운로드한 파일의 경로(사용한 경우) http_get_file(). 상태가 0인 경우에만 이 키를 얻을 수 있지만 이는 플랫폼에 따라 다릅니다.
  • "url ": 요청하신 전체 URL입니다.
  • "http_status ": 원시 http 상태 코드(사용 가능한 경우). 대부분의 브라우저에 대한 표준 웹 상태 코드를 반환합니다. 예: "수정되지 않음"의 경우 304 또는 "콘텐츠 없음"의 경우 204 등...

위는 당신이 사용할 때 얻는 것을 보여줍니다 http_post_string() 기능이지만 각각의 http_ 함수는 약간 다른 맵을 반환할 수 있으므로 반환되는 정확한 데이터를 찾으려면 각 함수의 수동 항목을 참조하십시오.

다운로드할 파일을 요청한 경우 이 지도에서 추가 데이터를 제공할 수도 있습니다. 이 경우 "상태"의 값은 1이고 DS 맵은 다음과 같은 추가 키를 보유합니다.

  • " ": 이것은 웹 서버가 수신해야 한다고 말한 파일의 크기입니다(서버가 이 데이터를 반환하지 않는 경우 -1일 수 있음).
  • " ": 이미 다운로드된 데이터의 크기입니다.

이벤트는 수신된 모든 단일 데이터 패킷에 대해 트리거되지 않고 기본 게임 루프 내에서 다운로드하는 동안 언제든지 업데이트됩니다. 또한 현재 이 기능은 일반 Windows 대상 플랫폼에서만 사용할 수 있습니다.
