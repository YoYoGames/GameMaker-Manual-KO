대화

Dialog 비동기 이벤트는 다음과 같은 특수 비동기 대화 함수 중 하나에서 콜백 을 받을 때만 트리거됩니다. get_login_async() (이 이벤트가 어떻게 사용되는지에 대한 확장 코드 예제는 이 함수를 참조하십시오).

이러한 이벤트는 이름, 로그인 세부 정보, 숫자 또는 색상 등의 사용자 입력 유형을 요청하는 이벤트입니다. 대부분의 장치는 회신을 기다리는 루프를 좋아하지 않기 때문에 다음을 수행해야 합니다. 비동기식이고 GameMaker 는 이러한 기능이 대화 상자를 열고 이 이벤트를 트리거하는 필수 사용자 입력을 얻을 때까지 백그라운드에서 계속 실행됩니다. 즉, 게임이 진행되는 동안 대화 상자가 열려 있으면 대화 콜백을 수신할 때까지 게임을 일시 중지하는 단계를 수행해야 합니다.

다시 말하지만, 특수 변수에 저장된 ID와 함께 DS Map 이 반환됩니다. async_load. 이 맵에 있는 값은 사용된 기능에 따라 다르며 자세한 내용은 이 설명서의 각 기능에 대한 개별 항목을 참조해야 합니다.

참고 비동기 대화 상자 기능은 디버깅 및 테스트 용도로만 사용되며 출시된 게임에서는 사용해서는 안 됩니다. 이를 위해서는 대화 상자의 모양과 동작을 완전히 제어할 수 있도록 플레이어로부터 입력을 수신하는 사용자 지정 사용자 인터페이스를 만들어야 합니다.
