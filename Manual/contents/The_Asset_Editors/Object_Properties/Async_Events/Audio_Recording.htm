오디오 녹음

이 이벤트는 audio_start_recording() 함수를 실행하고 변수에 저장된 DS Map 을 반환합니다. async_load, 이벤트를 트리거한 함수의 콜백 에 따라 다른 키/값 쌍을 포함합니다.

ds_map에서 다음 키를 사용할 수 있습니다.

  • "buffer_id " - 오디오 데이터를 검색하는 데 사용할 수 있는 임시 버퍼 의 ID
  • "channel_index " - 이 데이터를 가져온 호출 함수에 의해 반환된 녹화 채널 인덱스
  • "data_len " - 수신한 데이터의 길이(바이트)

참고: 이 페이지의 시작 부분에서 언급한 것처럼 async_load 변수는 비동기식 이벤트에서만 유효합니다. 가리키는 DS 맵은 이벤트 시작 시 생성되고 종료 시 다시 삭제되기 때문입니다. 그러나 모든 오디오가 지도와 함께 저장되는 임시 버퍼도 이벤트가 끝나면 제거되므로 나중에 사용하기 위해 유지하려면 사용자 지정 버퍼에 복사해야 합니다.

아래 사용 예에서는 녹음된 오디오를 저장하기 위한 사용자 지정 버퍼를 만들고 입력 소스 0에서 녹음하도록 게임을 설정합니다.

channel_index = audio_start_recording(0);
audio_buffer = buffer_create(len, buffer_fast, 1);

녹음이 감지된 모든 게임 단계는 다음과 같이 녹음된 입력을 처리할 수 있는 비동기 오디오 녹음 이벤트를 트리거합니다.

var channel = async_load[? "channel_index"];
if channel == channel_index
   {
   len = async_load[? "data_len"];
   buffer_copy(async_load[? "buffer_id"], 0, len, audio_buffer, 0);
   }

여기에서 "channel_index" 키를 확인하여 기록을 시작할 때 반환된 값과 일치하는지 확인하고 일치하는 경우 이 이벤트에 대해 생성된 임시 버퍼의 내용을 사용자 지정 버퍼에 복사합니다. 그런 다음 사용자 지정 버퍼로 원하는 작업을 수행할 수 있습니다. 전용 버퍼 오디오 기능을 사용하여 재생할 수 있고, 처리하고 네트워크를 통해 보낼 수 있으며, 디스크에 저장할 수 있습니다. 녹음된 오디오가 있으면 버퍼에서 기본적으로 원하는 모든 작업을 수행할 수 있습니다.
