이미지 로드됨

이 이벤트는 HTML5 또는 URL 을 통해 GameMaker 에 이미지를 로드할 때 트리거됩니다. 예를 들어, 스프라이트 이미지를 로드하고 인스턴스가 로드된 후에만 인스턴스의 현재 스프라이트를 새 스프라이트로 변경하려고 한다고 가정해 보겠습니다. Create 이벤트(또는 다른 이벤트)에는 다음과 같은 항목이 있습니다.

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

이제 이미지를 장치나 브라우저에 로드하기 시작하지만 파일이 로드되기를 기다리는 동안 GameMaker 를 차단하지 않습니다. 대신 GameMaker 는 이미지가 로드되고 콜백 이 DS Map 이 생성되어 특수 변수에 저장되는 Image Loaded Event 를 트리거할 때까지 정상적으로 계속 실행됩니다. async_load. 지도에는 다음 정보가 포함되어 있습니다.

  • "filename ": 요청한 파일의 전체 경로입니다.
  • "id ": 로드한 리소스의 ID입니다. 이것은 리소스를 할당한 변수와 동일합니다.
  • "status ": 오류에 대해 0보다 작은 값을 반환합니다.

그런 다음 이 이벤트에서 새로 로드된 이미지를 스프라이트에 할당합니다. 다음 코드 예제에서는 반환된 정보가 사용되는 방법을 보여줍니다.

if ds_map_find_value(async_load, "id") == spr
{
    if ds_map_find_value(async_load, "status") >= 0
    {
        sprite_index = spr;
    }
}

위의 코드는 먼저 id 생성된 DS 맵의 status 콜백의. 값이 0보다 크거나 같으면(성공 신호) 콜백의 결과가 스프라이트 인덱스를 새로 로드된 이미지로 설정하는 데 사용됩니다.
