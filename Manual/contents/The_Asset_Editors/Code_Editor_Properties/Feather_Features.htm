깃털 특징

Feather 는 스마트 리팩토링 옵션과 함께 GML 코드 스크립트에서 지능형 코드 완성 및 향상된 구문 검사를 제공합니다.

이 페이지는 Feather가 스크립트 및 개체 이벤트 에 대해 GML 코드 편집기에서 제공하는 기능을 나열합니다. 이러한 기능은 Feather가 enabled 경우에만 사용할 수 있습니다.

호버 정보

함수, 스프라이트, 색상 또는 기타 특수 값 위로 마우스를 가져가면 해당 정보가 표시됩니다. 아래에서 몇 가지 예를 볼 수 있습니다.

자산 리팩토링

자산 브라우저에서 자산의 이름을 바꾸면 Feather는 프로젝트 코드에서 자산에 대한 모든 언급을 자동으로 업데이트하므로 해당 참조가 손상되지 않습니다. 이것은 Feather 설정 에서 활성화 또는 비활성화할 수 있습니다.

코드 편집기를 통해 자산의 이름을 바꿀 수도 있습니다. 자산 참조에 텍스트 커서를 놓고 CTRL/CMD + SHIFT + R 을 누릅니다.

새 이름을 입력하고 " 적용 "을 누르십시오. 자산 자체와 프로젝트에서 자산에 대한 모든 참조의 이름이 바뀝니다.

빠른 수정

코드에 오류나 경고가 표시되면 보고된 줄에 텍스트 커서를 놓고 CTRL/CMD + Q 를 눌러 빠른 수정 메뉴를 불러옵니다. 또는 다음을 클릭할 수 있습니다. 거터에 깃털 아이콘입니다.

이 메뉴는 가능한 경우 빠른 수정을 보여주고 해당 행에 대한 규칙을 비활성화하거나 심각도 를 변경할 수 있습니다.

위의 예는 표면 대상이 변경된 후 재설정되지 않은 이벤트를 수정하는 데 사용되는 빠른 수정을 보여줍니다.

함수를 생성할 때 빠른 수정 메뉴를 사용하면 함수에 대한 JSDoc 주석을 자동으로 생성할 수 있습니다.

모든 참조 찾기

Feather를 사용하면 프로젝트 전체에서 변수에 대한 모든 참조를 찾을 수 있습니다.

변수에 텍스트 커서를 놓고 F3 키 를 누릅니다. 작업 공간 상단에 작은 검색 표시줄이 표시됩니다.

해당 화살표를 눌러 이전 또는 다음 참조로 이동하거나 닫습니다.

이 검색은 스마트하므로 단순히 텍스트 문자열을 검색하는 대신 선언된 범위에서 선택한 변수의 참조를 찾습니다. 즉, 이름은 같지만 서로 다른 두 개체에 별도로 존재하는 두 변수는 일치하지 않습니다.

모든 참조 나열

변수에 텍스트 커서를 놓고 SHIFT + F3 키 를 누릅니다. 그러면 검색 결과 창이 열리고 선택한 변수의 모든 참조가 목록에 표시됩니다.
