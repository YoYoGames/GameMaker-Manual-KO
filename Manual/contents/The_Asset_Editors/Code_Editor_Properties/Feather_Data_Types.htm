깃털 데이터 유형

Feather 는 데이터 유형을 사용하여 코드를 작성할 때 스마트한 구문 검사를 제공하여 변수 또는 함수 매개변수에 잘못된 데이터 유형을 사용하지 않도록 합니다.

또한 JSDoc 주석 을 사용하여 고유한 스크립트 함수 의 매개변수 및 반환 값에 대한 데이터 유형을 지정할 수 있습니다.

그만큼 @param 그리고 @return JSDoc 태그를 사용하면 다음 유형 중 하나를 지정할 수 있습니다.

참고 JSDoc 유형은 대소문자를 구분하지 않습니다. 즉, Id.DsList 와 id.dslist 가 동일합니다.

참고 * 지정자 예 열에는 일부 지정자 예가 나열되지만 전부는 아닙니다. 예를 들어 여기에 나열되지 않은 Id, Asset 및 Constant 그룹 아래에 가능한 유형이 더 있을 수 있습니다.

Feather를 사용할 때 IDE에서 Any * 유형을 볼 수 있습니다. 이는 현재 Feather에서 식별자 유형(변수, 매개변수, 반환 값 등일 수 있음)을 식별할 수 없음을 나타냅니다. 해당 식별자에 대해 더 많은 코드가 추가되면 특정 유형을 가정할 수 있습니다.

지정자

점을 사용하여 기본 데이터 유형 뒤에 지정자가 추가 됩니다., 해당 그룹의 정확한 데이터 유형을 지정합니다.

Id.DsList, Asset.GMObject 및 Constant.Color 와 같은 유형은 지정자를 사용합니다.

컬렉션 유형

여러 값을 포함하는 데이터 구조인 Array 및 Id.DsList 와 같은 형식은 모든 내용에 대해 단일 데이터 형식을 지정할 수 있습니다.

이것은 꺾쇠 괄호 <> 를 사용하여 유형에 추가되는 컬렉션 유형을 사용하여 수행됩니다.

예를 들어 문자열을 포함하는 배열은 Array<String> 이고 Reals를 포함하는 배열을 포함하는 DS 목록은 Id.DsList<Array<Real>> 입니다.

유형 검증

아래 표는 특정 유형(행)의 값을 데이터 유형(열)이 필요한 매개변수에 전달하면 어떻게 되는지 보여줍니다.

예를 들어, 문자열(두 번째 열)이 필요한 매개변수에 대해 정의되지 않음 (첫 번째 행)을 지정하면 오류가 발생합니다.
