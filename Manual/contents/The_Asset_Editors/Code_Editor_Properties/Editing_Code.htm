코드 편집

스크립트 편집기를 사용하기 시작하면 스크립트 자산 또는 이벤트에 대해 갖고 싶은 코드를 올바른 GML 구문 에 따라 입력하기만 하면 됩니다. 이 작업을 수행하는 동안 자동 완성 창에서 도움을 받을 수 있습니다.

이 창은 코드를 작성하는 동안 팝업되며(일반적으로 약간의 일시 중지 후, 그러나 기본 설정에서 설정할 수 있음) 알파벳 순서로 "가장 적합한" 답변을 선택할 수 있습니다.

코드 편집기를 사용하는 동안 다음을 클릭할 수 있습니다. 마우스를 끌어 코드 행을 선택하거나 그런 다음 클릭 드래그하여 코드 열을 선택합니다. 이 방법으로 선택한 모든 항목은 다음의 표준 키보드 단축키를 사용하여 복사, 잘라내기 및 붙여넣기 가 가능합니다. / + " C ", / + "X " 그리고 / + "V ".

다양한 방법으로 코드에 주석 을 달 수 있다는 점은 주목할 가치가 있습니다. 코드에 주석을 다는 것은 생산성을 유지하는 데 매우 중요합니다. 각 부분이 무엇을 하고 왜 존재하는지 명확하게 코드 섹션을 구분하기 때문입니다. 그것은 또한 당신이 팀에서 일하고 있다면 다른 구성원을 위해 메모를 남길 수 있고 그들이 당신이 한 일이나 코드의 특정 섹션이 무엇을 위한 것인지 정확히 알 수 있다는 것을 의미합니다. 솔로 개발자라도 코드에 주석을 남기는 것이 중요합니다. 특히 프로젝트를 종료하고 나중에 다시 돌아와야 하는 경우에는 더욱 그렇습니다.

"// " 텍스트 줄 앞에 또는 "를 사용하여 전체 단락을 주석 처리할 수 있습니다./* text here */ ". 선택한 코드 줄에 주석을 달고 주석을 해제하는 두 개의 키보드 단축키도 있습니다. / + "K " 댓글을 달고 / + + "K " 주석을 해제합니다. 다음 이미지는 이러한 방법과 일부 JS Doc 주석을 사용하여 작성된 코드를 보여줍니다(아래에서 더 자세히 설명).

스크립트에는 포함된 함수의 속성을 정의하기 위해 특정 JSDoc 스타일 주석이 추가되어 코드에서 함수를 사용할 때 인수 및 기타 세부 정보와 함께 자동 완성으로 표시될 수 있습니다. 자세한 내용은 JSDoc 스크립트 주석 섹션에서 이에 대해 자세히 알아볼 수 있습니다.

기본적으로 작성된 모든 코드에는 일치하는 각 중괄호 세트 옆에 코드 접기 아이콘이 있습니다. {}. 이것들은 약간의 플러스 형태를 취합니다. + 또는 마이너스 - 아이콘을 클릭하면 일치하는 중괄호 내에서 코드가 "접혀" 시각적 공간을 덜 차지합니다.

특수 기능을 사용하여 코드 접기 영역을 수동으로 설정할 수도 있습니다. #region 코드 섹션을 표시하는 태그. 이것이 작동하는 방식은 #region 태그(주석 텍스트와 함께)를 코드 섹션 상단에 추가한 다음 해당 #endregion 태그를 섹션 맨 아래에 붙인 다음 코드를 "접을" 수 있습니다. 즉, 편집기에서 한 번에 표시되는 코드 줄 수를 줄일 수 있도록 코드를 숨길 수 있습니다.

실제 코드는 다음과 같습니다.

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

눈치채셨겠지만 스크립트 텍스트의 일부는 색상이 지정되어 있습니다. 스크립트 편집기는 기존 개체, 내장 변수 및 기능 등에 대해 "알고" 그에 따라 색상 코드를 지정합니다. 색상 코딩은 이름의 철자가 틀리거나 키워드를 변수로 사용하거나 구문 오류가 있는 경우 즉시 확인할 수 있으므로 실수를 방지하는 데 많은 도움이 됩니다. 그러나 색상 코딩이 마음에 들지 않으면 기본 설정 에서 켜고 끌 수 있습니다(다음을 사용할 수도 있습니다. F10 편집기에서 켜거나 끄기) 사용되는 색상을 사용자 정의할 수 있습니다.

마우스 오른쪽 버튼을 클릭할 수 있습니다. 편집기의 아무 곳에서나 다음 메뉴를 열 수 있습니다.

코드 줄을 잘라내고, 복사하고, 붙여넣을 것으로 예상되는 일반 기능 외에도 선택한 텍스트에 주석을 달거나 주석 처리를 해제하고, 들여쓰기를 추가 또는 제거하고, 코드 창을 1, 2 또는 3으로 분할할 수 있는 옵션이 있습니다. 별도의 창( #regions 편집 중인 코드에서 정의하지 않은 경우 사용할 수 없음).

창으로 분할하는 마지막 옵션은 실제 창을 다른 부분으로 분할한 다음 스크립트 또는 이벤트를 드래그하여 비교 및 편집을 위해 다른 부분과 함께 표시할 수 있습니다.

이것은 두 개의 스크립트를 비교하거나 다른 창을 필요로 하지 않거나 탭 사이를 지속적으로 전환할 필요 없이 연결된 스크립트를 동시에 볼 때 매우 유용합니다.

이 메뉴의 다른 옵션은 Debugger 와 함께 사용하기 위해 중단점을 토글하고 GML 코드를 GML Visual로 또는 그 반대로 변환하는 것입니다. 코드를 GML Visual로 변환하는 경우 각 코드 줄에 대해 개별 작업을 생성 하지 않고 단일 코드 작업을 사용하여 이전에 이벤트 또는 스크립트에 있던 모든 것을 보유합니다.
