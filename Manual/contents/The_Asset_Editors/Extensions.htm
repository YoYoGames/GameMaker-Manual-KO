확장 편집기

확장 패키지 를 사용하여 많은 추가 기능과 상수 를 GameMaker Language(GML)에 추가하고 프로젝트 간에 쉽게 전송할 수 있습니다. 확장 기능은 게임 기능을 확장하는 데 사용할 수 있는 다른 언어(Android용 Java, iOS용 Objective-C 등)로 작성된 외부 SDK 또는 소스 코드 에 연결할 수도 있습니다. 광고, 인앱 구매 등).

확장은 모든 프로젝트에서 자주 사용하는 함수 라이브러리를 생성하는 데 사용할 수 있으므로 모든 프로젝트에 동일한 스크립트를 가져오는 대신 즐겨찾는 함수 및 상수 모음을 포함하는 하나의 확장 자산을 추가하기만 하면 됩니다. 이러한 기능은 특정 플랫폼에 따라 다를 수 있으며 확장 기능을 사용할 수 있는 플랫폼을 쉽게 선택할 수 있습니다.

확장 생성에 대한 단계별 예제는 확장 생성 을 참조하십시오.

대상 플랫폼을 기반으로 확장을 비활성화하려면 확장 을 비활성화하는 방법을 참조하십시오.

확장 가져오기

일반적으로 타사 확장은 GameMaker Marketplace 에서 제공되며 구독한 확장은 거기에서 설치할 수 있는 Marketplace 라이브러리 창에 나열됩니다. 마켓플레이스에서 확장 패키지를 받지 못한 경우에도 마우스 오른쪽 버튼을 클릭하여 프로젝트에 추가할 수 있습니다. 자산 브라우저 의 아무 곳에서나 기존 추가 를 선택한 다음 확장 위치를 찾아 선택합니다.

참고 다음이 있는 경우 .yymps /.yymp /.gmez 패키지를 IDE로 끌어서 가져오거나 "도구" 메뉴로 이동하여 "로컬 패키지 가져오기"를 선택할 수 있습니다.

대상 플랫폼

자산 브라우저에 확장을 추가한 후에는 이름과 버전 번호를 지정하고 내보낼 플랫폼도 설정해야 합니다. 선택한 플랫폼의 선택을 취소하면 해당 플랫폼에 대한 실행 파일의 일부로 확장을 내보내지 않도록 GameMaker 에 지시할 수 있습니다. 또한 확장 속성이 선택한 대상 플랫폼과 일치하는지 확인하고 다음을 기반으로 확장 파일에 대해 호환되는 내보내기 대상을 선택해야 합니다.

  • 모든 대상: GML 확장. 이들은 GameMaker Language(GML)를 사용하여 만든 확장이며 모든 대상 플랫폼과 호환됩니다. ( 이것은 Opera GX 대상이 현재 지원하는 유일한 확장 유형입니다. )
  • Windows(Standard, UWP, Xbox): Windows를 사용할 수 있습니다. DLL 파일(동적 링크 라이브러리). Windows 게임 옵션 에서 "x64 Windows 런타임 사용"을 선택한 경우 프로젝트의 모든 DLL 이 64비트와 호환되어야 합니다. 32비트 DLL은 64비트 실행 파일에서 작동하지 않습니다.
  • macOS: macOS 대상은 표준을 사용할 수 있습니다. DyLib 확장을 위한 파일.
  • Ubuntu(Linux): Ubuntu 대상의 경우 확장이 다음을 사용하는지 확인해야 합니다. .so 파일.
  • HTML5: 이러한 플랫폼은 다음을 사용할 수 있습니다. JavaScript 확장.
  • Android: 이 플랫폼은 확장 프로그램을 작성해야 합니다. Java 또는 Android Framework SDK.
  • iOS / tvOS: 이 플랫폼의 확장은 Objective C로 작성되었으며 다음이 필요합니다.*.mm 그리고*.h 파일 또는 프레임워크를 사용하는 경우 사전 컴파일된 네이티브*.a 파일.
  • PlayStation: PlayStation의 경우 확장 기능은 다음과 같아야 합니다. PRX 각 대상에 서로 다른 PRX 대상(PS3, PS4 또는 PS5)의 사양을 준수합니다. PSVita에서 작업하는 경우 확장은 다음과 같아야 합니다. SUPRX 대신에.

참고 일부 확장은 GameMaker가 설계된 플랫폼 및 CPU 아키텍처를 올바르게 식별하기 위해 특정 명명 규칙을 따라야 합니다. 자세한 내용은 명명 규칙 을 참조하세요.

확장에는 다양한 관련 파일이 있을 수 있으므로 여러 플랫폼에 대해 단일 확장을 생성할 수 있습니다. 이 경우 기본 확장 편집기에서 호환되는 모든 플랫폼을 지정한 다음(예: 옆에 있는 확인란 선택) 개별 플랫폼에서 확장의 각 부분에 대한 파일 속성에서 해당 파일을 내보내야 하는 플랫폼을 설정합니다.

특정 확장 파일 또는 자리 표시자의 대상 플랫폼을 변경하려면 먼저 확장 속성을 열어야 합니다(두 번 클릭 자산 브라우저에서 확장자)를 클릭한 다음 확장자 속성에서 편집하려는 오른쪽에서 리소스 파일을 선택합니다. 열리는 창에서 특정 파일을 내보낼 플랫폼을 선택할 수 있습니다.

확장 옵션

사용 중인 확장 프로그램에는 확장 프로그램 편집기 창 하단에 표시되는 옵션이 있을 수 있습니다.

확장 프로그램에 따라 다르며 모든 옵션이 있는 것은 아닙니다. 옵션에 대한 정보는 사용 중인 확장의 문서를 참조하십시오.

자신의 확장에 대한 옵션을 설정하려면 확장 옵션 생성 을 참조하십시오.

빌드 시 압축 해제

"Uncompress on build" 옵션은 특정 대상 플랫폼을 위한 압축된 아카이브 파일에 사용할 수 있습니다. 이 옵션이 활성화되면 해당 아카이브의 내용이 빌드된 패키지의 Included Files 디렉토리로 추출되지만 대상 플랫폼으로 복사된 후에만 추출됩니다.

예를 들어 이 기능을 활성화하면 .zip 확장에 파일을 추가한 다음 Windows에서 macOS 빌드를 시작합니다. .zip 아카이브는 Windows에서 macOS로 그대로 복사된 다음 대상 플랫폼(이 컨텍스트에서는 macOS)에서 컴파일하는 동안 압축이 해제됩니다.

확장 함수 호출

확장을 설정하고 나면 GameMaker 언어에서 사용되는 내장 함수 및 상수와 마찬가지로 확장에 대한 함수와 상수를 사용할 수 있습니다. 게임이 크로스 플랫폼인 경우(예: Windows 및 Android 대상에서 실행되도록 설계됨) 문제를 방지하기 위해 확장 특정 함수를 호출하기 전에 OS 상수와 함수 를 사용하여 현재 플랫폼을 확인해야 합니다. 예를 들면 다음과 같습니다.

switch(os_type)
{
    case os_android:
        call_android_extension_function(x, y, z);
    break;

    case os_windows:
        call_windows_extension_function(x, y, z);
    break;
}

다른 플랫폼의 경우 확장 프로그램이 함수 이름을 공유할 수 있으므로 Windows와 Android에서 메시지를 팝업하기 위한 별개의 파일이 있는 확장 프로그램이 있는 경우 두 확장 파일 모두 동일한 함수 이름을 공유할 수 있습니다. 둘 다 해당 플랫폼으로만 내보내도록 설정되었습니다.
