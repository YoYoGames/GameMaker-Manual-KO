변신된 스프라이트 그리기

이 함수는 Draw Sprite 작업과 유사하게 주어진 스프라이트를 그립니다. 그러나 그려지는 스프라이트의 크기, 혼합, 회전 및 프레임을 변경하는 추가 옵션이 있습니다. 이 값을 변경해도 리소스는 어떤 식으로든 수정되지 않으며 (그리는 방법만) 함수의 모든 인수에 대한 직접 값 대신 사용 가능한 스프라이트 속성 변수 를 사용할 수 있습니다. 아래 이미지는 다양한 값이 스프라이트 그리기에 어떤 영향을 미치는지 보여줍니다.

참고 색상 혼합은 WebGL이 활성화된 경우에만 HTML5 대상에 권장되지만 활성화되지 않은 경우 혼합 색상을 설정할 수 있으며 스프라이트가 정상적으로 혼합됩니다. 그러나 이러한 방식으로 모든 블렌딩은 중복 스프라이트를 생성한 다음 캐시에 저장하고 필요할 때 사용합니다. 이것은 최적이 아니며 여러 색상 변경을 사용하는 경우 WebGL을 활성화하지 않는 한 게임 성능이 느려집니다. WebGL을 사용하지 않으려면 글꼴 캐시 크기를 설정하여 기능을 사용하여 필요한 경우 이를 제한할 수 있습니다. sprite_set_cache_size().

참고: 이 작업은 다양한 그리기 이벤트 에서만 사용되며 다른 곳에서 사용되는 경우 아무 것도 그리지 않습니다.

동작 구문:

인수:

논쟁 설명
Sprite 그릴 스프라이트
Frame 그릴 스프라이트 프레임(내장 변수 사용 image_index 현재 프레임에 대해)
X 방 내에서 그릴 x 위치
Y 방 내에서 그릴 y 위치
Scale X X축에 대한 스케일 팩터(내장 변수 사용 image_xscale 현재 X 스케일의 경우)
Scale Y Y축에 대한 배율 인수(내장 변수 사용 image_yscale 현재 Y 스케일의 경우)
Rotation 스프라이트의 그리기 각도(0에서 360까지, 0이 오른쪽인 곳에서 시계 반대 방향으로 회전 - 내장 변수 사용 image_angle 현재 회전에 대해)
Colour 스프라이트와 혼합할 색상(기본값은 흰색이며 내장 변수를 사용할 수 있습니다. image_blend 현재 색상)

예시:

위의 작업 블록 코드는 변수를 확인하고 다음으로 설정되어 있는지 확인합니다. true 그런 다음 스프라이트는 x 및 y 축을 따라 임의 크기로 그려지고 빨간색과 혼합됩니다. 그렇지 않으면 스프라이트는 기본 변환을 사용하여 그려집니다.
