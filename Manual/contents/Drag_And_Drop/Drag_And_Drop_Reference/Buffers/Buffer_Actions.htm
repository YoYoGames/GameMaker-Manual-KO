버퍼 작업 라이브러리

버퍼 는 데이터가 한 장소에서 다른 장소로 이동되거나 어떤 식으로든 조작되는 동안 일시적으로 데이터를 보유하는 데 사용되는 물리적 메모리 영역입니다. 예를 들어 버퍼를 사용하여 게임 상태 데이터를 보유할 수 있습니다. GameMaker 에는 버퍼 생성, 수정 및 로드/저장과 관련된 여러 가지 특정 작업이 있지만 사용하기 전에 다음 사항에 유의하십시오.

  • 버퍼가 생성되면 자동으로 지워지고 0으로 채워집니다.
  • 버퍼 함수의 "크기" 인수는 항상 버퍼의 크기 (바이트 )를 나타냅니다.
  • GML Visual 버퍼는 항상 1바이트로 정렬 됩니다. "정렬"은 데이터가 버퍼 내에 저장되는 방식을 말하며 GML 버퍼 Functions 를 사용할 때 사용되지만 GML Visual 의 경우 관련이 없습니다.
  • 버퍼의 데이터는 순차적 방식으로 읽고 씁니다(예: 쓰기 버퍼 작업을 세 번 호출한 다음 첫 번째 데이터를 쓰고 앞으로 이동하고 두 번째 데이터를 오른쪽으로 이동하고 앞으로 이동한 다음 마지막 데이터를 씁니다). 버퍼에서 다음 읽기/쓰기가 수행될 위치를 알려주는 "탐색" 값이라는 것이 있습니다.
  • 모든 데이터는 버퍼에 기록되고 버퍼에서 문자열 로 반환되므로 실수를 저장한 경우 데이터 유형 작업 중 하나를 사용하여 문자열에서 값으로 변환해야 합니다.

버퍼 생성에 사용되는 메모리는 시스템 메모리이므로 게임에 포커스가 없어도 버퍼에 저장된 모든 데이터는 안전해야 하지만 앱을 닫거나 다시 시작하면 손실됩니다. 버퍼를 파일로 저장한 다음 파일 작업 을 사용하여 게임에 다시 로드할 수도 있다는 점도 주목할 가치가 있습니다.
