경로를 따라 시작

이 작업은 자산 브라우저에서 생성된 경로를 따라 인스턴스를 시작합니다. 그런 다음 이 작업을 사용하여 인스턴스에 따라갈 경로, 경로를 따라야 할 속도(게임 프레임당 픽셀 단위로 측정), 경로 끝에 도달했을 때 동작하는 방법, 절대 또는 상대를 따를지 여부를 알려줍니다. 경로 위치. 속도를 설정할 때 음수 값을 사용하여 인스턴스가 경로를 반대로 따라야 함을 나타낼 수 있으며 여기에서 설정한 속도는 각 경로 지점에 대해 경로 편집기에서 설정한 속도 수정자로 수정할 수 있습니다. 경로 속도를 2로 설정하고 경로에서 포인트 중 하나의 속도 수정자가 200%이면 해당 지점의 실제 속도는 게임 프레임당 4픽셀이 됩니다.

초기 경로 속도를 설정하고 나면 다음 옵션에서 인스턴스가 경로 끝에 도달할 때 작동하는 방식을 설정할 수 있습니다(인스턴스가 경로 끝에 도달하면 기타 - 경로 종료 됨도 트리거됩니다. 이벤트 ):

  • 중지: 경로를 종료하고 마지막 경로 지점에서 인스턴스를 중지합니다.
  • 다시 시작: 경로 편집기에서 경로가 닫힌 것으로 표시되지 않은 경우 시작 위치에서 다시 시작 위치로 이동하여 경로를 계속합니다.
  • Loop: 현재 위치에서 계속해서 경로를 다시 실행합니다.
  • 후진: 경로를 따라 다시 후진합니다(경로 추종 속도를 반대로 하여 달성). 경로의 시작 부분에 도달하면 다시 후진하는 등...

마지막으로 경로에 절대 여부를 표시할 수 있습니다. 기본적으로 인스턴스는 현재 위치(상대 위치)에서 경로를 따라 시작하지만 절대 확인란에 플래그를 지정하면 경로를 따라 디자인하고 경로 편집기에 배치한 것과 정확히 일치하기 시작합니다(절대 위치). 아래 이미지는 이를 보여줍니다.

위의 이미지에서 볼 수 있듯이 두 인스턴스는 이 작업으로 시작된 동일한 경로를 따르고 있지만 인스턴스 중 하나는 절대 경로를 따르고 다른 인스턴스는 상대 경로를 따릅니다. Absolute 에 플래그가 지정되면 인스턴스가 배치된 방의 위치에 관계없이 경로 시작 부분으로 "점프"됩니다.
