파일 작업 라이브러리

파일 작업 은 버퍼 파일과 Ini 파일의 두 가지 다른 파일 유형을 처리하며 기존 파일의 이름을 바꾸거나 복사하는 특정 일반 파일 작업도 있습니다.

버퍼 파일 은 이전에 생성한 버퍼에서 데이터를 저장하여 생성되며 이 데이터는 나중에 언제든지 버퍼에 다시 로드할 수 있습니다. 여기에서 버퍼에 대한 자세한 정보를 찾을 수 있습니다. 버퍼 작업.

Ini 파일 은 대부분의 플랫폼과 호환되는 작고 가벼운 파일입니다. 인터페이스 기본 설정, 로컬 최고 점수, 레벨 데이터 등과 같은 작은 정보를 저장하는 데 이상적이며 사용이 매우 쉽습니다. 이러한 작업을 사용하기 위해 이전에 Ini 파일을 만들 필요는 없으며 존재하지 않는 Ini 파일에서 읽을 수 있습니다. default 반환 값(귀하가 지정), 그러나 작업을 사용하기 전에 열고 수정할 수 있는 "기본" ini 파일을 최소한 생성하는 것이 좋습니다. 이 기본 ini 파일은 Open Ini File 액션을 호출한 다음 Close Ini File 을 호출하여 생성할 수 있습니다. 파일을 닫으면 디스크에 기록되기 때문입니다. 또는 Asset Browser의 Included Files 에 포함시킬 수 있습니다. 자산 브라우저에 포함된 파일을 기본 Ini로 사용 하는 경우 파일 시스템 작동 방식 에 대한 매뉴얼 섹션도 읽어야 합니다.

참고: 현지화 게임의 경우 외부 파일에서 악센트 문자를 읽을 수 있다는 것이 매우 중요합니다. 즉, 먼저 UTF8 형식 의 ini 파일을 생성한 다음 게임 실행 시 내보내지고 ini 함수에 의해 생성된 기본 ANSI 형식 ini 파일 대신 사용되도록 GameMaker 에 포함된 파일로 추가해야 합니다. 이전에 제공된 적이 없습니다. 이런 식으로 모든 악센트와 비로마 문자를 유지하면서 올바르게 읽고 쓸 수 있습니다.
