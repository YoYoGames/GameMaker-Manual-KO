스택 생성

이 작업은 새 스택 데이터 구조를 만들고 인덱스 값을 반환하므로 나중에 다른 데이터 구조 작업을 통해 스택에 액세스할 수 있습니다. 스택 인덱스는 Assign Variable 또는 Declare Temp 를 사용하여 이전에 생성할 수 있는 제공한 Target Variable로 반환되거나 "Temp" 확인란에 플래그를 지정하여 이름을 지정하고 값을 저장할 임시 로컬 변수를 생성할 수 있습니다. 스크립트 또는 이벤트의 끝. 새로 생성된 스택 데이터 구조는 "비어 있는" 것으로 간주됩니다. 즉, 항목이 포함되어 있지 않습니다. 더하기 아이콘을 클릭하여 추가 DS 스택을 생성할 수 있습니다. 액션 옆에 스택 ID를 저장할 다른 변수를 선택합니다.

중요한! 데이터 구조를 만들면 대상 플랫폼의 메모리가 소모되므로 Free Data Structure 작업을 사용하여 더 이상 필요하지 않을 때 모든 데이터 구조를 해제해야 합니다. 그렇지 않으면 게임 성능을 손상시키거나 충돌을 일으킬 수 있는 메모리 누수가 발생합니다. 데이터 구조 인덱스를 저장하기 위해 임시 로컬 변수를 사용할 때 특히 관련이 있습니다. 이러한 변수는 코드나 이벤트의 끝에서 제거되기 때문입니다. 그러나 그렇다고 데이터 구조도 제거되는 것은 아닙니다! 데이터 구조는 여전히 존재하지만 참조할 수 있는 방법이 없으므로 인스턴스 변수를 사용하고 나중에 구조를 해제하거나 인덱스가 다음에 저장된 경우 이벤트 또는 함수가 끝나기 전에 구조를 해제하십시오. 임시 변수.
