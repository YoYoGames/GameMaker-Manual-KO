게임패드 액션 라이브러리

GameMaker 에는 연결된 여러 게임 패드에서 아날로그 및 디지털 컨트롤을 모두 감지하는 데 사용할 수 있는 여러 가지 전용 작업이 있습니다. 이러한 작업을 수행하려면 선택 중인 게임패드를 나타내는 숫자(0부터 계산)인 게임패드 인덱스 값을 지정해야 합니다. 게임패드가 장치에 연결되면(또는 제거되면) 비동기 시스템 이벤트 가 트리거됩니다(그러나 이를 처리하려면 코드를 사용해야 할 수 있음).

게임패드 "슬롯"은 0부터 인덱싱되며 슬롯 0 - 3 은 Xinput 게임패드에만 해당됩니다 (예: Xbox360 컨트롤러 및 호환 장치). 그러나 DirectInput 게임패드에 대해 슬롯 4 - 11을 확인할 수도 있습니다. 즉, 이 슬롯을 통해 연결될 때 다른 많은 컨트롤러 모델을 감지할 수 있습니다.

입력 상수

게임패드 동작으로 작업할 때 입력은 축, 버튼 또는 모자 에서 올 수 있으며 GameMaker 는 다음 내장 상수에 할당합니다("모자"는 일반적으로 비표준 컨트롤러에서만 감지됨).

일정한 설명
gp_face1 상단 버튼 1(Xbox 360 컨트롤러의 "A" 와 PS 컨트롤러의 십자가 에 매핑됨)
gp_face2 상단 버튼 2(Xbox 360 컨트롤러의 "B" 와 PS 컨트롤러의 원 에 매핑됨)
gp_face3 상단 버튼 3(Xbox 360 컨트롤러의 "X" 와 PS 컨트롤러의 사각형 에 매핑됨)
gp_face4 상단 버튼 4(Xbox 360 컨트롤러의 "Y" 와 PS 컨트롤러의 삼각형 에 매핑됨)
gp_shoulderl 왼쪽 어깨 버튼
gp_shoulderlb 왼쪽 어깨 방아쇠
gp_shoulderr 오른쪽 어깨 버튼
gp_shoulderrb 오른쪽 어깨 방아쇠
gp_select 선택 버튼(DS4 컨트롤러에서 터치패드를 아래로 누를 때 트리거됨)
gp_start 시작 버튼(PS4 컨트롤러의 "옵션" 버튼)
gp_stickl 왼쪽 스틱 누름(버튼으로)
gp_stickr 오른쪽 스틱 누름(버튼으로)
gp_padu 방향 패드 위로
gp_padd 방향 패드 아래로
gp_padl 방향 패드 왼쪽
gp_padr 방향 패드 오른쪽
gp_axislh 왼쪽 스틱 가로축(아날로그)
gp_axislv 왼쪽 스틱 세로 축(아날로그)
gp_axisrh 오른쪽 스틱 가로축(아날로그)
gp_axisrv 오른쪽 스틱 세로축(아날로그)

각 상수가 컨트롤러의 어떤 부분을 나타내는지 정확히 이해하기 위해 표준 Xinput 게임패드의 다음 이미지를 참조할 수 있습니다.

다음 목록은 플랫폼 간의 현재 호환성을 보여줍니다(이는 향후 업데이트로 변경될 예정임).

  • Windows 는 한 번에 최대 11개의 연결 장치를 허용하여 완벽하게 지원됩니다. Windows에서 처음 4개의 게임패드 슬롯(0 - 3)은 XInput dll을 사용하여 처리됩니다. 즉, XBox 컨트롤러만 100% 호환될 수 있고 다른 컨트롤러 유형의 경우 나머지 게임패드 슬롯(4 - 11).
  • macOS 는 한 번에 최대 4개의 연결 장치가 허용되며 이러한 장치는 Playstation3 또는 Xbox 360 유형만 사용할 수 있습니다. Mac 게임 옵션에서 " Mac AppStore용 빌드 " 옵션을 해제해야 패드 지원이 작동합니다.
  • Ubuntu 는 GamePad 입력도 지원하지만 Ubuntu 리포지토리에서 추가 라이브러리를 설치해야 할 수도 있습니다. 명령줄 터미널을 열고 다음을 입력하면 쉽게 이 작업을 수행할 수 있습니다. 그러면 조이스틱과 조이스틱 자체에 대한 GUI 지원이 설치됩니다.

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5 게임은 Safari 를 제외한 대부분의 주요 브라우저에서 게임패드를 지원합니다.
  • 게임패드 지원은 iCade 캐비닛을 통해 iOS 로도 확장됩니다. 왼쪽 축은 스틱 컨트롤러에 매핑되고(입력은 아날로그가 아닌 디지털이지만) 4개의 "페이스" 버튼은 캐비닛 전면 버튼에 매핑되고 4개의 어깨 버튼은 캐비닛 후면에 매핑됩니다.
  • Android 내보내기는 NYKO 컨트롤러 및 일반 Bluetooth 컨트롤러를 지원하지만 활성화된 경우에만 가능합니다. 즉, Android 게임 옵션 의 일반 섹션에서 iCade/Bluetooth 옵션을 선택해야 합니다. 완전히 작동하려면 API 레벨 12가 필요하며 GameMaker 는 실제로 연결되었는지 여부에 관계없이 장치가 페어링된 모든 Bluetooth 장치를 연결된 것으로 등록합니다. 따라서 "슬롯"을 할당하고 확인할 때 이를 고려해야 합니다.
  • PS4 에서 터치 패드 추적을 사용하려면 다음 기능을 사용해야 합니다. device_mouse_* 버튼(이를 위한 GML 시각적 작업이 없음).
