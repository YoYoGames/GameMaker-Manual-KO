입자 유형 생성

프로젝트에서 방향 및 색상 등과 같은 고유한 속성을 가진 여러 개의 서로 다른 입자 유형 을 가질 수 있습니다. 이 유형은 서로 다른 입자 유형 작업을 사용하여 설정됩니다. 그러나 먼저 이 작업을 사용하여 입자 유형 을 만들어야 합니다. 기본적으로 모든 입자 세부 정보를 보유하기 위해 소량의 메모리를 할당하므로 나중에 언제든지 입자를 사용할 수 있습니다. 입자 유형에 대한 고유한 ID 값을 보유하는 변수를 제공하면 이 입자와 관련된 모든 향후 작업에서 변수를 호출할 수 있습니다. 이 변수를 임시 로컬 변수로 플래그 지정하도록 선택할 수 있지만 이는 작업 스크립트 또는 이벤트가 끝날 때 삭제됨을 의미합니다. 즉, 입자 유형은 여전히 존재하지만 액세스할 수단이 없습니다. 따라서 이것을 사용할 때 주의를 기울여야 합니다(일반적으로 입자 유형은 임시 로컬 변수에 저장할 필요가 없지만 표면에 입자를 그리는 것과 같은 특정 항목의 경우 필요할 수 있습니다).

참고: 파티클 유형은 한 번 생성되면 게임이 진행되는 동안 시스템 메모리에 남아 있으므로 언제든지 다시 생성해야 하는 경우(예: 게임 다시 시작 작업을 호출하는 경우) 제거해야 합니다. 현재 입자 유형 이 아니면 메모리 누수가 발생합니다. 이것은 리소스가 메모리에 추가된 다음 "역참조"될 때 발생합니다. 즉, GameMaker 는 더 이상 해당 리소스와 상호 작용할 수 없습니다. 이로 인해 액세스할 수 없는 리소스로 인해 메모리가 막혀 게임에 문제가 발생할 수 있습니다. 더 이상 필요하지 않을 때 입자 유형을 제거하려면 Destroy Particle Type 작업을 사용합니다.

입자에 사용할 혼합 모드를 설정할 수도 있습니다. 기본적으로 "일반" 블렌드 모드로 그려지므로 입자 아래에 그려진 색상으로 색상이 수정되지 않습니다. 그러나 "블렌드" 옵션이 활성화되면 추가 블렌딩을 사용하여 그려집니다. 즉, 입자 아래에 그려진 내용을 기반으로 색상이 변경되어 그려집니다. 애디티브 블렌딩은 매우 밝고 "빛나는" 효과를 제공하며 폭발, 레이저, 화재 등에 이상적이며 일반 블렌딩은 비, 연기 및 파편 효과에 더 좋습니다. 아래 이미지는 애디티브 블렌딩을 끔(왼쪽)과 켬(오른쪽)에서 동일한 효과를 보여줍니다.
