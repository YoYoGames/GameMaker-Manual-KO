2차 입자 설정

이 작업을 통해 기본 입자와 함께 두 번째 입자 유형을 사용할 수 있습니다. 기본 입자 유형 ID 값(입자 유형 생성 작업을 사용하여 입자를 생성할 때 사용한 값과 같아야 함)을 지정한 다음 존재하는 각 게임 프레임에서 보조 입자 유형을 방출하도록 선택할 수 있습니다. /또는 각 1차 입자의 수명이 끝날 때. "Secondary"에 대해 다른 입자 ID 값을 제공한 다음 "Count"에서 생성할 숫자를 지정해야 합니다.

선택한 이벤트가 "단계별"인 경우 화면에 1차 입자가 존재하는 각 게임 프레임은 주어진 수의 2차 입자를 방출합니다. 예를 들어 "불꽃" 입자가 있는 로켓이 있을 수 있으며 각 불꽃 입자는 이차 "연기" 입자. 이벤트가 "사망 시"로 설정되어 있으면 이제 2차 입자는 1차 유형의 입자가 수명이 다했을 때만 방출됩니다. 예를 들어 "폭발" 입자가 "먼지 " 수명이 다한 입자.

1차 입자와 동일한 입자 유형 ID를 2차 입자에 사용할 수 없으며 이 방법을 사용하면 수천 개의 입자를 매우 빠르게 생성하기 쉽기 때문에 이 작업을 사용할 때 성능 문제가 매우 빨리 발생할 수 있다는 점에 유의하는 것이 중요합니다. 1차 및 2차 입자 유형 모두에서 생성되는 입자의 수명과 수를 제한하도록 주의하십시오. 또한 보조 입자를 모든 입자 유형에 할당할 수 있으므로 한 입자가 각 단계마다 다른 입자를 생성한 다음 이 입자가 수명이 다했을 때 다른 입자를 생성하는 "입자 사슬"을 생성할 수 있습니다., 이러한 방식으로 파티클을 연결할 때 성능이 저하되지 않도록 주의해야 합니다. 또한 단일 입자 유형에 대해 이 작업을 두 번 호출하고 사망 시 버스트뿐만 아니라 단계적으로 스트리밍하도록 할 수도 있습니다.
