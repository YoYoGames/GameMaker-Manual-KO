파티클 이미터 만들기

파티클을 보려면 먼저 파티클 시스템 을 만든 다음 일부 파티클 유형 을 정의한 다음 마지막으로 이 작업을 사용하여 파티클 이미터 를 생성해야 합니다. 이미터를 만들려면 이 작업을 호출하고 이미터를 이전에 만든 입자 시스템에 할당한 다음( 파티클 시스템 참조) 대상 변수에 추가 작업에 사용할 고유 ID 값을 저장할 수 있습니다. 이 변수를 임시 로컬 변수로 플래그 지정하도록 선택할 수 있지만 이는 작업 스크립트 또는 이벤트가 끝날 때 삭제됨을 의미합니다. 즉, 이미터는 여전히 존재하지만 액세스할 수단이 없습니다. 이것을 사용할 때 주의를 기울여야 합니다(일반적으로 이미터는 임시 로컬 변수에 저장할 필요가 없지만 표면에 입자를 그리는 것과 같은 특정 항목의 경우 필요할 수 있습니다).

중요한! 파티클 시스템이 영구적인 경우 Destroy Particle Emitter 를 사용하여 각 방의 끝에서 모든 이미터를 파괴하고 필요에 따라 다음 방에서 다시 생성하거나 액세스할 수 있도록 해당 ID 값을 전역 변수에 저장해야 합니다. 어떤 방에서든 언제든지, 또는 영구 객체에 의해 생성되도록 하여 객체가 지속되고 인스턴스 변수를 통해 액세스할 수 있도록 합니다. 또한 Restart Game 액션을 호출하는 경우 파티클 시스템을 미리 파괴하여 할당된 시스템과 이미터가 다시 시작하기 전에 메모리에서 제거되도록 해야 합니다. 위에서 설명한 대로 시스템 메모리에서 이미터를 제대로 제거하지 못하면 메모리 누수 가 발생합니다. 이것은 리소스가 메모리에 추가된 다음 "역참조"될 때 발생합니다. 즉, GameMaker 는 더 이상 해당 리소스와 상호 작용할 수 없습니다. 이로 인해 액세스할 수 없는 리소스로 인해 메모리가 막혀 게임에 문제가 발생할 수 있습니다.
