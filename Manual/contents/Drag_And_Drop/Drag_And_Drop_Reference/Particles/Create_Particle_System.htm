입자 시스템 만들기

이 작업은 파티클 유형 및 이미터를 만들고 사용할 수 있도록 파티클 시스템을 초기화합니다. 레이어 이름을 문자열로 지정합니다(예: "Particle Layer"(룸 편집기에 있어야 함)). 그런 다음 시스템에 영구 여부에 플래그를 지정합니다. 시스템이 Persistent로 플래그가 지정되면 생성된 후 모든 회의실에서 활성 상태를 유지하고 초기 레이어가 후속 회의실에 없으면 관리되는 레이어가 할당됩니다. 지속성에 플래그가 지정 되지 않은 경우 시스템은 시스템과 연결된 이미터와 마찬가지로 생성된 공간이 끝날 때 자동으로 소멸됩니다. 메모리에서 시스템을 수동으로 제거하려면 Destroy Particle System 작업을 사용해야 합니다.

또한 고유한 입자 시스템 ID 값을 보유할 대상 변수를 제공해야 합니다. 이 ID 값은 시스템의 속성을 설정하거나 파티클을 생성할 때 사용할 시스템을 GameMaker 에 알려주기 위해 다른 기능에서 사용됩니다. 이 변수를 임시 로컬 변수로 플래그 지정하도록 선택할 수 있지만 이는 작업 스크립트 또는 이벤트가 끝날 때 해당 변수가 삭제됨을 의미합니다. 즉, 시스템은 여전히 존재하지만 액세스할 수단이 없기 때문에 이것을 사용할 때는 주의를 기울여야 합니다(일반적으로 시스템은 임시 로컬 변수에 저장할 필요가 없지만 표면에 입자를 그리는 것과 같은 특정 항목의 경우 필요할 수 있습니다).

참고: 영구 입자 시스템은 한 번 생성되면 게임 기간 동안 시스템 메모리에 남아 있으므로 언제든지 다시 생성해야 하는 경우(예: Restart Game 작업을 호출하는 경우) 다음을 수행해야 합니다. 현재 시스템을 먼저 파괴하십시오. 그렇지 않으면 메모리 누수가 발생합니다. 리소스가 메모리에 추가된 다음 "역참조"되면 메모리 누수가 발생합니다. 즉, GameMaker 는 더 이상 해당 리소스와 상호 작용할 수 없습니다. 이로 인해 액세스할 수 없는 리소스로 인해 메모리가 막혀 게임에 문제가 발생할 수 있습니다. 더 이상 필요하지 않을 때 영구 입자 시스템을 파괴하려면 Destroy Particle System 작업을 사용합니다.

인수:

논쟁 설명
Layer 이 시스템의 파티클이 그려질 레이어
Persistent 현재 방이 끝난 후에도 파티클 시스템이 활성 상태를 유지해야 하는지 여부
Target 생성된 시스템의 고유한 ID 값을 담을 대상 변수

예시:

위의 작업 블록은 전역 변수를 만든 다음 새 입자 시스템을 만들고 고유한 ID 값을 전역 변수에 할당합니다. 생성된 입자 시스템은 지속적으로 표시되므로 현재 방이 종료된 후에도 활성 상태를 유지합니다.
