임시 변수 선언

이 작업은 에서 선언 된 이벤트 또는 스크립트 기간 동안에만 존재 하는 변수인 임시 변수 ( 로컬 변수 라고도 함)를 선언하고 초기 값을 지정하는 데 사용됩니다. 임시 변수는 단일 작업에만 필요한 값을 저장하는 데 매우 유용하며 개체가 한 번만 사용할 수 있는 변수로 "어수선해"지는 것을 방지합니다. 또한 복잡한 작업을 분해하거나 작업 범위를 변경할 때 다른 인스턴스에서 값을 설정하는 데 매우 유용합니다. 이는 임시로 선언된 변수의 범위가 특정 인스턴스로 지정되지 않고 대신 이벤트 또는 스크립트 자체로 범위가 지정되기 때문입니다. 따라서 임시 변수를 만든 다음 적용 대상... 작업을 사용하여 작업 범위를 변경하면 여전히 문제 없이 작업 범위를 변경하지 않고 임시 변수를 확인합니다.

"값"에 대해 이야기할 때 변수는 문자열, 포인터, 리소스 ID 또는 함수가 반환하거나 사용할 수 있는 기타 모든 것이 될 수 있으므로 숫자 값만 의미하는 것이 아닙니다.. 자세한 내용은 데이터 유형 섹션을 참조하세요.

많은 작업에는 작업의 반환 값을 저장할 새 임시 로컬 변수를 생성할 수 있는 "Temp" 확인란이 있으며 이 변수는 이후의 다른 변수와 마찬가지로 작업 스크립트 또는 이벤트에서 액세스할 수 있습니다.. 또한 임시 로컬 변수의 범위는 해당 변수가 생성된 스크립트 또는 이벤트이므로 적용 대상을 사용할 때 동일한 블록의 다른 인스턴스에서 사용할 수 있습니다.

더하기 아이콘을 클릭하여 동일한 작업에 추가 변수를 추가할 수도 있습니다. 작업 옆에 다른 임시 지역 변수 이름과 값을 제공합니다. 변수에 대한 고급 정보는 변수 및 변수 범위 섹션을 참조하십시오.
