새 함수 선언

이 작업을 사용하여 나중에 코드에서 언제든지 호출할 수 있는 새 사용자 정의 함수를 만들 수 있습니다. 사용자 정의 함수는 기본 제공 GML 시각적 작업 또는 GML 실행기 함수에서 다루지 않는 작업을 수행하기 위해 정의한 함수이며 범위 내에서 전역, 인스턴스 또는 로컬일 수 있습니다(범위에 대한 자세한 내용은 여기 참조 ); 또한 구조체 를 생성하는 생성자 로 사용할 수도 있습니다. 이 작업을 사용하여 전역 함수를 만드는 방법에 대해 자세히 알아보려면 작업 함수 만들기 섹션을 참조하세요.

이 작업을 사용하려면 이벤트나 스크립트에 추가하고 새 기능에 이름을 지정할 수 있습니다. 그런 다음 함수가 사용할 필수 인수/매개변수와 선택적 인수의 기본값을 제공합니다.

함수를 생성자(구조체 생성에 사용)로 사용하려는 경우 생성자 옵션을 확인하고 나중에 새로 만들기 작업을 사용하여 함수에서 구조체를 생성해야 합니다. 함수를 static 으로 플래그 지정할 수도 있습니다. 즉, 이 작업이 실행된 횟수에 관계없이 한 번만 정의됩니다. 마지막으로, 함수가 동일한 이벤트에서 로컬로만 사용되고 다른 곳에서는 사용되지 않는 경우 Temp 옵션을 활성화할 수 있습니다.

이름과 매개변수를 정의한 후에는 코드에서 함수가 호출될 때 실행하려는 작업을 아래 이미지와 같이 작업 옆에 드롭하여 첨부할 수 있습니다.

아래의 "확장된 예" 섹션에서 이 작업을 일반 함수와 생성자 함수로 사용하는 예를 볼 수 있습니다.

동작 구문:

인수:

논쟁 설명
Name 새 함수의 이름입니다.
Argument [+] 함수에 전달할 수 있는 다양한 인수와 선택 사항인 경우 기본값.
Constructor 이 함수가 생성자인지 여부입니다.
Static 이 함수가 정적인지 여부입니다.
Temp 함수가 임시인지 여부(현재 이벤트 내에서만 사용됨을 의미).

확장된 예

확장된 예 1

위의 작업 블록은 인스턴스 또는 스크립트의 Create Event 에 들어가고 다음과 같은 새 함수를 선언합니다.set_vec 세 가지 인수: _x, _y 그리고 spd. 그런 다음 이러한 인수는 함수가 호출될 때 발생하는 인스턴스의 방향과 속도를 설정하기 위해 함수에 포함된 작업에 사용됩니다. 나중에 다음과 같이 후속 이벤트에서 함수 호출 작업을 사용하여 함수를 호출합니다.

위 이미지에서, set_vec 마우스의 X/Y 위치를 가리키도록 인스턴스의 방향을 변경하고 속도를 5로 설정하는 함수가 호출됩니다.

확장된 예 2

위의 액션 블록은 인스턴스 또는 스크립트의 이벤트 생성 에 들어가고 다음과 같은 새로운 생성자 함수를 선언합니다.init_char_struct 네 가지 인수: _name, _location, _hp 그리고 _mana, 마지막 두 개는 선택 사항이며 기본값은 100입니다(즉, 함수가 호출될 때 해당 인수가 지정되지 않으면 기본값이 대신 사용됨). 그런 다음 이러한 인수는 함수가 생성하는 구조체 내에서 일부 변수를 생성하는 데 사용됩니다.

나중에 New 작업을 사용하여 이 함수를 호출하여 다음과 같이 구조체를 만듭니다.

함수가 호출되고 배열에 저장된 새 구조체를 반환합니다. char, 구조체는 변수로 채워집니다. name, location, hp 그리고mana, 함수 호출에서 해당 인수로 전달된 값으로 설정합니다.
