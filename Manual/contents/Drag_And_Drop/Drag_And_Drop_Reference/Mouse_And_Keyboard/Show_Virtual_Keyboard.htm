가상 키보드 표시

이 작업을 사용하여 게임을 실행하는 장치에 가상 키보드를 표시할 수 있습니다. 이 작업을 호출하면 처음 세 인수 각각에 대해 다음 상수 중 하나를 제공하라는 메시지가 표시됩니다.

  • 키보드 유형: 이 상수는 가상 키보드에서 사용할 수 있는 키 세트를 설정하는 데 사용됩니다. 사용 가능한 키보드 유형은 다음과 같습니다.
    키보드 유형 설명
    kbv_type_default 현재 시스템의 기본 키보드 유형입니다.
    kbv_type_ascii ASCII 전용 키보드.
    kbv_type_url URL 입력에 최적화된 일반 키보드입니다. 일반적으로 ".com" 또는 기타 도메인 키와 "/" 및 "." 키.
    kbv_type_email 이메일 입력에 최적화된 일반 키보드입니다. 일반적으로 "@" 및 "." 문자.
    kbv_type_numbers 일반적으로 숫자 패드로 표시되는 숫자 전용 키보드입니다.
    kbv_type_phone 폰패드 키보드입니다. 일반적으로 "*" 및 "#" 키가 있는 숫자 전용입니다.
    kbv_type_phone_name 전화번호와 이름을 동시에 입력할 수 있도록 최적화된 키보드입니다. 일반적으로 ASCII 키보드와 유사하지만 특수 문자 선택이 제한됩니다.
  • 리턴 타입: 가상 키보드의 리턴/액션 키에 표시되는 내용을 설정하는 상수입니다. 사용 가능한 반환 유형은 다음과 같습니다.
    반환 유형 설명
    kbv_returnkey_default 현재 시스템의 기본 리턴 키 제목입니다.
    kbv_returnkey_go 리턴 키 제목을 "이동"으로 설정합니다.
    kbv_returnkey_google 리턴 키 제목을 "Google"로 설정하거나 경우에 따라 일반 검색 아이콘으로 설정합니다.
    kbv_returnkey_join 리턴 키 제목을 "이동"으로 설정합니다.
    kbv_returnkey_next 리턴 키 제목을 "다음"으로 설정합니다.
    kbv_returnkey_route 리턴 키 제목을 "경로"로 설정합니다.
    kbv_returnkey_search 리턴 키 제목을 "검색"으로 설정하거나 경우에 따라 일반 검색 아이콘으로 설정합니다.
    kbv_returnkey_send 리턴 키 제목을 "보내기"로 설정합니다.
    kbv_returnkey_yahoo 리턴 키 제목을 "Yahoo"로 설정하거나 경우에 따라 일반 검색 아이콘으로 설정합니다.
    kbv_returnkey_done 리턴 키 제목을 "완료"로 설정합니다.
    kbv_returnkey_continue 리턴 키 제목을 "계속"으로 설정합니다.
    kbv_returnkey_emergency 리턴 키 제목을 "긴급 호출"로 설정합니다.
  • Autocapitalization type: 이 상수는 가상 키보드를 통해 입력된 단어를 자동 대문자화하는 방법을 결정하는 데 사용됩니다. 사용 가능한 자동 대문자 유형은 다음과 같습니다.
    자동 대문자 유형 설명
    kbv_autocapitalize_none 자동 대문자 사용이 비활성화되었습니다.
    kbv_autocapitalize_words 단어는 자동 대문자로 표시됩니다.
    kbv_autocapitalize_sentences 문장은 자동 대문자로 표시됩니다.
    kbv_autocapitalize_characters 모든 문자는 대문자로 표시됩니다.

마지막 인수는 예측 텍스트를 활성화/비활성화하는 것이며 다음과 같이 설정됩니다. true 그것을 허용하고, false 그렇지 않으면 허용되지만 장치에 대한 사용자의 기본 설정에 따라 사용된다는 의미는 아닙니다. 예측 텍스트 모드에서 가상 키보드는 정상적인 GameMaker 키 누름 이벤트를 생성하지 않습니다. 대신 마지막으로 누른 문자와 키보드 문자열 변수만 업데이트합니다. 이것은 키 누름을 감지하는 데 사용되는 내부 텍스트 필드의 변경이 실제 가상 키보드 키에서 비롯된 것인지 또는 텍스트 제안에서 비롯된 것인지 감지할 수 없기 때문입니다. 이러한 경우에는 다음을 읽고 싶을 것입니다. keyboard_string 모든 종류의 원시 키 입력을 읽는 것과는 반대로 입력합니다.

사용자는 플랫폼 OS에 따라 기능이 다른 다양한 키보드를 얻게 되며 대상별로 다음과 같은 주의 사항이 적용됩니다.

  • UWP: UWP 구현이 지원하지 않습니다. return_key_type, autocapitalization_type 또는 predictive_text_enabled. 이 매개변수를 함수에 전달하면 오류가 발생하지 않으며 키보드에 영향을 주지 않습니다.
  • Android / Amazon Fire: Android의 경우 키보드 유형 kbv_type_phone_name 지원되지 않으며 대신 ASCII 키보드를 표시하며 리턴 키 유형은 kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send 또는 kbv_returnkey_done. 해당 플랫폼에서 다른 반환 키 유형이 사용되는 경우 시스템은 기본적으로 아래 나열된 대체 반환 키를 사용합니다.
    사용할 수 없는 리턴 키 교체 키
    kbv_returnkey_google
    kbv_returnkey_yahoo    		kbv_returnkey_search
    kbv_returnkey_join
    kbv_returnkey_route
    kbv_returnkey_emergency    		kbv_returnkey_go
    kbv_returnkey_continuekbv_returnkey_next

  • AndroidTV / FireTV: ASCII 키보드에서는 사용자 지정 반환 키 유형이 지원 되지 않습니다. 기본 반환 키가 항상 표시됩니다.
  • tvOS: 예측 텍스트 및 자동 대문자는 tvOS에서 지원되지 않으며 시스템 이벤트 및 함수에 의해 반환되는 키보드 높이 keyboard_virtual_height() 전체 화면에 걸쳐 있고 키보드 직사각형 치수 기능이 tvOS에서 비활성화되어 있기 때문에 기본적으로 화면 높이가 설정됩니다. 또한 가상 키보드가 열려 있지 않으면 물리적(Bluetooth) 키보드 이벤트가 브로드캐스트되지 않습니다.

이 함수를 호출하면 시스템 비동기 이벤트 가 생성되며, 여기서 async_load  DS 맵 은 다음 키/값 쌍으로 채워집니다.

  • " event_type " - 가상 키보드 동작에 의해 트리거될 때 " 가상 키보드 상태 "가 됩니다.
  • " screen_height " - 가상 키보드의 높이(픽셀 단위). 키보드가 보이지 않으면 0이 됩니다.
  • " keyboard_status " - 키보드의 현재 상태로, 다음 문자열 중 하나로 반환됩니다.
    • "숨김"
    • "숨겨진"
    • "전시"
    • "보이는"

동작 구문:

인수:

논쟁 설명
keyboard_type 가상 키보드에서 사용할 수 있는 키 집합을 결정합니다.
return_key_type 가상 키보드의 리턴/액션 키에 표시되는 내용을 결정합니다.
autocapitalization_type 가상 키보드를 통해 입력한 단어의 자동 대문자 사용 여부를 결정합니다.
predictive_text_enabled 로 설정 true /false 예측 텍스트 입력을 활성화/비활성화합니다.

예시:

위의 동작 블록 코드는 마우스 다운 동작을 확인하고, 감지되면 OS 가상 키보드가 표시되는지 확인하기 위해 확인을 수행합니다. 표시되지 않으면 숫자 키패드 유형을 사용하여 표시되도록 설정되고 이미 표시되어 있으면 숨겨집니다.
