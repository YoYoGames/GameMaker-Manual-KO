GML 시각적 개체를 코드로 변경

GML Visual의 또 다른 기능은 GML 코드로 변환할 수 있다는 것입니다. 이런 식으로 액션을 사용할 때 배후에서 정확히 어떤 일이 일어나는지 볼 수 있습니다. 그러나 실제로 작업을 코드로 변경하기 전에 먼저 마우스 오른쪽 버튼을 사용하여 작업을 미리 볼 수 있습니다. 메뉴 옵션 실시간 미리보기:

새 창이 열립니다.

GML Visual 작업을 추가, 변경 또는 제거하면 GML 라이브 미리보기가 업데이트되어 "뒤에서" 생성되는 실제 코드를 보여줍니다. 라이브 미리 보기의 코드는 직접 편집할 수 없지만 섹션을 선택하고 복사하여 GML 스크립트 또는 코드 작업(예:)에 붙여넣을 수 있습니다.

미리보기를 본 후 GML Visual을 코드로 변환하려는 경우 마우스 오른쪽 버튼을 클릭하여 다시 수행할 수 있습니다. 작업이 있는 모든 이벤트 작업 공간에서 GML 코드로 변환을 선택합니다.

이 작업을 처음 수행하면 작업을 코드로 변환할 수 있지만 나중에 개별 작업으로 다시 변환할 수 없기 때문에 이것이 단방향 변환이라는 경고 메시지가 표시됩니다. 여기에서 "확인"을 클릭하면 변환이 수행됩니다.

결과 코드는 다음을 사용합니다. {} 개별 작업을 구분하고 코드 내에서 어떤 함수 또는 변수 선언과 관련된 작업을 명확하게 볼 수 있습니다. GML Visual이 더 복잡하면 코드도 복잡해지지만 동일한 일반 규칙이 적용되고 코드는 GML Visual을 작성한 것과 동일한 구조가 됩니다. 때때로 코드에는 사용될 특정 값을 저장하기 위해 추가 로컬 (임시) 변수가 있습니다. 예를 들면 다음과 같습니다.

다음 코드가 됩니다.

여기서 코드는 우선 로컬(임시) 변수를 생성하고 다음으로 설정합니다. false, 그런 다음 인스턴스가 있는지 확인하고 지역 변수를 함수 호출의 반환으로 설정합니다. 그런 다음 지역 변수가 다음인지 확인합니다. true 또는 false 그리고 만약 그렇다면 true 나머지 코드가 실행됩니다.

GML Visual을 사용하여 프로그래밍하는 방법을 배울 때 이것은 나중 단계에서 GML 사용으로 넘어가는 데 중요한 도구가 될 수 있지만 이것이 의무 사항은 아니며 여전히 GML Visual을 사용하여 훌륭한 게임을 만들 수 있습니다! 변환 프로세스는 단방향이지만 작업을 코드로 변환한 후 오른쪽 클릭을 사용하여 GML Visual을 다시 사용할 수 있습니다. 코드 편집기의 메뉴에서 " GML Visual로 변환"을 선택합니다. 이렇게 하면 이전에 생성된 코드가 코드 실행 작업 내에 배치되고 이전과 같이 GML Visual을 계속 사용할 수 있습니다.
