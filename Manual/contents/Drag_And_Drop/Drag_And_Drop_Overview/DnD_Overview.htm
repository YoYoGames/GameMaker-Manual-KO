GML 시각적 개요

GML Visual 은 실제로 코드를 입력하지 않고도 게임을 만드는 데 사용할 수 있는 시각적 스크립팅 도구입니다. GML Visual 을 사용할 때 코드를 프로그래밍하는 것이 아니라 텍스트가 아닌 작업 의 "블록"을 사용하여 시각적인 방식으로 코드를 만들고 표시할 뿐입니다. 작업은 방에 배치될 때 개체 의 인스턴스가 무언가를 수행하도록 함께 "연결"할 수 있는 간단한 표현식 또는 명령문 입니다. 예를 들어 이벤트 만들기 에서 인스턴스를 임의의 방향으로 이동하는 작업을 수행하고 충돌 이벤트 에서 다른 작업을 사용하여 충돌하는 물체에서 튕겨 나오게 할 수 있습니다(사용 가능한 다양한 이벤트 및 방법에 대한 자세한 내용은 함께 작업하려면 개체 이벤트 섹션 참조).

위 이미지는 편집할 작업 공간 에 열려 있는 일반적인 개체를 보여줍니다. "Event" 창과 "Code" 창이 연결되어 있으며 코드 창에서 GML Visual을 구성한 위치를 볼 수 있습니다. 이 창을 좀 더 자세히 살펴보고 어떤 옵션을 사용할 수 있는지 살펴보겠습니다.

아래에서 표시된 각 섹션에 대한 자세한 정보를 찾을 수 있습니다.

이벤트 탭

개체에 이벤트 를 추가하면 해당 이벤트의 내용을 표시하기 위해 코드 창에 새 탭이 열립니다. 여러 이벤트를 추가하고 여러 탭을 동시에 열어 원하는 대로 전환하고 편집할 수 있습니다. 또한 언제든지 탭을 닫아 혼란을 줄일 수 있으며(객체에서 이벤트를 다시 선택하면 다시 열림) 다음을 클릭할 수 있습니다. 탭을 창 밖으로 끌어 자체 코드 창으로 가져갑니다. 이 경우 개체나 코드 창을 닫아도 새 창이 닫히지 않습니다. 탭을 왼쪽이나 오른쪽으로 드래그하여 재정렬할 수도 있습니다.

작업 개요

이벤트에 작업 블록을 추가하면 진행 상황에 대한 일반 영어 개요로 "약식" 형식으로 표시됩니다. 개요 순서는 해당 이벤트에 대해 각 작업 블록이 실행되는 순서이며 목록의 항목을 클릭하여 작업 블록 보기가 편집을 위해 해당 위치로 확대되도록 할 수 있습니다.

작업 블록 작업 공간

이 작업 영역은 도구 상자 에서 작업을 끌어서 GML Visual 코드를 생성하는 곳입니다. 딸깍 하는 소리 을 클릭하고 이 영역으로 끌어서 작업 목록에 추가하고 편집합니다. 대부분 의 작업에는 설정할 수 있는 다양한 인수 (매개변수)가 있으며 대부분은 키워드, 인스턴스 변수 또는 GML 함수 도 사용합니다. 예를 들어 방향 설정 작업을 호출하고 GML 함수를 사용할 수 있습니다. random(val) 임의의 방향을 설정합니다.

코드에 작업 블록을 추가하면 이 체인이 생성되는 방법은 사용된 초기 블록에 따라 다르지만 시각적으로 "연결"되어 다양한 작업이 발생하는 순서를 나타냅니다. 예를 들어 If Variable 블록에는 두 개의 체인이 있습니다. 하나는 검사 중인 조건이 다음과 같은 경우 호출될 블록입니다. true 그리고 그 뒤의 다음 블록. 작업 블록을 코드로 구성하는 방법에 대한 자세한 내용은 아래 링크된 섹션을 참조하세요.

동작 블록 작업 공간에 있는 동안 마우스 가운데 버튼을 길게 누를 수 있습니다. 팬 및 마우스 휠 작업 목록을 위 또는 아래로 스크롤합니다. 잡고 있으면 마우스 휠을 이동하는 동안 보기가 왼쪽/오른쪽으로 스크롤되며 누르고 있으면 / 마우스 휠을 움직이면 보기가 확대 및 축소되어 작업과 함께 더 많은 작업 공간을 볼 수 있습니다. 클릭할 수 있습니다. 왼쪽에 있는 작업 목록의 모든 작업에서 해당 작업에 초점을 맞추고 확대/축소 및 위치를 재설정합니다. 또한 마우스를 블록의 가장자리 쪽으로 이동한 다음 커서가 변경되면 필요한 방향으로 드래그하여 작업 블록의 크기를 조정할 수 있습니다.

당신은 또한 오른쪽 클릭 할 수 있습니다 GML Visual 작업 공간의 아무 곳이나 클릭하여 RMB 메뉴를 불러옵니다. 자세한 내용은 여기 에 설명되어 있습니다.

공구 상자

도구 상자 는 유사하거나 관련된 작업 모음(라이브러리)을 보유하는 다른 섹션과 함께 모든 GML 시각적 작업이 저장되는 곳입니다. 각 작업 모음은 특정 테마 또는 공통 목적과 관련이 있으며 클릭하기만 하면 됩니다. 도구 상자에서 필요한 작업을 작업 블록 작업 공간으로 끌어 현재 이벤트에 추가합니다. 작업이 많기 때문에 일을 더 간단하게 하려면 상단의 "검색" 막대를 사용하여 특정 작업을 검색하거나 보이는 작업을 필터링할 수 있습니다(예를 들어 모든 그리기 작업을 가져오려면 "그리기" 입력) 를 클릭할 수도 있습니다. 라이브러리의 아이콘을 즐겨찾기 폴더로 끌어서 자주 사용하는 작업을 함께 보관할 수 있습니다. 이것은 당신이 계속해서 하게 될 특정한 일이 있는 경우에 특히 유용합니다. 작은 "bin" 아이콘을 클릭하여 즐겨찾기 폴더를 지울 수 있습니다.

사용 가능한 작업의 전체 목록과 해당 작업이 취하는 인수 및 연결 방법은 GML 시각적 참조 섹션을 참조하세요.

마우스 오른쪽 버튼을 사용할 수 있습니다 도구 상자의 모든 작업에서 다음 메뉴를 엽니다.

여기에는 다음과 같은 옵션이 있습니다.

  • < action > 사용: 현재 이벤트 작업 공간에 마우스 오른쪽 버튼을 클릭했을 때 마우스 아래에 있던 작업을 추가합니다.
  • 즐겨찾기 만들기 / 제거: 도구 상자 상단의 즐겨찾기 라이브러리에 작업을 추가하거나 제거합니다. 이 라이브러리는 정기적으로 사용하는 사용자 지정 작업 그룹을 생성하여 실제 라이브러리에서 찾는 데 필요한 시간을 줄일 수 있는 곳입니다.
  • 즐겨찾기 지우기: 이 항목을 선택하면 즐겨찾기 라이브러리에서 모든 작업이 제거됩니다.
  • 최근에 사용한 항목 지우기: 이 항목을 선택하면 최근에 사용한 항목 라이브러리에서 모든 작업이 제거됩니다.
  • 모두 축소: 이 옵션은 모든 GML Visual 라이브러리를 축소하여 작업 노드가 아닌 라이브러리 제목만 표시되도록 합니다. 각 라이브러리는 클릭하여 다시 확장할 수 있습니다.
