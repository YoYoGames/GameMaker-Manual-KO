마우스 오른쪽 버튼 메뉴 옵션

GML Visual 코드에서 작업할 때 마우스 오른쪽 버튼을 클릭할 수 있습니다. 작업, 작업 공간 또는 왼쪽의 관심 목록에 있는 텍스트에서 오른쪽 마우스 버튼 메뉴를 엽니다.

이 메뉴에는 다음과 같은 옵션이 있습니다.

  • 잘라 내기/ 복사 / 붙여넣기: 작업 공간에서 선택한 작업을 잘라내거나 복사하거나 붙여넣을 수 있습니다. 사용할 수 있습니다. + 작업 외부에서 드래그하거나 사용 / + 개별 작업에서 둘 이상의 작업을 선택하고 모든 작업에 명령을 적용합니다.
  • 주석 / 주석 제거: 이 옵션을 사용하면 GML Visual 코드 블록에서 주석을 추가 및 제거할 수 있습니다. 주석을 선택하면 선택한 GML Visual 블록의 오른쪽에 텍스트 편집기가 열리며 여기에 필요한 텍스트를 추가할 수 있습니다. 여기에는 변경 알림, 튜토리얼에 대한 유용한 주석 또는 지침이 포함됩니다. 팀원을 위해:
    주석을 추가한 후에는 주석을 클릭하고 끌어 수직 및 수평으로 배치할 수 있습니다(부모 노드의 오른쪽에만 있음). GML Visual 노드를 축소하면 주석도 축소되고 내용 요약이 표시됩니다. (이 경우 위치를 변경할 수 없습니다). 주석이 더 이상 필요하지 않은 경우 주석의 모든 텍스트를 삭제하거나 메뉴에서 주석 제거 옵션을 사용할 수 있습니다. 참고로 노드를 복사하면 주석도 함께 복사되며, 노드를 GML Visual에서 GML로 변환하면 주석이 보존되어 생성된 GML에 추가됩니다. 아래 이미지와 같이:
    위에서 언급했듯이 주석은 단일 노드에만 적용할 수 있지만 확장을 위해 줄 바꿈을 추가하여 여러 노드를 포함하도록 확장할 수 있습니다. 노드:
    주석이 추가되면 RMB 메뉴에서 " 주석 " 옵션을 다시 선택하거나 두 번 클릭할 수 있습니다. 편집을 활성화합니다.
  • 비활성화 / 활성화: 이 명령은 선택한 작업을 비활성화하거나 활성화하는 데 사용할 수 있습니다. 작업이 비활성화되면 작업 공간에 계속 표시되지만 게임을 컴파일할 때 실행되지 않습니다. 이러한 방식으로 테스트 및 디버깅을 위한 작업을 추가/제거할 수 있습니다. 아래 이미지는 작업이 비활성화된 일부 GML Visual 코드를 보여줍니다.
    "If Score" 및 "Go To Next Room" 작업이 비활성화된 것을 볼 수 있습니다. 시각적으로 회색으로 표시되어 게임을 테스트하거나 컴파일할 때 건너뜁니다.
  • 중단점 전환: 이 명령은 선택한 작업에 대해 중단점을 켜거나 끕니다. 중단점은 단순히 게임이 디버그 모드에서 실행될 때 디버거 가 중지되어야 하는 GML Visual 코드의 한 지점입니다. 할당된 중단점이 있는 작업은 아래 이미지와 같이 강조 표시됩니다("If Health" 작업이 중단점으로 설정됨).
  • 모든 중단점 삭제: 이 명령을 사용하면 GML Visual 코드에서 모든 중단점을 제거할 수 있습니다.
  • 모두 축소/모두 확장: GML 시각적 작업은 축소되거나 확장될 수 있습니다. 축소된 작업은 단순히 작업의 이름을 표시하지만 내부 값은 표시하지 않으므로 작업 공간에 사용 가능한 공간을 최대화합니다. 예를 들어:
    보시다시피, 작업은 훨씬 적은 공간을 차지하며 오른쪽에 제공된 짧은 설명에서 여전히 수행하는 작업을 볼 수 있습니다. "모두 확장" 옵션을 선택하여 작업 공간의 모든 노드에 대해 이 작업을 실행 취소할 수 있습니다.
  • 레이아웃 재설정: GML Visual 작업 공간을 재설정하고 확대/축소 수준을 100%로 설정하고 현재 선택한 작업에 초점을 맞추고 모든(및 모든) 접힌 작업도 확장합니다.
  • 실시간 미리보기: 이 옵션을 선택하면 GML Visual to GML(GameMaker Language) 코드 미리보기 창을 활성화하거나 비활성화합니다. 이것은 GML Visual을 코드로 변경 페이지에 자세히 설명되어 있습니다.
  • GML 로 변환: 현재 이벤트의 모든 작업을 GML(GameMaker Language) 코드로 변환합니다. GML을 생성된 GML Visual 작업으로 다시 변환할 수 없으므로 매우 주의하십시오. 이벤트를 코드에서 다시 GML Visual 이벤트로 변환할 수 있지만 모든 코드는 단순히 코드 실행 작업으로 래핑됩니다. 이것은 GML Visual을 코드로 변경 페이지에 자세히 설명되어 있습니다.
  • 창: 이 섹션은 다음 하위 메뉴를 엽니다.
    여기에서 사용할 수 있는 메뉴 항목은 다음과 같습니다.
    • Go To Object: 이벤트를 호출하는 객체로 작업 공간을 이동합니다.
    • 상속된 이벤트 열기: 이 옵션은 편집 중인 개체가 상위 개체의 자식인 경우에만 사용할 수 있으며 상위 개체의 편집 중인 이벤트를 열고 작업 공간을 이동합니다.
    • 단일 창 / 2 열 / 3 열: 이 옵션을 사용하여 현재 작업 공간을 1, 2 또는 3개의 다른 "창"으로 분할할 수 있습니다. 각 창은 스크립트나 이벤트를 끌어올 수 있는 별도의 작업 공간이므로 한 번에 2개 이상의 GML Visual 코드 블록을 편집하고 비교할 수 있습니다. 아래 이미지는 자체 작업 영역 창에 표시된 한 창에 있는 두 개의 스크립트를 보여줍니다.

다음 두 옵션은 GML Visual 노드에서 RMB 메뉴를 열 때만 사용할 수 있습니다.

  • 팬: 노드에서 이 옵션을 선택하면 작업 공간이 팬하여 해당 노드를 중앙으로 링합니다.
  • 도움말 표시...: 이 항목을 선택하면 선택한 노드와 관련된 페이지의 새 작업 공간에서 매뉴얼이 열립니다. 다음을 사용하여 동일한 결과를 얻을 수도 있습니다. 노드에서.
