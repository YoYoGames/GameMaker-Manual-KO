액션 블록 코드 구성하기

객체에 동작을 추가하려면 Action Toolbox 에서 사용할 수 있는 다양한 라이브러리의 Actions 를 사용하여 코드를 구성할 수 있습니다. 시작하려면 새 GML Visual 프로젝트 를 만든 다음 새 개체 를 만들어야 합니다(필요한 경우 개체에 스프라이트 를 할당할 수도 있음). 새 개체에서 이벤트 추가를 시작할 수 있고 events 에서 GML Visual 코드 작업을 추가할 수 있습니다.

새 이벤트를 추가하면 주어진 이벤트에 대한 탭이 있는 "코드" 창이 열리고(위 이미지 참조) 이제 오른쪽 도구 상자에서 원하는 작업을 작업 블록 창으로 드래그할 수 있습니다. 코드 창의. 이제 모든 작업을 추가할 수 있다는 것은 사실이지만 모든 작업이 작동하거나 프로젝트가 함께 컴파일된다는 의미는 아닙니다. 일부 작업은 작동하려면 최소한 하나의 변수 가 필요하지만 그리기 작업과 같은 다른 작업은 특정 이벤트에서 사용되는 경우에만 작동합니다. 어떤 것을 사용해야 하는지 어떻게 압니까? 글쎄, 일반적으로, 그것은 단순히 논리를 사용하는 문제입니다... 액션에 변수가 필요하고 아직 변수를 정의하지 않았다면 변수를 생성하기 위한 액션을 추가할 때까지 사용해서는 안 됩니다.

도구 상자에서 작업을 기본 작업 블록 작업 공간으로 끌면 작업을 설정하기 위해 채우고 변경할 수 있는 사용 가능한 인수 (매개변수) 가 표시되도록 확장됩니다. 아래 이미지에서 도구 상자 공통 라이브러리 의 변수 할당 작업을 작업 블록 작업 공간으로 드래그했습니다.

새로운 액션이 코드 창의 왼쪽에도 축약형으로 표시되는 것을 볼 수 있습니다. 작업 개요라고 하는 이 작업 목록을 클릭하면 편집을 위해 해당 작업으로 빠르게 이동할 수 있습니다. 필요한 경우 이벤트에 작업을 계속 추가할 수 있습니다. 각 새 작업은 이전 작업에 "연결"되어 구성 중인 GML Visual 코드의 흐름을 보여줍니다. 추가 작업을 삭제할 수 있는 영역은 초기 작업 아래에 강조 표시되며, 사용 중인 작업에 따라 체인에서 추가할 수 있는 위치를 표시하기 위해 다른 영역이 강조 표시됩니다.

작업 공간에 작업을 추가하면 위의 작업에 "연결"되어 한 작업이 다른 작업으로 이어지는 GML Visual 코드의 흐름을 볼 수 있으며 개요 창에서 해당 작업을 실행 순서대로 축약형으로 표시합니다.:

일부 작업은 코드를 기본 흐름에서 떨어진 별도의 체인에 배치합니다. 예를 들어 If Variable 은 기본 체인을 계속 진행하기 전에 올바른 조건이 충족되는 경우 발생해야 하는 작업의 하위 체인 을 생성합니다.

이와 같이 사이드 체인 블록을 가질 수 있는 작업을 사용할 때 작업에는 추가 작업을 삭제하기 위해 강조 표시된 두 영역이 있습니다.

  • 평소처럼 행동 아래에 하나
  • 첫 번째 작업의 조건이 true를 반환할 때 실행할 별도의 작업 블록을 만들려는 것을 나타내는 작업 오른쪽에 하나

간단히 클릭하여 체인 내에서 작업의 위치를 변경할 수 있습니다. 필요한 새 위치로 드래그하고 클릭하면 잠시 누른 상태에서 마우스를 움직이면 작업 블록 체인 내에서 해당 위치를 유지하면서 작업 공간 내에서 작업을 이동할 수 있습니다.

GML Visual 작업에 주석 을 추가하는 것도 가능하므로 여러 블록 옆에 메모를 남겨 두 블록이 하는 일을 설명할 수 있습니다(팀에서 작업할 때 특히 유용함).

이 기능에 대한 자세한 내용은 오른쪽 마우스 버튼 메뉴 페이지에서 확인할 수 있습니다.

이것이 GML Visual 코드 편집기를 사용하는 기본 사항이지만 아래 섹션에 더 중요한 세부 정보가 설명되어 있습니다.

이제 이 대상 변수는 Assign Variable ( Instance Variable 생성) 또는 Declare Temporary Variable ( Local Temporary Variable 생성)을 사용하여 이전에 선언되어 있어야 합니다. "Temp" 옵션을 선택하면 간단히 추가할 수 있습니다. 변수 이름과 작업은 해당 변수를 만들고 반환 값을 유지하도록 설정합니다(임시 로컬 변수 만들기). 이후에 사용된 모든 작업은 이제 임시 변수의 값에 액세스할 수 있지만 동일한 Event 또는 Script 내에서만 가능합니다. 임시 변수는 생성된 범위 내에서만 사용할 수 있습니다. 변수 및 변수 범위에 대한 자세한 내용은 여기 를 참조하십시오.

자동 완성

GML Visual으로 작업할 때 작업의 다른 입력 필드에 변수와 표현식을 추가해야 합니다. 그러나 이렇게 하면 다음과 같은 자동 완성 창이 자주 나타납니다.

이 팝업 창에는 모든 내장 GML(GameMaker Language) 변수, 상수 및 함수는 물론 입력한 것의 첫 글자가 포함된 리소스가 나열됩니다. 직접 입력하지 않고도 작업에 필요한 리소스나 변수를 빠르게 찾는 데 사용할 수 있습니다. 예를 들어 모든 방에 "rm_ " 그런 다음 입력하고 잠시 기다리면 모든 자산이 "로 시작하는 자동 완성 창이 표시됩니다.rm_ ". 자동 완성 창에 표시되는 모든 내장 변수는 대부분의 GML 함수와 마찬가지로 작업에서 변수 또는 표현식이 예상되는 모든 곳에서 사용할 수 있습니다.
