장치 관리자

장치 관리자 는 다음을 클릭하여 액세스합니다. 연필 단추에 대상 목록 내:

이 버튼을 클릭하면 컴파일 및 테스트하려는 대상 플랫폼에 대해 다른 장치를 구성할 수 있는 장치 관리자 가 열립니다(사용 가능한 플랫폼은 보유한 라이선스 에 따라 다르며 모든 플랫폼이 표시되지 않을 수 있음). 아래에서 각 대상 플랫폼에 대한 장치 구성에 대한 세부 정보를 찾을 수 있습니다.

기계적 인조 인간

여기에서 새 장치 추가 버튼을 클릭하여 새 Android 장치를 목록에 수동으로 추가하거나 장치 감지 버튼을 클릭하여 GameMaker 가 유선 연결을 통해 연결된 모든 장치를 자동으로 감지하도록 할 수 있습니다. Run AVD 라고 표시된 버튼을 클릭하여 Android 가상 장치 를 설정할 수도 있습니다. 그러면 새 AVD를 만들 수 있는 Android 가상 장치 관리자가 열립니다(자세한 내용은 여기 참조). 생성한 후에는 AVD 관리자에서 실행한 다음 GameMaker 에서 장치 감지 버튼을 클릭해야 합니다. 그러면 장치 목록에 자동으로 추가됩니다.

장치(물리적 또는 가상)가 추가되면 연결 테스트 버튼을 사용하여 연결을 확인하거나 장치 제거 버튼을 클릭하여 목록에서 다시 제거할 수 있습니다.


HTML5

HTML5 장치 관리자는 실행 시 타겟팅할 장치 목록에 다른 브라우저를 추가하기 위한 것입니다. 시작하려면 브라우저 감지 버튼을 클릭해야 합니다. 그러면 컴퓨터에 설치된 각 브라우저에 대한 장치가 생성됩니다. 브라우저가 목록에 없으면 새 브라우저 추가 버튼을 클릭한 다음 장치에 이름을 지정하고 GameMaker 가 브라우저 실행 파일의 경로를 가리키도록 할 수도 있습니다.

사용 가능한 추가 필드를 설정하여 브라우저에서 프로젝트를 실행하는 방법을 제어할 수 있으며 다음 옵션을 사용할 수 있습니다.

  • 브라우저 명령줄 인수: 여기에서 브라우저 경로에 추가할 명령줄 인수를 추가할 수 있습니다. 주요 인기 브라우저에 대한 다음 링크에서 이러한 목록을 찾을 수 있습니다.
    • 크롬(크롬)
    • 파이어폭스
    • 인터넷 익스플로러
  • 웹 서버 주소: 대상 웹 서버 의 주소입니다(기본값은 127.0.0.1). 다른 API의 사양에 따라 이를 변경해야 할 수도 있습니다. 예를 들어 Facebook에서는 이를 다음으로 설정해야 합니다. http://localhost (그런 다음 앱 콘솔에서 허용 목록에 추가).
  • 웹 서버 포트: 웹 서버의 포트 번호 를 설정합니다(기본값은 51264).
  • 허용되는 웹 서버 IP / CIDR IP 범위: 필요에 따라 허용되는 웹 서버, 호스트 이름 및 CIDR IP 범위를 설정합니다.

iOS

iOS 기기를 설정하려면 먼저 Mac 기기를 설정하고 연결을 테스트해야 합니다(자세한 내용은 아래 참조). 완료되면 창 상단의 드롭다운 메뉴에서 대상으로 지정하려는 Mac을 선택한 다음 장치 감지 버튼을 클릭하여 GameMaker 가 시뮬레이터를 포함하여 연결된 iOS 장치를 자동으로 찾도록 할 수 있습니다. 연결되면 장치 세부 정보가 이 창의 목록에 표시되며 장치 이름을 편집하여 보다 사용자 친화적으로 만들 수 있습니다(이 변경 사항은 장치가 대상 관리자에 표시되는 방식에만 영향을 미치며 다른 것은 없음).


tvOS

tvOS 기기를 설정하려면 먼저 Mac 기기를 설정하고 연결을 테스트해야 합니다(자세한 내용은 아래 참조). 완료되면 창 상단의 드롭다운 메뉴에서 대상으로 지정하려는 Mac을 선택한 다음 장치 감지 버튼을 클릭하여 GameMaker 가 시뮬레이터를 포함하여 연결된 모든 tvOS 장치를 자동으로 찾도록 할 수 있습니다. 연결되면 장치 세부 정보가 이 창의 목록에 표시되며 장치 이름을 편집하여 보다 사용자 친화적으로 만들 수 있습니다(이 변경 사항은 장치가 대상 관리자에 표시되는 방식에만 영향을 미치며 다른 것은 없음).


맥 OS

관리자에 Mac 장치를 추가하려면 먼저 새 장치 추가 라고 표시된 버튼을 클릭해야 장치가 추가되고 연결을 활성화하기 위해 채워야 하는 몇 가지 빈 필드가 생성됩니다.

먼저 대상 목록에서 식별할 수 있도록 장치의 이름 을 지정한 다음 Mac의 호스트 이름 (또는 IP)을 지정해야 합니다. 그런 다음 연결하려는 사용자의 사용자 이름 과 사용한 암호 를 제공해야 합니다. 원하는 경우 Mac의 설치 폴더 를 GameMaker 에서 프로젝트의 빌드 파일을 저장하려는 위치로 변경할 수도 있습니다.

모든 작업이 완료되면 연결 테스트 버튼을 클릭하여 GameMaker 가 Mac과 통신할 수 있는지 확인할 수 있습니다.

참고: iOS용으로 개발 중인 경우 먼저 이 작업을 수행하고 개발에 사용하려는 Mac에 대한 작동 연결을 설정해야 합니다.

참고: 대상 Mac에서 원격 로그인 을 활성화 해야 합니다. 기본 설정 -> 공유 로 이동하여 로그인할 수 있도록 하려는 사용자에 대해 원격 로그인 활성화 를 수행할 수 있습니다.

우분투(리눅스)

Ubuntu(Linux) 장치를 관리자에 추가하려면 먼저 새 장치 추가 버튼을 클릭해야 장치가 추가되고 연결을 활성화하기 위해 채워야 하는 빈 필드가 생성됩니다.

먼저 대상 목록에서 식별할 수 있도록 장치 이름 을 지정한 다음 Ubuntu 시스템(또는 VM)의 호스트 이름 (또는 IP)을 제공해야 합니다. 그런 다음 연결하려는 사용자의 사용자 이름 과 사용한 암호 를 제공해야 합니다. 또한 프로젝트를 설치하는 데 사용할 Ubuntu 시스템에 설치 폴더 를 제공해야 합니다. 모든 작업이 완료되면 연결 테스트 버튼을 클릭하여 GameMaker 가 Ubuntu 시스템과 통신할 수 있는지 확인할 수 있습니다.

GameMaker에 의해 생성된 최종 실행 파일은 x64 또는 ARM 이 될 수 있는 대상 장치의 아키텍처에 따라 달라집니다. 예를 들어 라즈베리파이 3, 4를 연결하면 32비트 ARM 실행 파일이 생성되고, 리눅스 PC에 연결하면 64비트 x64 실행이 된다.

윈도우 UWP

Windows 10 UWP 대상용 장치를 추가하려면 먼저 개발자 모드 를 활성화하고 장치에서 장치 포털 을 설정해야 합니다. 이렇게 하지 않으면 GameMaker 가 장치와 통신할 수 없으며 UWP 앱을 제대로 생성할 수 없습니다.

개발자 모드를 활성화하는 가장 쉬운 방법은 Windows 검색에 " 개발자 모드 "를 입력한 다음 표시될 " 설정 " 항목을 클릭하는 것입니다. 그러면 다음 창이 열립니다.

개발자 모드 옵션(Windows에서 일부 추가 패키지를 다운로드하고 설치해야 할 수 있음)을 선택한 다음 그 아래에 있는 옵션인 장치 포털 활성화 를 선택해야 합니다. 장치 포털은 GameMaker 가 Windows 10 PC와 통신할 수 있도록 하며 활성화되면 사용자 이름 및 암호 를 제공해야 합니다.

사용자 이름과 암호는 포털에 고유해야 하며 Microsoft 계정이나 다른 Windows 자격 증명에서 사용하는 것이 아니어야 합니다. 이 정보를 제공했으면 적용 이라고 표시된 버튼을 클릭한 다음 설정 관리자를 닫고 GameMaker 로 돌아갑니다.

장치 포털에 대한 자세한 내용은 이 Microsoft 문서: 데스크탑용 장치 포털을 참조하십시오.

이제 GameMaker 에서 장치 관리자 를 열고 사용할 장치를 설정할 수 있습니다.

Windows 10 UWP 호환 장치를 관리자에 추가하려면 먼저 새 장치 추가 버튼을 클릭해야 장치가 추가되고 연결을 활성화하기 위해 채워야 하는 몇 가지 빈 필드가 생성됩니다.

먼저 대상 목록에서 식별할 수 있도록 장치의 이름 을 지정한 다음 호스트 를 지정 해야 합니다. 이것은 네트워크에서 게임을 호스트할 Windows 10 장치의 IP 주소이며 접미사가 " :50443 "를 사용하여 포트 번호를 설정합니다. 그런 다음 장치 포털에서 설정한 사용자 이름 과 암호 를 추가합니다(이전에 언급한 대로). UWP Windows 10 데스크톱에는 PIN이 필요하지 않으므로 PIN을 비워 둘 수 있습니다.

다양한 UWP 출력(데스크톱, XboxOne)을 다루는 보다 구체적인 설정 세부정보는 YYG 헬프데스크: UWP 설정 을 참조하세요.

모든 다른 장치 창에는 클릭하면 목록에서 연결된 모든 장치가 제거되는 장치 지우기 옵션이 하단에 있습니다.

장치를 설정했으면 GameMaker IDE 상단의 대상 옵션에서 장치 중 하나에 연결할 수 있으며 게임을 테스트/빌드하면 프로젝트가 해당 장치로 푸시됩니다.
