마켓플레이스 기본 설정

Marketplace 기본 설정 은 자산을 업로드하거나 구매하기 위한 Marketplace 인증서 를 설정하는 데 사용됩니다. 인증서 양식은 이미 인증서를 생성한 경우를 위한 다음 섹션으로 구성됩니다(인증서 생성 방법에 대한 자세한 내용은 이 페이지 아래 참조).

  • 인증서 파일: 이 섹션은 사용자가 지정하는 곳입니다. *.pem 마켓플레이스가 자산 인증에 사용하는 파일입니다. 이전에 생성한 인증서가 있는 경우 여기에서 파일 브라우저 버튼을 클릭하여 포럼에 추가할 수 있지만 인증서가 없고 인증서를 만들려면 여기에서 정보를 생략하고 대신 다음을 클릭해야 합니다. 새 인증서 만들기 버튼(아래 설명).
  • 인증서 비밀번호: 위에서 선택한 파일에 대한 인증서 비밀번호를 입력해야 합니다. 암호는 처음 생성할 때 생성됩니다. *.pem 파일(아래 의 새 인증서 만들기 참조)은 생성된 인증서 파일에 되돌릴 수 없도록 연결됩니다.
  • 확인: 위의 필드를 채우고 나면 이 버튼을 클릭하여 암호와 파일 선택이 정확하고 함께 작동하는지 확인해야 합니다.

확인 대화 상자

여기에서 자산 브라우저의 리소스로 작업할 때 확인 대화 상자의 동작을 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 오프라인 콘텐츠 삭제에 대한 자동 응답: 웹 연결 없이 라이브러리에서 마켓플레이스 자산을 삭제하면 일반적으로 계속 진행할 것인지 묻는 경고가 표시됩니다. 이 옵션을 "확인"으로 설정하면 계속 진행하기 위해 "확인" 버튼을 클릭한 것처럼 메시지가 표시되지 않습니다. 기본값은 "메시지 표시"입니다.

새 인증서 만들기

이전에 Marketplace를 사용하기 위한 인증서 파일을 만들지 않았다면 이 양식 하단에 있는 Create New Certificate 버튼을 클릭해야 합니다. 그렇게 하면 다음 정보를 입력하라는 새 창이 열립니다.

  • 게시자 이름: 자산의 게시자 이름으로 사용할 이름입니다. 필요한 경우 Marketplace 대시보드 내에서 다른 게시자를 설정할 수 있으며 각 게시자에는 해당하는 *.pem 인증서 파일이 있어야 합니다.
  • 인증서 비밀번호: 생성할 인증서 파일에 대한 고유 비밀번호를 입력해야 합니다. 나중에 변경할 수 없으며 위 섹션의 인증서 파일에 대한 올바른 비밀번호가 없으면 기존 Marketplace 자산을 업데이트할 수 없으며 새 자산을 업로드할 수도 없으므로 선택에 주의하십시오.

해당 세부 정보를 사용하여 인증서 생성 버튼을 클릭하여 생성할 수 있습니다. *.pem 파일. 저장 위치와 파일 이름을 지정하라는 메시지가 표시되며 나중에 참조할 수 있도록 이를 기록해 두어야 합니다. 그렇게 하면 새 파일과 암호가 위에 나열된 양식 필드에 자동으로 추가되고 확인 버튼을 클릭하여 인증서가 올바르게 생성되었는지 테스트할 수 있습니다.

참고: 실제 설정의 스크린샷을 찍고 복사본을 만드는 것이 좋습니다. *.pem  파일. 나중에 참조하거나 어느 시점에서 GameMaker 를 다시 설치해야 하는 경우를 대비하여 둘 다 안전한 위치에 함께 저장해야 합니다. 업데이트된 패키지가 원본을 만들 때 사용한 것과 동일한 인증서를 사용하여 서명하지 않으면 이미 업로드된 자산을 업데이트할 수 없습니다.
