텍스트 편집기 기본 설정

텍스트 편집기 기본 설정 은 다양한 코드/스크립트 편집기의 모양과 느낌을 정의하는 데 사용됩니다. 기본 페이지에는 다음과 같은 옵션이 있습니다.

  • 구문 검사 지연(ms): 구문 검사기가 실행되기 전의 지연 시간(밀리초)입니다. 이 기본값은 2000ms입니다.
  • 코드 완성 지연(ms): 지연 시간(밀리초)
  • 전체 화면 편집기에서 스크립트 열기: 이 옵션을 선택하면 스크립트 리소스가 새 작업 공간에서 자동으로 열립니다. 선택하지 않으면 현재 작업 공간 내의 창에서 열립니다. 이 옵션은 기본적으로 선택 해제되어 있습니다.
  • 전체 화면 편집기에서 셰이더 열기: 이 옵션을 선택하면 셰이더 리소스가 새 작업 공간에서 자동으로 열립니다. 선택하지 않으면 현재 작업 공간 내의 창에서 열립니다. 이 옵션은 기본적으로 선택 해제되어 있습니다.
  • 전체 화면 편집기에서 개체 이벤트 열기: 이 옵션을 선택하면 개체 이벤트 코드 편집기가 새 작업 공간에서 자동으로 열립니다. 선택하지 않으면 개체 이벤트 편집기에 연결된 창에서 열립니다. 이 옵션은 기본적으로 선택 해제되어 있습니다.
  • 동일한 창에서 이벤트 스크립트를 탭으로 열기: 이벤트에 코드 또는 GML Visual을 추가할 때 코드 창은 이벤트 창에 연결되며 이 옵션을 선택하면 편집을 위해 열려 있는 모든 이벤트가 동일한 창에 추가됩니다. 탭으로 표시됩니다. 이 옵션을 선택 취소하면 각 이벤트가 고유한 연결 창으로 열립니다. 이 옵션은 기본적으로 선택되어 있습니다.
  • 링크된 스크립트를 동일한 창에서 탭으로 열기: 링크 된 스크립트가 여러 개 있고 이를 열면 기본적으로 동일한 창에서 탭으로 그룹화됩니다. 이 옵션의 선택을 취소하면 각자의 창/작업 공간에서 강제로 열리게 됩니다. 이 옵션은 기본적으로 선택되어 있습니다.
  • 포커스가 있는 보기로 텍스트 편집기 이동: 개체 이벤트에서 텍스트 편집기를 열 때 이 옵션을 선택하면 자동으로 작업 공간을 이동하여 편집기를 경계 내로 가져옵니다. 이 옵션은 기본적으로 꺼져 있습니다.
  • 기본 코드 창 너비: 열릴 때 코드 창의 기본 너비(픽셀)를 설정합니다. 기본값은 900입니다.
  • 기본 코드 창 높이: 열릴 때 코드 창의 기본 높이(픽셀)를 설정합니다. 기본값은 600입니다.
  • 최소 코드 창 너비: 코드 창에서 허용하는 최소 너비(픽셀)를 설정합니다. 기본값은 300입니다.
  • 최소 코드 창 높이: 코드 창에서 허용하는 최소 높이(픽셀)를 설정합니다. 기본값은 200입니다.

위의 옵션 외에도 아래에 설명된 코드 편집기와 시각적 편집기의 두 가지 하위 범주가 있습니다.

코드 편집기

색상 기본 설정 은 코드와 스크립트 편집기가 다른 텍스트를 표시하는 방법을 변경하는 데 사용됩니다. 각 항목에는 위의 예제 이미지와 같이 개별 텍스트가 어떻게 표시되어야 하는지 정확하게 정의할 수 있는 여러 하위 항목이 있습니다. 일반적으로 각각의 다른 옵션에 대해 색상, 글꼴, 크기 및 스타일을 설정할 수 있습니다. 더 아래로 코드 대괄호의 줄 선택 등과 같은 것에 대해 보다 일반적인 스타일을 설정할 수도 있습니다.

색상 기본 설정 아래에는 다양한 스타일 옵션 이 있습니다. 이러한 기본 설정은 다음 옵션을 사용할 수 있는 코드/스크립트 편집기의 동작을 설정하는 데 사용됩니다.

  • 탭 유지: 선택하면 탭 키 누름이 탭 문자로 유지됩니다. - ' \t ' - 편집기에서 선택하지 않으면 탭이 공백으로 변환됨을 의미합니다. 기본값은 꺼져 있습니다.
  • 탭 크기: "Tab"을 눌러 코드를 들여써야 하는 문자 공간의 수입니다. 기본값은 4입니다.
  • 공백 표시: "표시" . " 공백을 표시합니다(아래 이미지 참조). 기본값은 꺼짐입니다.
  • 줄 번호 표시: 줄 번호를 왼쪽 아래로 표시할지 여부입니다. 기본값은 켜짐입니다.
  • 탭 줄 표시: "표시" .... " 탭 공간을 표시합니다. 기본값은 꺼져 있습니다.
  • 색상 코딩 활성화: 편집기의 색상 코딩을 활성화하거나 비활성화합니다. 꺼져 있으면 코드에 스타일이 지정되지 않고 켜져 있으면 위의 색상 섹션에 제공된 서식을 사용하여 스타일이 지정됩니다. 기본값은 켜짐입니다.
  • 문서 줄 제한: 주어진 편집기 문서에 대한 최대 줄 수를 설정합니다. 기본값은 0(0 이하는 본질적으로 무한함)이며 다른 값은 허용되는 라인을 제한합니다.
  • 새 줄에 들여쓰기 일치: 코드에 대한 자동 들여쓰기를 활성화 또는 비활성화합니다. 기본값은 켜져 있으며 새 줄이 추가될 때 줄에 설정된 모든 탭을 유지합니다.
  • 중괄호 뒤에 자동 들여쓰기: 중괄호를 추가할 때 코드에 대한 자동 들여쓰기를 활성화 또는 비활성화합니다. {}. 기본값은 켜져 있으며 새 중괄호가 표시될 때 다음 코드 줄을 탭합니다. { 그리고 새로운 라인이 추가됩니다.
  • 코드 완성 활성화: 코드 완성 창을 활성화 또는 비활성화합니다. 코드를 작성하고 이것이 활성화되면 현재 입력 텍스트를 기반으로 사용할 수 있는 기능을 보여주는 팝업 창이 나타납니다. 기본값은 켜짐입니다.
  • 자동 추가 기능 대괄호: 대괄호 자동 추가를 활성화 또는 비활성화합니다. () 자동 완성을 사용할 때. 코드를 작성하고 이것이 활성화되면 IDE는 자동으로 두 개의 대괄호를 추가합니다. () 함수가 필요로 하는 것, 함수가 인수를 취하는 경우 그 안에 커서를 배치하거나 인수를 취하지 않는 경우 인수 뒤에 커서를 배치합니다. 이 기능은 자동 완성에서 기능을 선택한 경우에만 작동합니다. 기본값은 켜짐입니다.
  • Undo / Redo Stack Limit: 여기에서 Undo/Redo 스택 제한을 설정할 수 있습니다. 0 미만의 값은 본질적으로 무한(사용 가능한 메모리에 의해 제한됨)인 반면 양수 값은 스택을 해당 수로 제한하고 제한에 도달하면 스택의 맨 뒤에 있는 모든 작업을 제거합니다. 기본값은 -1입니다.
  • 선택 일치 강조 표시: 다른 곳에서 복제된 코드 섹션을 가볍게 강조 표시합니다. 따라서 예를 들어 함수를 선택하면 현재 코드 창에서 이 함수의 모든 인스턴스도 강조 표시됩니다. 기본값은 켜짐입니다.
  • 사용되지 않는 기능에 대한 색상 코딩/코드 완성: 사용되지 않는 코드 섹션을 강조 표시합니다. 기본값은 켜짐입니다.

비주얼 에디터

주석 기본 설정은 GML Visual 스크립트에 추가된 주석에 대한 다양한 색상 및 서식 옵션을 제어합니다. 이러한 옵션은 다음과 같습니다.

  • 색상: 창에 쓰여지는 텍스트의 기본 색상입니다(기본값 ARGB #FF53E270).
  • 글꼴 이름: 사용할 글꼴의 이름(사용 가능한 글꼴은 시스템에 설치된 글꼴에 따라 다름).
  • 스타일: 굵게 또는 기울임꼴 과 같이 사용할 글꼴 스타일입니다(모든 창에서 동일한 스타일을 허용하는 것은 아니며 사용된 글꼴에 따라 다름).
  • Size: 주어진 출력 창에 대해 글꼴을 설정할 크기입니다.
  • 앤티 앨리어싱: 여기에서 출력 창 글꼴 렌더링에 대한 앤티 앨리어싱 모드를 설정할 수 있습니다. 기본값 사용 을 선택하여 IDE가 결정하도록 선택하거나 이러한 옵션 중 하나를 선택하여 강제로 켜 거나 끌 수 있습니다.
