컴파일 환경 설정

컴파일 기본 설정은 테스트 또는 패키지 생성을 위해 게임을 컴파일할 때 출력과 관련된 특정 옵션과 컴파일 프로세스의 특정 영역을 다룹니다. 옵션은 다음과 같습니다.

  • 감지 시 자동으로 구문 오류 창 열기: 컴파일 시 구문 오류 알림을 활성화 또는 비활성화합니다. 기본적으로 꺼져 있습니다.
  • 컴파일하는 동안 컴파일 창 표시: 기본적으로 컴파일 독이 닫혀 있어도(컴파일을 위해 열림) 프로젝트를 컴파일할 때 컴파일 창이 표시됩니다. 선택을 취소하면 이 동작이 꺼지고 컴파일 창이 열려 있는 경우에만 표시됩니다.
  • 컴파일 출력에 컴파일러 명령 표시: 기본적으로 선택되어 있으며 컴파일러가 컴파일 창에 모든 관련 명령을 표시하도록 합니다. 선택을 취소하면 출력이 작아지고 정보가 줄어듭니다.
  • 자세한 컴파일러 출력 표시: 이 옵션을 선택하면 컴파일러에서 컴파일 프로세스와 관련된 추가 명령을 강제로 표시합니다. 일반적으로 이것은 디버깅할 때만 필요하며 문제가 있는 경우 지원팀에서 요청할 수 있습니다. 기본적으로 꺼져 있으며 권장되는 설정입니다.
  • 드라이브 경로에 subst 사용: 이 옵션을 선택하면 GameMaker 가 전체 파일 경로를 대신할 가상 드라이브를 생성하고 더 이상 필요하지 않을 때 다시 제거합니다. 이러한 가상 드라이브는 파일 경로가 OS에 대해 너무 긴 잠재적인 문제(특히 Android용으로 컴파일하거나 확장을 사용할 때)를 우회하므로 이를 활성화하는 것이 좋습니다. 그러나 GameMaker 가 이와 같은 가상 드라이브를 생성하는 데 문제가 있는 경우 옵션을 비활성화할 수 있습니다. 기본적으로 활성화되어 있습니다.
  • 빌드 완료 시 탐색기에 패키지 표시: 이 옵션을 선택하면 GameMaker 가 최종 실행 패키지가 생성된 위치에서 OS 탐색기 창을 엽니다. 기본적으로 켜져 있습니다.
  • Clear output window on compile start: 기본적으로 컴파일러 창은 각 빌드가 시작될 때 컴파일러 출력 창에서 모든 이전 메시지를 지웁니다. 그러나 선택을 취소하면 이전 컴파일러 로그가 유지됩니다. 이것은 기본적으로 켜져 있으며 큰 컴파일러 로그는 컴파일 시간을 늦출 수 있으므로 권장되는 구성입니다.
  • 패키징하기 전에 자동으로 실행 중지: 기본적으로 게임이 이미 실행 중이고 실행 파일을 만들려고 하면 오류 메시지가 표시되며 다음을 수행해야 합니다. 실행 파일을 만들기 전에 게임을 수동으로 중지하십시오. 이 옵션을 활성화하면 이 오류 메시지를 항상 건너뛰고 "실행 파일 생성"을 선택했을 때 게임이 실행 중인 경우 자동으로 게임을 중지할 수 있습니다. 기본적으로 꺼져 있습니다.
  • 최대 병렬 컴파일 작업: 이 값은 프로젝트 컴파일 전용으로 사용할 최대 CPU 코어 수를 설정합니다. 기본값은 8이며 일반적으로 이 값을 변경할 필요가 없지만 랩톱이나 저사양 컴퓨터를 사용하는 경우 이 값을 낮추는 것이 좋습니다. 기본값은 8로 설정됩니다.
