경로 기본 설정

경로 기본 설정은 GameMaker IDE의 요소와 필요한 외부 편집기에 필요한 다양한 파일 경로를 제어합니다. IDE에는 다음과 같은 옵션이 있습니다.

  • 임시 디렉토리: 모든 임시 폴더를 저장할 위치입니다. 기본적으로 Windows에서 이것은

    \Users\<Username>\AppData\Local\Temp\GameMaker

    macOS에서는 다음과 같습니다.

    /var/folders/<Path_Hash>/GameMaker

  • IDE 캐시 디렉토리: IDE 캐시를 저장할 위치입니다. 기본적으로 Windows에서 이것은

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    macOS에서는 다음과 같습니다.

    /Users/<Username>/.config/GameMaker/Cache

  • 자산 캐시 디렉토리: 프로젝트별 자산 캐시를 저장할 위치입니다. 기본적으로 Windows에서 이것은

    \Users\<Username>\AppData\Roaming\GameMaker\Cache

    macOS에서는 다음과 같습니다.

    /Users/<Username>/.config/GameMaker/Cache

  • "내 프로젝트" 위치: GameMaker 가 처음에 새 프로젝트를 생성할 위치입니다. 기본적으로 Windows에서 이것은

    \Users\<Username>\Documents\GameMaker

    그리고 macOS에서는

    /Users/<Username>/GameMaker

  • 닫을 때 임시 디렉터리 자동 삭제: 이 옵션을 선택하면 GameMaker 가 컴파일 등을 위해 프로젝트별로 생성한 Temp 폴더를 자동으로 삭제합니다... 이 설정은 기본적으로 켜져 있으며 선택을 취소하면 꺼집니다(즉, 나중에 임시 파일을 수동으로 제거해야 함).
  • 닫기 시 자산 캐시 자동 삭제: 이 옵션을 선택하면 GameMaker 를 종료할 때 자산 컴파일러 캐시 폴더가 제거됩니다. 기본적으로 꺼져 있으며 활성화하면 프로젝트를 로드하고 실행할 때마다 캐시를 다시 빌드해야 합니다(게임 크기에 따라 시간이 걸릴 수 있음).
  • 닫기 시 IDE 캐시 자동 삭제: 이 옵션을 선택하면 GameMaker 를 종료할 때 IDE 컴파일러 캐시 폴더가 제거됩니다. 이것은 기본적으로 꺼져 있으며 활성화하면 GameMaker 를 시작할 때마다 캐시를 다시 빌드해야 합니다.
  • 임시 폴더 삭제: 이 버튼을 클릭하면 프로젝트의 임시 폴더가 삭제됩니다.
  • 자산 캐시 삭제: 이 버튼을 클릭하면 프로젝트의 컴파일러 자산 캐시가 삭제됩니다.
  • IDE 캐시 삭제: 이 버튼을 클릭하면 IDE 캐시가 삭제됩니다.

외부 편집기에 대한 경로를 설정하기 위한 다음 옵션이 있습니다.

  • SWF 파일용 외부 편집기/뷰어 경로: SWF 형식 스프라이트로 작업하는 경우 이를 원하는 뷰어/편집기의 경로로 설정할 수 있으며 스프라이트 편집기에서 이미지 편집 버튼을 클릭하면 지정된 파일이 열립니다. 프로그램. 여기서 기본값은 경로가 없는 것입니다.
  • Spine 파일용 외부 편집기/뷰어 경로: Spine 형식 스프라이트로 작업하는 경우 이를 원하는 뷰어/편집기의 경로로 설정할 수 있으며 스프라이트 편집기에서 이미지 편집 버튼을 클릭하면 지정된 파일이 열립니다. 프로그램. 여기서 기본값은 경로가 없는 것입니다.
