파일 감시자 기본 설정

파일 감시자 는 외부 소프트웨어(예: 외부 편집기에서 이미지 편집) 또는 스크립트 자산 편집과 같은 내부 변경으로 인한 변경 사항에 대해 프로젝트와 관련된 파일을 감시하는 IDE 부분에 부여된 이름입니다. 일반적으로 이러한 설정은 편집할 필요가 없어야 하지만 SSD를 프로젝트 파일의 저장 드라이브로 사용하는 시스템과 같은 일부 시스템에서는 단순히 저장하는 것과 같은 작업을 수행할 때 파일 감시자가 변경 사항을 알려줄 수 있습니다. 프로젝트 또는 프로젝트 자산을 편집할 때마다 이러한 설정을 변경해야 할 수 있습니다.

사용 가능한 설정은 다음과 같습니다.

  • 파일 감시자 활성화: 파일 감시자 시스템을 활성화하거나 비활성화하는 데 사용할 수 있습니다. 이것이 비활성화되면 IDE는 외부 소스에 의해 변경된 파일을 더 이상 선택하지 않으므로 이 기능을 비활성화할 때 주의해야 합니다. 파일 감시자는 기본적으로 활성화되어 있습니다.
  • IDE에서 저장 후 MS에서 파일 감시자 허용 오차 설정: 이것은 파일을 저장하고 파일 감시자가 변경 사항을 확인하는 사이의 시간을 조정합니다. 프로젝트를 저장할 때 파일 감시자가 무언가 변경되었다는 잘못된 메시지가 나타나면 이 값을 조정합니다. 이 경우 메시지가 더 이상 나타나지 않을 때까지 지연 시간이 늘어납니다. 기본값은 2000ms이지만 범위는 1ms에서 최대 10,000ms입니다.
