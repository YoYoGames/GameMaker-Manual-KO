Windows 게임 옵션

이 섹션에서는 Windows 게임 프로젝트가 컴파일되는 방법을 제어하는 데 사용할 수 있는 다양한 옵션에 대해 설명합니다. 다른 섹션은 다음과 같습니다.

일반적인

Windows 게임의 일반 페이지는 게임이 설치되고 실행될 때 Windows가 표시하는 데 필요한 정보를 제공해야 하는 곳입니다. 이 정보는 제품 정보 섹션에 있으며 완전히 채워야 합니다.

이 창의 옵션 섹션에서는 게임 패키지를 설치할 위치를 "%localappdata% " 또는 "%appdata% " 폴더, 그리고 Copy exe 를 output 폴더 로 설정할지 여부. 일반적으로 이 옵션은 절대 필요하지 않지만 서로 및 게임과 통신하는 다양한 DLL이 있는 사용자를 위해 제공되며 활성화되면 유지됩니다. 테스트할 때 게임 EXE를 Temp 폴더에 넣지 않고 출력 폴더에 넣습니다.

다음으로 Windows 절전 여백 을 설정할 수 있습니다. 이 옵션은 특정 시스템에서 게임을 실행할 때 끊김 현상을 줄이는 것과 관련이 있습니다. 기본적으로 게임이 실내 속도보다 빠르게 실행되는 경우 GameMaker 는 남은 시간 동안 "절전"하지만 이 절전 모드는 매우 정확하지 않을 수 있으며 종종 필요한 것보다 더 오래 절전 모드를 종료하여 프레임이 더 오래 걸리게 되어 끊김 현상이 발생할 수 있습니다.. 이 문제를 해결하기 위해 우리는 더 짧은 시간 동안 잠을 잔 다음 더 정확하게 만들기 위해 나머지 시간 동안 빡빡한 루프에 앉아 있을 수 있습니다. 루프에 앉아 있을 때의 문제는 CPU 사용량을 증가시켜 결과적으로 CPU 온도가 올라가고 팬이 더 빨리 작동합니다. 기본적으로 이 값은 10으로 설정되어 있고 99.99%의 경우에는 문제가 없지만 저사양 머신이나 백그라운드 프로세스가 많이 실행되는 머신의 경우 이상적인 솔루션이 아닐 수 있습니다. 매우 고급 기계도 포함) 1 또는 15 또는 그 이상의 값이 필요할 수 있습니다. 이것은 매우 시스템 특정 구성이며 빌드 시스템에서 작동하는 것이 다른 사용자에게는 적합하지 않을 수 있으므로 의심스러운 경우 10으로 설정한 상태로 두십시오.

참고: 절전 여백 외에도 런타임에 스레드 스케줄러의 해상도 를 조정하여 Windows에서 성능과 전력 소비를 추가로 제어할 수 있습니다.

다음 옵션은 게임에 Steam을 활성화 하는 것입니다. 이 옵션을 활성화하면 Steamworks SDK가 설치되어 있고( 여기 참조) Steam 앱 ID 를 게임의 일반 옵션 에 추가했는지 확인하세요. Windows 및 Steam 버전의 IDE에서 GameMaker 를 실행하고 Steam용 게임을 개발하는 경우 Steam 응용 프로그램용 대체 실행 프로그램 사용 옵션도 확인해야 할 수 있습니다. 이 옵션이 하는 일은 대체 실행 프로그램을 사용하여 테스트할 때 게임을 실행하여 Steam에서 게임 고유의 애플리케이션 ID 대신 GameMaker 의 애플리케이션 ID로 플래그를 지정하는 것을 방지하는 것입니다. Steam용으로 개발할 때와 게임 실행에 문제가 있는 경우에만 이 기능을 활성화해야 합니다.

마지막 두 번째 옵션은 파일 시스템 샌드박스 비활성화 옵션입니다. 이 옵션을 선택하면 파일 저장 및 로드를 위한 GameMaker 샌드박싱이 비활성화되어 게임이 게임을 실행하는 시스템의 어디에서나 파일에 액세스할 수 있습니다. 이 옵션은 기본적으로 선택 해제되어 있습니다. 모든 시스템의 파일에 대해 GameMaker 샌드박스를 유지 관리하는 것이 좋으며 선택 시 OS 수준 권한에 따라 일부 저장/로드 영역을 계속 사용하지 못할 수 있기 때문입니다. 샌드박스에 대한 자세한 내용은 GameMaker 파일 시스템 페이지를 참조하십시오.

마지막으로 GameMaker 가 Windows 프로젝트용 x64 실행 파일을 생성하도록 하는 옵션이 있습니다. 이 옵션을 활성화하면 게임이 컴파일될 때 64비트 실행 파일이 생성되며 모든 새 프로젝트에서 이 옵션을 사용하는 것이 좋습니다. 이것은 32비트 러너가 단계적으로 중단됨에 따라 장기적으로 Windows 실행 파일의 기본(유일한) 형식이 될 것입니다. 활성화되면 프로젝트에서 사용 중인 확장 dll도 64비트여야 합니다(자세한 내용은 여기 참조).

제도법

그래픽 옵션은 게임에서 대상 Windows PC의 그래픽 카드를 사용하는 방법을 결정하기 위해 구성해야 하는 옵션입니다. 수정할 수 있는 다음 옵션이 포함되어 있습니다.

  • Display Cursor: 이 옵션을 선택하면 일반 창 커서가 표시되고 선택을 취소하면 게임 코드에서 커서를 생성하지 않는 한 커서가 표시되지 않습니다. 이것은 기본적으로 켜져 있습니다.
  • 전체 화면 시작: 이 옵션을 선택하면 게임이 전체 화면 모드로 시작되며 그렇지 않으면 창으로 시작됩니다. 기본적으로 꺼져 있습니다.
  • 전체 화면 전환 허용: 이 옵션을 선택하면 사용자는 표준 Windows 바로 가기를 사용하여 전체 화면에서 창으로 전환했다가 다시 전환할 수 있습니다. 기본적으로 꺼져 있습니다.
  • 픽셀 간 색상 보간: 기본적으로 픽셀을 "매끄럽게"하는 선형 보간 을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야 하지만 멋진 알파 블렌드와 부드러운 가장자리 그래픽이 있는 경우 켜진 상태로 두는 것이 좋습니다. 기본적으로 켜져 있습니다.
  • 찢어지지 않도록 동기화 사용: 이것은 v-sync 를 켜거나 끕니다. 방 속도가 120인 게임이 있고 플레이어에 새로 고침 빈도가 60인 모니터가 있는 경우 이 옵션을 켜면 게임 속도도 60으로 잠깁니다. 기본적으로 꺼져 있습니다.
  • 창 크기 조정 허용: 이 옵션을 선택하면 사용자가 게임 창의 크기를 변경할 수 있습니다. 이 옵션은 기본적으로 꺼져 있습니다.
  • 테두리 없는 창: 이 옵션을 선택하면 게임이 테두리 없는 창으로 실행되고 게임 제목은 물론 일반 최소화, 최대화 및 닫기 버튼이 제거됩니다. 기본적으로 꺼져 있습니다.
  • 크기 조정: 여기에서 가로 세로 비율을 유지하거나(4:3 공간은 16:9에서 "레터 박스"로 표시됨) 전체 크기 조정(전체 화면에 맞도록 이미지 확장)을 선택할 수 있습니다.

    경고! 응용 프로그램 화면을 끄면 다시 켤 때까지 Windows 게임 옵션에 설정된 모든 크기 조정 옵션이 비활성화됩니다. 자세한 내용은 애플리케이션 표면을 참조하세요.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본(그리고 가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 이 플랫폼에 대한 텍스처 페이지를 생성한 다음 창을 열어 어떻게 보이는지 확인할 수 있는 미리보기 라고 표시된 버튼도 있습니다. 이것은 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요 이상으로 크거나 작은 것을 방지하려는 경우 매우 유용할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조하십시오.

참고: 텍스처 페이지의 크기가 클수록 사양이 낮은 PC에서는 게임의 호환성이 떨어집니다.

이미지

이미지 섹션은 게임에 필요한 이미지를 제공하는 곳입니다. Windows의 경우 아이콘 파일(다음 위치에 있어야 함)을 제공해야 합니다. .ico 형식) 및 시작 화면( .png, .bmp, .jpg 또는 .gif 체재). 스플래시 화면은 게임이 로드되는 동안 표시되며 Use Splash Screen 옵션(기본적으로 꺼져 있음)을 선택하여 활성화 또는 비활성화할 수 있습니다.

GameMaker 에는 게임이 컴파일되는 다양한 대상 플랫폼에 필요한 모든 이미지를 자동으로 생성하는 데 사용할 수 있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 생성된 이미지를 수정하여 필요한 이미지인지 확인해야 합니다.

설치 프로그램

Windows 게임은 설치 프로그램 으로 생성할 수 있으며 여기에서 GameMaker 가 게임을 생성할 때 사용할 다양한 설정과 그래픽을 정의할 수 있습니다. 우선 설치 프로그램 자체의 모양을 정의하는 그래픽 옵션이 있습니다.

  • 완료됨: 설치 프로그램의 "완료됨" 페이지에 표시되는 그래픽입니다. 에 있어야 합니다. *.bmp 형식을 지정하고 164x314픽셀을 측정합니다.
  • 헤더: 설치 프로그램이 실행되는 동안 나타나는 그래픽입니다. 에 있어야 합니다. *.bmp 형식을 지정하고 150x57px로 측정합니다.

특정 라이선스 계약 파일( GameMaker 에 예제가 제공됨)과 사용자 지정 NSI 스크립트 파일 (자세한 내용은 여기 )을 가리키거나 필요한 경우 적절한 버튼을 클릭하여 GameMaker 와 함께 제공되는 기본 파일로 돌아갈 수도 있습니다.
