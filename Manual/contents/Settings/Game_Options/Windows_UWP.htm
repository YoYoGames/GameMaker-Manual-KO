Windows UWP 게임 옵션

이 섹션에서는 UWP(유니버설 Windows 플랫폼) 게임 프로젝트가 컴파일되는 방법을 제어하는 데 사용할 수 있는 다양한 옵션에 대해 설명합니다. 다른 섹션은 다음과 같습니다.

일반적인

UWP 옵션의 일반 섹션에서는 제품 정보 를 설정할 수 있습니다.

  • 표시 이름: 사용자에게 표시되는 앱의 친숙한 이름을 지정합니다(이 문자열은 현지화 가능).
  • 패키지 이름: 이것은 당신이 생성한 패키지의 이름이며 게임을 위해 생성한 개발자 키에 사용된 것과 일치해야 합니다(아래 패키징 섹션 참조).
  • 패키지 표시 이름: 패키지의 "짧은 이름"이며 이상적으로는 표시 이름과 동일해야 합니다.
  • 게시자 표시 이름: 게임 게시자로 연결하려는 회사 또는 사람의 이름입니다.
  • 버전: 게임의 버전 번호입니다.

이를 설정한 후 최고의 사용자 경험을 위해 게임을 실행할 수 있는 다양한 방향 을 선택할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • 풍경
  • 초상화
  • 가로 뒤집힌
  • 세로 뒤집힌

기본적으로 이러한 모든 항목이 선택되며 필요하지 않은 항목은 선택 취소해야 합니다.

방향 옵션을 설정한 후 다음 옵션에서 UWP 앱에 필요한 다양한 권한 을 알릴 수 있습니다.

  • 인터넷 권한 활성화: 게임이 인터넷에 연결되어 있음을 표시합니다. 기본적으로 꺼져 있습니다.
  • 마이크 권한 활성화: 게임에 장치의 마이크에 대한 액세스 권한이 있는 것으로 플래그를 지정합니다. 기본적으로 꺼져 있습니다.
  • IAP 샌드박스 활성화: 샌드박스가 적용된 인앱 구매를 사용하는 것으로 게임에 플래그를 지정합니다. 기본적으로 꺼져 있습니다.

장치 제품군

여기에서 실행 파일을 만들 때 Windows UWP를 내보낼 대상 장치 제품군 을 선택할 수 있습니다. 기본적으로 "범용 장치 제품군 지원" 옵션이 선택되어 있습니다. 즉, 앱이 모든 장치와 제품군을 대상으로 합니다. 특정 장치 제품군만 지원하도록 앱을 제한하려면 범용 옵션을 선택 취소 하고 적절한 장치 제품군을 선택해야 합니다. 또한 범용 장치 제품군을 지원하지만 다른 장치 제품군을 특정 플랫폼 버전으로만 제한할 수도 있습니다. 이 경우 범용 장치 제품군을 선택한 상태로 유지하고 다른 플랫폼 버전을 지원하려는 다른 장치 제품군도 확인합니다. 해당하는 텍스트 상자에 해당 제품군의 버전 세부 정보를 입력합니다. 이 논리가 작동하는 방식에 대해서는 다음 Microsoft 지원 페이지를 참조하십시오.

  • 대상 장치 제품군: 예

일반적으로 대상 및 최소 플랫폼 버전을 건드리지 않고 기본값으로 유지해야 하지만 빌드 문제가 있는 경우(특히 기본적으로 설정된 Windows SDK를 설치할 수 없는 경우) 설치한 것으로 변경해야 합니다. 설정할 값이 확실하지 않은 경우 Visual Studio에서 새 UWP 프로젝트를 만들고 값을 설정하는 항목을 확인하세요.

제도법

그래픽 옵션은 게임에서 대상 UWP 장치의 그래픽 카드를 사용하는 방법을 결정하기 위해 구성해야 하는 옵션입니다. 수정할 수 있는 다음 옵션이 포함되어 있습니다.

  • 픽셀 간 색상 보간: 기본적으로 픽셀을 "매끄럽게"하는 선형 보간 을 켭니다. 선명한 픽셀 그래픽의 경우 꺼져 있어야 하지만 멋진 알파 블렌드와 부드러운 가장자리 그래픽이 있는 경우 켜진 상태로 두는 것이 좋습니다. 기본적으로 꺼져 있습니다.
  • 커서 표시: 이 옵션이 활성화되면 기본 OS 커서가 게임에 표시됩니다. 비활성화된 경우 마우스가 게임 창 위에 있는 동안 게임 프로젝트의 일부로 생성하지 않는 한 커서가 보이지 않습니다(기본적으로 켜져 있음).
  • 전체 화면 시작: 이 옵션을 선택하면 게임이 전체 화면 모드로 시작되며 그렇지 않으면 창으로 시작됩니다. 기본적으로 꺼져 있습니다.
  • 전체 화면 전환 허용: 이 옵션을 선택하면 사용자는 표준 Ubuntu OS 바로 가기를 사용하여 전체 화면에서 창으로 전환했다가 다시 전환할 수 있습니다. 기본적으로 꺼져 있습니다.
  • 크기 조정: 여기에서 가로 세로 비율을 유지하거나(4:3 공간은 16:9에서 "레터 박스"로 표시됨) 전체 크기 조정(전체 화면에 맞게 이미지 확장)을 선택할 수 있습니다.

    경고! 응용 프로그램 표면을 끄면 다시 켜질 때까지 Windows 게임 옵션에 설정된 모든 크기 조정 옵션이 비활성화됩니다. 자세한 내용은 애플리케이션 표면을 참조하세요.

마지막으로 텍스처 페이지 의 크기를 설정하는 옵션이 있습니다. 기본(그리고 가장 호환되는) 크기는 2048x2048이지만 256x256에서 최대 8192x8192 사이에서 선택할 수 있습니다. 이 플랫폼에 대한 텍스처 페이지를 생성한 다음 창을 열어 어떻게 보이는지 확인할 수 있는 미리보기 라고 표시된 버튼도 있습니다. 이것은 텍스처 페이지가 어떻게 구성되어 있는지 확인하고 텍스처 페이지가 필요한 것보다 크거나 작은 것을 방지하려는 경우 매우 유용할 수 있습니다. 텍스처 페이지에 대한 자세한 내용은 여기 를 참조하십시오.

참고: 텍스처 페이지의 크기가 클수록 사양이 낮은 PC에서는 게임의 호환성이 떨어집니다.

이미지

이미지 섹션의 첫 번째 부분에는 UWP가 실행될 수 있는 다양한 장치에서 게임이 사용할 다양한 타일 을 설정할 수 있는 옵션이 있습니다. 다음을 설정할 수 있습니다. *.png 형식 파일):

  • 작은 타일 이미지는 검색창 및 UI의 다른 영역에서 앱 이름 옆에 나타나는 로고 이미지입니다. 44 x 44픽셀이어야 합니다.
  • 중간 타일 이미지는 71x71 픽셀 이미지여야 합니다.
  • 다음 타일 이미지는 Store 로고 타일 이며 50 x 50픽셀이어야 합니다.
  • 와이드 이미지 타일 은 타일이 와이드 형식일 때 나타나는 타일이며 310 x 150픽셀이어야 합니다. 이 이미지가 제공되지 않으면 타일은 정사각형 형식으로만 표시될 수 있으며 넓은 템플릿 유형에 기반한 알림을 수락할 수 없습니다. 타일 형식을 사용자가 결정할 수 있으므로 넓은 이미지 타일을 포함하는 것이 좋습니다. 또한 와이드 이미지 타일이 제공되는 경우 타일이 처음에는 와이드 형식으로 나타날 수 있습니다.
  • 큰 이미지 타일 은 큰 라이브 타일 형식이며 310 x 310픽셀이어야 합니다. 이것은 데스크탑 OS에서만 사용됩니다.

다음 섹션은 게임 로고 를 정의하는 것입니다. 이것은 본질적으로 게임이 UWP 생태계 내에서 사용할 아이콘이며 이러한 이미지가 투명할 수 있다는 점은 주목할 가치가 있습니다. .png 이 경우 이 섹션에 지정된 배경색 위에 그려집니다. 이 배경색은 앱 소유 대화 상자의 버튼 색상과 디바이스 스토어의 앱 설명 페이지에도 적용됩니다.

로고를 설정한 후 게임의 시작 화면 을 설정할 수 있습니다. 이것은 게임을 시작할 때 표시되는 화면이며 620 x 300픽셀보다 클 수 없으며 로고처럼 투명할 수 있습니다. .png 이 경우 선택한 배경색 이 파일에 사용됩니다.

GameMaker 에는 게임이 컴파일되는 다양한 대상 플랫폼에 필요한 모든 이미지를 자동으로 생성하는 데 사용할 수 있는 프로젝트 이미지 생성기 도구가 있습니다. 이 도구를 사용하는 경우 생성된 이미지를 수정하여 필요한 이미지인지 확인해야 합니다.

포장

GameMaker 는 Windows UWP 개발을 위한 기본 개발자 키 를 제공하지만 직접 생성하여 여기에서 가리키는 것이 좋습니다. 이렇게 하려면 Visual Studio에서 빈 프로젝트를 만든 다음 자동으로 생성된 *.pfx 빈 프로젝트에 포함될 파일(개발자 키임). 그런 다음 링크해야 합니다. *.pfx 여기 파일.

파일에 연결한 후 관련 설치 버튼을 클릭한 다음 열리는 명령 프롬프트의 지침을 따라야 합니다. 이렇게 하지 않으면 실행 /디버그 버튼 사용(또는 F5 키 누르기)이 실패하고 UWP 플랫폼에서 게임을 테스트할 수 없습니다. *.pfx 아직 등록되지 않았습니다.)

엑스 박스 라이브

UWP 게임 옵션의 Xbox Live 섹션에서는 Xbox One 본체에서 사용할 UWP 프로젝트를 활성화할 수 있습니다. 여기에서 Xbox용 프로젝트를 활성화하면 프로젝트가 Xbox One에서 실행될 때 다음을 수행할 수 있습니다.

  • 앱 내에서 Xbox Live에 로그인
  • 프로젝트에서 Xbox Live 사용자 ID 사용(예: 게이머태그 또는 게이머 사진 표시)

그러나 이러한 작업을 수행하기 전에 Xbox Live 활성화 확인란을 클릭했는지 확인해야 하며, Microsoft Creators Program 에 속한 경우 Creators Program App 옵션도 확인해야 합니다. 크리에이터 프로그램의 일부인 경우 타이틀 ID 와 서비스 구성 ID 를 추가해야 합니다(자세한 내용은 Microsoft 에서 찾을 수 있음).

현재 프로젝트에서 Xbox Live 기능을 테스트하려면 UWP로 실행하는 대신 실행 파일 만들기 를 사용해야 합니다. Xbox 출력은 패키징 정보(위 참조)의 일부로 제공하는 Windows 스토어 서명 인증서를 사용해야 합니다. GameMaker 와 함께 제공되는 임시 것입니다. 이는 Run을 누를 때 사용됩니다.

XBox Live를 활성화하면 프로젝트에서 XBox Live 기능을 사용할 수 있지만 가장 기본적인 기능을 사용하려면 Creator 프로그램 에 참여해야 하고 고급 기능을 사용하려면 ID@ 가 필요합니다. 엑스박스 계정..
