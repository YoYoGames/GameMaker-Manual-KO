구성

크로스 플랫폼 개발 요구와 다양한 클라이언트, 장치 또는 플랫폼에 대해 하나의 제품을 사용 가능하고 사용자 정의해야 하는 필요성으로 인해 GameMaker 에는 작업을 더 간단하게 만드는 구성 이 포함되어 있습니다. 이 시스템을 사용하면 서로 다른 시작 화면, 아이콘 및 포함된 파일 사이를 빠르게 전환하고 대상 플랫폼 또는 클라이언트와 관련된 항목만 언제든지 동일한 기본 코드에서 테스트하거나 내보낼 수 있습니다.

클라이언트 및 플랫폼의 다양한 요구를 처리할 수 있도록 자산 브라우저 의 추가 메뉴에서 액세스할 수 있는 구성 편집기 를 사용할 수 있습니다.

GameMaker 에서 만든 모든 게임은 "기본" 구성으로 시작하며, 처음 열 때 편집기에 나열됩니다.

구성을 설정하려면 먼저 프로젝트에 대해 구성 가능한 모든 자산을 여기에 통합하여 이 "기본" 구성을 준비하는 것이 좋습니다. 이렇게 하려면 다음 옵션 목록을 살펴보고 기본적으로 필요한 옵션으로 설정하기만 하면 됩니다.

  • 확장 옵션: 다양한 "복사 대상..." 필드를 구성에 필요한 플랫폼으로 설정합니다. 여기에서 생성된 모든 "프록시" 파일에는 구성에 연결된 "대상" 플랫폼도 있습니다.
  • 스프라이트 편집기: 스프라이트를 추가할 텍스처 그룹을 설정합니다.
  • Tileset Editor: 타일셋 을 추가할 텍스처 그룹을 설정합니다.
  • 글꼴 편집기: 글꼴을 추가할 텍스처 그룹을 설정합니다.
  • 텍스처 그룹 관리자: 여기에서 텍스처 그룹이 속한 상위 그룹을 포함하여 구성별 기반에 대한 속성을 설정할 수 있습니다.
  • 오디오 그룹 관리자: 다른 오디오 그룹에 할당된 사운드는 여기에서 구성할 수 있습니다.
  • 포함된 파일: "복사 대상..." 필드를 구성에 필요한 플랫폼으로 설정합니다.

기본 구성을 설정했으면 마우스 오른쪽 버튼을 클릭하여 구성 편집기에 새 구성을 추가할 수 있습니다. 구성 폴더에서 구성 추가 를 선택하거나 창 하단의 " 만들기 " 버튼을 클릭합니다. 새 구성은 기본 구성의 "하위"로 표시됩니다. 모든 사용자 정의 구성은 기본 구성의 하위 구성이므로 적절한 이름을 지정할 수 있습니다.

구성을 편집하려면 두 번 클릭해야 합니다. 그것에 또는 마우스 오른쪽 버튼을 사용 메뉴 옵션 " Set Config "는 편집기에서 구성을 선택하고 완료되면 계속해서 사용자 지정할 수 있습니다. 이것은 위에 나열된 옵션을 살펴보고 새로 생성된 구성에 대해 원하는 설정으로 변경하기만 하면 됩니다. GameMaker 는 구성에 대해 이러한 값을 자동으로 저장하며 RMB를 사용하여 언제든지 구성 간에 변경할 수 있습니다. 메뉴 옵션 또는 더블 클릭.

추가 구성은 생성할 때 편집기에서 강조표시된 구성에 따라 기본 구성의 하위 또는 생성한 사용자 정의 구성의 하위로 추가할 수 있습니다. 아래 예제 이미지에서 세 개의 사용자 정의 구성이 정의되어 있음을 볼 수 있습니다. 그 중 두 개는 기본 구성의 자식이고 다른 하나는 사용자 정의 구성의 자식입니다.

프로젝트를 컴파일하거나 테스트할 때 지정된 대상 플랫폼에서 지정된 구성을 사용할 수 있다는 점은 주목할 가치가 있습니다. Target Manager 의 Config 섹션에서 필요한 구성을 선택하면 됩니다.

또한 오른쪽 상단의 메뉴 버튼을 클릭하여 Target Manager에서 언제든지 Configuration Editor를 열 수 있습니다(위 이미지에서 강조 표시됨).

이러한 방식으로 구성을 설정하면 테스트를 위해 또는 최종 자산 패키지를 생성하기 위해 대상으로 컴파일할 때마다 선택한 구성이 사용됩니다. 대상 창에 대한 자세한 내용은 컴파일 섹션을 참조하십시오.
