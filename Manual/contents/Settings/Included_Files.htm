포함된 파일

GameMaker 는 자산 브라우저에 추가하는 자산에만 국한되지 않습니다. 게임이 실행되는 동안 액세스할 수 있도록 프로젝트에 파일을 포함된 파일 로 추가할 수도 있습니다. 이것들은 단순히 게임 패키지에 포함된 파일이므로 원하는 대로 사용할 수 있습니다.

추가하는 파일의 종류는 수행하려는 작업에 따라 다르지만 기본 *.ini 파일, 스프라이트 및 배경 이미지 또는 스프레드시트는 일반적으로 포함 파일로 사용됩니다.

포함된 파일 추가

포함된 파일을 추가하는 가장 간단한 방법은 탐색기/Finder에서 IDE로 끌어오는 것입니다. 끌어온 파일을 Included Files 에 추가해야 하는지 확인하는 메시지가 표시됩니다. 예 를 누르면 방금 추가한 파일이 포함된 포함된 파일 창이 열립니다.

포함된 파일 창

자산 브라우저의 기타 메뉴 열기 (그림 1) 포함된 파일 창을 수동으로 열려면(그림 2):

이 창에는 프로젝트에 있는 포함된 파일만 표시됩니다. 파일을 재정렬하거나 이름을 바꾸려면 이러한 파일이 저장된 실제 디렉토리를 편집해야 합니다.

탐색기/파인더에서 열기 를 클릭하면 포함된 모든 파일이 포함된 프로젝트 디렉토리의 DLL>데이터 파일 폴더가 열립니다. 이 디렉토리에서 원하는 대로 파일을 추가 및 제거할 수 있으며 GameMaker의 인터페이스는 DyLib>데이터 파일 폴더의 구조를 반영하도록 자동으로 업데이트됩니다.

편집기에 파일을 추가했으면 두 번 클릭할 수 있습니다. 파일 속성 을 열려면 다음을 수행하십시오.

이 창에는 파일 이름, 디스크상의 크기, 게임을 컴파일/테스트할 때 내보내야 하는 플랫폼이 표시됩니다. "모두" 또는 "없음"을 선택하는 버튼이 있거나 목록 아래로 이동하여 파일을 내보낼 대상을 선택/선택 취소할 수 있습니다.

포함 파일이 게임과 함께 저장되는 방법과 액세스 방법에 대한 자세한 내용은 파일 시스템 을 참조하십시오.

중요 사운드 파일을 포함할 때 파일 이름이 스트리밍 사운드 자산과 충돌하지 않도록 해야 합니다. 예를 들어 " main_music "이라는 사운드 자산은 프로젝트가 실행될 때 " main_music.ogg "라는 포함 파일이 정의되지 않은 상태가 되도록 합니다. 스트리밍된 파일도 메인 프로젝트 번들 외부에 저장되기 때문에 이러한 상황에 주의해야 합니다.

다음을 포함할 수도 있습니다.external_define() (Windows에서) 또는external_call() (macOS에서) 파일을 Included Files로 만든 다음 다음을 사용하여 처리할 사용자 정의 함수를 생성합니다. . 그런 다음 호출하는 데 사용할 수 있습니다. 함수.

참고 포함된 파일을 사용하여 이러한 방식으로 추가 기능을 생성하는 것은 확장에 추가된 파일과 함께 external_*() 함수를 사용할 수 없기 때문에 확장 시스템과 호환되지 않습니다. 확장 자산에는 이미 상수와 함수를 추가하는 방법이 있습니다.
