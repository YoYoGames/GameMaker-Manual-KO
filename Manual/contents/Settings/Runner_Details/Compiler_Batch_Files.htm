컴파일러 배치 파일/스크립트

GameMaker 컴파일러를 사용하면 배치 파일(Windows) 또는 Shell 스크립트(macOS 및 Linux의 경우)를 프로젝트 디렉토리와 프로젝트의 각 확장에 대한 디렉토리에 배치할 수 있습니다.

놓기

스크립트 파일은 프로젝트의 .yyp 파일과 함께 프로젝트 디렉토리의 루트에 위치할 수 있습니다.

스크립트 파일은 <project_directory>/extensions/<extension_folder>/ 와 같은 확장의 디렉토리에 위치할 수도 있습니다.

스크립트 파일의 확장 버전이 먼저 실행되고 루트 버전이 마지막에 실행됩니다. 확장이 실행되는 순서는 제어할 수 없으므로 확장 개발자는 한 확장의 스크립트가 다른 확장의 스크립트에 의존하지 않는지 확인해야 합니다.

실행

위에 설명된 디렉토리에 있는 특정 이름의 스크립트 파일은 컴파일 프로세스 중 다양한 지점에서 실행됩니다.

Windows 배치 파일 이름 macOS/Linux 셸 스크립트 이름 설명
pre_run_step.bat pre_run_step.sh 게임 배포가 시작되기 전에 실행됩니다.
post_run_step.bat post_run_step.sh 게임이 준비되고 실행할 준비가 되면 실행됩니다. 스크립트 실행 후 게임이 시작됩니다.
pre_build_step.bat pre_build_step.sh 게임 빌드를 위해 자산 컴파일러가 시작되기 전에 실행됩니다.
post_build_step.bat post_build_step.sh 게임 빌드를 위해 자산 컴파일러가 시작된 후에 실행됩니다.
pre_package_step.bat pre_package_step.sh 이것은 모든 파일이 준비되었지만 최종 ZIP 파일 또는 저장소 패키지가 생성되려는 최종 패키징 단계 전에 실행됩니다.
post_package_step.bat post_package_step.sh 최종 패키징 단계가 완료되고 최종 ZIP 파일 또는 스토어 패키지가 준비된 후 실행됩니다.
post_textures.bat post_textures.sh 이것은 게임의 텍스처가 생성된 후에 실행됩니다. 이 단계를 사용하여 게임 텍스처를 수동으로 최적화할 수 있습니다.

이 스크립트는 생성된 텍스처의 경로를 저장하는 TexturesDir 이라는 환경 변수를 받습니다.

이러한 스크립트 파일은 모든 대상 플랫폼에서 빌드할 때 지원됩니다.

확장에 대한 스크립트 비활성화

예를 들어 Steamworks 및 GDK와 같이 스크립트를 실행하는 여러 확장을 사용하는 경우 GDK로 내보낼 때 Steamworks를 비활성화하고 그 반대의 경우도 마찬가지입니다.

이에 대해서는 확장 기능을 비활성화하는 방법을 참조하십시오.
