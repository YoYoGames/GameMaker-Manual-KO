debug_event

이 함수는 게임이 디버그 모드 에서 실행될 때 디버거의 그래프 보기에 표시될 사용자 정의 디버그 이벤트를 생성합니다. 디버그 모드가 아닐 때 메시지를 표시해야 하는 경우 다음을 사용하십시오.show_debug_message().

이 함수는 Visual Studio와 같은 외부 응용 프로그램을 사용하여 디버깅을 수행하는 데 도움이 되도록 두 개의 예약된 문자열도 사용합니다. 이러한 문자열은 다음과 같습니다.

  • "OutputDebugOn " - 전화를 걸 수 있습니다. OutputDebugString 즉, 모든 디버그 출력(출력 창 에 표시되는 모든 것)은 Visual Studio 또는 타사 앱에서 볼 수 있습니다.
  • "OutputDebugOff " - 위에서 설명한 동작을 비활성화합니다.
  • "BreakOnError " - 이 옵션은 Windows YYC 빌드 전용이며 런타임에 오류가 감지되면 프로젝트에 일반적인 코드 오류 화면이 표시 되지 않고 대신 계속 진행되고 충돌이 발생합니다. 이를 통해 디버깅하는 경우 Visual Studio 내에서 스택 추적을 볼 수 있습니다..
  • "ResourceCounts " - 데이터 구조, 시간 소스, 표면 등과 같은 현재 활성 리소스를 모두 나열합니다.
  • "DumpMemory " - 현재 메모리 사용량에 대한 정보를 제공합니다.

통사론:

debug_event(string)

논쟁 유형 설명
string 끈 The custom debug event string to use.

보고:

N/A

예시:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

위의 코드는 표면이 존재하는지 확인하고 존재하지 않으면 디버거의 그래프 보기에서 디버그 이벤트가 트리거되고(이를 보려면 게임이 디버그 모드에서 실행되어야 함) 표면이 다시 생성됩니다.
