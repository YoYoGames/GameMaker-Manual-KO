fps_real

GameMaker 에는 게임 실행 속도를 알려주는 데 사용할 수 있는 두 가지 주요 방법이 있습니다. 그만큼game speed (게임 옵션에 지정된 대로) 및 fps (초당 프레임). 이러한 값은 종종 혼동되지만 기본적으로 하나는 GameMaker 가 1초에 완료해야 하는 게임 단계 수(게임 속도)이고 다른 하나는 GameMaker 가 1초에 실제로 완료하는 CPU 단계 수(실제 fps), 이 값은 일반적으로 게임 속도보다 훨씬 높지만 게임이 더 복잡해지고 설정된 실내 속도를 유지하기 위해 더 많은 처리 능력을 사용할수록 떨어집니다.

이 읽기 전용 변수는 현재 fps를 정수 값으로 반환합니다. 이 기능은 게임의 모든 단계에서 한 번만 업데이트되므로 한 값에서 다른 값으로 "점프"하는 것처럼 보일 수 있지만 이는 매우 정상적인 현상입니다.

참고: HTML5 대상에서 이 변수는 대신 GameMaker 가 타이밍 및 발송을 위해 브라우저에 의존해야 하므로 모니터 재생 빈도의 배수를 반환합니다.

통사론:

fps_real

보고:

예시:

if debug_mode
{
    draw_text(32, 32, "FPS = " + string(fps_real));
}

위의 코드는 게임이 디버그 모드인지 확인하고 디버그 모드인 경우 화면에 현재 실제 fps를 표시합니다.
