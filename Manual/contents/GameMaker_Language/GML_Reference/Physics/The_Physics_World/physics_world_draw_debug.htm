physics_world_draw_debug

이 함수는 물리 장면이 제대로 작동하도록 하는 데 문제가 있을 때 디버깅을 돕기 위해 방의 물리 세계 시스템의 표현을 그립니다. 때때로 문제는 게임의 렌더링 부분에 있을 수 있습니다. 예를 들어 스프라이트가 잘못된 위치나 회전으로 그려져 잘못된 물리 현상이 나타나거나 계산이 올바르지 않을 수 있습니다. 어느 쪽이든, 이 기능은 물리 시스템에서 일어나는 일을 그려서 문제를 찾고 해결하는 데 도움이 될 것입니다. 이렇게 하려면 시스템의 어느 부분을 그릴지 알려야 하며 이는 다양한 "플래그" 상수에 저장된 비트에서 생성된 값을 전달하여 수행됩니다.

물리 디버그 플래그
깃발 설명
phy_debug_render_aabb 이것은 방 축과 관련하여 각 조명기의 절대 경계 상자를 보여줍니다.
phy_debug_render_collision_pairs 이것은 현재 충돌하고 있는 모든 조명기를 보여줄 것입니다
phy_debug_render_coms 이것은 방에 있는 각 설비의 질량 중심을 표시합니다.
phy_debug_render_core_shapes 방의 비품을 구성하는 기본 모양을 보여줍니다.
phy_debug_render_joints 이것은 방에 있는 모든 비품의 각 조인트를 그립니다.
phy_debug_render_obb 이것은 방의 설비에 대한 상대적 경계 상자를 보여줍니다.
phy_debug_render_shapes 이것은 방 안의 비품을 구성하는 실제 모양을 보여줍니다

따라서 이러한 옵션 중 두 개를 그리려면 두 옵션에 대해 비트 단위 "또는" ( GameMaker 에서는 | 로 표시됨)을 수행하고 함수가 사용할 결과 변수를 저장해야 합니다. 다음은 방에 있는 모든 설비의 조인트와 질량 중심만 그리는 함수와 함께 사용할 변수를 설정하는 방법의 예입니다.

flag = phy_debug_render_coms | phy_debug_render_shapes;

다음과 같은 것을 줄 것입니다.

위의 코드에서 볼 수 있듯이 화면에는 모양과 질량 중심만 표시됩니다(또한 상태를 나타내는 색상도 마찬가지입니다. 자세한 내용은 Box 2D 온라인 설명서를 확인하세요). 같은 방법으로 디버그하려는 상수를 더 많이 또는 모두 표시할 수 있습니다... 그냥 "또는" 모두 함께 표시하십시오! 한 가지 추가 사항에 유의해야 합니다. 이 함수는 디버그하려는 것보다 깊이가 낮은 인스턴스의 그리기 이벤트 에서만 호출해야 합니다. 그렇지 않으면 디버그 정보를 볼 수 없습니다.

통사론:

physics_world_draw_debug(flag)

논쟁 유형 설명
flag 물리 디버그 플래그 The flag value for showing different debug features

보고:

N/A

예시:

flag = phy_debug_render_aabb | phy_debug_render_collision_pairs | phy_debug_render_obb;
physics_world_draw_debug(flag);

위의 코드는 모든 절대 경계 상자, 상대 경계 상자 및 현재 물리 시스템 내에서 충돌하는 고정 장치를 그립니다.
