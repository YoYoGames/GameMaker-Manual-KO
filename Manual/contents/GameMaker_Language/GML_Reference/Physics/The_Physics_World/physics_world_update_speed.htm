physics_world_update_speed

GameMaker 는 실내 속도를 기반으로 업데이트하므로 이를 30으로 설정하면 GameMaker 는 1초 동안 30단계를 실행합니다. 그러나 물리 기능이 충분하지 않고 안정성이나 정밀도를 높이기 위해 약간 더 빠른 속도로 업데이트되기를 원할 수 있습니다. 이를 위해 우리는 함수를 사용합니다 physics_world_update_speed() 이는 실내 속도와 독립적 으로 물리 시스템의 업데이트 속도를 설정합니다. 즉, 실내 속도를 30으로 설정할 수 있지만 물리를 60으로 설정하면 단계가 업데이트되는 속도와 비교하여 물리 시스템이 업데이트하고 계산을 수행하는 속도를 효과적으로 두 배로 늘릴 수 있습니다.

참고: 현재는 이 속도를 실내 속도보다 낮게 설정할 수 없지만 향후 업데이트에서는 이를 변경할 수 있습니다.

통사론:

physics_world_update_speed(speed)

논쟁 유형 설명
speed 진짜 The number of times per second that the physics system updates

보고:

N/A

예시:

physics_world_update_speed(room_speed * 2);

위의 코드는 물리 시스템 업데이트 속도를 실내 속도의 2배로 설정합니다.
