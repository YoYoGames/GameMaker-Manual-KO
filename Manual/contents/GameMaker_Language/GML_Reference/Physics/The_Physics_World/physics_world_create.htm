physics_world_create

객체에 대한 물리학을 정의한 경우에도 객체를 배치할 세계가 없으면 기본적으로 기존 GameMaker 충돌 시스템이 사용되므로 객체 이벤트 또는 방 자체의 생성 코드(방 편집기에서 코드를 사용하지 않고 정의할 수도 있음). 이 함수는 물리 "세계"를 방과 연결하고 방에서 생성된 모든 인스턴스는 그들과 세계 자체에 대해 정의된 물리를 사용하여 동작합니다.

픽셀 크기와 미터 간의 비율을 정의하는 인수가 있음을 알 수 있습니다. 이는 물리적 기능이 실제 측정을 사용하여 작동하므로 "픽셀에서 미터 단위로"를 사용하여 픽셀 좌표와 픽셀 크기를 이러한 실제 측정으로 변환해야 하기 때문입니다. 사용 중인 개체의 평균 픽셀 크기가 대략적으로 적절한 크기의 시뮬레이션된 물리 개체로 변환될 때까지 pixeltometrescale 설정을 조정하고 싶을 것입니다. 위에서 1미터에 해당하는 32픽셀로 설정했음을 알 수 있습니다.. 물리 시스템은 더 크고 더 작은 비율로 작동하지만 최상의 성능은 물체의 실제 측정이 0.1미터 이상 50미터 이하일 때입니다.

통사론:

physics_world_create(pixeltometrescale);

논쟁 유형 설명
pixeltometrescale defines the ratio of pixels on screen to metres in the real world. A ratio of 32:1 will be specified as 1/32 (or 0.03125).

보고:

예시:

physics_world_create(1/32);

위의 코드는 1미터에 해당하는 32픽셀의 축척 비율로 현재 방에 물리 세계를 만듭니다.
