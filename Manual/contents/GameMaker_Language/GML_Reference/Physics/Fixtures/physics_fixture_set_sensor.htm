physics_fixture_set_sensor

고정 장치가 있는 두 인스턴스가 충돌할 때 게임이 감지하고 인식하지만 충돌에 대한 물리적 반응이 없는 경우가 있습니다. 이것은 고정물을 센서로 변환하여 수행할 수 있습니다. 즉, 기본적으로 충돌 이벤트를 생성하지만 물리적 응답은 없으므로 이러한 인스턴스를 게임룸에서 다른 이벤트가 발생하도록 "트리거"로 사용할 수 있습니다. 모든 고정 장치는 센서로 플래그를 지정할 수 있으며 바인딩된 인스턴스가 정적이든 이동 중이든 차이가 없습니다.

참고: 센서 고정 장치는 충돌 이 처음 발생할 때만 충돌 이벤트를 발생시킵니다. 즉, 두 개의 본체가 계속 겹치기 때문에 충돌 이벤트 스트림을 얻지 못합니다(전통적으로 발생함). 중첩을 멈추고 나중에 중첩되면 다른 충돌 이벤트가 트리거됩니다.

통사론:

physics_fixture_set_sensor(fixture, state)

논쟁 유형 설명
fixture 물리 픽스처 ID the index of the fixture
state 부울 whether a fixture is a sensor (true) or not (false)

보고:

N/A

예시:

physics_fixture_set_sensor(fix_Cloud, 1);

위의 코드는 "fix_Cloud"에 인덱싱된 조명기의 센서 상태를 true로 바꿉니다.
