physics_fixture_bind_ext

픽스쳐를 정의한 후에는 인스턴스에 바인딩해야 합니다. 이것은 그 속성 이 실제 조명기 자체가 아니라 선택된 인스턴스로 전송된다는 것을 의미하므로 모두가 동일한 속성을 가지려면 하나의 조명기가 여러 인스턴스에 바인딩될 수 있습니다. 대상에 대한 객체 인덱스를 지정할 수 있으며 그 시간에 방에 있는 모든 인스턴스는 조명기 속성을 수신합니다(그러나 나중에 생성된 객체의 새 인스턴스는 아님), 또는 특수 키워드 other 및 all 을 사용할 수 있습니다. 부모 개체를 지정할 수도 있으며 해당 부모가 있는 모든 자식 인스턴스도 조명기를 받습니다. 설비가 필요한 모든 인스턴스에 바인딩되면 더 이상 필요하지 않은 경우 삭제할 수 있으며 해당 설비 속성이 있는 인스턴스는 영향을 받지 않고 해당 속성을 유지합니다.

일반적으로 고정 장치는 인스턴스의 원점에 위치하는 질량 중심으로 인스턴스에 바인딩되지만 이것이 항상 필요한 것은 아니므로 이 기능을 사용하면 고정 장치가 있는 x 및 y 위치를 오프셋할 수도 있습니다. 주어진 양으로 제한됨(필요하지 않은 경우 다음을 사용하십시오.physics_fixture_bind() 대신에). 단일 고정 장치에 여러 오프셋을 추가하는 것은 Box2D에서 지원되지 않기 때문에 고정 장치는 단일 오프셋만 지원할 수 있다는 점에 유의하는 것이 중요합니다.

이 함수는 또한 바인딩된 조명기에 대해 고유한 "id" 값을 반환합니다( 조명기 자체가 아님). 이 값은 함수를 사용하여 인스턴스에서 물리적 속성을 제거("바인딩 해제")하는 데 사용할 수 있습니다.physics_remove_fixture(). 이를 통해 객체를 파괴하고 다시 생성하지 않고도 인스턴스에서 물리적 속성을 추가 및 제거할 수 있습니다.

참고: 더 이상 필요하지 않을 때 고정 장치를 삭제해야 합니다. 삭제하지 않으면 메모리 누수가 발생하여 속도가 느려지고 결국 게임이 중단될 수 있습니다.

통사론:

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)

논쟁 유형 설명
fixture 물리 픽스처 ID The fixture that is to be bound
target 인스턴스 ID 또는 개체 자산 The target instance that is to receive the fixture (can be an instance id, an object id, other, or all )
xoffset 진짜 The offset along the x-axis
yoffset 진짜 The offset along the y-axis

보고:

Real

예시:

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

위의 코드는 고정물을 생성하고 그것의 인덱스를 변수 "fix"에 할당할 것입니다. 그런 다음 스프라이트의 너비와 높이를 기반으로 한 오프셋에서 인스턴스에 바인딩하기 전에 조명기의 모양과 밀도를 정의합니다. 바인딩된 조명기의 인덱스는 "my_fix" 변수에 저장됩니다. 마지막으로 조명기 자체가 더 이상 필요하지 않으므로 메모리 누수를 방지하기 위해 삭제됩니다.
