physics_apply_local_impulse

GameMaker 의 물리학을 사용하여 물체에 힘과 중력을 적용할 수 있을 뿐만 아니라 임펄스를 적용할 수도 있습니다. 이것은 힘이 가해지면 즉시 속도에 영향을 미치고 잠재적으로 물체의 토크(또는 "회전")에 영향을 미친다는 점에서 힘과 약간 다릅니다. 특히 충격을 적용하기 위해 선택한 지점에 그렇지 않은 벡터가 있는 경우 질량 중심과 정렬됩니다(참고: 질량 중심이 원점과 반드시 같을 필요는 없습니다!). 이 함수는 인스턴스에 로컬 로 임펄스를 적용합니다. 이것이 의미하는 바는 임펄스의 강도와 방향이 게임 룸이나 물리 세계에서 가질 수 있는 방향이나 회전을 고려하지 않고 인스턴스의 원점(또는 스프라이트가 없는 경우 위치)을 기반으로 계산된다는 것입니다.. 다음은 예시입니다.

보시다시피, 이미지의 개구리는 아래에서 충동을 받아 "점프"합니다. 정확한 임펄스는 xpos/ypos 좌표와 관련하여 ximpulse/yimpulse 구성 요소에서 얻은 벡터에 의해 정의됩니다. 이는 단순히 임펄스가 xpos/ypos에서 ximpulse/yimpulse까지의 거리(뉴턴 단위 및 방향)로 계산됨을 의미합니다. xpos/ypos에서 ximpulse/yimpulse까지의 각도입니다.

참고: 물리학 세계는 모든 계산에 미터를 사용하며 미터당 픽셀 비율은 룸 편집기 또는 다음을 사용하여 정의됩니다. physics_world_create().

통사론:

physics_apply_local_impulse(xpos, ypos, ximpulse, yimpulse)

논쟁 유형 설명
xpos The x coordinate relative to the origin where the impulse will be applied
ypos The y coordinate relative to the origin where the impulse will be applied
ximpulse the x component of the impulse vector (relative to the xpos coordinate)
yimpulse the y component of the impulse vector (relative to the ypos coordinate)

보고:

예시:

if jump
{
    physics_apply_local_impulse(0, 20, 0, 50);
}

위의 코드는 "jump" 변수가 true인 경우 인스턴스 원점을 기준 으로 30뉴턴의 힘과 아래쪽에서 위쪽 방향으로 임펄스를 적용합니다.
