physics_joint_friction_create

마찰 관절은 생성된 연결이 인스턴스의 위치나 움직임을 제한하지 않고 속도와 회전을 제한한다는 점에서 물리 시뮬레이션의 다른 모든 관절과 약간 다릅니다. 이것은 힘과 토크에 대한 최대 입력 값을 취하고 두 번째 고정 장치에 적용하여 속도와 각운동량을 첫 번째 인스턴스와 동일한 값으로 낮추는 방식으로 작동합니다. 따라서 정지된 인스턴스와 움직이는 인스턴스가 있는 경우 마찰 조인트로 연결하면 움직이는 인스턴스도 정지될 때까지 점차 느려집니다. 두 인스턴스가 모두 이동 중이었다면 두 번째 인스턴스의 이동 속도는 첫 번째 인스턴스의 이동 속도와 일치하도록 수정됩니다.

대부분의 다른 관절과 마찬가지로 함께 결합할 인스턴스와 공간 내에서 생성될 관절의 위치를 제공합니다. 그런 다음 최대 힘과 최대 토크(방향 및 회전 마찰)를 제공하고 "col" 값을 다음으로 설정하면 true 두 인스턴스는 서로 상호 작용하고 충돌할 수 있지만 충돌 이벤트가 있는 경우 에만 가능합니다. false, 그들은 무슨 일이 있어도 충돌하지 않습니다.

통사론:

physics_joint_friction_create(inst1, inst2, anchor_x, anchor_y, max_force, max_torque, col)

논쟁 유형 설명
inst1 인스턴스 ID The first instance to connect with the joint
inst2 인스턴스 ID The second instance to connect with the joint
anchor_x 진짜 The x coordinate for the joint, within the game world
anchor_y 진짜 The y coordinate for the joint, within the game world
max_force 진짜 The maximum frictional force that will be applied
max_torque 진짜 The maximum rotational force that will be applied
col 부울 Whether the two instances can collide (true) or not (false)

보고:

Physics Joint ID

예시:

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_friction_create(id, o_id, x, y, 10, 0.3, true);
physics_fixture_delete(mainFixture);

위의 코드는 고정 장치를 만든 다음 마찰 조인트를 사용하여 두 인스턴스를 연결하기 전에 두 인스턴스에 바인딩합니다.
