키보드 입력

GameMaker 에서 키보드를 다룰 때 눌림 또는 떼기와 같은 다양한 키보드 상태를 인식하는 데 사용할 수 있는 다양한 기능이 있습니다. 또한 모든 키 누름을 문자열로 저장하거나 마지막으로 누른 키가 무엇인지 알려줄 수 있는 것도 있고, 키보드 상태를 완전히 지울 수 있게 하는 것도 있습니다.

참고 이러한 기능은 Windows/Mac/Ubuntu 데스크탑 플랫폼 전용으로 설계되었습니다. 내장 변수와 상수 중 일부는 다른 플랫폼에서 유효하지 않으며 많은 기능이 모바일에서 작동하지 않을 수 있습니다.

키(또는 여러 키)의 각 입력 문자는 숫자 값인 UTF8 코드로 정의됩니다. 이 값은 다음을 사용하여 모든 문자에 대해 검색할 수 있습니다.ord() 기능이지만 GameMaker 에는 constants 가장 많이 사용되는 키보드 특수 키 및 특수 기능. 일반적으로 다음 조합을 사용합니다. ord() 와 더불어 keyboard_check*() 다음과 같은 기능:

if keyboard_check(ord("A"))
{
    hspeed = -5;
}

따라서 위의 내용은 "A" 키를 확인하고 이 키를 누르면 개체의 수평 속도가 -5로 설정됩니다. 참고로 사용 ord() 이 방법은 입력 문자열의 길이가 1자이고 0에서 9 사이의 숫자이거나 A에서 Z까지 대문자 로 된 로마 문자인 경우에만 올바르게 작동합니다. ord() 전체 UTF8 값을 반환하지만 keyboard_check*() 기능은 A-Z 및 0-9만 감지 합니다. 하지만 화살표 키를 사용하려면 어떻게 해야 할까요? 또는 "shift" 키를 사용하여 작업을 수정하려면? 음, GameMaker 에는 다음 시리즈가 있습니다. vk_ 상수(vk_ 대신 사용할 수 있는 가상 키 ) ord():

다음은 사용 방법에 대한 작은 예입니다. vk_ 상수:

if keyboard_check_pressed(vk_tab)
{
    instance_create_layer(x, y, "Controllers", obj_Menu);
}

위의 코드는 "Tab" 키가 눌렸는지 감지하고 만약 그렇다면 개체 "obj_Menu"의 인스턴스를 생성합니다.

0 - 9, A - Z 또는 VK 상수 중 하나가 아닌 키 문자를 확인해야 하는 경우 다음 중 하나를 확인해야 합니다. keyboard_* 변수 keyboard_lastchar 예를 들어:

var _key = keyboard_lastchar;
if ord(_key) == ord("ç")
{
    show_debug_message("ç key pressed");
}
