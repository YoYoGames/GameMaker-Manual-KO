virtual_key_add

이 기능을 사용하면 화면 영역의 "터치"를 키보드 이벤트에 매핑할 수 있습니다. 즉, 가상 키에 영역을 할당하면 해당 영역에 대한 모든 터치가 해당 영역에 매핑한 키에 해당하는 키보드 이벤트를 트리거합니다. 정의한 각 가상 키를 변수에 할당할 수도 있습니다. 그러면 추가 가상 키 기능에서 이를 표시, 숨기기 및 삭제할 수 있습니다. 이 키는 방 별로 할당되며 방을 변경할 때 GameMaker 에 의해 자동으로 제거됩니다.

가상 키의 실제 위치는 방 위치가 아닌 화면 위치를 기반으로 하므로 x/y 값은 화면에서 절대적입니다. 즉, 뷰 사용이나 상대 방 좌표에 대해 걱정할 필요가 없으며 객체의 Create Event 에서 키를 간단히 정의할 수 있습니다(실당 한 번만 가상 키를 정의하면 됨). Draw GUI Event 에서 키 스프라이트를 그립니다. GUI 레이어의 크기를 화면과 동일하게 조정합니다.

통사론

virtual_key_add(x, y, w, h, keycode);

논쟁 유형 설명
x 진짜 The x coordinate (left side) of the virtual key on the screen
y 진짜 The y coordinate (top side) of the virtual key on the screen
w 진짜 The width of the virtual key
h 진짜 The height of the virtual key
keycode 가상 키 상수(vk_*) Which keyboard key event should be triggered by touching this area

보고:

Virtual Key ID

예시:

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

위의 코드는 화면의 (32, 32)에 위치한 64x64 픽셀 정사각형 가상 키를 생성하여 터치 시 vk_left 이벤트를 트리거하고 이 가상 키의 인덱스를 전역 변수에 할당합니다.
