가상 키 및 가상 키보드

터치스크린 장치용 게임을 개발할 때 종종 사용자로부터 키보드 입력을 받는 것이 중요합니다. GameMaker 를 사용할 때 다음 두 가지 방법 중 하나로 수행할 수 있습니다.

  • 입력을 받기 위해 화면에 가상 키 를 직접 생성
  • OS 가상 키보드 를 띄우고 입력을 받으십시오.

사용하는 항목은 게임과 필요한 항목에 따라 크게 달라지지만 일반적으로 플레이어 제어 또는 메뉴용 화면 버튼을 원할 때 가상 키를 사용하고 플레이어 입력이 필요할 때 가상 키보드를 사용합니다., 이름을 얻거나 값을 설정하는 것과 같습니다.

터치 스크린이 있는 모바일 장치에서 게임이 제대로 작동하려면 사용자가 터치할 때 특정 항목에 반응하도록 화면 영역을 할당할 수 있어야 합니다. 이것은 내부적으로 매핑된 표준 키보드 키에 "가상 키"를 할당한 다음 애플리케이션을 제어하기 위해 생성하는 키보드 이벤트를 사용하여 수행됩니다. 이러한 "가상 키"는 방 위치가 아닌 화면 위치에 매핑되므로 x/y 값은 디스플레이 크기에 따라 절대적입니다. 즉, 뷰 사용이나 상대 방 좌표에 대해 걱정할 필요가 없으며 객체의 이벤트 생성에서 키를 간단히 정의할 수 있습니다(실당 가상 키를 한 번만 정의하면 됨). Draw GUI Event 에서 키 스프라이트를 그립니다. GUI 레이어의 크기를 화면과 동일하게 조정합니다.

참고 이러한 기능은 마우스 클릭을 "터치"로 수신하더라도 터치 스크린을 사용하는 대상 플랫폼인 Windows, Ubuntu(Linux) 또는 MacOS에서 작동하지 않습니다.

이러한 기능은 HTML5 및 Opera GX 대상에서 지원됩니다.

사용 가능한 가상 키 기능은 다음과 같습니다.

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

게임에서 터치스크린 장치의 텍스트 또는 숫자 입력이 필요한 경우 제공된 가상 키보드 기능을 사용하여 장치 OS 키보드를 사용할 수 있습니다. 이러한 기능은 OS 키보드를 호출하거나 숨기고 입력을 수락하지만 일반 키보드 이벤트를 트리거 하지 않고 오히려 업데이트합니다. keyboard_string (이것은 가상 키보드가 단일 문자가 아닌 전체 단어를 입력할 수 있기 때문입니다).

가상 키보드를 열거나 닫으면 시스템 비동기 이벤트 가 생성됩니다. async_load DS 맵은 다음 키/값 쌍으로 채워집니다.

  • " event_type " - 수신된 시스템 이벤트의 유형으로, 가상 키보드의 경우 " virtual keyboard status "가 됩니다.
  • " screen_height " - 가상 키보드의 높이(픽셀 단위). 키보드가 보이지 않으면 0이 됩니다.
  • " keyboard_status " - 키보드의 현재 상태로, 다음 문자열 중 하나로 반환됩니다.
    • "숨김"
    • "숨겨진"
    • "전시"
    • "보이는"

참고 이러한 기능은 Android ( AndroidTV 포함), iOS ( tvOS 포함) 및 UWP 대상 플랫폼에서만 유효합니다.

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
  • keyboard_virtual_height
