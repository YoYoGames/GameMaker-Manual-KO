게임패드 입력

GameMaker 에는 연결된 여러 게임패드에서 아날로그 및 디지털 컨트롤을 모두 감지하는 데 사용할 수 있는 여러 전용 기능이 있습니다. 이 기능은 연결된 최대 4개의 다른 XInput 게임 패드(및 최대 8개의 DirectInput 게임패드)를 감지하고 동일한 기능을 사용하여 각각의 입력을 처리할 수 있다는 점에서 Device Inputs 와 유사하게 작동합니다. 게임패드가 장치에 연결되거나 제거되면 적절한 기능을 사용하여 상황을 처리할 수 있는 비동기 시스템 이벤트 가 트리거됩니다.

게임패드 "슬롯"은 0부터 위쪽으로 인덱싱되며 게임패드에 할당된 실제 슬롯은 다양한 요인에 따라 달라집니다. 특히 프로젝트가 실행되는 OS가 그 중 하나입니다. Windows 대상에서 슬롯 0 - 3은 Xinput 게임패드 전용입니다 (예: Xbox360 컨트롤러 및 호환 장치). 그러나 DirectInput 게임패드에 대해 슬롯 4 - 11을 확인할 수도 있습니다. 즉, 이 슬롯을 통해 연결될 때 다른 많은 컨트롤러 모델을 감지할 수 있습니다. 다른 플랫폼에서는 OS가 할당 한 슬롯에서 패드가 감지될 수 있으며, 슬롯 3 또는 슬롯 20 이상이 될 수 있습니다. 예를 들어, Android 기기는 블루투스 게임패드를 슬롯에 저장한 다음 연결 여부에 관계없이 향후 해당 게임패드용 슬롯을 예약하므로 연결된 단일 게임패드가 슬롯 0에 연결되어 있다고 가정할 수 없습니다. 그렇지 않습니다.

Windows에서 DirectInput 게임패드를 사용하거나 다른 플랫폼에서 일반 게임패드를 사용할 때 API가 단편화되고 표준화되지 않은 방식으로 인해 아래에 제공된 상수 가 눌렀을 때 예상한 버튼과 정확히 일치하지 않을 수 있습니다. 컨트롤러 제조업체에서 구현합니다. 이 때문에 사람들이 연결된 장치의 입력을 기반으로 게임패드 버튼을 재정의하여 문제를 완화할 수 있는 일종의 게임패드 설정 화면을 게임에 두는 것이 좋습니다(이를 도울 수 있는 게임패드 "매핑" 기능이 있습니다) Windows 데스크톱, Ubuntu, macOS 및 Android 대상에서는 코드를 사용하여 직접 수행해야 합니다.

입력 상수

게임패드 기능으로 작업할 때 입력은 축, 버튼 또는 모자 에서 올 수 있으며 GameMaker 는 다음 내장 상수에 할당합니다("모자"는 일반적으로 비표준 컨트롤러에서만 감지됨).

게임패드 버튼 상수
일정한 설명
gp_face1 상단 버튼 1(Xbox 360 컨트롤러의 "A" 와 PS 컨트롤러의 십자가 에 매핑됨)
gp_face2 상단 버튼 2(Xbox 360 컨트롤러의 "B" 와 PS 컨트롤러의 원 에 매핑됨)
gp_face3 상단 버튼 3(Xbox 360 컨트롤러의 "X" 와 PS 컨트롤러의 사각형 에 매핑됨)
gp_face4 상단 버튼 4(Xbox 360 컨트롤러의 "Y" 와 PS 컨트롤러의 삼각형 에 매핑됨)
gp_shoulderl 왼쪽 어깨 버튼
gp_shoulderlb 왼쪽 어깨 방아쇠
gp_shoulderr 오른쪽 어깨 버튼
gp_shoulderrb 오른쪽 어깨 방아쇠
gp_select 선택 버튼(DS4 컨트롤러에서 터치패드를 아래로 누를 때 트리거됨)
gp_start 시작 버튼(PS4 컨트롤러의 "옵션" 버튼)
gp_stickl 왼쪽 스틱 누름(버튼으로)
gp_stickr 오른쪽 스틱 누름(버튼으로)
gp_padu 방향 패드 위로
gp_padd 방향 패드 아래로
gp_padl 방향 패드 왼쪽
gp_padr 방향 패드 오른쪽
게임패드 축 상수
일정한 설명
gp_axislh 왼쪽 스틱 가로축(아날로그)
gp_axislv 왼쪽 스틱 세로 축(아날로그)
gp_axisrh 오른쪽 스틱 가로축(아날로그)
gp_axisrv 오른쪽 스틱 세로축(아날로그)
아래 상수는 PS4 또는 PS5의 DualSense 게임패드에서만 사용할 수 있습니다.
gp_axis_acceleration_x* X축에서 게임패드의 가속도
gp_axis_acceleration_y* Y축에서 게임패드의 가속도
gp_axis_acceleration_z* Z축에서 게임패드의 가속도
gp_axis_angular_velocity_x* X축에서 게임패드의 각속도
gp_axis_angular_velocity_y* Y축에서 게임패드의 각속도
gp_axis_angular_velocity_z* Z축에서 게임패드의 각속도
gp_axis_orientation_x* 게임패드의 X 방향
gp_axis_orientation_y* 게임패드의 Y 방향
gp_axis_orientation_z* 게임패드의 Z 방향
gp_axis_orientation_w* 게임패드의 W 방향

참고 게임패드의 방향 은 Quaternion 이므로 4가지 값(X, Y, Z 및 W)이 있습니다.

* 이 상수는 PS4 및 PS5에서만 지원되며 다른 플랫폼에서 사용하면 입력 함수가 반환됩니다. 0, DualSense 게임패드를 사용하는 경우에도 마찬가지입니다.

각 상수가 컨트롤러의 어떤 부분을 나타내는지 정확히 이해하기 위해 표준 XInput 게임패드의 다음 이미지를 참조할 수 있습니다.

다음 게임패드 기능도 존재하며 기본 제공 상수를 주어진 게임패드의 직접적인 물리적 입력에 다시 매핑하는 데 사용됩니다. 이러한 기능은 Windows 데스크톱, Ubuntu, macOS 및 Android 대상 플랫폼에만 해당되며 Windows에서는 직접 입력 장치에만 유효합니다. GameMaker 는 SDL 게임패드 컨트롤러 DB 를 기반으로 하는 다양한 게임패드에 대한 매핑과 함께 제공되지만 컨트롤러 유형과 브랜드가 엄청나게 많기 때문에 GML 상수를 모든 제조사와 모델의 올바른 입력에 매핑하는 것은 불가능합니다. 따라서 이러한 기능을 사용하면 고유한 사용자 지정 매핑을 만들 수 있습니다.

  • gamepad_get_mapping
  • gamepad_test_mapping
  • gamepad_remove_mapping

Direct Input 게임패드는 협동 모드 에서 실행된다는 점은 주목할 가치가 있습니다. 즉, 게임은 전경 응용 프로그램일 때만 해당 패드에 액세스할 수 있으므로 게임이 포커스를 잃으면 직접 입력 컨트롤러가 "잃어버리게" 됩니다. 다시 초점이 맞춰지면 "찾음"이 표시됩니다(시스템 이벤트에서 감지하여 처리할 수 있음). 마찬가지로 게임에 초점이 맞지 않는 동안에는 게임패드의 입력이 감지되지 않으므로 이 기능을 사용하는 것이 좋습니다.os_is_paused() 또는 window_has_focus() 이를 감지하고 게임을 일시 중지하거나 게임이 포커스를 잃을 때 누르고 있는 버튼은 게임이 포커스를 다시 얻을 때까지 누르고 있는 상태를 유지합니다.

호환성

다음 목록은 플랫폼 간의 현재 호환성을 보여줍니다(이는 향후 업데이트로 변경될 예정임).

  • Windows 는 한 번에 최대 12개의 연결 장치가 허용되도록 완벽하게 지원됩니다(0에서 11까지 번호가 지정되며 0 - 3은 XInput 장치이고 4 - 11은 DirectInput). 컨트롤러 상수의 재매핑도 허용됩니다.
  • macOS 는 한 번에 최대 4개의 연결 장치가 허용되며 이러한 장치는 Playstation3 또는 Xbox 360 유형만 사용할 수 있습니다. Mac 게임 옵션의 "Mac AppStore용 빌드" 옵션이 꺼져 있어야 합니다. 패드 지원이 작동합니다. 컨트롤러 상수의 재매핑도 허용됩니다.
  • Ubuntu 는 게임패드 입력도 지원하지만 Ubuntu 리포지토리에서 추가 라이브러리를 설치해야 할 수도 있습니다. 명령줄 터미널을 열고 다음을 입력하여 이 작업을 쉽게 수행할 수 있습니다(조이스틱/게임패드는 물론 조이스틱/게임패드 API 자체에 대한 GUI 지원을 설치합니다 - 컨트롤러 상수의 재매핑은 허용 되지 않음 ).

    sudo apt-get install jstest-gtk
    sudo apt-get install joystick.

  • 게임패드를 사용하는 HTML5 게임은 Safari 를 제외한 대부분의 주요 브라우저에서 지원됩니다. 컨트롤러 상수의 재매핑은 허용 되지 않습니다.
  • 웹 브라우저는 버튼을 누르거나 축을 이동할 때만 게임패드를 감지하므로 게임을 바로 시작할 때 사용하지 못할 수 있습니다. 이는 Opera GX 및 HTML 대상 플랫폼에 적용됩니다.
  • 게임패드 지원은 iCade 캐비닛을 통해 iOS 로도 확장됩니다. 왼쪽 축은 스틱 컨트롤러에 매핑되고(입력은 아날로그가 아닌 디지털이지만) 4개의 "페이스" 버튼은 캐비닛 전면 버튼에 매핑되고 4개의 어깨 버튼은 캐비닛 후면에 매핑됩니다.
  • Android (및 Amazon Fire ) 내보내기는 NYKO 컨트롤러 및 일반 Bluetooth 컨트롤러를 지원하지만 활성화된 경우에만 가능합니다. 즉, Android 게임 옵션의 일반 섹션에서 iCade/Bluetooth 옵션을 선택해야 합니다. 완전히 작동하려면 API 레벨 12가 필요하며 GameMaker 는 실제로 연결되었는지 여부에 관계없이 장치가 페어링된 모든 Bluetooth 장치를 연결된 것으로 등록합니다. 따라서 "슬롯"을 할당하고 확인할 때 이를 고려해야 합니다. 컨트롤러 상수의 재매핑도 허용됩니다.
  • PS4 에서 터치 패드 추적을 사용하려면device_mouse_* 버튼. 컨트롤러 상수의 재매핑은 허용 되지 않습니다.
  • Windows UWP, XBoxOne 및 Nintendo Switch 대상에서 게임패드는 완전히 지원되지만 컨트롤러 상수의 재매핑은 허용 되지 않습니다.

이상적으로는 모든 대상 플랫폼에서 사용 가능한 게임패드 "슬롯" 목록을 열거한 다음 다음과 같이 감지된 장치가 있는지 확인하려고 합니다.

var _maxpads = gamepad_get_device_count();

for (var i = 0; i < _maxpads; i++)
{
    if (gamepad_is_connected(i))
    {
        // do stuff with pad "i"
    }
}
