gif_add_surface

이 기능을 사용하면 표면에서 프레임을 GIF 파일로 저장할 수 있습니다. 함수에서 반환된 GIF 파일 ID를 제공합니다.gif_open() ) 및 표면 ID(함수에 의해 반환됨)surface_create() ), 프레임 사이의 지연 시간뿐만 아니라. 지연 시간은 1/100초 단위로 계산되지만 GIF가 생성된 후 판촉 목적으로 사용하는 경우 브라우저마다 프레임 지연을 약간 다르게 해석할 수 있습니다. 자세한 내용은 여기 를 참조하세요. 선택적으로 GIF에 추가하는 데 사용되는 표면에 대해 X 및 Y 오프셋 값을 제공할 수 있습니다. 이 경우 GIF에 그려진 부분은 기본(0,0) 위치가 아니라 오프셋에서 시작하여 다음을 수행할 수 있습니다. 추가할 표면의 단면만 선택합니다. 0과 3 사이의 정수인 양자화 값을 제공할 수도 있습니다. 여기서 0은 전체 양자화이고 3은 0 양자화입니다(값이 낮을수록 품질이 저하되지만 더 작은 GIF가 생성됨). 기본 품질 2와 최고 품질 3 사이에는 큰 차이가 없으며 3을 사용하면 GIF 생성 속도가 크게 느려지므로 최대 값을 사용할 때 주의해야 합니다.

프레임 수 또는 GIF의 해당 프레임 크기에는 기본 제공 제한이 없지만 메모리를 너무 많이 사용하면 함수가 실패하고 -1을 반환하고, 그렇지 않으면 0을 반환합니다.

통사론:

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

논쟁 유형 설명
gif_index GIF ID The ID of gif to add the surface data to
surface 표면 ID The ID of the surface to use as the added frame
delay_time 진짜 The delay time for the frame (in 100ths of a second)
[xoffset] 진짜 OPTIONAL The offset along the X axis of the surface to add from
[yoffset] 진짜 OPTIONAL The offset along the Y axis of the surface to add from
[quantization] 진짜 OPTIONAL The quantization amount from 0 to 3 (0 lowest quality, 3 highest quality, default is 2)

예시:

if save_gif == true
{
    if count == 0
    {
        gif_image = gif_open(room_width, room_height);
    }
    else if count < 30
    {
        gif_add_surface(gif_image, application_surface, 6/100);
    }
    else
    {
        gif_save(gif_image, "GameCapture.gif");
        count = 0;
        save_gif = false;
    }
    count++;
}

위의 코드는 응용 프로그램 표면에서 가져온 30프레임의 GIF 이미지 파일을 만든 다음 저장합니다.
