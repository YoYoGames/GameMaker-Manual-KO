window_get_visible_rects

이 기능을 사용하면 연결된 각 디스플레이에서 (x1,y1) ~ (x2,y2)로 정의된 사각형의 겹치는 영역을 찾을 수 있습니다. 이 함수는 디스플레이당 8개의 값이 있는 배열을 반환합니다(예: 디스플레이가 2개 있는 경우 배열의 길이는 16개 인덱스). 여기서 값 [0... 3]은 오버랩x1, 오버랩핑1, 오버랩x2, 오버랩 2 - 이 디스플레이에서 오버랩 영역을 정의하고 오버랩이 없으면 0,0,0,0으로 설정됩니다. 값 [4... 7]은 모니터 x1, 모니터 1, 모니터 x 2, 모니터 2에 해당합니다. - 좌표 가상 디스플레이 공간의 디스플레이. 예를 들어 저장된 창 위치가 표시되는지 여부를 테스트하는 데 사용할 수 있습니다(사용자가 외부 모니터의 연결을 끊거나 유효하지 않은 것으로 저장된 창 위치를 남겨둔 창을 화면 밖으로 이동했을 수 있음).

통사론:

window_get_visible_rects(x1, y1, x2, y2);

논쟁 유형 설명
x1 The left edge of the rectangle to check
y1 The top edge of the rectangle to check.
x2 The right edge of the rectangle to check
y2 The bottom edge of the rectangle to check.

보고:

예시:

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

위의 코드는 변수에 보관된 1D 배열을 생성합니다. display_data 디스플레이에 대한 정보를 포함하고 변수를 생성합니다. display_num 발견된 활성 디스플레이의 수를 유지합니다.
