camera_create

이 기능은 "빈" 카메라, 즉 정의된 보기 또는 투영 설정이 없는 카메라를 만듭니다. 카메라를 생성하고 나면 다른 행렬 함수 를 사용하여 보기 및 투영 행렬을 설정할 수 있습니다. 이 기능을 사용하여 카메라를 만든 다음 다른 camera_set_* 기능을 사용하여 보기를 설정하지만 최소한 위치와 보기 크기를 제공 해야 합니다. 그렇지 않으면 결과 카메라가 유효하지 않고 보기 포트에 적용되는 경우 이상한 결과를 제공합니다(다음 개체를 설정하지 않으면 테두리 또는 이동 속도는 룸 편집기에서 뷰 카메라의 기본값으로 설정됩니다.

카메라는 모든 동적 리소스와 같으며 생성될 때 메모리를 차지합니다. 즉, 불필요하게 카메라를 생성하지 않도록 주의하고 함수를 호출하여 필요하지 않은 카메라를 파괴하지 않도록 주의해야 합니다.camera_destroy(). 이것은 또한 카메라가 할당된 경우 방에서 방으로 유지된다는 것을 의미합니다. global 변수(인스턴스 또는 로컬 변수에 할당될 때도 유지되지만 방을 변경한 후에는 액세스할 수 없으므로 메모리 누수), 예를 들어 새 카메라를 만들고 전역 변수에 할당할 수 있습니다., 그런 다음 각 방의 시작 부분에서 필요한 보기 및 투영 행렬과 사용해야 하는 보기 포트를 설정합니다.

통사론:

camera_create();

보고:

Camera ID

예시:

view_camera[0] = camera_create();
var viewmat = matrix_build_lookat(640, 240, -10, 640, 240, 0, 0, 1, 0);
var projmat = matrix_build_projection_ortho(640, 480, 1.0, 32000.0);
camera_set_view_mat(view_camera[0], viewmat);
camera_set_proj_mat(view_camera[0], projmat);

위의 코드는 새 카메라를 만들고 해당 ID를 view[0]에 할당합니다. 그런 다음 카메라를 설정하는 데 사용되기 전에 뷰 및 투영 행렬이 생성되어 로컬 변수에 저장됩니다.
