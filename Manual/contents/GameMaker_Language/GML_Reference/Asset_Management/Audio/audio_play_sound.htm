audio_play_sound

이 기능을 사용하면 게임에서 모든 사운드 리소스를 재생할 수 있습니다. 사운드 인덱스를 제공하고 우선 순위를 지정하면 재생되는 사운드 수가 기능에서 설정한 제한을 초과할 때 사운드를 처리하는 방법을 결정하는 데 사용됩니다.audio_channel_num(). 우선 순위가 낮은 사운드는 중지되어 우선 순위가 높은 사운드를 선택하고 우선 순위 값은 임의의 실수일 수 있습니다(실제 값은 임의적이며 GameMaker 가 동일한 우선 순위를 지정하므로 0에서 1 또는 0에서 100일 수 있음). 우선 순위를 다룰 때 숫자가 높을 수록 우선 순위가 높아져 우선 순위 100의 사운드가 우선 순위 1의 사운드보다 선호됩니다. 세 번째 인수는 사운드 루프를 만들고 다음으로 설정하는 것입니다. true 멈출 때까지 사운드 루프를 만들고 다음으로 설정합니다. false 소리를 한 번만 재생합니다.

이 함수는 또한 재생 중인 사운드에 대한 고유 인덱스 번호를 반환하며, 이 번호는 변수에 저장되어 적절한 기능으로 일시 중지하거나 중지할 수 있습니다. 즉, 한 번에 동일한 사운드의 여러 인스턴스가 재생되는 경우 해당 사운드의 한 인스턴스만 대상으로 지정하여 오디오 기능을 사용하여 처리할 수 있습니다.

통사론:

audio_play_sound(index, priority, loop, [gain], [offset], [pitch], [listener_mask]);

논쟁 유형 설명
index 사운드 자산 The index of the sound to play.
priority 진짜 Set the channel priority for the sound.
loop 부울 Sets the sound to loop or not.
gain 진짜 OPTIONAL Value for the gain.
offset 진짜 OPTIONAL The time (in seconds) to set the start point to. Values longer than the length of the given sound are ignored.
pitch 진짜 OPTIONAL The pitch multiplier (default 1).
listener_mask 진짜 OPTIONAL The bitmask data to set for the sound. On the HTML5 target this will have no effect as the target does not support more than one listener.

보고:

Sound Instance ID

예 1:

if health <= 0
{
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
}

위의 코드는 "health" 전역 변수를 확인하고 0보다 작거나 같으면 "lives" 전역 변수에서 1을 제거하고 소리를 재생합니다.

예 2:

if bbox_left > room_width
{
    audio_play_sound(snd_Goodbye, 10, false, 1.1, 0, 2);
}

위의 코드는 인스턴스가 방의 오른쪽 경계를 넘었는지 확인하고 게인 1.1과 피치 2(일반 피치의 두 배)로 사운드를 재생합니다.
