audio_sync_group_debug

그의 함수는 주어진 동기화 그룹에 대한 디버그 정보를 표시하는 데 사용할 수 있습니다(그룹 인덱스는 함수를 사용하여 그룹을 만들 때 반환된 값입니다.audio_create_sync_group() ). -1 값으로 함수를 호출하여 오버레이를 끌 수 있습니다.

오디오 동기화 그룹을 디버깅할 때 오버레이에 다음이 표시됩니다.

  • 디버깅 중인 동기화 그룹
  • 몇 개의 채널이 있는지
  • 동기화 그룹이 실패한 횟수(즉, 너무 많은 작업을 시도했음을 의미)
  • 채널 목록
    • 빨간색은 이 채널이 현재 작업을 하고 있지 않음을 의미합니다.
    • 노란색은 채널이 초기 상태에 있음을 의미합니다(아직 디코딩 또는 재생되지 않음).
    • 녹색은 트랙이 현재 탐색/디코딩 중임을 의미합니다.
    • 보라색은 채널에 데이터가 부족함을 의미합니다.
  • 디코딩 스레드가 수행하는 작업을 보여주는 프로파일링 막대
    • 파란색은 파일을 처음으로 읽고 있음을 의미합니다.
    • 빨간색은 스레드가 읽을 위치를 찾기 위해 파일을 읽고 있음을 의미합니다.
    • 녹색은 스레드가 데이터를 디코딩 중임을 의미합니다.
    • 보라색은 스레드가 작동해야 할 때까지 잠자기 상태임을 의미합니다.

오디오의 일반적인 디버깅을 위해 다음 기능도 있습니다.audio_debug().

참고: 이 기능은 HTML5 대상 플랫폼에서 사용할 수 없습니다.

통사론:

audio_sync_group_debug(group_index);

논쟁 유형 설명
group_index 오디오 동기화 그룹 ID The group index to debug, or -1 to switch off the debugging.

예시:

if (debug_mode)
{
    audio_sync_group_debug(sg);
}
else
{
    audio_sync_group_debug(-1);
}

위의 코드는 게임이 디버그 모드에서 실행되는지 여부에 따라 변수 "sg"에 인덱싱된 동기화 그룹을 표시하는 오디오 동기화 그룹 디버그 오버레이를 켜거나 끕니다.
