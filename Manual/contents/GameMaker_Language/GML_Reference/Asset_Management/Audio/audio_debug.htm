audio_debug

이 기능은 오디오 시스템에 대한 디버그 정보를 표시하는 데 사용할 수 있습니다. true 그것을 켜고 false 끄려면. 이러한 방식으로 오디오를 디버깅하면 오버레이에 다음 정보가 표시됩니다.

  • 핸들 - OpenAL 소스 핸들(재생되는 모든 사운드에 대해 고유)
  • 데이터 핸들 - OpenAL 버퍼 핸들(동일한 소스 핸들을 가진 여러 사운드에 대해 동일한 값 표시)
  • 처리된 버퍼 - 재생된 버퍼 수
  • Queued Buffers - 얼마나 많은 버퍼가 큐에 들어갔는지(그리고 여전히 재생 대기 중)
  • 게인 - 오디오의 게인 값(0 - 65536)
  • 재생 오프셋 - 현재 버퍼로의 샘플 위치(현재 재생 위치)
  • Queued Buffers Bars - 파란색 선으로 그려진 재생 오프셋(현재 위치)이 있는 처리/대기 중인 버퍼의 작은 상자 다이어그램

동기화된 오디오 그룹의 경우 다음 기능도 있습니다.audio_sync_group_debug().

참고 이 기능은 HTML5 대상 플랫폼에서 작동하지 않습니다.

통사론:

audio_debug(enable);

논쟁 유형 설명
enable Enable (set to true ) or disable (false ) audio debugging.

보고:

예시:

if (debug_mode)
{
    audio_debug(true);
}
else
{
    audio_debug(false);
}

위의 코드는 게임이 디버그 모드에서 실행 중인지 여부에 따라 오디오 디버그 오버레이를 켜거나 끕니다.
