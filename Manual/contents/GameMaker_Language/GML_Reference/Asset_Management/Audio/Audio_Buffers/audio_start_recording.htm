audio_start_recording

이 기능은 인덱싱된 레코더 소스에서 오디오 녹음을 시작합니다. 함수를 사용하여 레코더 소스의 수를 얻을 수 있습니다audio_get_recorder_count(), 그리고 녹음을 시작하면 오디오가 임시 버퍼에 저장되고 오디오 녹음 비동기 이벤트 트리거가 시작됩니다. 이 이벤트는 기록이 발생하는 모든 단계에서 트리거되며 변수에 특수 DS 맵 을 생성합니다. async_load 다음 키/값 쌍으로:

  • " buffer_id " - 오디오 데이터를 검색하는 데 사용할 수 있는 임시 버퍼의 ID
  • " channel_index " - 이 데이터를 가져온 기록 채널 인덱스(함수에 의해 반환됨)
  • " data_len " - 수신한 데이터의 길이(바이트)

비동기 이벤트가 트리거된 후 임시 버퍼의 오디오가 지워지므로 해당 데이터를 이전에 생성한 사용자 지정 버퍼에 복사해야 합니다.

참고 대부분의 플랫폼은 어떤 형태로든 오디오 녹음을 지원하지만 플랫폼이 지원하더라도 모든 장치에서 허용한다는 의미는 아니므로 항상 다음을 확인해야 합니다.audio_get_recorder_count() 이 함수는 나머지 녹음 기능을 사용하기 전에 녹음 장치를 사용할 수 있는지 확인하기 위해 0보다 큰 값을 반환합니다.

통사론:

audio_start_recording(recorder_index);

논쟁 유형 설명
recorder_index 진짜 The index of the recorder source to use.

보고:

Buffer ID

예시:

audio_record = audio_start_recording(0);

위의 코드는 레코더 소스 0에서 녹음을 시작하여 비동기 오디오 녹음 이벤트에 사용하기 위해 "audio_record" 변수에 녹음의 채널 인덱스를 저장합니다.
