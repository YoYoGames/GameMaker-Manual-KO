audio_create_buffer_sound

이 기능을 사용하면 버퍼의 내용에서 새 사운드를 만들 수 있습니다. 버퍼는 이전에 생성되었으며(이를 수행하는 방법에 대한 자세한 내용은 버퍼 함수 참조) 데이터가 추가되거나 로드되었습니다. 그런 다음 데이터 형식으로 이 함수에 전달합니다. buffer_u8 또는 buffer_s16 현재 지원됨), 샘플 속도(1000hz와 48000hz 사이일 수 있음), 데이터를 가져올 버퍼로의 오프셋. 또한 버퍼의 샘플 수와 사운드에 필요한 채널을 제공해야 합니다. 이러한 채널은 다음 상수 중 하나로 정의됩니다.

오디오 채널 유형
일정한 설명
audio_mono 모노(단일 채널) 오디오.
audio_stereo 스테레오(듀얼 채널) 오디오.
audio_3D 3D(5.1) 오디오.

사운드를 생성한 후 함수를 사용하여 더 이상 필요하지 않을 때 사운드와 연결된 포인터 인덱스를 해제해야 합니다.audio_free_buffer_sound(). 이렇게 하지 않고 변수를 다시 할당하거나 방을 변경하는 등의 작업을 수행하면 사운드 ID가 손실되고 메모리 누수가 발생합니다. 또한 버퍼에서 사운드가 생성되었고 사운드가 먼저 해제되지 않은 경우 버퍼를 삭제할 수 없습니다. 따라서 먼저 사운드(또는 사운드)를 해제한 다음 버퍼를 삭제합니다. 또한 버퍼에 무엇이든 추가하거나 버퍼 크기를 변경하면 사운드가 생성된 후 예기치 않은 결과가 발생하므로 권장하지 않습니다. 버퍼에서 사운드 생성을 시작한 후에는 조작해서는 안 됩니다. 나중에 다른 방법으로.

통사론:

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);

논쟁 유형 설명
bufferId 버퍼 ID The ID of the buffer to use.
bufferFormat 버퍼 데이터 유형 상수 The format of the data in the buffer (buffer_u8 or buffer_s16 ).
bufferRate 진짜 The sample rate of the data in the buffer.
bufferOffset 진짜 The offset into the buffer to read the sample data from (in bytes).
bufferLength 진짜 The length of the buffer (the number of bytes of audio data, excluding the header).
bufferChannels 오디오 채널 유형 The channels to use from one of the constants listed above.

보고:

Sound Asset

예시:

var rate = 44100;
var hertz = irandom_range(220, 880);
var samples = 44100;

var bufferId = buffer_create(rate, buffer_fast, 1);
buffer_seek(bufferId, buffer_seek_start, 0);

var num_to_write = rate / hertz;
var length = buffer_get_size(bufferId);
var val_to_write = 1;

for (var i = 0; i < (samples / num_to_write) + 1; i++)
{
    for (var j = 0; j < num_to_write; j++)
    {
        buffer_write(bufferId, buffer_u8, val_to_write * 255);
    }
    val_to_write = (1 - val_to_write);
}

soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, length, audio_stereo);

위의 내용은 버퍼를 만든 다음 절차에 따라 데이터로 채웁니다. 그런 다음 이 데이터를 사용하여 "soundId" 변수에 저장되는 새 사운드를 생성합니다.
