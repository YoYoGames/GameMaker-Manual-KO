audio_listener_set_velocity

이 기능을 사용하여 주어진 청취자에게 도플러 효과를 주고 주어진 상대적 x, y 및 z 위치에서 해석되는 벡터를 기반으로 오디오 모션을 시뮬레이션할 수 있습니다. 기본 리스너 인덱스는 0이지만 함수를 사용할 수 있습니다.audio_get_listener_info() 대상 플랫폼에 사용할 수 있는 다양한 인덱스를 가져옵니다.

주어진 리스너가 절대 움직이지 않거나 움직임이 일정한 움직임이 아닌 경우 일반적으로 이러한 값을 설정할 필요가 없지만 예를 들어 플레이어가 바닥에 일정한 위치에 있는 스크롤링 게임을 만드는 경우 위쪽 이동 및 적들이 일정한 위쪽에서 아래쪽으로 이동하는 경우 리스너 및 이미터 속도를 설정합니다(이미터의 경우 사용할audio_emitter_velocity() )를 적절한 벡터에 추가하여 플레이어 인스턴스를 지나갈 때 올바른 도플러 효과를 시뮬레이션합니다.

아래 이미지는 위에 제공된 예제 게임에 대해 이것이 어떻게 설정될 수 있는지 보여줍니다.

통사론:

audio_listener_set_velocity(index, x, y, z);

논쟁 유형 설명
index 실제 또는 오디오 리스너 ID The listener to change the velocity of (default 0).
x 진짜 The x velocity of the listener (default 0).
y 진짜 The y velocity of the listener (default 0).
z 진짜 The z velocity of the listener (default 0).

보고:

N/A

예시:

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
{
    var info = audio_get_listener_info(i);
    var data = audio_listener_get_data(info[? "index"]);
    if data[? "vx"] != 0
    {
        audio_listener_set_velocity(info[? "index"], 0, 0, 0);
    }
    ds_map_destroy(info);
    ds_map_destroy(data);
}

위의 코드는 사용 가능한 리스너의 수를 확인한 다음 루프를 돌며 x 속도가 0이 아니면 속도 값을 0, 0, 0으로 설정합니다.
