path_start

경로는 함께 연결되어 인스턴스의 이동을 계획하는 데 사용되는 일련의 정의 지점에서 생성됩니다. 코드로 만들 거나 경로 편집기에서 만들 수 있으며 게임에서 사용할 인스턴스에 할당됩니다. 그런 다음 이 함수를 사용하여 인스턴스에 따라갈 경로, 경로를 따라야 할 속도(단계당 픽셀 단위로 측정), 경로 끝에 도달했을 때 동작하는 방법, 절대 경로 또는 상대 경로를 따를지 여부를 알려줍니다. 위치. 이 마지막 부분은 경로 편집기 에 디자인하고 배치한 대로 정확히 경로를 시작하고 따르거나(절대) 인스턴스가 생성된 위치(상대)에서 경로를 시작하고 따릅니다.

위 이미지에서 볼 수 있듯이 두 인스턴스는 다음으로 시작하는 동일한 경로를 따르고 있습니다. path_start(), 그러나 인스턴스 중 하나는 절대 경로를 따르고 다른 하나는 상대 경로를 따릅니다. 상대가 false(절대)로 설정되면 인스턴스는 방의 어디에 배치되든 경로의 시작 부분으로 "점프"합니다.

경로 끝에 도달할 때 인스턴스의 동작을 정의하는 데 사용되는 상수는 이 페이지에 나와 있습니다.

통사론:

path_start(path, speed, endaction, absolute);

논쟁 유형 설명
path 경로 자산 The path index to start.
speed 진짜 The speed of which to follow the path in pixels per step, negative meaning going backwards.
endaction 경로 종료 동작 상수 What to do when the end of the path is reached (see the constants listed above).
absolute 부울 Whether the calling instance should follow the absolute path as it is in the editor (true) or a relative path to its current position (false).

보고:

N/A

예시:

path_start(path, 4, path_action_reverse, 0);

이렇게 하면 호출 인스턴스가 단계당 4픽셀의 속도로 변수 "경로"에 인덱싱된 경로를 따르게 됩니다. 자신의 위치를 기준으로 경로를 따라가다가 끝에 도달하면 방향을 바꿔 왔던 길로 되돌아갑니다.
