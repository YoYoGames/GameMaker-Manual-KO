script_execute

이 함수는 원래 동적 방식으로 스크립트 자산 을 호출하도록 설계되었지만 사용자 정의 함수가 도입되면서 이제 두 가지 방법으로 사용할 수 있습니다.

  • 첫 번째 방법은 각 스크립트가 단일 함수이고 스크립트 제목이 함수 이름인 레거시 스크립트를 지원하는 것입니다. 이런 식으로 사용하면 이전과 같이 작동하고 사용자가 만든 스크립트 자산을 실행하고 주어진 인수를 전달합니다. 이 경우 실제로 스크립트 를 호출하는 것이 아니라 스크립트 내에서 동일한 이름의 스크립트 함수 를 호출한다는 점에 유의하십시오.
  • 두 번째 방법은 여러 함수를 포함하는 스크립트 자산에서 함수를 호출하는 것입니다. 이 경우 스크립트 자산 이름 대신 함수 이름과 필요한 인수를 제공합니다. 스크립트 기능에만 할당된 스크립트 ID가 있기 때문에 이것은 메서드 가 아닌 스크립트 기능 에 대해서만 작동합니다.

이 기능은 주로 레거시 지원을 위해 설계되었지만 네트워킹 및 원격 프로시저 호출 상황에서 여전히 사용되기 때문에 더 이상 사용되지 않습니다. 또한 이 기능은 느리고 많이 사용하면 성능에 부정적인 영향을 줄 수 있다는 점에 유의해야 합니다.

참고 이 기능은 내장 기능과 함께 사용할 수 없습니다.

통사론:

script_execute(scr, arg0, arg1, arg2..., etc...);

논쟁 유형 설명
scr 스크립트 자산 또는 스크립트 기능 The function/script that you want to call
arg0, arg1, arg2..., etc... 변하기 쉬운 The different arguments that you want to pass through to the function/script

보고:

Variable (Will depend on the return value from the script/function being called)

예시:

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

위의 예제 코드는 script_execute 4개의 사용자 정의 함수(이 경우 이동과 관련됨) 중 하나를 선택하고 첫 번째 인수로 임의의 정수를 전달합니다.
