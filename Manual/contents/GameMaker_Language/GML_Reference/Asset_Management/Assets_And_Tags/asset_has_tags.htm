asset_has_tags

이 기능을 사용하면 자산 브라우저에서 자산에 하나 이상의 태그 문자열이 할당되었는지 확인할 수 있습니다. 자산 이름(문자열) 또는 자산 색인은 물론 단일 태그 문자열 또는 각 항목이 단일 태그 문자열인 배열을 제공합니다. 자산 인덱스 값을 제공하는 경우 선택적인 자산 유형 인수(상수)를 제공해야 합니다. 다른 유형의 자산은 동일한 이름을 가질 수 없더라도 동일한 인덱스를 가질 수 있기 때문입니다. 사용 가능한 자산 유형은 아래 표에 나열되어 있습니다.

일정한 설명
asset_object 주어진 이름은 객체 를 나타냅니다.
asset_sprite 주어진 이름은 스프라이트 를 나타냅니다.
asset_sound 주어진 이름은 소리 를 나타냅니다.
asset_room 주어진 이름은 room 을 나타냅니다.
asset_tiles 주어진 이름은 타일 세트 를 나타냅니다.
asset_path 주어진 이름은 경로 를 나타냅니다.
asset_script 주어진 이름은 script 를 나타냅니다.
asset_font 주어진 이름은 글꼴 을 나타냅니다.
asset_timeline 주어진 이름은 타임 라인 을 나타냅니다.
asset_shader 주어진 이름은 셰이더 를 나타냅니다.
asset_animationcurve 주어진 이름은 애니메이션 곡선 을 나타냅니다.
asset_sequence 주어진 이름은 Sequence 를 참조합니다.

함수가 성공하고 자산에 대해 지정된 모든 태그가 있으면 반환됩니다. true 그렇지 않으면 반환됩니다 false. 모든 태그가 아닌 선택된 태그 중 하나를 확인해야 하는 경우 함수를 사용하십시오. asset_has_any_tag().

통사론:

asset_has_tags(name_or_index, tags, [asset_type]);

논쟁 유형 설명
name_or_index 문자열 또는 자산 The name of the asset (a string) or its index value (an integer).
tags 문자열 또는 문자열 배열 A single asset tag string or an array with various asset tags.
[asset_type] 자산 유형 상수 OPTIONAL! The type of asset to check the tags for, only used when supplying an index for the first argument.

보고:

Boolean

예시:

var _a = array_create(3);
_a[0] = "enemy";
_a[1] = "level_" + string(global.Level);
_a[2] = "boss";
if asset_has_tags(object_index, _a, asset_object)
{
    instance_create_layer(0, 0, "Overlay", obj_Boss_Text);
}

위의 코드는 태그 배열을 생성한 다음 모든 태그가 지정된 객체에 적용되었는지 확인하고 적용된다면 다른 객체의 인스턴스를 생성합니다.
