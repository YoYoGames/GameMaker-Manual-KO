sprite_save

이 기능은 스프라이트의 모든 하위 이미지를 디스크에 저장하고 지정된 파일 이름을 지정하는 데 사용할 수 있습니다. 스프라이트는 런타임에 추가 되어야 하며(IDE를 통해 추가된 스프라이트는 저장할 수 없음) 파일은 *.png 확대.

참고: 선택한 대상 플랫폼에 따라 파일을 저장하고 로드할 수 있는 위치가 제한됩니다. 자세한 내용은 파일 처리 를 참조하십시오.

통사론:

sprite_save(ind, subimg, fname);

논쟁 유형 설명
ind 스프라이트 자산 The index of the sprite to save.
subimg 진짜 The sub-image of the sprite to save.
fname 끈 The filename for the saved sprite.

보고

N/A

예시:

var surf, spr_custom;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);
sprite_save(spr_custom, 0, "Player_Custom_Sprite.png");
sprite_delete(spr_Custom);

위의 코드는 표면을 생성하고 그 인덱스를 지역 변수 "surf"에 저장합니다. 그런 다음 해당 표면을 대상으로 지정하고 지우고 서로 위에 여러 스프라이트를 그립니다. 표면에 그려진 합성 이미지에서 새 스프라이트를 만들고 표면에서 사용하는 메모리를 해제하기 전에 해당 인덱스를 로컬 변수 "spr_Custom"에 할당합니다. 마지막으로 결과 스프라이트는 파일에 저장된 다음 메모리에서도 제거됩니다.
