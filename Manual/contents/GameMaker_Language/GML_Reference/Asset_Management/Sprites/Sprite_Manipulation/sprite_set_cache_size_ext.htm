sprite_set_cache_size_ext

이미지 블렌딩을 수행할 때 HTML5는 실행 파일이 수행되는 것과 같은 방식으로 동적으로 수행할 수 없습니다. 따라서 GameMaker 는 이미지의 혼합 사본을 임시로 저장하고 로드해야 합니다. 이 기능은 이전 항목을 덮어쓰기 전에 지정된 스프라이트의 혼합 사본을 캐시할 수 있는 수를 설정합니다. 기본값은 4입니다. 이것은 스프라이트의 주어진 단일 하위 이미지에 적용됩니다.

통사론:

sprite_set_cache_size_ext(ind, index, max);

논쟁 유형 설명
ind 스프라이트 자산 The index of the sprite to change the cache size of.
index 진짜 The sub-image of ind to change the cache size of.
max 진짜 The maximum number of cached copies of the sprite that can be stored.

보고

N/A

예시:

sprite_set_cache_size_ext(sprite0, 0, 2);

이것은 sprite0의 첫 번째 하위 이미지의 스프라이트 캐시를 2개의 복사본으로 설정합니다.
