sprite_add

이 기능을 사용하면 이미지를 스프라이트로 추가할 수 있으며 로드할 이미지 파일이 항상 둘 중 하나에 있어야 하는 외부 소스에서 로드할 수 있습니다. *.png, *.gif, *.jpg/jpeg 또는 *.json 형식(*json 파일은 Spine으로 만든 스켈레톤 애니메이션 스프라이트 를 로드하는 데 사용됩니다. 이 함수는 스프라이트와 관련된 모든 추가 코드에서 사용해야 하는 새 스프라이트 인덱스를 반환합니다. HTML5에서 이 함수를 사용하거나 URL에서 이미지를 가져오는 경우 이 함수는 Image Loaded 비동기 이벤트도 생성합니다. 자세한 내용은 이 페이지 를 참조하십시오.

애니메이션 스프라이트로 변환할 모든 이미지 - 다음을 제외하고 *.gif (아래 참조) - "스트립" 형식이어야 하며(아래 이미지 참조) sprite width = strip width / sub images 규칙에 따라 지정된 수의 하위 이미지로 분할됩니다.

위 이미지에서 볼 수 있듯이 스프라이트는 어두운 자주색 배경에 배치되었으며 "removeback" 인수를 다음으로 설정하여 제거할 수 있습니다. true. 이것은 스프라이트의 왼쪽 하단 픽셀에서 색상을 확인한 다음 제거할 색상으로 사용하는 방식으로 작동합니다. 예를 들어 위의 이미지에서 왼쪽 하단 픽셀 색상을 녹색으로 지정했다면 스프라이트의 모든 녹색 부분이 제거되고 나머지 보라색 배경은 무시되었을 것입니다.

"removeback" 옵션을 선택하면 GameMaker 가 "smooth" 인수를 다음으로 설정하여 스프라이트의 가장자리를 매끄럽게 만들 수도 있습니다. true. 이 모든 작업은 배경이 제거된 스프라이트의 가장자리 주위에 반투명 테두리를 만드는 것입니다.

마지막으로 스프라이트의 x 및 y 원점 을 지정할 수도 있습니다. 이것은 스프라이트가 그것을 사용하는 인스턴스에 "고정"되는 지점이며 항상 하나의 스프라이트 하위 이미지의 왼쪽 상단 모서리 0,0을 기준으로 계산됩니다. 예를 들어, 이 값이 (16,16)으로 설정된 32 x 32 픽셀의 스프라이트는 중앙에 원점이 있습니다.

추가하는 경우 *.gif 형식 이미지를 사용하는 경우 파일 이름과 이미지 번호(첫 번째 프레임의 경우 1이어야 함) 만 지정하고 GIF의 첫 번째 이미지만 로드 된다는 점에 유의하여 다른 모든 인수는 0으로 둘 수 있습니다. 을위한 *.json 스프라이트의 경우 이미지 번호는 1이어야 하며 "제거" 및 "부드럽게" 설정은 무시됩니다.

기본적으로 모든 새 스프라이트에는 경계 상자가 자동으로 계산됩니다(정확한 경계 상자는 스프라이트의 크기와 투명도에 따라 다름).sprite_collision_mask().

이런 식으로 스프라이트를 추가할 때 사용되는 메모리는 예상보다 클 것입니다. GameMaker 가 스프라이트를 텍스처 페이지 로 저장하고 GPU 메모리에도 저장하기 때문에 총 메모리가 동일한 스프라이트의 이미지 파일에 대해 예상되는 것보다 더 큽니다.

참고: GameMaker 에 스프라이트를 로드할 때 스프라이트를 다시 제거해야 합니다(sprite_delete() ) 더 이상 필요하지 않을 때 메모리 누수의 위험이 있어 속도가 느려지고 결국 게임이 중단됩니다.

참고: iOS 에서 폴더에서 포함된 파일을 로드하는 경우(예: "Backgrounds/background1.png ", 폴더를 경로의 일부로 포함할 필요가 없으며 단순히 파일 이름만 제공하면 됩니다. 다른 모든 플랫폼에서는 파일 경로의 일부로 폴더를 제공해야 합니다.

참고: HTML5 에서 보안 서버에서 스프라이트를 로드하는 경우 교차 출처 유형을 설정하고 절대 경로 대신 상대 경로를 사용해야 할 수도 있습니다. 보다 http_set_request_crossorigin() 자세한 사항은.

통사론:

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig);

논쟁 유형 설명
fname 끈 The name (a string file path) of the file to add.
imgnum 진짜 Use to indicate the number of sub-images (1 for a single image or for a *.gif ).
removeback 부울 Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth 부울 Indicates whether to smooth the edges if transparent.
xorig 진짜 Indicate the x position of the origin in the sprite.
yorig 진짜 Indicate the y position of the origin in the sprite.

예시:

spr = sprite_add("player_5.png", 16, true, true, 0, 0);

위의 코드는 게임에 스프라이트를 로드하고 "spr" 변수에 인덱스를 저장합니다.
