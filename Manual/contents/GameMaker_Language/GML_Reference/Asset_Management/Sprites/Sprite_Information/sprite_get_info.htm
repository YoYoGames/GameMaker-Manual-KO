sprite_get_info

이 함수는 주어진 스프라이트에 대한 정보를 검색하는 데 사용됩니다. 스프라이트 인덱스( 자산 브라우저 를 통해 추가된 자산 또는 런타임 에 추가 된 스프라이트일 수 있음)를 제공하면 함수가 다음 변수가 있는 구조체 를 반환합니다.

스프라이트 정보 구조체
변하기 쉬운 유형 설명
너비 진짜 스프라이트의 너비(픽셀)
키 진짜 스프라이트의 높이(픽셀)
xoffset 진짜 스프라이트의 X 오프셋/원점(픽셀 단위)
요오프셋 진짜 스프라이트의 Y 오프셋/원점(픽셀 단위)
투명한 부울true 스프라이트가 투명으로 표시된 경우, 그렇지 않으면 false
(이것은 다음을 통해서만 지정할 수 있습니다. sprite_add() 또는 유사한 기능, 그리고 false IDE에서 생성된 스프라이트의 경우)
매끄러운 부울true 스프라이트가 부드러운 것으로 표시된 경우, 그렇지 않은 경우 false
(이것은 다음을 통해서만 지정할 수 있습니다. sprite_add() 또는 유사한 기능, 그리고 false IDE에서 생성된 스프라이트의 경우)
유형 진짜 스프라이트의 유형:
0 - 비트맵(일반 스프라이트)
1 - SWF
2 - 척추
bbox_left 진짜 경계 상자의 왼쪽 가장자리 위치(픽셀 단위)
bbox_top 진짜 경계 상자의 위쪽 가장자리 위치(픽셀 단위)
bbox_right 진짜 경계 상자의 오른쪽 가장자리 위치(픽셀 단위)
bbox_bottom 진짜 경계 상자의 아래쪽 가장자리 위치(픽셀 단위)
이름 끈 스프라이트의 이름
num_subimages 진짜 스프라이트의 하위 이미지(또는 프레임) 수
use_mask 부울true 이 스프라이트가 충돌 마스크를 사용하는 경우, 그렇지 않으면 false
num_masks 진짜 이 스프라이트의 마스크 수(스프라이트가 프레임당 마스크를 사용하는 경우 1보다 큼)
나인슬라이스 구조체 이 스프라이트의 Nine Slice 구조체, 또는 undefined 9개의 슬라이스 데이터가 없는 경우
메시지 정렬 각 브로드캐스트 메시지는 메시지에 대한 정보를 포함하는 구조체인 이 스프라이트에 대한 브로드캐스트 메시지 배열입니다(자세한 내용은 "일반 스프라이트 데이터" 참조).
프레임 정보 정렬 이 스프라이트의 프레임 배열로, 각 프레임은 타이밍에 대한 정보를 포함하는 구조체입니다(자세한 내용은 "일반 스프라이트 데이터" 참조).

이 추가 변수는 Bitmap(일반) 스프라이트에만 사용할 수 있습니다.

스프라이트 정보 구조체
변하기 쉬운 유형 설명
프레임 정렬 이 스프라이트의 프레임 배열로, 각 프레임은 텍스처에 대한 정보를 포함하는 구조체입니다(자세한 내용은 "비트맵 스프라이트 데이터" 참조).

다음 추가 변수는 Spine 스프라이트에만 사용할 수 있습니다.

스프라이트 정보 구조체
변하기 쉬운 유형 설명
num_atlas 진짜 사용된 아틀라스 텍스처의 수
atlas_texture 정렬 아틀라스에 사용되는 텍스처 ID의 배열
미리 곱한 부울true 이 스프라이트가 미리 곱한 것으로 표시되어 있으면 그렇지 않으면 false
animation_names 정렬 이 스프라이트의 각 애니메이션 이름을 포함하는 배열
skin_names 정렬 이 스프라이트의 각 스킨 이름을 포함하는 배열
뼈 정렬 이 스프라이트의 각 뼈에 대한 구조체를 포함하는 배열(자세한 정보는 "스파인 스프라이트 데이터" 참조)
슬롯 정렬 이 스프라이트의 각 슬롯에 대한 구조체를 포함하는 배열(자세한 정보는 "스파인 스프라이트 데이터" 참조)

함수가 반환됩니다 undefined 주어진 스프라이트가 존재하지 않는 경우. 또한 이 구조체에서 반환된 정보는 이러한 변수를 수정해도 스프라이트에 영향을 미치지 않으므로 읽기 전용 으로 간주해야 합니다.

아래 섹션에는 스프라이트 유형에 따라 반환된 구조체에 포함된 배열 및 구조체에 대한 정보가 포함되어 있습니다.

일반 스프라이트 데이터

이 섹션에는 모든 종류의 스프라이트에 대한 구조체에 포함된 변수에 대한 정보가 포함되어 있습니다.

그만큼 messages 변수는 주어진 스프라이트에 존재하는 브로드캐스트 메시지에 대한 정보를 포함하는 배열입니다. 이 배열의 각 브로드캐스트 메시지는 다음 변수를 포함하는 구조체입니다.

변수 이름 데이터 형식 설명
액자 진짜 애니메이션 시작부터 이 브로드캐스트 메시지의 타이밍(프레임 단위)
메시지 끈 브로드캐스트 메시지 문자열

그만큼 frame_info 변수는 스프라이트 프레임의 타이밍에 대한 정보를 포함하는 배열입니다. 이 배열의 각 프레임은 다음 변수를 포함하는 구조체입니다.

변수 이름 데이터 형식 설명
액자 진짜 이 프레임의 시작 타이밍(프레임 단위)
지속 진짜 이 프레임의 지속 시간(프레임 단위)
이미지 인덱스 진짜 이 프레임의 이미지 인덱스

비트맵 스프라이트 데이터

그만큼 frames 변수는 스프라이트 프레임의 텍스처에 대한 정보를 포함하는 배열입니다. 이 배열의 각 프레임은 다음 변수를 포함하는 구조체입니다.

y_offset 진짜 원본 이미지의 위쪽 가장자리에서 잘린 부분의 위쪽 가장자리까지의 Y 오프셋

통사론:

sprite_get_info(index);

논쟁 유형 설명
index 스프라이트 자산 The index of the sprite to get the information for.

보고

Sprite Info Struct (or undefined)

예시:

var _info = sprite_get_info(sprite_index);

if (_info != undefined && _info.type == 0)
{
    var _messages = _info.messages;
    var _messageCount = array_length(_messages);
    
    for (var i = 0; i < _messageCount; i ++)
    {
        var _message = _messages[i];
        show_debug_message("Message at frame " + string(_message.frame) + ": " + _message.message);
    }
}

위의 코드는 인스턴스의 스프라이트에 대한 정보 구조체를 가져온 다음 그렇지 않은지 확인합니다. undefined 유형이 0(비트맵 스프라이트임을 의미)입니다. 이 경우 해당 스프라이트에 대한 브로드캐스트 메시지 배열을 가져온 다음 for 루프를 실행하여 각 브로드캐스트 메시지(프레임과 함께)를 출력 로그에 인쇄합니다.
