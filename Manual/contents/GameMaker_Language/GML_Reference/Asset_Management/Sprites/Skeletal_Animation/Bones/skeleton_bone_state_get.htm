skeleton_bone_state_get

뼈대 애니메이션은 애니메이션 프로그램에서 정의하고 이름을 지정한 여러 "뼈"로 구성되며 이 기능을 사용하여 언제든지 명명된 뼈에 대한 특정 데이터를 얻을 수 있습니다. 이 데이터는 사용된 애니메이션 세트에 따라 스켈레톤의 현재 포즈를 참조하며, 이 함수에는 이전에 생성된 DS 맵 이 필요합니다. 그러면 함수 호출 후 다음 키와 해당 값이 포함됩니다.

  • "x": 상위 뼈대를 기준으로 한 뼈대의 로컬 x 위치입니다.
  • "y": 상위 뼈대를 기준으로 한 뼈대의 로컬 y 위치입니다.
  • "각도": 상위 뼈에 대한 뼈의 로컬 회전입니다.
  • "xscale": 부모 뼈를 기준으로 뼈의 로컬 수평 스케일입니다.
  • "yscale": 부모 뼈를 기준으로 뼈의 로컬 수직 스케일입니다.
  • "worldScaleX": 월드 스케일 X의 크기(항상 양수)입니다( 읽기 전용 값임).
  • "worldScaleY": 월드 스케일 Y의 크기(항상 양수)( 읽기 전용 값임).
  • "worldAngleX": X축의 세계 회전입니다( 읽기 전용 값임).
  • "worldAngleY": Y축의 세계 회전입니다( 읽기 전용 값임).
  • "worldX": 세계 X 위치( 읽기 전용 값임).
  • "worldY": 세계 Y 위치( 읽기 전용 값임).
  • "appliedAngle": 스켈레톤 뼈의 포즈를 지정하는 데 사용되는 로컬 회전입니다.
  • "parent": 부모 뼈의 이름(문자열)입니다.

반환된 지도 데이터는 다음을 사용할 때 기본 포즈에 대해 반환된 것과 유사합니다.skeleton_bone_data_get(), 이제 추가 "세계" 키가 있습니다. 이것들은 뼈대 애니메이션 스프라이트의 루트 (원점)에 상대적인 뼈의 위치를 참조하고 반환된 값은 다음과 같은 내장 스프라이트 변수를 설정하여 수행된 크기 조정 또는 회전을 고려하지 않습니다. image_xscale 또는 image_angle. 세계 값은 읽기 전용 이며 설정할 수 없습니다.

이 기능은 애니메이션 데이터가 화면에 그려지기 전에 "가로채기"하고 수정할 수 있도록 제공되며, 따라서 이 이벤트는 그리기 직전에 트리거되기 때문에 기타 - 애니메이션 업데이트 이벤트에서 사용하고 싶을 것입니다. 이벤트.

통사론:

skeleton_bone_state_get(bone, map);

논쟁 유형 설명
bone 끈 The name (as a string) of the bone.
map DS 맵 ID The (previously created) DS map that stores the bone data.

보고:

N/A

예시:

var map = ds_map_create();
skeleton_bone_state_get("head", map);
var xx = ds_map_find_value(map, "worldX");
var yy = ds_map_find_value(map, "worldY");
var deltax = mouse_x - (x + xx);
var deltay = mouse_y - (y + yy);
var angle = -radtodeg(arctan2(deltay, deltax));
ds_map_replace(map, "angle", angle);
skeleton_bone_state_set("head", map);
ds_map_destroy(map);

위의 코드는 DS 맵을 만든 다음 "head"라는 뼈대의 데이터로 채웁니다. 그런 다음 뼈의 세계 위치를 추출하고 해당 데이터를 사용하여 뼈의 "각도"를 설정하여 게임에서 마우스 위치를 추적합니다.
