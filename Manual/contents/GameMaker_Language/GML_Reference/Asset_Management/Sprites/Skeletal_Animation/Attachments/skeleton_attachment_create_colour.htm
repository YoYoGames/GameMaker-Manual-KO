skeleton_attachment_create_colour

뼈대 애니메이션 스프라이트를 만들 때 첨부 슬롯 과 첨부 파일 을 할당할 수 있습니다. 이것들은 애니메이션과 별개인 단순한 이미지(스프라이트)이지만 부착되면 부착된 뼈와 함께 이동합니다. 일반적으로 애니메이션 프로그램(Spine)에서 이러한 첨부 파일을 할당하지만 이 기능을 사용하면 게임의 스프라이트 자산을 사용하여 런타임에 자신만의 첨부 파일을 만들 수 있습니다. 이 기능을 사용하려면 첨부 파일에 이름(문자열으로)을 지정한 다음sprite_index 그리고image_index x 및 y 원점(스프라이트 속성에서 스프라이트에 의해 정의된 것과 다를 수 있음)을 사용할 수 있으며 첨부될 때 이미지에 적용하려는 변환을 설정할 수도 있습니다. 이미지와 혼합할 색상과 알파(투명도) 값입니다. 첨부 파일이 성공적으로 생성되면 함수가 반환됩니다. 1 그렇지 않은 경우(잘못된 스프라이트 인덱스를 제공했거나 기본 스프라이트가 Spine 스프라이트가 아닌 경우) 반환됩니다. -1.

인스턴스 스프라이트는 블렌드 색상과 알파 설정을 가질 수 있습니다(image_blend 그리고image_alpha ), 부착 슬롯도 가능합니다(기능 참조skeleton_slot_colour_set() ), 그리고 슬롯에 할당된 어태치먼트 스프라이트가 가질 최종 색상과 알파는 이 모든 값의 합성이 됩니다.

통사론:

skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha);

논쟁 유형 설명
name 끈 The name (as a string) of the attachment to create.
sprite 스프라이트 자산 The sprite_index to get the attachment image from.
ind 진짜 The image_index to get the attachment image from.
xorigin 진짜 The x origin for the image being used.
yorigin 진짜 The y origin for the image being used.
xscale 진짜 The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale 진짜 The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot 진짜 The rotation of the image: 0 = normal, 90 = turned 90° counter-clockwise etc.
Colour 색깔 The colour value to use (A constant, integer or hex value).
Alpha 진짜 The alpha value to use (from 0 to 1).

보고:

Real (1 if successful, -1 if not)

예시:

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

위의 코드는 "slot_leftHand"라는 슬롯에 대해 현재 할당된 부착 이름을 확인하고 빈 문자열이 반환되면 새 스프라이트가 부착됩니다.
