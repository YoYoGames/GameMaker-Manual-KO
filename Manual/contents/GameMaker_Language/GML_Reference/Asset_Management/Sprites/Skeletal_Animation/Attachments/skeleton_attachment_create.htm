skeleton_attachment_create

뼈대 애니메이션 스프라이트를 만들 때 첨부 슬롯 과 첨부 파일 을 할당할 수 있습니다. 이들은 단순히 애니메이션과 별개의 이미지이지만 부착되면 부착된 뼈와 함께 이동합니다. 일반적으로 애니메이션 프로그램(Spine)에서 이러한 첨부 파일을 할당하지만 이 기능을 사용하면 게임의 스프라이트 자산을 사용하여 런타임에 자신만의 첨부 파일을 만들 수 있습니다. 이 기능을 사용하려면 첨부 파일에 이름(문자열)을 지정한 다음 사용할 sprite_index 및 image_index와 x 및 y 원점(이는 sprite 속성에서 sprite가 정의한 것과 다를 수 있음)을 설정해야 합니다. 또한 첨부할 때 이미지에 적용하려는 변환을 설정할 수도 있습니다. 첨부 파일이 성공적으로 생성되면 함수가 반환됩니다. 1 그렇지 않은 경우(잘못된 스프라이트 인덱스를 제공했거나 기본 스프라이트가 Spine 스프라이트가 아닌 경우) 반환됩니다. -1.

통사론:

skeleton_attachment_create(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot);

논쟁 유형 설명
name 끈 The name (as a string) of the attachment to create.
sprite 스프라이트 자산 The sprite_index to get the attachment image from.
ind 진짜 The image_index to get the attachment image from.
xorigin 진짜 The x origin for the image being used.
yorigin 진짜 The y origin for the image being used.
xscale 진짜 The horizontal scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
yscale 진짜 The vertical scaling of the image, as a multiplier: 1 = normal scaling, 0.5 is half etc...
rot 진짜 The rotation of the image. 0=normal, 90=turned 90 degrees counter-clockwise etc.

보고:

Real (1 if successful, -1 if not)

예시:

skeleton_attachment_create("sword", spr_Weapons, 0, 0, 80, 1, 1, 90);
skeleton_attachment_create("knife", spr_Weapons, 1, 0, 45, 1, 1, 90);
skeleton_attachment_create("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

위의 코드는 "slot_leftHand"라는 슬롯에 대해 현재 할당된 부착 이름을 확인하고 빈 문자열이 반환되면 새 스프라이트가 부착됩니다.
