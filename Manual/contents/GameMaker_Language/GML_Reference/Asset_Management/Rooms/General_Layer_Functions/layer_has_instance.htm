layer_has_instance

이 기능은 주어진 인스턴스가 현재 주어진 레이어에 할당되어 있는지 확인하는 데 사용할 수 있습니다. 레이어 ID를 제공합니다(다음을 사용하여 레이어를 생성할 때 얻는layer_create() ) 또는 레이어 이름(문자열로 - 성능에 영향을 미침) 및 확인할 인스턴스의 인스턴스 ID입니다. 당신은 또한 줄 수 있습니다 object_index (예: 자산 브라우저의 개체 이름) 및 함수는 해당 개체의 인스턴스가 지정된 레이어에 있는지 확인합니다. 함수가 반환됩니다 true 인스턴스가 레이어에 있고 false 그렇지 않은 경우.

통사론:

layer_has_instance(layer_id, instance_id)

논쟁 유형 설명
layer_id 문자열 또는 레이어 ID The unique ID value of the layer to target (or the layer name as a string)
instance_id 인스턴스 ID The unique instance ID or the object index of the instance to check for

보고:

Boolean

예시:

if !layer_has_instance(global.Bullet_Layer, obj_Bullet_Parent)
{
    layer_destroy(global.Bullet_Layer);
}

위의 코드는 주어진 레이어에 "obj_Bullet_Parent" 개체의 인스턴스가 포함되어 있는지 확인하고 그렇지 않은 경우 레이어를 파괴합니다.
