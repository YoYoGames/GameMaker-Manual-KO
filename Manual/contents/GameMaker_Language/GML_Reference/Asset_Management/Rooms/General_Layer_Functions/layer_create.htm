layer_create

이 기능은 현재 방 안에 새 레이어를 생성하는 데 사용됩니다. 새 레이어가 차지할 깊이를 지정하면 함수가 고유한 레이어 ID 값을 반환하며 이 값은 이후 기능에서 레이어에 액세스하는 데 사용할 수 있습니다. 또한 선택적 "name" 인수를 제공하여 레이어에 특정 이름(문자열으로)을 부여할 수 있으며 레이어 ID를 사용하여 레이어에 액세스하는 대신 이 이름 문자열을 사용할 수 있습니다. 이름 문자열을 사용하면 ID 값만 사용하는 것보다 성능에 더 큰 영향을 미친다는 점에 유의해야 합니다. 레이어 이름을 지정하지 않으면 생성된 레이어에 "_layer_XXX ", 여기서 "XXX"는 레이어에 고유한 이름을 지정하는 데 사용되는 16진수 값입니다.

깊이 값을 지정하면 레이어가 카메라에서 "멀수록" 더 높고 레이어가 카메라에 "가까울수록" 낮아집니다. 따라서 깊이가 -100, 0, 100인 3개의 레이어가 있는 경우 레이어는 100, 0, -100 순서로 그려지므로 "상단" 레이어(즉, 카메라 뷰에 가장 가깝고 다른 모든 것 위에 그려짐)는 -100 깊이의 레이어가 됩니다.

다음 이미지는 레이어에 대한 깊이가 작동하는 방식의 개략도를 보여줍니다.

단일 깊이에 둘 이상의 레이어를 할당할 수 있지만 동일한 깊이의 레이어가 주어진 순서로 그려지는 것을 보장할 수 있는 방법은 없으므로 하나의 레이어만 할당하는 것이 가장 좋습니다. 주어진 깊이까지. 또한 최소 및 최대 레이어 깊이는 -16000 ~ 16000이며 이 깊이를 벗어나는 것은 렌더링되지 않습니다. 해당 범위를 벗어난 깊이가 필요한 경우 함수를 사용해야 합니다.layer_force_draw_depth().

통사론:

layer_create(depth, [name])

논쟁 유형 설명
depth 진짜 The depth that the layer has to be created at.
name 끈 OPTIONAL The name to give the layer (a string).

보고:

Layer ID

예시:

global.Bullet_Layer = layer_create(-100);

위의 코드는 깊이 값이 -100인 새 레이어를 만들고 해당 ID 값을 전역 변수에 할당합니다.
