layer_set_target_room

이 함수를 호출하면 GameMaker 에 모든 추가 레이어 함수가 주어진 방에 적용되어야 한다고 말하는 것입니다. 이러한 방식으로 현재 룸이 아닌 룸에서 레이어 및 레이어 콘텐츠를 절차적으로 변경하거나 생성할 수 있습니다. 방에 레이어 또는 레이어 요소 추가를 마쳤으면 함수를 호출합니다.layer_reset_target_room() 방 목표를 재설정하려면(또는 방 인수 -1로 이 함수를 다시 호출하십시오). 이 기능은 현재 방이 아닌 다른 방 에서만 사용할 수 있으며, 현재 실행 중인 방이 아닌 다른 방에 레이어 및 레이어 요소를 추가/제거할 수 있도록 설계되었습니다.

경고! 다른 방을 대상으로 하는 동안 다음을 사용하여 인스턴스를 생성할 수 없다는 점을 제외하고 모든 일반 레이어 기능을 사용할 수 있습니다. instance_create_layer() 또는 instance_create_depth(), 레이어도 작동하지 않습니다 layer_add_instance() 는 가능하다.

통사론:

layer_set_target_room(room)

논쟁 유형 설명
room The room to target for all further layer functions

보고:

예시:

layer_set_target_room(rm_Game);
var l = layer_get_id("SpriteAssets");
repeat(50)
{
    var _x = irandom(1000);
    var _y = irandom(1000);
    layer_sprite_create(l, _x, _y, spr_Trees);
}
layer_reset_target_room();

위의 코드는 대상 방을 "rm_Game" 방으로 설정한 다음 해당 방의 "SpriteAssets"라는 레이어에 대한 레이어 ID를 가져옵니다. 그런 다음 이 레이어 ID는 레이어 대상이 현재 방으로 재설정되기 전에 50개의 임의 스프라이트 자산을 레이어에 추가하는 데 사용됩니다.
