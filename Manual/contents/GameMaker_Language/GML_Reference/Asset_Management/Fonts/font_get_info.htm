font_get_info

이 함수는 주어진 글꼴에 대한 정보를 검색하는 데 사용됩니다. 글꼴 인덱스( 자산 브라우저 를 통해 추가된 자산 또는 런타임 에 추가 된 글꼴일 수 있음)를 제공하면 함수가 다음 변수가 있는 구조체 를 반환합니다.

글꼴 정보 구조체
변수 이름 데이터 형식 설명
ascenderOffset 진짜 기준선에서 글꼴 상단까지의 최대 오프셋(픽셀)
size 진짜 글꼴의 대략적인 크기(픽셀 단위)
spriteIndex 스프라이트 자산 스프라이트에서 생성된 글꼴의 스프라이트 인덱스, 그렇지 않으면 -1
texture 바늘 글꼴이 스프라이트에서 생성된 경우 -1, 그렇지 않으면 글꼴의 텍스처 ID
name 끈 글꼴의 이름
bold 부울true 글꼴이 굵으면 그렇지 않으면 false (아래 참고 참조)
italic 부울true 글꼴이 기울임꼴이면 그렇지 않으면 false (아래 참고 참조)
glyphs 글꼴 문양 구조 글꼴의 각 글리프에 대한 정보를 포함하는 구조체(자세한 정보는 아래에 있음)

참고 bold 그리고italic 변수는 글꼴에 대한 사용자 설정만 반영하므로 기본적으로 굵게 또는 기울임꼴로 표시되는 글꼴에는 정확하지 않을 수 있습니다.

제공된 글꼴이 없으면 함수는 다음을 반환합니다. undefined.

또한 이러한 변수 또는 내부에 포함된 변수의 값을 변경하면 glyphs 구조체는 글꼴이 렌더링되는 방식을 변경하지 않으므로 이 정보는 읽기 전용 으로 간주되어야 합니다.

글리프 구조체

그만큼 glyphs 반환된 구조체의 변수는 자체적으로 구조체가 되며 글꼴에 포함된 각 글리프에 대한 정보를 포함합니다. 이 구조체의 각 변수는 해당 글리프에 대한 정보를 포함하는 구조체를 보유하는 글리프의 문자 이름이 됩니다. 이러한 중첩 구조체를 다음 형식(주 구조체에서 시작)으로 상상할 수 있습니다.

{
    glyphs:
    {
        A: {},
        B: {},
        C: {},
        // ...other glyphs
    },
    // ...other font info
}

다음 코드와 같이 글리프의 문자 이름을 사용하여 구조체에서 해당 정보를 검색할 수 있습니다.

var _font_info = font_get_info(Font1);

var _info_A = _font_info.glyphs[$ "A"]; // $ is a struct accessor

show_debug_message(_info_A);

위의 코드는 "A" 문자에 대한 글리프 구조체를 가져와서 출력 로그에 인쇄합니다. 문자열("A" ) 문자열을 보유하는 변수를 사용하여 런타임에 해당 변수를 통해 모든 글리프에 대한 정보를 얻을 수 있습니다.

글리프에 대해 반환된 구조체에는 다음 변수가 포함됩니다.

참고 HTML5에서 glyphs 구조체는 로드된 파일 글꼴(예: *.ttf 또는 *.otf ) 런타임에는 해당 글리프에 대한 정보가 없기 때문입니다.

글꼴 문양 구조
변수 이름 데이터 형식 설명
char 진짜 글꼴이 스프라이트에서 생성된 경우 해당 스프라이트의 글리프 이미지 인덱스가 되고, 그렇지 않으면 유니코드 문자 번호가 됩니다.

참고: 글꼴이 스프라이트에서 생성된 경우 이 아래의 모든 변수는 구조체에 존재하지 않습니다.
x 진짜 텍스처 페이지에서 글리프의 X 위치(텍셀)
y 진짜 텍스처 페이지에서 글리프의 Y 위치(텍셀)
w 진짜 텍스처 페이지의 글리프 너비(텍셀)
h 진짜 텍스처 페이지의 글리프 높이(텍셀)
shift 진짜 다음 문자로 이동할 때 오른쪽으로 이동할 픽셀 수(왼쪽으로 이동할 때 음수일 수 있음)
offset 진짜 이 글리프의 렌더링을 수평으로 오프셋할 픽셀 수(양수 또는 음수일 수 있음)
kerning 정렬 커닝 정보를 쌍(또는 2개 그룹)으로 포함하는 정수 배열입니다. 쌍의 첫 번째 정수는 문자에 대한 유니코드 값이고 두 번째 정수는 이 문자 모양의 문자가 선행하는 경우 해당 문자의 시프트 값(양수 또는 음수일 수 있음)에 추가할 양입니다.

참고 런타임에 로드된 파일 글꼴의 경우x 그리고 y 글리프의 값은 아직 렌더링되지 않은 경우 -1이 될 수 있습니다. 캐싱에 사용되는 텍스처 페이지에서 위치가 없기 때문입니다.

통사론:

font_get_info(font);

논쟁 유형 설명
font 글꼴 자산 The font for which information should be retrieved.

보고:

Font Info Struct (or undefined)

예시:

var _info = font_get_info(my_font);

if (_info != undefined && _info.bold)
{
    image_xscale *= 2;
    image_yscale *= 2;
}

이 코드는 파일에 저장된 글꼴에 대한 정보를 검색합니다. my_font 변하기 쉬운. 반환된 값이 아닌 경우undefined 주어진 글꼴이 굵게 표시되면 인스턴스의 크기가 두 배가 됩니다.
