font_texture_page_size

이 내장 변수 는 함수를 사용할 때 텍스처 페이지 크기를 가져오거나 설정하는 데 사용할 수 있습니다.font_add(). 해당 기능을 사용하여 글꼴을 추가하면 GameMaker 는 이 변수에 의해 정의된 크기까지 필요한 글리프의 텍스처 페이지 캐시를 생성합니다. 프로세스는 다음과 같습니다.

  • 사용하는 각 문자는 글꼴에서 텍스처 페이지로 캐시됩니다.
  • 텍스처 페이지가 가득 차면(큰 캐릭터나 많은 다른 캐릭터를 사용하기 때문에) GameMaker 는 캐시에서 캐릭터를 제거하고 필요한 새 캐릭터로 교체합니다.
  • GameMaker 는 모든 장치가 큰 텍스처를 처리할 수 있는 것은 아니기 때문에 텍스처를 너무 크게 만들지 않으려고 하고 텍스처에 문자를 제거했다가 다시 추가하면 CPU 시간이 걸리고 게임이 느리게 실행되기 때문에 텍스처를 너무 작게 만들지 않으려고 합니다.

GameMaker 가 사용하는 기본 1024px보다 크거나 작은 텍스처 페이지 크기가 필요하다고 생각되면 이 내장 변수를 사용하여 값을 설정할 수 있습니다. 사용하지 않는 경우 font_add() 기능을 사용하는 경우 IDE에 추가된 글꼴이 일반 텍스처 페이지( 글꼴 편집기 및 대상 플랫폼의 게임 옵션 에서 제어할 수 있음)에 캐시되므로 이를 설정해도 게임 성능에 영향을 미치지 않습니다.

통사론:

font_texture_page_size;

보고:

Real (maximum width/height in pixels)

예시:

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128);

위의 코드는 글꼴 캐시 텍스처 페이지를 최대 크기 512px로 설정한 다음 게임에 새 글꼴을 추가합니다.
