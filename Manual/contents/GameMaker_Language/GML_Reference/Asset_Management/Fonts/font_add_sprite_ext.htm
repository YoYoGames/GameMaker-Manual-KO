font_add_sprite_ext

이 기능으로 "스프라이트 스트립"을 사용할 수 있습니다(스프라이트 자체 는 자산 브라우저의 스프라이트 자산이거나 다음을 사용하여 게임 리소스에 추가한 스프라이트여야 합니다. sprite_add ) 각 하위 이미지가 개별 기호 또는 문자가 되는 새 글꼴 자산을 생성합니다. 평소와 달리font_add_sprite() 스프라이트의 하위 이미지에 대한 특정 순서가 있는 경우 이 함수는 함수의 "string_map" 인수를 기반으로 스프라이트 하위 이미지를 매핑합니다. 이 인수는 GameMaker 에 스프라이트 글꼴의 하위 이미지 순서를 알려주는 데 사용할 수 있는 문자열이며 텍스트를 작성할 때 자동으로 매핑합니다. 따라서 예를 들어 "AaBbCcDdEeFfGgHh"의 문자열 맵이 있는 경우 스프라이트 글꼴에는 이러한 방식으로 정렬된 하위 이미지가 있어야 합니다. "공백"을 원하는 문자로 정의할 수 있습니다. 예를 들어 한 줄에 원하는 간격으로 지정하면 텍스트의 모든 공백이 해당 너비로 렌더링됩니다(그러나 선택한 이미지는 절대 렌더링되지 않음). 또는 공간에 대한 스프라이트를 제공하지 않으면 스프라이트 글꼴에서 가장 넓은 문자의 너비가 대신 사용됩니다.

글꼴의 간격을 비례하도록 설정할 수도 있습니다(true ) 여부(false ), 여기서 비례 글꼴은 문자의 실제 너비를 기준으로 간격을 두는 반면(예: 문자 "i"는 문자 "w"보다 공간을 적게 차지함) 비례하지 않는 글꼴은 하위 기준에 따라 간격이 지정됩니다. 고정 폭 글꼴과 같은 이미지 너비. 마지막으로, 쓸 때 각 문자 사이에 공백을 남길 수 있습니다. 이것은 임의의 정수 값이 될 수 있으며 0은 공백이 없습니다(문자가 비례하는 경우 "접촉").

이 함수는 이 글꼴을 참조하는 모든 추가 코드에 필요하므로 변수에 저장해야 하는 인덱스 값을 반환하거나 어떤 이유로든 함수가 실패하면 -1을 반환합니다.

글꼴 정렬 기능이 작동하려면(예:draw_set_halign() ) 그런 다음 글꼴 스프라이트의 원점은 왼쪽 상단 모서리 로 설정되어야 합니다. 다른 값을 사용하는 경우 글꼴을 사용하여 텍스트를 그릴 때 원점 오프셋을 고려해야 합니다.

참고 GameMaker 에 글꼴을 로드할 때 글꼴을 다시 제거해야 합니다(font_delete() ) 더 이상 필요하지 않을 때 메모리 누수의 위험이 있어 속도가 느려지고 결국 게임이 중단됩니다.

통사론:

font_add_sprite_ext(spr, string_map, prop, sep);

논쟁 유형 설명
spr 스프라이트 자산 The sprite to add a font based on.
string_map 끈 String from which sprite sub-image order is taken.
prop 부울 Set as proportional font or not.
sep 진짜 The space to leave between each letter.

보고:

Font Asset

예시:

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

위의 코드는 "spr_CalcFont" 변수에 인덱싱된 스프라이트에서 새 글꼴 자산을 만들고 변수 "global.Font"에 새 글꼴의 인덱스를 저장합니다. 스프라이트 글꼴의 하위 이미지는 지정된 문자열에 매핑됩니다.
