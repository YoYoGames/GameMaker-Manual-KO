font_add_sprite

이 기능을 사용하면 "스프라이트 스트립"을 사용하여 각 하위 이미지가 개별 기호 또는 문자가 되는 새 글꼴 자산을 만들 수 있습니다. 인덱스는 사용할 첫 번째 문자의 UTF8 맵 번호이며 사용할 범위는 "!"로 시작하는 일반 글꼴과 동일합니다. 그러나 필요한 것만 포함해야 하며ord() 시작 문자에 대한 올바른 UTF8 값을 가져오는 함수입니다. 그러면 생성된 스프라이트 글꼴의 하위 이미지가 글꼴의 개별 글리프로 설정되며, 여기서 "첫 번째" 글리프의 image_index는 0입니다. "공백"을 원하는 문자로 정의할 수 있습니다(예: 단일 원하는 공간의 크기를 줄이면 텍스트의 모든 공간이 해당 너비로 렌더링됩니다(그러나 선택한 이미지는 렌더링 되지 않음 ). 또는 공간에 대한 스프라이트를 제공하지 않으면 가장 넓은 너비의 너비 대신 스프라이트 글꼴의 문자가 사용됩니다.

글꼴의 간격을 비례하도록 설정할 수도 있습니다(true ) 여부(false ), 여기서 비례 글꼴은 문자의 실제 너비를 기준으로 간격을 두는 반면(예: 문자 "i"는 문자 "w"보다 공간을 적게 차지함) 비례하지 않는 글꼴은 하위 기준에 따라 간격이 지정됩니다. 고정 폭 글꼴과 같은 이미지 너비. 마지막으로, 쓸 때 각 문자 사이에 공백을 남길 수 있습니다. 이것은 임의의 정수 값이 될 수 있으며 0은 공백이 없습니다(문자가 비례하는 경우 "접촉").

이 함수는 이 글꼴을 참조하는 모든 추가 코드에 필요하므로 변수에 저장해야 하는 인덱스 값을 반환하거나 어떤 이유로든 함수가 실패하면 -1을 반환합니다.

또한 문자표에는 공백 이 없어야 하므로 예를 들어 '.' 및 '0'이지만 '/'가 아닌 경우, 어쨌든 '/'를 포함하거나 사용된 두 문자 사이에 빈 하위 이미지를 남겨두어야 합니다.

스프라이트 자체는 자산 브라우저의 스프라이트 자산이거나 다음을 사용하여 파일에서 로드된 자산일 수 있습니다. sprite_add 먼저 로드된 스프라이트의 인덱스를 가져옵니다(그 다음 함수에서 사용됨). 글꼴 정렬 기능(예:draw_set_halign() ) 스프라이트 글꼴을 사용할 때 예상대로 작동하려면 스프라이트의 원점이 왼쪽 상단 으로 설정되어야 합니다. 다른 값을 사용하는 경우 글꼴을 사용하여 텍스트를 그릴 때 원점 오프셋을 고려해야 합니다.

참고 GameMaker 에 글꼴을 로드할 때 글꼴을 다시 제거해야 합니다(font_delete() ) 더 이상 필요하지 않을 때 메모리 누수의 위험이 있어 속도가 느려지고 결국 게임이 중단됩니다.

통사론:

font_add_sprite(spr, first, prop, sep);

논쟁 유형 설명
spr 스프라이트 자산 The sprite to add a font based on.
first 진짜 The index of the first available character.
prop 부울 Set as proportional font or not.
sep 진짜 The space to leave between each letter.

보고:

Font Asset

예시:

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

위의 코드는 스프라이트 자산에서 새 글꼴 자산을 만들고 나중에 사용할 수 있도록 "global.Font" 변수에 새 글꼴의 인덱스를 저장합니다.
