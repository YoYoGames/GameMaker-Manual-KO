시퀀스 인스턴스 구조체

방에 배치될 때 시퀀스는 레이어의 요소 로 배치되며 이 요소에는 시퀀스 인스턴스 가 포함됩니다. 이 시퀀스 인스턴스는 구조체 에 다음 속성을 갖습니다.

시퀀스 인스턴스 구조체
변하기 쉬운 유형 설명
sequence 시퀀스 객체 구조체 이것은 시퀀스 인스턴스가 생성된 시퀀스 개체 구조체의 ID입니다(자세한 내용은 여기 참조). 이 구조체를 가져오거나 설정할 수 있으므로 인스턴스가 참조하는 기본 시퀀스를 변경할 수 있습니다.
headPosition 진짜 시퀀스 인스턴스의 현재 재생 헤드 위치(프레임 단위)입니다. 이 값을 얻거나 설정할 수 있지만 현재 정의된 재생 영역보다 크거나 작은 값으로 설정하면 시퀀스에 대해 설정된 재생 유형에 따라 다른 효과가 나타납니다. 보다layer_sequence_headpos() 자세한 내용은.
headDirection 시퀀스 방향 상수 이것은 시퀀스 인스턴스의 현재 재생 방향입니다. 다음 표에 제공된 상수를 사용하여 이 값을 가져오거나 설정할 수 있습니다. layer_sequence_get_headdir() 그리고layer_sequence_headdir() 구조체에 직접 액세스할 필요 없이 이 값을 가져오거나 설정합니다.
speedScale 진짜 이 속성은 재생 속도 스케일을 가져오거나 설정하는 데 사용할 수 있습니다. 속도 척도는 승수 입니다. 여기서 1은 기본 재생 속도이고 1보다 작은 값은 재생 속도를 늦추고 1보다 큰 값은 속도를 높입니다. 예: 값 0.5는 재생 속도의 절반이고 값 2는 재생 속도입니다. 두 배의 재생 속도가 될 것입니다. 레이어 기능을 사용할 수 있습니다.layer_sequence_get_speedscale() 그리고layer_sequence_speedscale()구조체에 직접 액세스할 필요 없이 이 값을 가져오거나 설정합니다.
paused 부울 시퀀스가 일시 중지되었는지 여부를 확인하기 위해 이 속성을 확인할 수 있습니다. true 그것이 있다면, 또는 false 그렇지 않으면. 이것은 읽기 전용 속성이지만 레이어 기능을 사용할 수 있습니다.layer_sequence_pause() 재생을 일시 중지하려면layer_sequence_play() 필요한 경우 다시 시작합니다. 레이어 기능을 사용하여 이 속성을 확인할 수도 있습니다.layer_sequence_is_paused() 구조체의 속성을 직접 확인하는 대신.
finished 부울 시퀀스가 재생을 완료했는지 여부를 확인하기 위해 이 속성을 확인할 수 있습니다. true 재생이 끝나면 false 그렇지 않으면. 이것은 읽기 전용 속성이며 다음에만 반환됩니다. true 루프 또는 핑퐁으로 설정되지 않은 트랙의 경우. 레이어 기능을 사용하여 이 속성을 확인할 수도 있습니다. layer_sequence_is_finished() 구조체의 속성을 직접 확인하는 대신.
elementID 시퀀스 요소 ID 이 속성은 시퀀스 요소 의 ID를 보유합니다. 이 ID는 룸 편집기의 레이어와 연결된 간단한 식별 값이며, 적절한 레이어 기능 과 함께 사용하여 시퀀스가 할당된 레이어를 찾거나 처리할 필요 없이 특정 시퀀스 속성을 변경할 수 있습니다. 구조체.
activeTracks 시퀀스 활성 트랙 구조체 의 배열 이 속성은 시퀀스의 각 자산 트랙 (그래픽, 시퀀스, 오디오 등...)의 현재 상태에 대한 정보를 포함하는 "평가" 구조체의 배열을 보유합니다. 이들은 최상위 트랙이며 반환된 배열은 읽기 전용 입니다. 각 배열 항목에서 반환되는 자산 트랙 구조의 내용은 아래 섹션을 참조하십시오.

시퀀스 방향 상수
끊임없는 설명 값
seqdir_right 시퀀스는 왼쪽에서 오른쪽으로 증분 순서로 프레임을 재생합니다. 1
seqdir_left 시퀀스는 오른쪽에서 왼쪽으로 감소하는 순서로 프레임을 재생합니다. -1

activeTracks 구조체

이 속성은 Sequence Create 이벤트의 시퀀스 인스턴스 구조체에 이미 존재하지만 첫 번째 Sequence Begin Step 이벤트 이후의 트랙으로만 채워집니다. 따라서 이 구조체에서 값을 처음 발견하는 것은 시퀀스 단계 이벤트의 첫 번째 반복 동안입니다.

다음 목록에는 활성 트랙 구조에 액세스할 때 사용할 수 있는 속성이 포함되어 있습니다( activeTracks 위에서 설명한 시퀀스 인스턴스 구조체의 배열). 각 자산 트랙 구조는 트랙이 사용하는 자산 유형에 따라 다음 속성 중 하나를 가질 수 있으며( 읽기 전용 으로 지정된 경우를 제외하고 이러한 모든 속성을 수정할 수 있음) 반환된 모든 값은 현재 재생 헤드 위치에 대한 것입니다.:

시퀀스 활성 트랙 구조체
변하기 쉬운 유형 설명
matrix 행렬 배열 이것은 트랙(스프라이트, 인스턴스 및 시퀀스 트랙)을 그리는 데 사용되는 변환 행렬입니다.
posx 진짜 현재 재생 헤드 위치(모든 자산)에 있는 트랙의 X축을 따라 시퀀스에서 자산의 위치입니다.
posy 진짜 현재 재생 헤드 위치(모든 자산)에 있는 트랙의 Y축을 따라 시퀀스에서 자산의 위치입니다.
scalex 진짜 현재 재생 헤드 위치(스프라이트, 인스턴스 및 시퀀스 트랙)에 있는 트랙의 X축을 따라 시퀀스에 있는 자산의 크기입니다.
scaley 진짜 현재 재생 헤드 위치(스프라이트, 인스턴스 및 시퀀스 트랙)에 있는 트랙의 Y축을 따른 시퀀스의 자산 배율입니다.
xorigin 진짜 트랙에 대한 자산의 X 원점(모든 자산).
yorigin 진짜 트랙에 대한 자산의 Y 원점(모든 자산).
width 진짜 트랙(스프라이트, 인스턴스 및 시퀀스 트랙)에 대한 자산의 너비(픽셀)입니다.
height 진짜 트랙(스프라이트, 인스턴스 및 시퀀스 트랙)에 대한 자산의 높이(픽셀)입니다.
imageindex 진짜 현재 재생 헤드 위치(스프라이트 및 인스턴스 트랙)에 있는 시퀀스의 트랙에 있는 자산에 대한 이미지 인덱스입니다.
imagespeed 진짜 현재 재생 헤드 위치(스프라이트 및 인스턴스 트랙)에서 시퀀스의 트랙에 있는 자산에 대한 이미지 속도입니다.
colourMultiply 색깔 현재 재생 헤드 위치(스프라이트, 인스턴스 및 시퀀스 트랙)에 있는 시퀀스의 트랙에 있는 자산에 대한 색상 곱셈 값입니다. 이 값은 [A, R, G, B] 형식의 4개의 ARGB 값 배열 입니다. 각 구성 요소의 값은 0과 1 사이로 표현됩니다. 여기서 0은 HEX 값 #00에 해당하고 1은 HEX 값 #FF에 해당합니다(시퀀스의 색상 곱하기 트랙에 대한 색상 선택기에 표시된 대로 0 - 255 편집자).
emitterIndex 오디오 이미터 ID 이 트랙(오디오 트랙)에 속한 오디오 이미터의 인덱스입니다.
gain 진짜 현재 재생 헤드 위치(오디오 트랙)에서 트랙의 게인(볼륨)입니다.
pitch 진짜 현재 재생 헤드 위치에서 트랙의 피치(오디오 트랙).
frameSizeX 진짜 텍스트 프레임(텍스트 트랙)의 가로 크기입니다.
frameSizeY 진짜 텍스트 프레임(텍스트 트랙)의 세로 크기입니다.
characterSpacing 진짜 문자 간격 값(텍스트 트랙).
lineSpacing 진짜 줄 간격 값(텍스트 트랙).
paragraphSpacing 진짜 단락 간격 값(텍스트 트랙).
track 시퀀스 트랙 구조체 이 트랙 의 기반이 되는 트랙 구조입니다(모든 자산, 읽기 전용 값).
parent 시퀀스 인스턴스 구조체 트랙의 상위 시퀀스 인스턴스 ID(모든 자산, 읽기 전용 값).
activeTracks 시퀀스 트랙 구조체 의 배열 이것은 자산 트랙에 포함된 각 매개변수 트랙에 대한 평가 구조체의 배열입니다(모든 자산, 읽기 전용 값). 배열에 있는 각 구조체의 내용은 매개변수 Track Struct 섹션에 나열됩니다.
