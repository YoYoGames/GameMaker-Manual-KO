object_set_persistent

이 기능을 사용하여 특정 개체의 지속성을 설정할 수 있습니다. 영속 객체는 방에서 방으로 "이동"되고 항상 속성을 유지합니다(지속성에 대한 자세한 내용은 개체 편집기 섹션 참조). 객체의 지속성을 설정한다는 것은 지속성이 변경된 후에 생성된 이 객체의 모든 인스턴스가 지속적으로 생성된다는 것을 의미하지만 이미 룸에 있는 인스턴스는 영향을 받지 않을 수 있습니다.

방에서 지속성이 활성화된 인스턴스를 생성하면 레이어 또는 깊이가 할당됩니다(객체 인스턴스 생성에 사용한 기능에 따라 다름). 룸이 변경되고 다음 룸에 할당된 것과 동일한 이름이나 깊이를 가진 레이어가 없으면 룸 전체에 지속되는 인스턴스에 대해 새 레이어가 생성됩니다. 인스턴스를 생성할 때 명명된 레이어를 지정했다면 새 레이어의 이름도 원래 레이어와 동일하게 지정되고, 인스턴스에 깊이를 지정했다면 새 레이어의 이름은 "_layer_XXX ", 여기서 "XXX"는 레이어에 고유한 이름을 지정하는 데 사용되는 16진수 값입니다. 이러한 레이어는 Room End의 룸에서 제거됩니다. 레이어 및 영구 인스턴스에 대해 주의해야 할 또 다른 사항은 인스턴스에 이름을 지정한 경우 생성할 때 레이어가 생성되고 다음 룸에 같은 이름의 다른 레이어가 있는 경우 레이어 의 깊이에 관계없이 동일한 이름의 레이어에 지속 인스턴스가 할당됩니다.마지막으로 지속 인스턴스가 방으로 이동하는 경우 인스턴스가 생성된 것과 동일한 깊이의 레이어가 있는 경우 이 레이어에 할당 되지 않고 대신 동일한 깊이에서 새 레이어가 생성됩니다(이전에 설명한 명명 규칙에 따름).

이것은 인스턴스 함수가 아니라는 점에 유의하십시오! 다음을 사용하여 개별 인스턴스의 영구 플래그를 설정할 수 있습니다.persistent 변수이므로 문제의 객체가 IDE에서 플래그를 false로 설정하더라도 10개의 인스턴스가 모두 영구 플래그가 지정됩니다. 이 함수를 사용하여 객체의 영구 플래그를 true로 변경하더라도 현재 방에 있는 모든 인스턴스는 그대로 유지되며, 함수를 호출한 후에 생성된 인스턴스만 true로 플래그가 지정된 퍼시스턴트로 시작됩니다.

통사론:

object_set_persistent(index, pers);

논쟁 유형 설명
index 개체 자산 The index of the object to change.
pers 부울 Whether the object should be flagged as persistent (true ) or not (false ).

보고:

N/A

예시:

if (room == rm_final)
{
    object_set_persistent(obj_Player, false);
}
instance_create_layer(32, 32, "Instances", obj_Player);

위의 코드는 인스턴스가 현재 어느 방에 있는지 확인하고 지정된 "rm_final"에 있으면 개체 "obj_Player"의 지속성을 false로 설정한 다음 인스턴스를 만듭니다.
