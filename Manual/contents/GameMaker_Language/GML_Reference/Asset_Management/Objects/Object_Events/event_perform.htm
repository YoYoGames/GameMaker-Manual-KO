event_perform

이 함수는 코드를 실행하는 인스턴스에 대해 지정된 인수와 함께 지정된 이벤트를 수행합니다. 여기에는 가능한 모든 이벤트의 완전한 시뮬레이션을 허용하는 많은 옵션이 있지만, 이것은 문자 그대로 이벤트의 모든 코드를 수행하고 게임은 예를 들어 키보드 수행을 선택하는 경우와 같이 수동으로 트리거되도록 아무것도 수정하지 않습니다. press 이벤트가 발생하면 이벤트가 발생하지만 해당 키가 눌린 것으로 인식되지 않습니다. 또는 알람 이벤트를 수행하면 알람 카운터가 -1로 설정되지 않고 계속 카운트다운됩니다.

비동기 이벤트는 다음을 사용하여 호출할 수 있습니다.event_perform_async().

아래 표에는 이벤트 유형과 해당 숫자 값을 참조하기 위해 GameMaker 에 포함된 모든 상수가 요약되어 있습니다. 이벤트 자체는 굵은 텍스트 로 표시되며 이벤트의 반환 값과 동일합니다.event_type 변수(이 함수의 첫 번째 인수로 전달됨), 숫자 값은 기울임꼴 텍스트 로 표시되고 변수의 반환 값과 동일합니다.event_number 변수(이 함수의 두 번째 인수로 전달됨):

---- ev_gesture_double_tap 인스턴스 에 대해 두 번의 빠른 터치/클릭 및 릴리스가 감지되었습니다.
---- ev_gesture_drag_start 예를 들어 드래그 제스처의 시작이 감지되었습니다.
---- ev_gesture_drag 예를 들어 터치/클릭이 유지되고 이동되었습니다.
---- ev_gesture_drag_end 예를 들어 드래그에서 터치/클릭 해제가 감지되었습니다.
---- ev_gesture_flick 드래그에서 터치/클릭을 떼면 인스턴스 에 대해 플릭 이벤트가 감지될 만큼 충분한 움직임이 있었습니다.
---- ev_gesture_pinch_start 예를 들어 두 번의 터치와 직선 이동이 감지되었습니다.
---- ev_gesture_pinch_in 예를 들어 두 터치 사이의 움직임이 안쪽으로 감지되었습니다.
---- ev_gesture_pinch_out 예를 들어 두 터치 사이의 움직임이 바깥쪽으로 감지되었습니다.
---- ev_gesture_pinch_end 예를 들어 핀치에 대한 하나(또는 둘 다) 터치의 해제가 감지되었습니다.
---- ev_gesture_rotate_start 예를 들어 두 터치 사이의 움직임이 회전으로 감지되었습니다.
---- ev_gesture_rotating 예를 들어 두 터치 간의 움직임이 회전하는 것으로 감지되었습니다.
---- ev_gesture_rotate_end 예를 들어 회전에 대한 하나(또는 둘 다) 터치의 해제가 감지되었습니다.
---- ev_global_gesture_tap 방 안 어디에서나 클릭/터치 및 해제가 감지되었습니다.
---- ev_global_gesture_double_tap 두 번의 빠른 터치/클릭 및 릴리스가 방의 어느 곳에서나 감지되었습니다.
---- ev_global_gesture_drag_start 드래그 제스처의 시작이 방의 어디에서나 감지되었습니다.
---- ev_global_gesture_draging 터치/클릭을 누르고 방의 아무 곳으로나 이동했습니다.
---- ev_global_gesture_drag_end 드래그로 인한 터치/클릭 해제가 방의 어느 곳에서나 감지되었습니다.
---- ev_global_gesture_flick 드래그에서 터치/클릭을 떼면 플릭 이벤트가 방의 어느 곳에서나 감지될 수 있을 만큼 충분한 움직임이 있었습니다.

---- ev_global_gesture_pinch_end 핀치에 대한 하나(또는 둘 다) 터치의 해제가 방의 어느 곳에서나 감지되었습니다.

통사론:

event_perform(type, numb);

논쟁 유형 설명
type 이벤트 유형 상수 The type of event to perform (see the table above).
numb 실수 또는 이벤트 수 상수 The specific event constant or value, if one is necessary (otherwise, just use 0).

확장된 예

이 기능을 사용하는 방법을 이해하는 데 도움이 되도록 다음 몇 가지 예를 살펴보세요.

event_perform(ev_keypress, ord("W"));

이것은 Keyboard Check Pressed "W" 키와 관련된 이벤트를 수행합니다(실제로 키보드 누름 생성 없이).

event_perform(ev_step, ev_step_begin);

이것은 단계 시작 이벤트를 수행합니다(단계 이벤트에서 호출되는 경우 단계 시작 이벤트 코드가 두 번 실행됨).

event_perform(ev_create, 0);

이렇게 하면 인스턴스의 Create 이벤트가 수행됩니다.
