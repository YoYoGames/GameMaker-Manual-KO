셰이더 상수

GLSL ES(OpenGL ES Shading Language) 참조 페이지 에 정의된 셰이더 함수 및 상수 외에도 GameMaker 에 고유한 여러 셰이더 상수도 사용할 수 있습니다.

다음 디스플레이 행렬 상수는 셰이더 배열 상수를 사용할 때 배열 인덱스로 사용할 수 있습니다.gm_Matrices:

일정한 설명
MATRIX_VIEW 이 배열 인덱스 상수는 현재 뷰 행렬에 대한 인덱스를 보유합니다. 반환된 인덱스는 다음을 호출할 때 배열 값으로 사용됩니다. gm_Matrices 셰이더 코드 내에서 상수입니다.
MATRIX_PROJECTION 이 배열 인덱스 상수는 현재 투영 행렬에 대한 인덱스를 보유합니다. 반환된 인덱스는 다음을 호출할 때 배열 값으로 사용됩니다. gm_Matrices 셰이더 코드 내에서 상수입니다.
MATRIX_WORLD 이 배열 인덱스 상수는 현재 세계 행렬에 대한 인덱스를 보유합니다. 월드 스페이스에 조명 정보가 있는 경우 조명과 같은 작업에 사용할 수 있습니다. 반환된 인덱스는 다음을 호출할 때 배열 값으로 사용됩니다. gm_Matrices 셰이더 코드 내에서 상수입니다.
MATRIX_WORLD_VIEW 이 배열 인덱스 상수는 함께 곱한 세계 및 보기 행렬의 결과에 대한 인덱스를 보유합니다. 이것은 안개와 같은 것에 자주 사용됩니다. 반환된 인덱스는 다음을 호출할 때 배열 값으로 사용됩니다. gm_Matrices 셰이더 코드 내에서 상수입니다.
MATRIX_WORLD_VIEW_PROJECTION 이 배열 인덱스 상수는 함께 곱한 세계, 보기 및 투영 행렬의 결과에 대한 인덱스를 보유합니다. 정점 위치에 사용되는 일반 변환 행렬입니다. 반환된 인덱스는 다음을 호출할 때 배열 값으로 사용됩니다. gm_Matrices 셰이더 코드 내에서 상수입니다.
MATRIX_MAX 이것은 배열 인덱스가 아니라 정점 셰이더에서 행렬 배열의 크기를 반환합니다.
MAX_VS_LIGHTS 이것은 배열 인덱스가 아니라 정점 셰이더의 조명 수를 반환합니다.

조명에 대해 다음 상수도 사용할 수 있습니다.

일정한 설명
MAX_VS_LIGHTS 셰이더에서 사용할 수 있는 포인트 및 디렉셔널 라이트의 최대 수

다음과 같은 사전 정의된 매트릭스 유니폼 및 상수를 셰이더에서 사용하여 GameMaker 특정 값에 액세스할 수 있습니다.

일정한 설명
gm_Matrices[matrix] 이 배열 상수는 변환 행렬을 반환하며 GameMaker 가 셰이더 코드 편집기 내에서 사용하기 위해 생성하는 사용 가능한 사전 정의 유니폼 중 하나입니다. 배열 인덱스는 위에 나열된 상수 중 하나에서 선택됩니다.
gm_BaseTexture 이것은 GameMaker 에서 설정한 대로 현재 개체의 텍스처를 반환하는 2D 샘플러 상수입니다. 따라서 셰이더를 호출하지 않고 그릴 때 일반적으로 사용되는 현재 스프라이트, 표면 또는 텍스처가 됩니다.
gm_LightingEnabled 이것은 3D를 사용할 때 GameMaker 조명을 가져오거나 설정하는 데 사용할 수 있습니다.
gm_FogStart 이것은 폴리곤이 안개 색상과 혼합되기 시작하는 거리를 얻는 데 사용할 수 있습니다.
gm_RcpFogRange 이것은 안개가 최대이고 더 이상 아무것도 보이지 않는 거리를 얻는 데 사용할 수 있습니다.
gm_PS_FogEnabled 이것은 돌아올 것이다 true 또는 false GPU에 픽셀 포그가 활성화되어 있는지 여부.
gm_FogColour GameMaker 에서 사용하는 안개 색상을 가져오는 데 사용할 수 있습니다.
gm_VS_FogEnabled 이것은 돌아올 것이다 true 또는 false GPU에 정점 안개가 활성화되어 있는지 여부.
gm_AlphaTestEnabled 이것은 셰이더에서 알파 테스트를 수행하는 데 사용할 수 있습니다. 보다gpu_set_alphatestenable() 알파 테스트에 대한 자세한 내용은
