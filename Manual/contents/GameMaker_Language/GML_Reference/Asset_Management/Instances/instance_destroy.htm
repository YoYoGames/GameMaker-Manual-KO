instance_destroy

인스턴스를 "파괴"하고 싶을 때마다 이 함수를 호출합니다. 일반적으로 Destroy Event 및 Clean Up Event 도 트리거합니다. 이렇게 하면 룸이 다시 시작될 때까지 룸에서 제거됩니다(룸이 지속되지 않는 경우). 인수 없이 함수를 호출하면 현재 범위에 있고 코드를 실행 중인 인스턴스가 소멸되지만 선택적 "id" 인수를 제공하고 인스턴스를 사용하여 특정 인스턴스를 대상으로 지정할 수 있습니다. id 값을 사용하거나object_index. 예를 들어:

instance_destroy(other);      // destroy the "other" instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

두 번째 선택적 플래그를 사용하면 파괴되는 인스턴스에 대한 Destroy Event를 "스위치 끄기"할 수 있습니다. 기본적으로 Destroy Event는 항상 수행되지만 이 플래그를 다음으로 설정하면 false 그런 다음 인스턴스를 자체적으로 파괴하고 해당 이벤트 수행을 건너뛸 수 있습니다.

중요한! 이렇게 하면 Destroy Event를 건너뛰지만 Clean Up 이벤트는 계속 호출됩니다.

인스턴스를 파괴할 때 destroy 함수를 호출하는 코드 또는 작업 직후 에 해당 destroy 이벤트가 호출된다는 점은 주목할 가치가 있습니다. 또한 destroy 이벤트가 수행되더라도 인스턴스 가 게임에서 즉시 제거되지 않고 현재 이벤트에 포함된 코드를 계속 수행합니다. 현재 이벤트가 종료된 경우에만 게임에서 제거됩니다.

예를 들어 다음 코드가 있는 경우:

if (hp <= 0) instance_destroy();
score += 10;

변수 "점수" 는 instance_destroy 함수가 호출되었으며 이벤트가 끝나면 인스턴스가 마침내 게임에서 제거됩니다. (예를 들어) 데이터 구조와 같은 인스턴스에 대한 동적 리소스를 생성한 다음 destroy 함수 또는 작업 후에 해당 리소스에 대한 참조가 있는 경우 destroy 이벤트에서 해당 리소스를 파괴한 것처럼 이것을 주의하십시오. 제거 이벤트가 게임에서 제거되었기 때문에 "알 수 없는 리소스" 오류가 발생합니다.

통사론:

instance_destroy([id, execute_event_flag]);

논쟁 유형 설명
id 인스턴스 ID 또는 개체 자산 OPTIONAL The instance ID to destroy, or the object asset to destroy instances of (optional, default is the calling instance)
execute_event_flag 부울 OPTIONAL Set to true or false to perform the Destroy event or not (optional, default is true )

보고:

N/A

예시:

if (bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height)
{
    instance_destroy(id, false);
}

위의 코드는 인스턴스가 방 경계 밖에 있는 경우 Destroy Event를 호출하지 않고 코드를 실행하는 인스턴스를 파괴합니다.
