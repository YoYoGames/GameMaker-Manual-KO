instance_create_depth

이 기능을 사용하면 방 내의 주어진 지점과 지정된 깊이에서 지정된 객체의 새 인스턴스를 생성할 수 있습니다. 깊이는 모든 값이 될 수 있습니다. 깊이가 낮을수록 카메라 사물이 "가까이" 그려지고 깊이가 높을수록 더 멀어지므로 깊이 -200의 인스턴스가 깊이 +300의 인스턴스 위에 그려집니다( 예를 들어).

모든 인스턴스가 룸의 레이어에 있어야 하기 때문에 이 함수는 실제로 인스턴스에 대한 룸 레이어를 생성하지만 이것은 관리되는 레이어이기 때문에(예: 코드를 통해 또는 룸에서 생성한 것이 아니라 GameMaker 가 자동으로 생성한 것). 해당 레이어에 액세스할 수 없으며layer 인스턴스 변수는 -1을 반환합니다.

중요 최소 및 최대 레이어 깊이는 -16000 ~ 16000입니다. 모든 이벤트는 여전히 정상적으로 실행되지만 해당 범위 밖의 깊이에 배치된 모든 항목은 그려지지 않습니다.

이 함수는id 그런 다음 변수에 저장하고 해당 인스턴스에 액세스하는 데 사용할 수 있는 새 인스턴스의 이 함수는 함수를 호출한 이벤트에 대한 코드 또는 작업을 계속 하기 전에 생성 중인 인스턴스의 이벤트 생성 도 호출합니다.

선택적 구조체

마지막 인수인 speed>var_struct 는 선택 사항이며 새 인스턴스에 대한 추가 변수가 포함된 구조체를 사용합니다.

이 구조체의 변수는 Create 이벤트가 실행 되기 전에 새 인스턴스에 적용되지만 변수 정의 가 설정된 후에 적용됩니다.

즉, 해당 구조체의 값은 새 인스턴스의 Create 이벤트에서 읽을 수 있습니다. 하단의 예 2 를 참조하십시오.

이 구조체를 통해 새 인스턴스에 적용된 값은 method 변수를 포함하여 모든 유형이 될 수 있습니다. 내장 변수 도 변경할 수 있습니다.

참고 구조체의 변수는 새 인스턴스에 "얕게 복사"됩니다. 즉, 모든 배열, 구조체 및 기타 리소스가 참조로 복사되고 복제되지 않습니다.

통사론:

instance_create_depth(x, y, depth, obj);

논쟁 유형 설명
x 진짜 The x position the object will be created at
y 진짜 The y position the object will be created at
depth 진짜 The depth to assign the created instance to
obj 개체 자산 The object index of the object to create an instance of
var_struct 구조체 OPTIONAL A struct with variables to assign to the new instance

보고:

Instance ID

예 1:

var inst = instance_create_depth(x, y, -100, obj_Bullet);
with (inst)
{
    speed = other.shoot_speed;
    direction = other.image_angle;
}

위의 코드는 direction>-100 깊이에서 obj_bullet 개체의 새 인스턴스를 만들고 인스턴스 ID를 변수에 저장합니다. 이 변수는 다음을 할당하는 데 사용됩니다. 그리고 새 인스턴스에.

이렇게 하면 먼저 인스턴스를 만들고 Create 이벤트를 실행 한 다음 변수에 값을 할당합니다.

Create 이벤트가 실행 되기 전에 일부 변수를 할당하려면 아래 예를 참조하세요.

예 2:

var inst = instance_create_depth(x, y, -100, obj_bullet,
{
    speed : shoot_speed,
    direction : image_angle
});

위의 코드는 obj_bullet 의 인스턴스를 만들고 마지막 인수로 구조체를 전달합니다.

그 구조체에는 속도와 방향에 대한 변수가 있습니다. other 를 사용할 필요 없이 호출 인스턴스에서 값을 가져옵니다.

이러한 변수는 Create 이벤트가 실행되기 전에 새 인스턴스에 적용됩니다.

기존 구조체를 저장하는 변수를 전달하거나 constructor 에서 new 구조체를 생성할 수도 있으므로 구조체 리터럴에 제한되지 않습니다.
