id

이 읽기 전용 변수는 인스턴스의 고유 식별 번호를 보유합니다. 코드를 통해 생성하거나 Room Editor에서 Room에 추가하여 생성하는 모든 인스턴스에는 이 인스턴스와 변수를 식별하는 데 내부적으로 사용되는 번호가 지정됩니다. id 참조하는 데 사용할 수 있습니다. 그만큼 id 다음을 사용하여 인스턴스를 생성할 때도 반환됩니다(변수에 저장할 수 있음). instance_create_layer() 또는 instance_create_depth(), 뿐만 아니라 다른 인스턴스 기능.

이 변수는 이제 단순히 실수가 아닌 인스턴스에 대한 참조입니다. 다음을 호출하여 인스턴스 ID 번호를 얻을 수 있습니다.real() 또는int64() 이 변수에.

참고 이 변수의 값은 인스턴스가 룸 편집기에서 제공되는 식별자와 동일 하지 않으며 instance_id 현재 활성화된 모든 인스턴스의 모든 ID를 포함하는 배열입니다.

통사론:

id;

보고:

Instance ID

예시:

for (var i = 0; i < instance_count; i++;)
{
    if (instance_id[i] != id)
    {
        instance_id[i].scr += 5;
    }
}

위의 코드는 코드를 실행하는 인스턴스를 제외한 방의 모든 활성 인스턴스에 대해 "scr" 변수에 5를 추가합니다. 모든 활성 인스턴스를 반복하고(각 활성 인스턴스 ID를 반환하기 위해 "instance_id" 배열을 사용) 이를 코드를 실행하는 원래 인스턴스의 ID인 기본 제공 "id" 변수와 비교하여 이를 수행합니다.
