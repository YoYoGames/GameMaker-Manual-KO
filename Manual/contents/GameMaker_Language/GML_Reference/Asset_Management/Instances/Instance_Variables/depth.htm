depth

개체를 생성할 때 게임이 실행되는 동안 해당 개체의 인스턴스가 방에 그려지는 방식을 정의하는 초기 깊이 를 할당할 수 있으며 이 변수는 게임이 실행되는 동안 해당 깊이 값을 가져오고 변경하는 데 사용할 수 있습니다. 달리기. 일반적으로 특정 깊이로 설정되는 개별 레이어 에 추가할 인스턴스를 설정해야 하므로 이 변수를 사용할 필요가 없지만 이 값을 사용하여 인스턴스의 깊이를 변경하고 싶을 수도 있습니다. 이 경우 "임시 레이어"가 지정된 깊이의 인스턴스에 대해 특별히 생성됩니다. 동일한 깊이에 인스턴스가 없는 경우 이 임시 레이어는 메모리에서 제거됩니다(아무것도 없는 경우에도 남아 있는 일반 레이어와 달리).

중요한! 수정할 때 depth 변수 및 GameMaker 는 내장된 레이어를 관리합니다.layer 관리되는 레이어는 코드를 통해 조작할 수 없기 때문에 변수는 레이어 ID가 아닌 -1을 반환합니다.

GameMaker 에서 인스턴스의 깊이 값이 낮을수록 해당 인스턴스가 "카메라에 더 가깝게" 그려지고 깊이 값이 높을수록 인스턴스가 "카메라에서 더 멀리" 그려집니다. 즉, -1000이 그려집니다. -100 위에, 0 위에 그려지는 식입니다.

동일한 깊이를 가진 개체의 인스턴스는 룸 편집기에서 다른 개체 위에 배치된 것처럼 보이더라도 서로 위 또는 아래 에 그릴 수 있습니다. 다른 모든 것 위에 또는 아래에 무언가가 그려지도록 하려면 항상 깊이(또는 레이어)를 명시적으로 설정해야 합니다. 또한 최소(-16000) 및 최대(+16000) 깊이 값이 있으며 이 값을 벗어나면 인스턴스가 그려지지 않지만 여전히 존재하고 이벤트를 처리합니다.

참고 그리기 이벤트에서 인스턴스의 깊이를 설정할 수 없습니다 (다른 모든 이벤트는 괜찮음).

인스턴스의 깊이를 수정하면 적용되는 필터 및 효과 가 변경될 수 있습니다. 깊이를 FX 레이어의 깊이보다 낮게 변경하면 인스턴스에 더 이상 효과가 적용되지 않기 때문입니다.

통사론:

depth;

보고:

Real (single precision floating point value)

예시:

if y != yprevious
{
    depth = -y;
}

위의 코드는 y 위치가 변경되었는지 확인하고 변경되면 깊이도 이에 상응하도록 설정됩니다.
