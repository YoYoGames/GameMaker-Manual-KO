인스턴스 변수

새 개체를 만들 때 이미 기본값으로 초기화된 특정 변수가 함께 제공됩니다. 이러한 변수를 우리는 인스턴스 변수 라고 부릅니다. 왜냐하면 게임 룸에 있는 개체의 모든 인스턴스에 대해 생성되며, 일단 인스턴스가 생성되면 이러한 변수의 값은 해당 인스턴스 및 해당 인스턴스 에만 고유하기 때문입니다. 이러한 변수 중 일부는 다음과 같이 많이 사용할 것입니다. x 그리고 y, 다른 것들은 수행하려는 작업에 따라 덜 필요하지만 일반적으로 매우 유용하며 가능한 경우 사용자 정의 변수를 만드는 것보다 사용하는 방법을 찾아야 합니다.

대부분의 인스턴스 변수는 읽기는 물론 설정도 가능하므로 특정 변수의 값을 조정하는 것만으로 인스턴스의 속성과 동작을 변경할 수 있습니다. 예를 들어 단순히 다음을 설정하여 인스턴스가 그리는 것을 방지할 수 있습니다. visible 내장 변수 false. 아래에서 게임에 있는 모든 개체의 모든 인스턴스에 대해 초기화되는 다양한 변수를 찾을 수 있습니다.

중요 인스턴스가 sequence 에 있는 경우 이러한 변수 중 일부는 다음과 같습니다.image_xscale /image_yscale / image_angle /x /y - 재생이 시작된 후 각 단계마다 시퀀스가 업데이트되면 덮어쓰게 됩니다.

이러한 변수는 일반 인스턴스 속성을 처리합니다.

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

다음 변수는 인스턴스 위치 및 이동을 처리합니다.

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

개체 속성

다음 변수는 인스턴스가 생성된 개체의 고유 인덱스 ID를 보유합니다.

  • object_index

이 변수는 모두 인스턴스에 할당된 스프라이트와 관련이 있으며 그려지는 항목과 방법을 변경하는 데 사용할 수 있습니다.

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

다음 변수는 충돌 마스크를 처리합니다.

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

이러한 변수는 경로 및 할당된 경우 인스턴스가 경로와 상호 작용하는 방식과 관련됩니다.

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

다음 변수는 타임라인을 사용하도록 인스턴스를 설정하기 위한 것입니다.

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

시퀀스

다음 변수는 주어진 시간에 인스턴스를 제어할 수 있는 시퀀스에 대한 정보를 저장합니다.

  • in_sequence
  • sequence_instance
