인스턴스 비활성화

GameMaker 는 더 이상 어떤 식으로든 처리되지 않도록 인스턴스를 "스위치 끄기"할 수 있는 가능성을 제공합니다. 기술적으로 그들은 비활성화 프로세스 자체 내의 포인터를 제외하고는 더 이상 존재하지 않습니다. 즉, 비활성화된 인스턴스는 다시 활성화될 때까지 어떤 식으로든 조작하거나 변경할 수 없습니다. 따라서 이러한 기능은 제대로 사용하지 않으면 문제가 발생할 수 있으므로 특히 영속 객체의 경우 문제를 일으킬 수 있으므로 각별히 주의해야 합니다. 비활성화된 영속 객체는 먼저 다시 활성화(효과적으로 게임에서 삭제). 활성화 및 비활성화는 즉각적이지 않으며 이러한 방식으로 상태가 변경된 인스턴스 는 함수가 호출된 이벤트가 끝날 때까지 활성/비활성으로 간주되지 않습니다.

일반적으로 게임의 모든 단계에서 인스턴스를 비활성화할 필요는 없으며 실제로 게임에 성능 문제가 발생할 수 있습니다. 대신 알람의 몇 단계마다 또는 카메라 보기의 위치가 변경된 경우에만 이러한 기능을 실행하는 것이 좋습니다. 게임의 심각한 오류.

비활성화된 인스턴스는 게임에서 더 이상 존재하지 않지만 개별 인스턴스는 여전히 변수에 액세스할 수 있습니다. 당신은 사용할 수 없습니다 with(object) 또는 with(instance) 그러나 대신 다음 형식의 고유 ID를 사용하여 인스턴스에 직접 액세스해야 합니다.

val = inst.variable;

위의 예에서 "inst"는 비활성화된 인스턴스의 ID를 보유하는 변수이며, 이것이 비활성화된 인스턴스가 이전에 활성화되지 않고 참조될 수 있는 유일한 방법입니다.

참고 회의실 시작 시 인스턴스를 비활성화하는 경우(예: 회의실 생성 코드에서 또는 회의실 내 인스턴스의 인스턴스 생성 이벤트에서) 회의실 편집기에서 회의실 내에 배치된 모든 인스턴스는 이전에 생성 이벤트를 계속 실행합니다. 비활성화 중입니다.

경고: 물리학이 활성화된 인스턴스를 비활성화해도 픽스쳐가 물리학 시뮬레이션 내에서 상호 작용하는 것을 중지하지 않습니다. 이를 위해 truestyle="font-size: 14px">phy_active 변수를 다음으로 설정해야 합니다. false 또는 인스턴스를 활성화/비활성화할 때.

인스턴스 비활성화를 위해 다음과 같은 기능이 있습니다.

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
