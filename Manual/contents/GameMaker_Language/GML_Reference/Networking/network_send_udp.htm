network_send_udp

이 기능을 사용하면 UDP를 사용하여 네트워크를 통해 서버에 데이터를 보낼 수 있습니다. 이 함수는 연결할 소켓 ID, 연결할 URL 및 사용할 포트를 사용합니다. 그런 다음 보낼 데이터가 포함된 버퍼 ID (버퍼에 대한 자세한 내용은 참조 - 버퍼 참조 )와 마지막으로 데이터 크기(바이트)를 제공해야 합니다. UDP는 실제로 연결을 수행하지 않고 IP에 직접 패킷을 보내고 서버는 IP 주소에서 들어오는 데이터를 가져와 "있는 그대로" 처리해야 한다는 점에서 "연결되지 않음"입니다. 이 함수는 전송된 데이터의 바이트 수를 반환하거나 전송이 실패한 경우 0보다 작은 수를 반환합니다. 이 함수가 반환하는 전송되는 데이터의 최종 크기에는 추가 12바이트인 GameMaker 헤더 정보도 포함됩니다.

이 기능은 HTML5 대상에서 실행 중인 프로젝트에서 사용할 때 작동하지 않으며 HTML5 프로젝트에서도 UDP를 수신할 수 없습니다.

통사론:

network_send_udp(socket, url, port, bufferid, size);

논쟁 유형 설명
socket 네트워크 소켓 ID The id of the socket to use.
url 끈 The url or IP to connect to (a string).
port 진짜 The port to connect to.
bufferid 버퍼 ID The id of the buffer to get the data from.
size 진짜 The size (in bytes) of the data.

보고:

Real

예시:

network_send_udp(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

위의 코드는 포트 6510의 URL에 의해 정의된 서버로 UDP 패킷을 보낼 것입니다. 데이터는 변수 "buff"에 색인된 버퍼에서 가져옵니다.
