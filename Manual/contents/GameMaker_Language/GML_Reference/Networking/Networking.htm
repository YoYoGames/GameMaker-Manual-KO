네트워킹

GameMaker 네트워킹 기능은 모든 플랫폼의 산업 표준인 소켓 을 기반으로 합니다. 소켓을 사용하면 클라이언트/서버 모델(나중에 멀티플레이어를 쉽게 추가할 수 있는 것)을 사용하여 싱글 플레이어 게임도 만들 수 있는 간단한 GML을 사용하여 클라이언트와 서버를 모두 생성할 수 있습니다.

기본적으로 소켓은 네트워크의 포트 를 전송, 수신, 연결 및 수신할 수 있는 개체입니다. 이러한 기능은 인터넷이 기반으로 하는 TCP/IP 연결만 처리합니다(더 정확하게는 IPv4). 본질적으로 이것은 IP 주소(예: 192.168.1.10)를 사용하여 하나의 소켓을 다른 IP 주소(예: 192.168.1.11)의 다른 소켓에 연결할 수 있습니다. 일단 연결되면 이 두 소켓은 다음을 사용하여 데이터를 주고받을 수 있습니다. network_send_* 기능(예:network_send_packet() ) 및 이에 의해 트리거되는 전용 비동기 이벤트입니다. 이 비동기 이벤트는 수신된 데이터와 기타 세부 정보가 포함된 DS 맵 을 생성하며 네트워크 비동기 이벤트 섹션에서 전체 세부 정보를 찾을 수 있습니다.

프로그램이 기계로 들어오는 모든 네트워크 트래픽을 읽고 처리해야 하는 대신, IP 주소는 0에서 65535까지 번호가 매겨진 포트도 처리합니다. 이를 통해 각 프로그램은 관심 있는 데이터 패킷만 가져올 수 있습니다. 모든 프로그램에서 보낸 모든 것이 아닙니다. 이것은 또한 데이터 처리를 심각하게 줄이기 때문에 상당한 CPU 시간을 절약하므로 GameMaker 를 사용하여 포트를 "수신"할 수도 있습니다. 따라서 서버는 소켓을 만든 다음 수신할 특정 포트에 바인딩합니다. 그런 다음 해당 소켓에서 연결, 연결 해제 및 데이터 전송에 대한 알림을 받습니다.

데이터 "패킷" 자체는 버퍼에서 가져온 이진 데이터로 구성됩니다. 이 버퍼는 GameMaker 버퍼 기능 을 사용하여 생성하고 조작할 수 있습니다.

HTML5 대상에 대한 프로젝트를 생성하는 경우 브라우저 제한으로 인해 서버 생성 기능을 사용할 수 없으며 Electron 또는 NodeJS 와 같은 것을 사용하여 이 플랫폼에 대한 자체 서버를 작성해야 합니다. 또한 게임이 크로스 플랫폼이고 모바일 또는 데스크톱 장치에서 실행되는 서버에서 게임의 HTML5 인스턴스에 연결할 것으로 예상하는 경우 서버는 Web Socket 프로토콜을 사용하여 HTML5 인스턴스와만 통신할 수 있습니다. 상수를 사용하여 network_socket_ws 또는network_socket_wss - 보다network_create_server() 자세한 내용은). 이것은 서버가 다른 포트에 두 개의 소켓을 생성해야 할 수 있음을 의미합니다. 하나는 비 HTML5 대상(Android, Windows 등...)에 연결하고 다른 하나는 HTML5 대상에 연결하기 위한 것입니다.

네트워킹에 사용할 수 있는 기능은 다음과 같습니다.

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • network_connect_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
