network_set_config

이 기능을 사용하여 다양한 네트워크 구성을 설정할 수 있습니다.

참고: 이 기능은 HTML5 대상에서 실행 중인 프로젝트에서 사용할 때 작동하지 않습니다.

이러한 구성은 상수 로 제공되며 설정은 선택한 상수에 따라 달라집니다. 아래 표에는 사용 가능한 상수와 해당 설정이 나와 있습니다.

네트워크 구성 상수
일정한 설명 환경
network_config_connect_timeout 연결 시간 초과 값 설정 A value in milliseconds
network_config_use_non_blocking_socket GameMaker 에 연결 시 차단하지 않도록 지시합니다. 0 = block, 1 = don't block
network_config_enable_reliable_udp 기존 UDP 소켓에 대해 "신뢰할 수 있는 UDP" 프로토콜을 활성화합니다. The socket ID to target
network_config_disable_reliable_udp 기존 UDP 소켓에 대해 "신뢰할 수 있는 UDP" 프로토콜을 비활성화합니다. The socket ID to target
network_config_avoid_time_wait 설정SO_LINGER 기존 TCP 소켓에 대해 시간 초과 값을 0으로 설정 The socket ID to target


첫 번째 옵션(network_config_connect_timeout ) 단순히 서버에 연결하기 위한 시간 초과를 설정하지만 기다려야 하는 시간을 제외하고 연결 동작을 변경하지 않습니다. 이 값을 너무 낮게 설정하지 말고 LAN 전용 게임의 경우 약 1000, 인터넷의 경우 약 4000 정도로 유지하는 것이 좋습니다. 그렇지 않으면 게임이 무작위로 연결되지 않을 수 있습니다. 패킷 송수신에 대한 시간 초과 값을 설정하려면 다음 기능을 사용하십시오.network_set_timeout().

두 번째 옵션(network_config_use_non_blocking_socket )는 다음을 의미합니다. network_connect_* 함수(비비동기)는 모두 다음을 반환합니다. socket_id 즉시, 그러나 비동기 네트워크 이벤트를 수신할 때까지 해당 소켓에서 보내거나 받을 수 없습니다. 트리거된 이벤트에는 "유형" 키가 다음으로 설정됩니다. network_type_non_blocking_connect ( 네트워킹 비동기 이벤트 페이지에서 더 자세한 정보를 찾을 수 있습니다). GameMaker 는 하나의 응용 프로그램에서 차단 및 비차단 혼합을 지원하지 않기 때문에 이는 전역 설정입니다.

세 번째 및 네 번째 옵션(network_config_enable_reliable_udp, network_config_disable_reliable_udp ) GameMaker 신뢰할 수 있는 UDP 프로토콜을 활성화 또는 비활성화하기 위한 것이므로 UDP 연결에서만 사용해야 합니다. 이 경우 "신뢰할 수 있는" 것은 연결의 양쪽 끝에서 활성화된 경우 GameMaker 가 패킷이 올바르게 도착했는지 확인하고 도착하지 않은 패킷을 다시 보내려고 시도한다는 것입니다(참고: 패킷은 전송 순서대로 도착합니다. 신뢰할 수 있는 소켓 유형을 사용하는 경우 GameMaker 에서 패킷 오류를 확인하고 누락된 패킷을 다시 보내는 데 필요한 정보가 포함된 12바이트 헤더가 모든 udp 패킷에 추가됩니다. 이 옵션을 사용할 때 다음 사항에 유의해야 합니다.

  • 데이터를 보내기 전에 양쪽에서 프로토콜을 활성화/비활성화해야 하며 옵션이 활성화/비활성화되기 전에 보내거나 받은 데이터는 영향을 받지 않습니다.
  • 서버 측과 클라이언트 측 모두에서 적절하게 관리되는 한 주어진 소켓 ID에 대해 프로토콜을 여러 번 활성화하거나 비활성화할 수 있습니다.
  • 두 개의 개별 UDP 소켓을 가질 수 있습니다. 하나는 안정적인 UDP가 활성화되어 있고 다른 하나는 안정적인 UDP가 비활성화되어 있습니다.

마지막 옵션,network_config_avoid_time_wait TCP 연결 전용 이며 설정하는 데 사용할 수 있습니다. SO_LINGER 값을 TIME_WAIT>0 으로 설정 하여 많은 수의 연결이 상태, 서버에서 사용 가능한 모든 리소스를 묶습니다. 이 변경 사항은 소켓이 활성 상태인 동안 영구적이며 취소할 수 없습니다.

통사론:

network_set_config(config_value, setting);

논쟁 유형 설명
config_value 네트워크 구성 상수 The config constant to set (refer to the table at the top of this page).
setting 변하기 쉬운 The setting value of the config.

보고:

String

예시:

network_set_config(network_config_connect_timeout, 1000);
network_set_config(network_config_use_non_blocking_socket, 1);

위의 코드는 네트워크 연결 시간 초과를 1000ms로 설정하고 GameMaker 에 연결 시 차단하지 않도록 지시합니다.
