network_send_broadcast

이 기능을 사용하면 버퍼에서 IP 주소 범위로 로컬로 데이터를 브로드캐스트할 수 있습니다(버퍼에 대한 자세한 내용은 참조 - 버퍼 참조 ). 범위는 서버를 실행하는 장치의 범위로 제한되므로 장치의 IP가 92.168.11.130인 경우 데이터는 92.168.11.* 범위를 통해 브로드캐스트됩니다. 이 함수는 전송된 데이터의 바이트 수를 반환하거나 전송이 실패한 경우 0보다 작은 수를 반환합니다.

이 기능은 UDP와 함께 사용할 때만 작동합니다. 서버는 TCP여야 하고 클라이언트에는 UDP 클라이언트 소켓이 있어야 합니다. network_create_socket_ext() 서버에서 보낸 모든 브로드캐스트를 수신하기 위해.

이 기능은 HTML5 대상에서 실행 중인 프로젝트에서 사용할 때 작동하지 않습니다.

통사론:

network_send_broadcast(socket, port, bufferid, size);

논쟁 유형 설명
socket 네트워크 소켓 ID The id of the socket to use.
port 진짜 The port that the server will use.
bufferid 버퍼 ID The id of the buffer to get the data from.
size 진짜 The size (in bytes) of the data.

보고:

Real

예시:

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer));

위의 코드는 현재 서버의 이름 문자열("global.ServerName"에 저장됨)을 작성한 다음 ID가 "broadcast_buffer"인 바이너리 버퍼에 씁니다. 그런 다음 이 데이터는 IP 범위(장치 IP는 현재 브로드캐스트 기본 범위로 암시됨)에서 포트 6511로 로컬로 브로드캐스트됩니다.
