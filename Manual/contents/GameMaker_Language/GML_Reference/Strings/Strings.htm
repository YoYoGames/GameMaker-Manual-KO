문자열

게임을 만들 때 텍스트를 사용해야 하는 경우가 있습니다. 게임의 텍스트는 문자열 함수(문자열은 텍스트 줄을 말하는 또 다른 방법임)를 사용하여 처리되며 GameMaker 에는 하나의 문자열 삽입을 포함하여 다양한 방법으로 문자열을 조작할 수 있는 완전한 기능 세트가 있습니다. 다른 하나는 문자열 복사 및 문자열에 포함된 숫자나 문자의 문자열을 구문 분석하는 기능입니다. 일반적으로 문자열은 큰따옴표 " " 안에 텍스트를 추가 해야만 생성할 수 있으며 작은따옴표 문자열은 허용되지 않습니다. 또한 문자열을 여러 줄로 분할하고 GameMaker 가 줄 바꿈이 개행인 것처럼 렌더링할 것으로 기대할 수 없습니다( 문자열 이 아닌 경우 정확한 @ 식별자가 아래에 설명된 대로 사용됩니다.

주로 이스케이프 문자 사용과 관련된 문자열을 생성할 때 사용할 수 있는 특정 규칙이 있다는 점은 주목할 가치가 있습니다. "가 앞에 오는 문자입니다.\ " 기호입니다. 예를 들어 문자열 안에 인용 부호를 넣으려면 다음과 같이 됩니다.

str = "Hello\"World\"";

또한 GameMaker 는 4바이트 너비의 유니코드 문자를 완벽하게 지원하므로 표준의 상한에서 유니코드 문자를 디코딩하고 인코딩할 수 있습니다(이모티콘을 포함하되 이에 국한되지 않음). 유니코드 문자를 처리하려면 "\ " 모든 유니코드 리터럴 앞에 오기 - 16진수 숫자 앞에 "u ", 예를 들어 "\u00e2 " for "á"- 여기서 숫자는 유니코드 문자의 번호입니다. 이러한 방식으로 유니코드로 작업 할 때 GameMaker 는 "u ", 따라서 예를 들어 "áa"를 쓰고 싶다면 다음을 사용해야 합니다.

"\u00e2\a"

또는

"\u00e2\u61"

또는

"\u00e2" + "a"

"를 사용하는 것처럼\u00e2a " 실제로는 유니코드 문자가 됩니다 "ส "(본질적으로 "\ue2a ").

GameMaker 는 16진수 리터럴도 처리할 수 있습니다. 일반적으로 "0x ", 예를 들어 "0xff ", 여기서 숫자는 사용할 문자의 번호입니다. GameMaker 에서는 "\x " 그리고 16진수 값입니다. 이러한 문자 및 기타 미리 정의된 이스케이프 문자는 아래 표에 나열되어 있습니다.

끊임없는 설명
\n 개행
\r 캐리지 리턴
\b 백스페이스(0x08)
\f 양식 피드(0x0c)
\t 가로 탭(0x09)
\v 세로 탭(0x0b)
\\ 백슬래시 자체(0x5c)
\a 경고(0x07)
\u[Hex Digits] 유니코드 문자 삽입
\x[Hex Digits] 16진수 리터럴 문자 삽입
\[Octal Digits] 8진수 유니코드 문자 삽입

참고: 문자열은 양식 피드, 세로 탭 등을 지원하지만 렌더링 이 지원된다는 의미는 아니며 문자열을 그릴 때 이러한 문자는 무시될 수 있습니다.

전체 문자열 앞에 @ 캐릭터:

var test = @"
Line breaks
over multiple
lines
";

위의 코드는 줄 바꿈 이스케이프 문자가 포함된 것처럼 여러 줄에 걸쳐 문자열을 렌더링합니다. 축자 문자열 리터럴은 이전 GameMaker 버전 문자열 리터럴과 유사하지만 큰따옴표나 작은따옴표도 사용하며 접두사 @ 기호, 코드 파일에서 여러 줄로 나눌 수 있으며 이스케이프 문자를 지원하지 않습니다. @"Hello\World" W on World를 탈출 하지 않고 그대로 저장됩니다. 그러나 이와 같은 문자열 리터럴을 사용할 때 문자열의 일부로 따옴표를 포함하려면 문자열을 끊어야 합니다. 예:

var test = @"Hello " + "\"" + @"World" + "\""

주목해야 할 또 다른 사항은 유니코드 문자 9647(▯)이 그리기 이벤트에서 렌더링할 때 지정된 글꼴에 있을 수 있는 누락된 글리프를 대체하는 데 사용된다는 것입니다. 예를 들어 글꼴에 ° 기호가 없는 경우 90°를 쓰면 실제로 90▯이 됩니다.
