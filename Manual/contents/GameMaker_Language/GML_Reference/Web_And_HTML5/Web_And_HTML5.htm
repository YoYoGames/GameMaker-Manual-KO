웹과 HTML5

GameMaker 는 인터넷 또는 서버와 상호 작용하기 위한 다양한 기능을 제공하며 사용 가능한 모든 대상 플랫폼에서 사용할 수 있습니다. 이러한 기능은 다음과 같습니다.

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

위에 나열된 일반 기능 외에도 HTML5 대상 에는 대상을 지정하는 게임에서 사용하도록 특별히 설계된 몇 가지 추가 웹 기능이 있습니다.

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

HTML5 대상 모듈에 대한 이러한 기능 외에도 html5game 폴더의 내용이 완전히 다른 위치(URL)에 저장되는 드문 경우에 사용할 수 있는 특수 JavaScript 변수 도 있습니다. 일반적으로 이 경로는 JavaScript 파일에 하드 코딩되어 있으며 JavaScript가 호출된 위치에 상대적이지만 변수를 설정할 수 있습니다. g_GameMakerHTML5Dir 게임 스크립트를 호출하기 전에 게임이 새로 정의된 위치에서 실행됩니다. 예를 들어:

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

접두사를 사용하여 스크립트 함수에 플래그를 지정할 수 있다는 점도 주목할 가치가 있습니다.gmcallback_ 즉, HTML5용 프로젝트를 컴파일할 때 난독화되지 않습니다. 이것은 위에 나열된 것을 사용할 때 매우 유용합니다. clickable_* 기능(자세한 내용은 여기 참조).
