md5_string_utf8

암호화에서 MD5(Message-Digest algorithm 5)는 128비트 해시 값을 가진 널리 사용되는 암호화 해시 함수이며 다양한 보안 응용 프로그램에 사용되었습니다. 또한 일반적으로 파일 및 문자열의 무결성을 검사하는 데 사용됩니다. 이 함수는 입력 UTF-8 문자열(문자당 가변 바이트 수를 가짐)을 사용하고 해당 문자열에 고유한 32자의 16진수 MD5 해시를 반환합니다. 이러한 방식으로 저장하고 외부 서버(예: 외부 서버)로 보내거나 받는 정보의 무결성을 확인하는 데 사용할 수 있는 보안 키를 생성할 수 있습니다.

참고: MD5 인코딩에는 UTF-8 및 유니코드의 두 가지 형식이 있습니다. 둘 다 서로 다른 서버 설정과의 통신을 용이하게 하기 위해 제공되지만 가장 일반적으로 사용되는 것은 유니코드입니다.

참고: MD5는 완전히 안전하지 않으며 파손될 수 있습니다. 자세한 내용은 이 페이지 를 참조하세요.

통사론:

md5_string_utf8(string)

논쟁 유형 설명
string The string to hash.

보고:

예시:

var hash, str;
str = json_encode(hiscore_map);
hash = md5_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

위의 코드는 DS 맵을 JSON 문자열로 인코딩합니다. 그런 다음 MD5 해시가 생성되어 ini 파일에 저장되어 나중에 동일한 정보가 서버에서 수신되는 경우 JSON의 무결성을 확인하는 데 나중에 사용할 수 있습니다. 그런 다음 JSON이 전송됩니다.
