md5_string_unicode

암호화에서 MD5(Message-Digest algorithm 5)는 128비트 해시 값을 가진 널리 사용되는 암호화 해시 함수이며 다양한 보안 응용 프로그램에 사용되었습니다. 또한 일반적으로 파일 및 문자열의 무결성을 검사하는 데 사용됩니다. 이 함수는 입력 유니코드 문자열(각 문자에 대해 16비트)을 사용하고 해당 문자열에 고유한 32자의 16진수 MD5 해시를 반환합니다. 이러한 방식으로 저장하고 외부 서버(예: 외부 서버)로 보내거나 받는 정보의 무결성을 확인하는 데 사용할 수 있는 보안 키를 생성할 수 있습니다.

참고 MD5 인코딩에는 UTF-8과 유니코드의 두 가지 형식이 있습니다. 둘 다 서로 다른 서버 설정과의 통신을 용이하게 하기 위해 제공되지만 가장 일반적으로 사용되는 것은 유니코드입니다.

참고: MD5는 완전히 안전하지 않으며 파손될 수 있습니다. 자세한 내용은 이 페이지 를 참조하세요.

통사론:

md5_string_unicode(string)

논쟁 유형 설명
string The string to hash.

보고:

예시:

var hash, str;
str = base64_encode(game_data);
hash = md5_string_unicode(str);
http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?hash=" + hash); http_get("http://www.MacSweeneyGames.com/CatchTheHaggis/gamedata?data=" + str);

위의 코드는 문자열을 base64로 인코딩한 다음 MD5 해시를 생성합니다. 마지막으로 해시와 인코딩된 문자열이 모두 서버로 전송됩니다.
