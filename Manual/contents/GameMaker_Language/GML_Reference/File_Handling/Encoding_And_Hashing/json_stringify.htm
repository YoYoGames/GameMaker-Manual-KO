json_stringify

이 함수를 사용하면 단일 또는 중첩 구조체 및 배열을 유효한 JSON 문자열로 변환할 수 있습니다. 사용할 초기 값(배열 인덱스 또는 구조체 참조)을 제공하면 함수가 이를 "문자열화"하여 JSON 문자열로 변환합니다(GameMaker 배열을 JSON 배열로, GameMaker 구조체를 JSON 객체로 변환).

이 기능을 사용할 때 주의해야 할 몇 가지 중요한 사항이 있습니다.

  • 이 함수는 DS 맵, 목록 또는 기타 데이터 구조를 JSON으로 변환 하지 않고 단순히 구조에 대한 내부 인덱스 값(인덱스가 게임 실행 사이에 변경되기 때문에 거의 사용되지 않음)을 저장합니다. 데이터 구조에 대한 참조를 포함할 수 있는 값을 문자열화하려고 해서는 안 됩니다.json_encode() ).
  • 이 함수는 거의 모든 값을 "유효한" JSON 문자열로 변환하지만 문자열에 포함된 실제 값은 JSON 표준이 작동하는 방식으로 인해 예상한 것과 정확히 일치하지 않을 수 있습니다.

    "JSON은 숫자에 대해 불가지론적입니다. 모든 프로그래밍 언어에는 고정 또는 부동, 2진수 또는 10진수 등 다양한 기능과 보완의 다양한 숫자 유형이 있을 수 있습니다. 이로 인해 서로 다른 프로그래밍 언어 간의 상호 교환이 어려워질 수 있습니다. JSON은 대신 표현만 제공합니다. 인간이 사용하는 숫자: 일련의 숫자. 모든 프로그래밍 언어는 내부 표현에 동의하지 않더라도 숫자 시퀀스를 이해하는 방법을 알고 있습니다."

    자세한 내용은 ECMA JSON 표준 을 참조하십시오.
  • JSON으로 변환할 값에 int64를 포함하면 int32의 유효한 범위에 있으면 int 로, 정밀도를 잃지 않고 수행할 수 있으면 double 로 씁니다. 두 경우 모두 해당되지 않는 경우 ) 식별자가 있는 문자열 로 "@i64@ " 그 전에 "$i64$ " 이후. JSON을 다시 구문 분석하려고 할 때 GameMaker 는 이러한 식별자를 선택하고 값을 다시 int64로 다시 변환합니다. 이는 JSON이 서버 또는 일부 다른 비 GameMaker 대상을 위한 것이라면 다음을 의미합니다. 값은 적절하지 않으므로 피해야 합니다.
  • 포함하는 경우 infinity, -infinity 또는 NaN JSON으로 변환할 값에서 이러한 값은 JSON 표준의 일부가 아니므로 문자열로 인코딩되며 결과 JSON을 구문 분석할 때 런타임 값으로 다시 변환 되지 않고 문자열로 유지됩니다. 참고 NaN 문자열화 시 "nan"(소문자)으로 변환됩니다.
  • 반환된 JSON 문자열을 ini 파일로 ini 사양은 따옴표 및 이스케이프 문자를 사용할 때 문제를 일으킬 수 있습니다. 기능 보기ini_write_string() 자세한 내용은.

통사론:

json_stringify(val)

논쟁 유형 설명
val 구조체 또는 배열 The reference value for a struct or array to convert into a JSON string

보고:

String

예시:

var _contents =
{
    version : "1.0.0",
    data:
    {
        coins : 4,
        mana : 15,
        playername : "Gurpreet",
        items :
        [
            ITEM.SWORD,
            ITEM.BOW,
            ITEM.GUITAR
        ]
    }
};

var _json_string = json_stringify(_contents);

위의 코드는 _contents 구조체를 JSON 문자열로 변환하고 문자열을 변수에 저장합니다. 반환된 문자열은 다음과 같습니다.

{ "data": { "items": [ 0.0, 1.0, 2.0 ], "coins": 4.0, "mana": 15.0, "playername": "Gurpreet" }, "version": "1.0.0" }
