json_encode

중요한! 이 기능은 여전히 유효하지만 기능으로 대체되었습니다. json_stringify(), 레거시 지원에만 이 기능을 사용하는 것이 좋습니다.

JSON(JavaScript Object Notation)은 사람과 기계 모두에서 읽고 쓰기 쉬운 경량 데이터 교환 형식입니다. 두 가지 기본 구조를 기반으로 합니다.

  • 라고 하는 이름/값 쌍의 모음DS Map GameMaker 에서는 있지만 다른 프로그래밍 언어에서는 "사전" 또는 "객체"라고도 합니다.
  • 라고 하는 정렬된 값 목록 DS List GameMaker 에 있지만 다른 프로그래밍 언어에서는 "배열" 또는 "시퀀스"라고 부를 수도 있습니다.

json_encode() 이전에 생성한 DS 맵 또는 배열을 가져와서 JSON 문자열로 인코딩한 다음 이를 (예를 들어) 파일의 일부로 사용할 수 있습니다.http_post_string() 호출하거나 - 외부에 저장할 수 있도록 - 파일에 쓸 수 있습니다. 배열을 최상위 구조로 사용하는 경우 배열은 데이터 구조 가 아닌 유효한 값이나 다른 배열만 포함할 수 있습니다. 이를 위해 적절한 DS 기능을 사용해야 합니다.

중요한! JSON은 숫자에 대해 불가지론적입니다. 모든 프로그래밍 언어에는 고정 또는 부동, 2진수 또는 10진수, 다양한 용량 및 보완의 다양한 숫자 유형이 있을 수 있습니다. 이는 서로 다른 프로그래밍 언어 간의 교환을 어렵게 만들 수 있습니다. 대신 JSON은 인간이 사용하는 숫자 표현인 숫자 시퀀스만 제공합니다. 모든 프로그래밍 언어는 내부 표현에 동의하지 않더라도 숫자 시퀀스를 이해하는 방법을 알고 있습니다. 자세한 내용은 ECMA JSON 표준 을 참조하십시오.

따옴표와 이스케이프 문자를 사용할 때 ini 사양이 문제를 일으킬 수 있으므로 ini 파일에 JSON을 작성할 때 주의해야 합니다. 기능 보기ini_write_string() 자세한 내용은. 또한 int64를 JSON으로 인코딩하면 int32의 유효한 범위에 있으면 int 로, 정밀도를 잃지 않고 그렇게 할 수 있는 경우 double 로 또는 (두 경우 모두 해당되지 않는 경우) 다음과 같이 작성합니다. 식별자가 있는 문자열 "@i64@ " 그 전에 "$i64$ " 이후. JSON을 다시 맵으로 디코딩하려고 할 때 GameMaker 는 이러한 식별자를 선택하고 값을 int64로 다시 변환합니다. 이는 JSON이 서버 또는 다른 비 GameMaker 대상을 위한 것인지를 의미합니다., 이러한 값은 적절하지 않으므로 피해야 합니다.

참고: JSON의 계층적 기능은 특수 DS 맵 및 DS 목록 기능을 통해 사용할 수 있습니다(예: ds_map_add_list() 또는 ds_list_mark_as_map() ), 따라서 하위 목록과 지도를 인코딩할 수 있습니다.

통사론:

json_encode(map)

논쟁 유형 설명
map DS 맵 ID A DS map with the information to encode (or an array)

보고:

String

예시:

var hiscore_map, i, str;
hiscore_map = ds_map_create();
for (i = 0; i < 10; i ++;)
{
    ds_map_add(_hiscore_map, name[i], score[i]);
}
str = json_encode(hiscore_map);
get[0] = http_post_string("http://www.angusgames.com/game?game_id=" + string(global.game_id), str)
ds_map_destroy(hiscore_map);

위의 코드는 DS 맵을 만든 다음 이름 및 점수 배열을 반복하여 각 키/값 쌍을 새 맵에 추가합니다. 다음으로 이 지도는 다음을 사용하여 인코딩됩니다. json_encode() 변수 "str"에 문자열로 저장됩니다. 이 문자열은 다음을 사용하여 웹 서버로 전송됩니다. http_post_string() DS 맵은 더 이상 필요하지 않으므로 메모리 누수를 방지하기 위해 파괴됩니다.
