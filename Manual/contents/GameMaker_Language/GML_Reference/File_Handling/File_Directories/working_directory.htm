working_directory

Working_directory 실제로 무엇을 사용하는지에 따라 두 개의 다른 값을 반환할 수 있습니다.

디스크에 파일을 쓰는 경우, working_directory 대상 장치에서 게임용으로 예약된 로컬 저장소 영역을 가리킵니다(Windows에서는 "%LOCALAPPDATA%/gamedir/ " 여기서 "gamedir"은 게임 이름이 있는 디렉터리입니다.

그러나 디스크에서 읽을 때, working_directory 로컬 저장소 (game_save_id ) 또는 포함된 파일 이 저장되는 영역입니다.

참고 working_directory 변수는 마지막 백슬래시 를 포함 하는 경로를 반환합니다.

예를 들어 기본값이 있는 경우 *.txt 게임에 포함된 파일을 사용하여 메모리로 읽습니다. working_directory, 그러면 포함된 파일이 저장된 영역에서 해당 파일을 가져옵니다. 그런 다음 해당 정보를 다음을 사용하여 텍스트 파일에 쓰기로 선택한 경우 working_directory, 이렇게 하면 파일을 대상 플랫폼의 로컬 저장소에 쓰고 GameMaker 에 파일을 삭제하도록 지시하지 않는 한 다음 번에 읽을 수 있습니다. working_directory 포함된 파일이 저장된 영역을 다시 한 번 가리킵니다.

일반적으로 이것을 사용할 필요가 없으며 파일 이름 자체(추가 경로 정보 포함)만 지정하면 충분합니다. 예를 들면 다음과 같습니다.

file_text_open_read(working_directory + "my_file.txt");

다음과 같이 수행하는 것이 좋습니다.

file_text_open_read("my_file.txt");

경고 이 기능은 GameMaker 가 샌드박스 처리되어 있기 때문에 예상대로 작동하지 않을 수 있습니다! 자세한 내용은 파일 시스템 섹션을 참조하십시오.

예시:

ini_open(working_directory + "temp_ini.ini");

이렇게 하면 게임의 작업 디렉터리에서 ini 파일이 열립니다(아직 없는 경우 생성). 이것은 로컬 저장소일 수도 있고, 찾고 있는 파일이 어느 위치에 있는지에 따라 포함된 파일이 있는 영역일 수 있습니다.
