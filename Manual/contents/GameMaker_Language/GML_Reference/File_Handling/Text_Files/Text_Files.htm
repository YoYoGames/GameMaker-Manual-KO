텍스트 파일

텍스트 파일은 게임 외부에 있는 많은 양의 데이터를 저장하는 데 유용한 방법입니다. ini 파일과 달리 한 번에 최대 32개의 텍스트 파일을 열 수 있으며 많은 양의 데이터를 읽고 쓸 수도 있습니다. 그러나 다음과 같이 "섹션" 또는 "키" 구조가 설정되어 있지 않으므로 해당 데이터를 구조화하는 방법을 결정하는 것은 사용자의 몫입니다. *.ini파일.

현지화 게임의 경우 외부 파일에서 악센트 문자를 읽을 수 있다는 것이 매우 중요합니다. 즉, UTF8 형식의 txt 파일을 먼저 생성한 다음 게임메이커에 포함된 파일로 추가해야 게임 실행 시 내보내 지고 GameMaker 파일 기능에 의해 생성되는 기본 ANSI 형식 txt 파일 대신 사용됩니다. 파일이 이전에 제공되지 않은 경우. 이런 식으로 모든 악센트와 비로마 문자를 유지하면서 올바르게 읽고 쓸 수 있습니다.

파일을 처리하는 다음 함수가 존재합니다.

  • file_text_open_read
  • file_text_open_write
  • file_text_open_append
  • file_text_open_from_string
  • file_text_read_real
  • file_text_read_string
  • file_text_readln
  • file_text_write_real
  • file_text_write_string
  • file_text_writeln
  • file_text_eoln
  • file_text_eof
  • file_text_close
