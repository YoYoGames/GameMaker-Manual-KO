file_text_open_write

이 함수는 쓰기 전용 으로 지정된 파일 이름으로 텍스트 파일을 열고(파일이 존재하지 않으면 생성됨), 모든 추가 작업에 사용될 변수에 저장되어야 하는 파일의 고유 ID 를 반환합니다. 그 파일로 할 수 있습니다.

파일을 생성 할 수 없는 경우(예: 잘못된 파일 이름으로 인해) 함수는 -1을 반환합니다.

파일이 이미 존재하는 경우 이 기능을 사용하면 데이터를 저장할 때 파일을 덮어씁니다. 기존 내용을 그대로 유지하면서 파일에 데이터를 추가하려면 다음을 사용하십시오.file_text_open_append().

참고: 한 번에 최대 32개의 파일만 열 수 있습니다. 또한 정보를 기록하고 파일과 관련된 메모리를 해제하므로 완료되면 항상 파일을 닫아야 합니다.

경고! 이 기능은 GameMaker 가 샌드박싱되어 있기 때문에 예상대로 작동하지 않을 수 있습니다! 자세한 내용은 파일 시스템 섹션을 참조하십시오.

통사론:

file_text_open_write(fname);

논쟁 유형 설명
fname 끈 The name of the file to write to.

보고:

Text File ID or -1

예시:

var file;
file = file_text_open_write(working_directory + "level.txt");
file_text_write_string(file, level_data);
file_text_close(file);

위의 코드는 쓰기를 위해 "level.txt" 파일을 열고 마지막으로 파일을 다시 닫기 전에 "level_data" 변수에 저장된 문자열을 씁니다.
