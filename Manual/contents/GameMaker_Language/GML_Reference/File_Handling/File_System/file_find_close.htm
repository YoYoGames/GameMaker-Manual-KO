file_find_close

이 함수는 다음을 사용하여 열린 파일을 처리한 후에 호출해야 합니다.file_find_first() 그리고file_find_next() 메모리를 해제하는 기능. 파일 찾기 기능은 파일 디렉토리에 대한 핸들을 열고 이것은 (시간이 지남에 따라) 합산되는 최소한의 메모리를 차지합니다. 따라서 이러한 핸들을 "닫는" 데 필요한 파일을 찾은 후에는 항상 이 함수를 호출해야 합니다.

통사론:

file_find_close();

보고

예시:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

이 코드는 지정된 디렉토리가 존재하는지 확인하고 존재하는 경우 그곳으로 이동하여 첫 번째 "읽기 전용"을 반환합니다. .doc 파일을 찾았습니다. 그런 다음 두 개의 파일을 더 찾고 파일 찾기를 닫습니다.

while 루프를 사용하여 원하는 수의 파일을 찾을 수 있습니다.

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

위의 코드는 찾은 모든 파일 이름을 저장할 빈 배열을 만들고 읽기 전용 찾기를 시작합니다. .doc 파일. 해당 파일 이름이 빈 문자열이 아닌 경우 파일 이름으로 푸시합니다. files 배열을 찾은 다음 다음 파일을 찾으려고 시도합니다. 루프는 빈 문자열이 발견될 때까지(즉, 일치하는 파일이 더 이상 없을 때까지) 루프를 종료하고 끝에서 파일 찾기를 닫을 때까지 계속 반복됩니다.
