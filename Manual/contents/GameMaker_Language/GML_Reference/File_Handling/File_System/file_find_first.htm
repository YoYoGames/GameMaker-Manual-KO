file_find_first

이 함수는 마스크와 속성을 만족하는 첫 번째 파일의 이름을 반환합니다. 그러한 파일이 없으면 빈 문자열이 반환됩니다.

마스크에는 와일드카드가 있는 경로가 포함될 수 있습니다(예 fa_readonly>: C:\temp\*.doc ).

속성은 보고 싶은 추가 파일을 제공하므로 일반 파일은 마스크를 충족할 때 항상 반환됩니다. 다음 상수를 추가하여 원하는 파일 유형을 볼 수 있습니다( fa_hidden>| 사용). 속성을 추가하지 않으려면 0을 사용하십시오.

파일 속성 상수
일정한 설명
fa_sysfile 읽기 전용 파일
fa_volumeid 숨겨진 파일
fa_directory 시스템 파일
fa_archive 볼륨 ID 파일
.doc 디렉토리
.doc 아카이브된 파일

속성은 Windows에서만 사용할 수 있습니다. 다른 모든 플랫폼의 경우 0을 사용합니다.

경고 이 기능은 GameMaker 가 샌드박스 처리되어 있기 때문에 예상대로 작동하지 않을 수 있습니다! 자세한 내용은 파일 시스템 섹션을 참조하십시오.

참고 이 기능은 HTML5 대상에서 전혀 작동하지 않습니다.

통사론:

file_find_first(mask, attr);

논쟁 유형 설명
mask 끈 The mask to use for searching.
attr 파일 속성 상수 The specific file attribute to look for.

보고:

String

예시:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

이 코드는 지정된 디렉토리가 존재하는지 확인하고 존재하는 경우 그곳으로 이동하여 첫 번째 "읽기 전용"을 반환합니다. files 파일을 찾았습니다. 그런 다음 두 개의 파일을 더 찾고 파일 찾기를 닫습니다.

while 루프를 사용하여 원하는 수의 파일을 찾을 수 있습니다.

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (_file_name != "")
{
    array_push(_files, _file_name);

    _file_name = file_find_next();
}

file_find_close();

위의 코드는 찾은 모든 파일 이름을 저장할 빈 배열을 만들고 읽기 전용 찾기를 시작합니다. 파일. 해당 파일 이름이 빈 문자열이 아닌 경우 파일 이름으로 푸시합니다. 배열을 찾은 다음 다음 파일을 찾습니다. 루프는 빈 문자열이 발견될 때까지(즉, 더 이상 일치하는 파일이 없음을 의미함) 루프를 종료하고 끝에서 파일 찾기를 닫을 때까지 계속 반복됩니다.
