round

그것이 말하는 것처럼, round() 실수를 취해 가장 가까운 정수로 반올림하거나 내림합니다. 제공된 숫자가 정확히 반정수(1.5, 17.5, -2.5 등)인 특수한 경우 숫자는 가장 가까운 짝수 값으로 반올림됩니다. 예를 들어 2.5는 2로 반올림되고 3.5는 반올림됩니다. 4로 반올림됩니다. 이러한 유형의 반올림을 뱅커 반올림 이라고 하며 많은 수의 반복에 대해 또는 부동 소수점 수학을 사용할 때 ".5를 초과하면 반올림하고 그렇지 않으면 반올림" 접근 방식보다 통계적으로 더 나은 반올림을 제공합니다.

이것이 의미하는 바는 값의 분수가 0.5이면 반올림된 결과는 입력 값에 가장 가까운 짝수 정수라 는 것입니다. 예를 들어, 24.5와 마찬가지로 23.5는 24가 되고 -23.5는 -24.5와 마찬가지로 -24가 됩니다. 이 방법은 양수 값과 음수 값을 대칭적으로 처리하므로 부호 편향이 없으며 더 중요한 것은 값의 합리적인 분포를 위해 반올림된 숫자의 예상(평균) 값이 원래 숫자의 값과 동일하다는 것입니다.

통사론:

round(n);

논쟁 유형 설명
n The number to round.

보고:

예시:

score += round(hp / 5);

위의 코드는 점수 값에 반올림된 정수를 추가합니다.
