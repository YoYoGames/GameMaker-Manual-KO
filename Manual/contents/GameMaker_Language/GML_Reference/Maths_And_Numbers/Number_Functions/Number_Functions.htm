숫자 함수

GameMaker 의 실수는 배정도 부동 소수점 숫자로 간주 됩니다. 즉, 소수점 앞이나 뒤에 고정된 자릿수가 없는 소수점이 있습니다. 소수점 값이 없습니다. 예를 들어 2는 정수이지만 2.01은 부동 소수점 실수입니다.

참고: HTML5 대상에서 모든 실수는 두 배입니다.

정수와 부동 소수점 사이의 이러한 구별은 크로스 플랫폼 개발을 다룰 때 매우 중요합니다. Windows PC에서 수행되는 계산의 정밀도가 모바일 장치에서 수행되는 동일한 계산의 정밀도와 같지 않기 때문입니다. 즉, 비교할 때 특히 주의해야 합니다. 예를 들면 다음과 같습니다.

if (image_index == 1)
{
    do something...
}

위의 예에서 설정한 경우 image_speed 0.25까지 - 예를 들어 - 4단계 후에 다음과 같이 가정할 수 있습니다. image_index 값은 1이지만 부동 소수점 수학이 작동하는 방식으로 인해 1.0000002와 같은 값 이 될 수 있으므로 평가는 true. 이러한 유형의 오류는 디버그하기가 매우 어려울 수 있으므로 항상 이를 인식하고 미리 계획하고 값을 확인하는 다른 수단을 사용하거나 적절한 바닥 또는 반올림 함수(아래 나열)를 사용하여 숫자를 다음으로 변환하는 것이 좋습니다. 정수로 확인하십시오(부동 소수점 수학에 대한 자세한 내용과 이것이 문제인 이유는 여기를 참조하십시오 ). 예를 들어 위의 코드는 다음과 같이 작성할 수 있습니다.

if (floor(image_index) == 1)
{
    do something...
}

또한 YoYo Compiler 대상을 사용할 때 모든 표현식과 함수는 왼쪽에서 오른쪽으로 평가 되지만 다른 모든 대상 플랫폼에서는 오른쪽에서 왼쪽 으로 평가됩니다. 플랫폼에서:

val = max(num, ++num, num++);

참고: 자세한 내용은 평가 순서 섹션을 참조하십시오.

GameMaker 에서 사용 가능한 특수 기능을 사용하여 부동 소수점 수학에 대한 엡실론 값을 설정할 수도 있습니다. 실수가 가장 가까운 부동 소수점 숫자로 반올림되면 엡실론("머신 엡실론"이라고도 함)이 상대 오차의 상한을 형성하고 다음 함수를 사용하여 엡실론 값을 가져오고 설정할 수 있습니다.

  • math_set_epsilon
  • math_get_epsilon

이 함수는 모두 난수와 값을 사용합니다.

참고 무작위 기능을 사용할 때 GameMaker 는 게임을 시작할 때마다 동일한 무작위 시드를 유지합니다. 이렇게 하면 임의의 함수가 처음에 항상 동일한 값을 반환하도록 보장하므로 디버깅이 훨씬 쉬워지지만, 이러한 일이 발생하지 않도록 하려면 먼저 게임을 시작할 때 다음 중 하나를 사용하여 새로운 임의의 시드를 설정해야 합니다.randomise() 또는random_set_seed().

  • choose
  • random
  • random_range
  • irandom
  • irandom_range
  • random_set_seed
  • random_get_seed
  • randomise

다음은 어떤 방식으로든 값을 반올림하거나 다양한 주어진 값에서 단일 값을 선택하는 모든 함수입니다.
