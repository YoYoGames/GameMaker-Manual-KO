dot_product_3d_normalised

내적은 두 벡터 사이의 각도 관계를 나타내는 값으로 두 벡터를 취하여 곱한 다음 결과를 더하여 구합니다. "내적"이라는 이름은 이 연산을 지정하는 데 자주 사용되는 중심 점 "·"에서 파생되었습니다(대체 이름 "스칼라 곱"은 결과의 벡터 특성보다 스칼라를 강조합니다).

실제 수학 공식은 다음과 같이 작성할 수 있습니다.

2D에서 벡터의 내적 a[x1,y1] 그리고 b[x2,y2] ~이다 x1x2 + y1y2, 따라서 3D에서 벡터의 내적 a[x1,y1,z1] 그리고 b[x1,y1,z1] ~이다 x1x2 + y1y2 + z1z2. 따라서 GameMaker의 dot_product_3d 는 다음과 같이 계산됩니다.

a · b = (x1*x2) + (y1*y2) + (z1*z2);

정규화된 내적은 어떻습니까? 정규화된 내적은 반환 값을 -1과 1의 범위로 가져오는 방식으로 수정되었습니다(자세한 내용은 벡터 참조). 이는 특히 조명 및 기타 3D를 처리할 때 많은 상황에서 매우 유용합니다. 기능.

통사론:

dot_product_3d_normalised(x1, y1, z1, x2, y2, z2)

논쟁 유형 설명
x1 The x coordinate of the first vector.
y1 The y coordinate of the first vector.
z1 The z coordinate of the first vector.
x2 The x coordinate of the second vector.
y2 The y coordinate of the second vector.
z2 The z coordinate of the second vector.

보고:

예시:

var x1, y1, x2, y2;
x1 = 0;
y1 = 1;
z1 = 0;
x2 = o_Player.x - x;
y2 = o_Player.y - y;
z2 = o_Player.z - z;
if dot_product_3d_normalised(x1, y1, z1, x2, y2, z2) > 0 above=true else above=false;

위의 코드는 인스턴스 y(위쪽) 축을 따라 벡터 법선을 만든 다음 플레이어 개체 "o_Player"의 벡터를 자체적으로 가져옵니다. 마지막으로 이 두 벡터의 내적을 계산하고 tan 0보다 크면 변수 "above"를 true로 설정하고 0보다 작거나 같으면 변수를 "false"로 설정합니다.
