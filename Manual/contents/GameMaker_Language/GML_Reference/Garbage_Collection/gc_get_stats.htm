gc_get_stats

이 기능을 사용하면 가비지 수집기의 현재 상태에 대한 정보를 검색할 수 있습니다. 이 함수는 다음과 같은 멤버 변수를 포함하는 구조체 를 반환합니다(여기서 "객체"는 가비지 수집될 수 있는 모든 것을 나타내며 자산 브라우저에 정의된 일반 객체 인스턴스가 아님 ).

GC 통계 구조체
변하기 쉬운 유형 설명
objects_touched 진짜 이것은 이전 프레임에서 찾은 가비지 수집기의 활성 개체 수입니다. 수집된 세대에 따라 다릅니다.
objects_collected 진짜 가비지 수집기가 결정한 개체 수는 이전 프레임에서 활성화되지 않았으므로 삭제할 수 있습니다.
traversal_time 진짜 이것은 가비지 수집기가 어떤 개체가 활성 상태인지 파악하는 데 걸린 시간(메인 스레드에서)입니다.
collection_time 진짜 가비지 수집기가 비활성화된 것으로 간주되는 개체를 정리하는 데 걸린 시간(별도 스레드에서)은 마이크로초 단위입니다.
gc_frame 진짜 이것은 가비지 수집 패스가 발생할 때마다 증가하는 카운터입니다. 가비지 수집이 비활성화되어 있으면 증가하지 않습니다.
generation_collected 진짜 마지막으로 수집된 세대의 인덱스입니다. 0은 가장 어린 세대이고 3은 현재 가장 오래된 세대입니다.
num_generations 진짜 이것은 가비지 수집 생성의 총 수입니다.
num_objects_in_generation 실수 배열 이것은 배열입니다(크기 num_generations ) 각 세대의 개체 수를 포함합니다.

참고: HTML5 대상 플랫폼에서 가비지 수집은 JavaScript 엔진에 의해 처리되므로 해당 플랫폼에서 이 함수를 사용하는 경우 위 구조체의 멤버 변수는 모두 0을 반환합니다.

이 기능을 사용할 때 객체 에 대해 표시되는 정보는 전체 생성 처리가 완료된 경우에만 업데이트 되며 프레임 시간 설정에 따라 여러 프레임이 소요될 수 있습니다(프레임 타이밍에 대한 자세한 내용은 여기 참조).

통사론:

gc_get_stats();

보고:

GC Stats Struct

예시:

if (global.debug == true)
{
    var _s = gc_get_stats();
    var _t = _s.traversal_time;
    var _c = _s.collection_time;
    show_debug_message("Traversal time = " + string(_t))
    show_debug_message("Collection time = " + string(_c))
}

위의 코드는 전역 변수를 확인하고 다음과 같은 경우 true Garabge 수집기에서 정보를 가져와 디버그 메시지로 콘솔에 출력합니다.
