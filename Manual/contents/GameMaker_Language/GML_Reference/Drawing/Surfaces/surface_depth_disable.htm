surface_depth_disable

이 기능은 비활성화된 후 생성되는 모든 표면에 대한 자동 깊이 버퍼 생성을 비활성화합니다. 일반적으로 모든 표면에는 깊이 버퍼가 있으므로 표면에 3D 개체를 그리면 표면에 대한 렌더링과 백 버퍼에 대한 렌더링 간의 기능적 패리티를 유지하기 위해 깊이에 따라 적절하게 정렬되지만 깊이 버퍼를 할당하면 기본적으로 메모리 양이 두 배로 늘어납니다. 특히 게임이 메모리 집약적이며 2D 게임의 경우 일반적으로 필요하지 않은 경우 과도하고 불필요한 오버헤드가 될 수 있는 표면에 필요합니다.

이 기능을 사용하려면 표면을 생성 하기 전에 호출한 다음 (필요한 경우) 후에 다시 활성화할 수 있습니다. 깊이 버퍼가 비활성화되었을 때 생성된 모든 표면은 그려질 때 더 이상 깊이 버퍼를 사용하지 않습니다. 그 결과 깊이 패리티가 있는 표면에 그려지는 사물이 됩니다. 아래 이미지는 실제로 이것을 보여줍니다.

위 이미지에서는 3D에서 정점 버퍼를 사용하여 직사각형과 삼각형을 그리고 있습니다. 빨간색 삼각형의 좌표는 녹색 사각형 뒤에 배치되지만 삼각형을 그리는 코드는 사각형에 대한 코드 뒤에 호출되며 깊이 버퍼가 활성화되면 삼각형은 여전히 3D 깊이가 사각형 뒤에 그려집니다. 존중받고 있다. 그러나 표면에 대한 깊이 버퍼를 비활성화하면 코드 순서만 존중되므로 삼각형이 사각형 위에 그려집니다.

참고: 표면을 사용할 때 3D에서 깊이를 적절하게 유지하려면 깊이 버퍼를 활성화해야 할 뿐만 아니라 z-write 및 z-testing도 활성화해야 합니다. 기능 보기gpu_set_ztestenable() 그리고gpu_set_zwriteenable() 자세한 내용은.

참고: 게임이 100% 2D이거나 표면 깊이 버퍼가 필요하지 않은 경우 게임 시작 시 이 함수를 한 번만 호출하여 버퍼를 비활성화하는 것이 좋습니다.

통사론:

surface_depth_disable(disable);

논쟁 유형 설명
disable If set to true the depth buffer will be disabled for created surfaces.

보고:

예시:

var _depth_setting = surface_get_depth_disable();
surface_depth_disable(false);
surface_with_depth = surface_create(room_width, room_height);
surface_depth_disable(true);
surface_without_depth = surface_create(room_width, room_height);
surface_depth_disable(_depth_setting);

위의 코드는 현재 깊이 버퍼 설정을 변수에 저장한 다음 원래 설정을 다시 복원하기 전에 깊이 버퍼링이 활성화된 표면과 비활성화된 표면의 두 가지 표면을 만듭니다.
