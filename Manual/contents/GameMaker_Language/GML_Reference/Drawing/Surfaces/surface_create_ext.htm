surface_create_ext

이 기능을 사용하면 웹 페이지에 이미 있는 캔버스 요소에 표면을 연결할 수 있습니다. 즉, 게임의 일부를 효과적으로 분할하여 페이지 내의 다양한 위치에 그릴 수 있습니다. 이를 위해서는 캔버스 요소를 내에서 올바르게 정의 해야 합니다. *.html 만들고자 하는 표면에 해당하는 올바른 크기와 이름을 사용하여 게임의 페이지에 표시합니다. 따라서 "기본" 캔버스가 있고 이 기능을 사용하여 올바른 표면에 할당되는 보조 표면 캔버스 요소가 있습니다. 다음 이미지는 세 개의 캔버스 요소가 있는 페이지를 설정하는 방법의 예입니다.

페이지 레이아웃은 특히 모든 요소를 올바르게 정렬하려는 경우 까다로울 수 있지만 레이아웃을 만드는 어려운 작업이 완료되면 다음을 추가할 수 있습니다. html 게임 옵션 의 HTML5 탭 을 사용하여 게임의 기본 페이지 파일로 파일을 만듭니다. 다음으로 해야 할 일은 방과 보기를 설정하는 것입니다. 각 표면은 게임 이미지를 "캡처"하기 위해 특정 보기와 연결되어야 하기 때문입니다(보기 변수 참조 view_surface_id[0...7] ). 아래 이미지는 위의 캔버스 예제에 대한 게임룸이 어떻게 설정되는지 보여줍니다.

마지막으로 이 함수를 사용하여 캔버스 요소에 사용된 이름과 동일한 캔버스의 크기에 해당하는 크기로 표면을 생성합니다. 이 함수는 향후 함수 호출을 위해 변수에 저장해야 하는 표면의 인덱스를 반환합니다. 표면이 처음 생성될 때 기본적으로 목적을 위해 따로 보관된 메모리 영역이기 때문에 "노이즈"가 포함될 수 있으므로(그 메모리에는 여전히 정보가 포함될 수 있음) 같은 기능draw_clear_alpha().

참고: 이 기능은 HTML5 모듈에서만 사용할 수 있습니다.

통사론:

surface_create_ext(name, w, h);

논쟁 유형 설명
name The name of the canvas element to link the surface to.
w The width of the surface to be created.
h The height of the surface to be created.

보고:

Surface ID

예시:

s1 = surface_create_ext("surface1", 192, 550);
s2 = surface_create_ext("surface2", 608, 186);
view_surface_id[1] = s1;
view_surface_id[2] = s2;

위의 코드는 크기가 다른 두 개의 표면을 만들고 각각을 다른 캔버스 요소에 할당한 다음 해당 표면을 두 개의 보기에 할당하여 방의 올바른 부분을 캡처합니다.
