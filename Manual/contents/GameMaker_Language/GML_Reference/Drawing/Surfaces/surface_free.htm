surface_free

표면으로 작업할 때 사용을 마칠 때마다 항상 이 기능을 사용해야 합니다. 표면은 메모리 공간을 차지하므로 일반적으로 방 끝에서 제거해야 하지만 사용 용도에 따라 언제든지 제거할 수 있습니다. 그렇게 하지 않으면 메모리 누수가 발생하여 결국 게임 속도가 느려지고 충돌이 발생할 수 있습니다.

참고: 표면으로 작업할 때 표면이 텍스처 메모리에 저장되어 있기 때문에 언제든지 존재하지 않을 가능성이 있습니다. 다음을 사용하여 표면이 존재하는지 항상 확인해야 합니다.surface_exists() 직접 참조하기 전에.

통사론:

surface_free(surface_id);

논쟁 유형 설명
surface_id The ID of the surface to be freed.

보고:

예시:

if keyboard_check_pressed(vk_escape)
{
    surface_free(surf);
    room_goto(rm_Menu);
}

위의 코드는 키 누름을 확인하고 키 누름이 감지되면 변수 "surf"에 인덱싱된 표면용으로 예약된 메모리를 해제한 다음 공간을 변경합니다.
