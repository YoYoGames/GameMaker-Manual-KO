surface_exists

이 기능은 표면의 휘발성으로 인해 표면으로 작업할 때 필수적입니다. 표면은 항상 텍스처 메모리에 보관되므로 한 순간에 파괴될 수 있습니다(예: 화면 보호기가 Windows에서 시작되거나 Android 기기에서 최소화될 때). 따라서 표면이 존재하기 전에 항상 확인해야 합니다. 그것으로 무엇이든 할 수 있습니다(여기에는 화면에 그리는 것도 포함됩니다). 아래 예제 코드는 표면을 확인하고 제거된 경우 다시 생성하기 위해 인스턴스의 그리기 이벤트에서 이 명령의 일반적인 사용을 보여줍니다(표면은 원래 개체의 생성 이벤트에서 생성됨을 유의하십시오. ).

통사론:

surface_exists(surface_id);

논쟁 유형 설명
surface_id The ID of the surface to check.

보고:

예시:

if !surface_exists(surf)
{
    surf = surface_create(1024, 1024);
}
else
{
    if view_current = 1
    {
        draw_surface(surf,0,0);
    }
}

위의 코드는 "surf" 변수에 인덱싱된 표면이 존재하는지 확인하고 존재하지 않으면 다시 생성합니다. 존재하는 경우 현재 그려지고 있는 뷰를 확인하고 뷰[1]이면 표면을 그립니다.
