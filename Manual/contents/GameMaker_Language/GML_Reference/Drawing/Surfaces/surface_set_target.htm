surface_set_target

이 기능을 사용하면 모든 추가 그리기를 화면이 아닌 대상 표면으로 설정하고 이러한 방식으로 GameMaker 가 지정된 표면에 특정 항목만 그리도록 지시할 수 있습니다. 전화를 하지 않으시면 참고하세요surface_reset_target() 작업을 마친 후에는 모든 추가 그리기(심지어 다른 경우에도)가 표면에서 수행되므로 화면에 아무 것도 그려지지 않습니다. 또한 인스턴스의 그리기 이벤트에서 표면 자체가 화면에 그려지지 않으면 아무 것도 표시되지 않음을 알아야 합니다. 이 함수의 반환 값을 디버그 도구로 확인하여 표면 대상이 설정되었는지 여부를 확인할 수도 있습니다. 반환 값 0은 대상 설정 실패이고 다른 양수 값은 성공입니다.

한 가지 주목해야 할 점은 표면이 스택 되어 대상에서 대상으로 점프한 다음 끝에 일반 그리기 대상으로 재설정할 수 없다는 것입니다. 대신 렌더링 대상을 열고 닫아야 합니다. 예를 들어 다음은 올바르게 작동 하지 않습니다.

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();

대신, 코드 블록에 여는 대괄호 {}를 사용해야 하는 것처럼 설정한 각 표면에 대한 대상을 재설정해야 합니다. 따라서 위의 내용은 다음과 같이 작성되어야 합니다.

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_reset_target();
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();

또는 다음과 같이:

surface_set_target(surf1);
draw_text(32, 32, "surface1");
surface_set_target(surf2);
draw_text(32, 64, "surface2");
surface_reset_target();
surface_reset_target();

참고: 표면으로 작업할 때 표면이 텍스처 메모리에 저장되어 있기 때문에 언제든지 존재하지 않을 가능성이 있습니다. 다음을 사용하여 표면이 존재하는지 항상 확인해야 합니다.surface_exists() 직접 참조하기 전에.

통사론:

surface_set_target(surface_id);

논쟁 유형 설명
surface_id The ID of the surface to set as the drawing target.

보고:

예시:

if view_current = 0
{
    surface_set_target(surf);
    with (obj_Effect)
    {
        draw_self();
    }
    surface_reset_target();
}
else
{
    draw_surface(surf, 0, 0);
}

위의 코드는 현재 어떤 뷰가 그려지고 있는지 확인하고 그것이 view[0]인 경우 그리기 대상을 표면으로 설정하고 그리기 대상을 다시 재설정하기 전에 "obj_Effect" 개체의 모든 인스턴스를 그립니다. 보기가 보기[0]가 아니면 표면이 화면에 그려집니다.
