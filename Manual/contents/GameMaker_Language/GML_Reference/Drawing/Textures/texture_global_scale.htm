texture_global_scale

이 기능을 사용하면 최종 게임 실행 파일의 일부인 WAD 파일에서 로드 시 텍스처 페이지의 크기를 제어할 수 있습니다. 입력 값은 2의 거듭제곱 값이어야 하며 다음과 같이 작동합니다.

  • 텍스처 페이지를 1:1로 설정하려면 texture_global_scale(1)
  • 반 질감에 항상 사용 texture_global_scale(2)
  • 4분의 1로 사용 texture_global_scale(4), 등...

이런 식으로 메모리 문제가 있는 플랫폼에서 텍스처 페이지 메모리 사용량을 더 잘 제어할 수 있습니다. 텍스처 페이지는 WAD에서 요청 시 생성되므로 게임 시작 시 이를 호출하여 스케일된 모든 텍스처를 로드하거나 특정 시간에 호출하여 특정 텍스처를 로드할 수 있습니다. 이 경우 아마 전화하고 싶어draw_texture_flush() 이 함수를 호출하기 전에 프리페치 함수를 사용하여 메모리에 올바른 텍스처 페이지를 가져옵니다.

통사론:

texture_global_scale(pow2integer);

논쟁 유형 설명
pow2integer The scale factor to use (1, no scale, 2, half scale, 4, quarter scale, etc...)

보고:

예시:

draw_texture_flush();
texture_global_scale(2);
sprite_prefetch(spr_Trees);

위의 코드는 메모리에서 모든 텍스처를 플러시한 다음 텍스처 스케일링을 2로 설정하고(그래서 텍스처 페이지는 절반 크기임) 마지막으로 스프라이트 "spr_Trees"가 포함된 텍스처 페이지를 미리 가져옵니다.
