draw_path

이 기능을 사용하면 GameMaker 가 화면에 대한 경로를 그리도록 할 수 있습니다. 경로는 단순한 선으로 그려지며 호출 인스턴스에 상대적이거나 경로 편집기에서 또는 코드를 통해 생성된 절대 위치에 있을 수 있습니다. 이 함수는 동적 경로를 디버깅할 때 매우 유용합니다(예: mp_grid_path() 함수).

참고: 셰이더를 사용하여 모양을 그리고 싶다면 대부분의 셰이더가 정점, 질감, 색상과 같은 입력을 예상한다는 점을 알아야 합니다. 그러나 이 기능을 사용할 때 정점과 색상 데이터만 전달되기 때문에 셰이더는 아무 것도 그리지 않을 수 있습니다. 이러한 방식으로 모양을 그려야 하는 경우 이를 염두에 두고 셰이더를 사용자 정의해야 합니다.

통사론:

draw_path(path, x, y, absolute);

논쟁 유형 설명
path The path to draw
x The x coordinate of where the path is drawn
y The y coordinate of where the path is drawn
absolute Whether the path is drawn at the absolute position (true ) or the relative position (false )

보고:

예시:

if mp_grid_path(grid, path, x, y, obj_Player.x, obj_Player.y, 1)
{
    draw_path(path, x, y, false);
}

위의 코드는 mp_grid_path 경로를 생성하고 변수 "경로"에 저장하는 함수입니다. 경로가 성공적으로 생성되면 코드를 실행하는 인스턴스에 상대적인 위치에 화면이 그려집니다.
