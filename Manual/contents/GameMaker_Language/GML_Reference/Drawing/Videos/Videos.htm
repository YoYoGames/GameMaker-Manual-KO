비디오 재생

비디오를 재생하는 방법?

런타임에 비디오를 로드하고 재생하려면 다음 단계를 따르십시오.

  • 프로젝트의 포함된 파일 ( video_open()>/datafiles 디렉토리)에 비디오 파일을 추가하십시오.
  • 다음을 사용하여 비디오 로드 video_draw().
    • 또는 URL에서 비디오를 로드합니다.
  • 다음을 사용하여 Draw 이벤트에 그립니다. draw_surface().
    • 이 함수는 비디오를 내부 표면에 그려 반환 배열의 일부로 반환합니다.
    • 다음을 사용하여 해당 표면을 수동으로 그려야 합니다. video_close() 동영상이 표시되도록 합니다.
      • YUV 비디오 를 그리려면 추가 단계가 필요합니다.
  • 비디오가 완료되면 호출하여 메모리에서 제거하십시오. video_close().
    • 비디오가 끝나면 비동기 콜백 을 사용하여 코드를 실행합니다.

비디오 코덱 및 라이선스

GameMaker 는 코덱과 함께 제공되지 않으며 게임이 실행되는 플랫폼에 있는 코덱을 사용합니다. 즉, 비디오는 의도한 대상 플랫폼에서 디코딩할 수 있는 형식으로 인코딩되어야 합니다.

특정 형식을 사용하는 경우 관련 코덱의 라이선스 및 게임 수익에 따라 라이선스 비용을 지불해야 할 수 있으며, 이 경우 코덱 소유자에게 지불해야 하는 모든 수수료에 대한 법적 책임은 개발자에게 있습니다. 코덱도 플랫폼마다 라이선스가 다르기 때문에 여러 플랫폼에 배포되는 게임에 주의가 필요합니다.

비디오 형식(예: .mp4, .avi, .mkv 등)은 비디오 및 오디오 스트림을 래핑하는 단순한 컨테이너이며, 둘 다 별도로 인코딩될 수 있으며 별도의 라이선스가 필요할 수 있습니다.

콘솔 플랫폼에 배포하려면 계정 담당자에게 문의하여 추가 라이선스 비용을 지불하지 않고도 사용할 수 있는 형식을 알아보세요.

형식

게임에서 로드하고 재생할 수 있는 비디오 형식은 대상 플랫폼과 해당 플랫폼에서 지원되는 코덱에 따라 다릅니다.

기능

다음 기능을 사용하여 런타임에 비디오 파일을 로드, 재생 및 조작할 수 있습니다.

  • video_open
  • video_close
  • video_draw
  • video_set_volume
  • video_pause
  • video_resume
  • video_enable_loop
  • video_seek_to

다음 함수는 현재 로드된 비디오에 대한 정보를 가져오는 데 사용됩니다.

  • video_is_looping
  • video_get_volume
  • video_get_duration
  • video_get_position
  • video_get_status
  • video_get_format

비동기 콜백

Async Social 이벤트는 async_load 맵의 "type" 키가 지정된 값으로 설정된 아래 주어진 단계에서 트리거됩니다.

  • "video_start": 동영상 재생이 시작될 때, 즉 상태 가 video_status_preparing 에서 video_status_playing 으로 변경됩니다.
  • "video_end": 비디오가 종료될 때. 비디오가 루프로 설정되어 있으면 트리거되지 않습니다.

다음은 비디오 콜백을 수신하는 비동기식 소셜 이벤트의 예입니다.

var _type = async_load[? "type"];

if (_type == "video_start")
{
    obj_video.display_video = true;
}
else if (_type == "video_end")
{
    obj_video.display_video = false;
    video_close();
}

위의 코드는 async_load 맵에서 유형을 읽습니다. 비디오가 시작되면 비디오를 표시할 수 있도록 개체의 변수를 true 로 설정합니다.

비디오가 끝나면 동일한 변수를 false 로 설정한 다음 메모리에서 비디오를 제거합니다.
