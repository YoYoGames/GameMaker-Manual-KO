part_system_create_layer

이 기능은 주어진 레이어에 새로운 파티클 시스템을 생성합니다. 함수에서 반환된 고유 레이어 ID를 제공합니다. layer_create() 또는 문자열로 사용할 레이어의 이름(예: "instance_layer")을 입력한 다음 시스템에 지속성 여부를 표시합니다. 시스템이 지속적으로 플래그가 지정되지 않은 경우 시스템이 생성된 방의 끝에서 자동으로 소멸됩니다(이는 함수를 호출한 것과 동일합니다.part_system_destroy() 또한 시스템과 관련된 모든 이미터도 파괴합니다. 단, 퍼시스턴트로 플래그가 지정되면 방이 변경될 때 시스템이 다음 방으로 옮겨지며, 다음 방에 할당된 것과 동일한 이름이나 깊이의 레이어가 없으면 새 레이어가 생성됩니다. 룸 전체에 지속되는 시스템의 경우 원래 레이어와 동일한 이름이 지정됩니다. 방을 변경할 때 다음 방에 같은 이름의 다른 레이어가 있는 경우 해당 레이어 의 깊이에 관계없이 해당 이름의 레이어에 지속 인스턴스가 할당됩니다. 마지막으로, 지속형 시스템이 시스템이 생성된 레이어와 동일한 깊이의 레이어가 있는 방으로 이동하면 이 레이어에 할당 되지 않고 대신 동일한 깊이에 새 레이어가 생성됩니다( 원본 레이어와 동일한 이름).

이 함수는 시스템 ID를 제공해야 하는 모든 추가 함수 호출에 사용해야 하는 입자 시스템에 대한 고유 ID 값을 반환합니다.

중요한! 파티클 시스템에 영구 플래그를 지정하면 사용하지 않을 때 적절한 파괴 기능을 사용하여 파티클 시스템(및 할당된 모든 이미터)을 수동으로 정리해야 합니다. 그렇지 않으면 최종 게임에 부정적인 영향을 주는 메모리 누수 위험이 있습니다.

통사론:

part_system_create_layer(layer_id, persistent);

논쟁 유형 설명
layer 문자열 또는 레이어 ID The layer ID (or name) to assign the particle system to (can be any layer type)
persistent 부울 Flag the particle system as persistent (set to true ) or not (set to false )

보고:

Particle System ID

예시:

global.p_sys = part_system_create_layer("effects_layer", true);

위의 코드는 주어진 레이어에 새로운 파티클 시스템을 생성하고 후속 방에서 지속되는 것으로 플래그를 지정합니다. 파티클 시스템의 ID는 나중에 참조할 수 있도록 전역 범위 변수에 저장됩니다.
