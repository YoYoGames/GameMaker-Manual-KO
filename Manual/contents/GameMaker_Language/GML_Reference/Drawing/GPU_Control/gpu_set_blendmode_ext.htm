gpu_set_blendmode_ext

GameMaker 가 픽셀을 그릴 때 소스 색상(그리려는 픽셀의 색상)과 대상 색상(그리려는 픽셀에 이미 있는 색상)이 있으므로 새 색상을 결정할 때 픽셀의 소스 및 대상 색상은 선택한 혼합 모드에 따라 계산됩니다. 색상의 각 구성 요소는 0과 1 사이의 부동 소수점 값으로 저장되며, 새로운 색상은 소스 색상의 각 구성 요소에 몇 가지 요소를 곱하고 대상 색상의 각 구성 요소에 다른 요소를 곱한 다음 추가하여 계산됩니다. 결과는 구성 요소별로 구성됩니다.

이 기능을 사용하면 사용자 정의 블렌드 모드를 생성하기 위해 함께 인수되어야 하는 다양한 구성 요소를 설정할 수 있습니다. 소스와 대상에는 모두 빨강, 녹색, 파랑 및 알파 구성 요소가 있으며 다음 차트에서 대상의 RGBA는 (Rd, Gd, Bd, Ad)인 동안 소스의 RGBA(Rs, Gs, Bs, As)로 간주됩니다. 이 함수에서 사용할 때 사용할 수 있는 11개의 상수는 소스 또는 대상(또는 둘 다)일 수 있습니다.

일정한 혼합 요소(빨간색, 녹색, 파란색, 알파)
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-A)
bm_src_alpha (대로, 그대로, 그대로, 그대로)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (광고, 광고, 광고, 광고)
bm_inv_dest_alpha (1-광고, 1-광고, 1-광고, 1-광고)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) 여기서 f = min(As, 1-Ad)

중요한! WebGL이 활성화 되지 않은 HTML5는 다음 모드를 올바르게 표시할 수 없습니다.

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

이 함수에 두 개의 개별 인수를 제공하거나 인수 배열(함수에서 반환된 대로)을 제공할 수 있습니다.gpu_get_blendmode_ext() 예를 들어). 배열을 제공하는 경우 다음 두 가지 요소가 있어야 합니다.

  • [0] = 소스 혼합 모드(기본값은 bm_src_alpha )
  • [1] = 대상 혼합 모드(기본값은 bm_inv_src_alpha )

블렌드 모드를 최대한 활용하고 작동 방식과 화면에 그려지는 최종 이미지에 미치는 영향을 이해하려면 다음 가이드를 읽는 것이 좋습니다.

  • 블렌드 모드 사용 가이드

통사론:

gpu_set_blendmode(src, dest);

논쟁 유형 설명
src 혼합 모드 계수 상수 Source blend mode factor (see constants above).
dest 혼합 모드 계수 상수 Destination blend mode factor (see constants above)

보고:

N/A

예시:

gpu_set_blendmode_ext(bm_src_alpha, bm_one);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal);

이렇게 하면 검정색이 투명하게 되어 흰색이 외부에서 강해지고 원 중심에서 점점 약해지면서 '광택' 효과가 생성됩니다. 블렌드 모드는 원을 그린 후 재설정되어 추가 블렌딩이 그 이후의 다른 모든 항목에도 적용되지 않도록 합니다.
