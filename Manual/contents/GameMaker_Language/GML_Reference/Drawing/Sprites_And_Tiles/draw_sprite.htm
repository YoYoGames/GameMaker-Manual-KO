draw_sprite

이 함수는 게임 룸 내 위치에 주어진 스프라이트와 하위 이미지를 그립니다. 스프라이트의 경우 인스턴스 변수를 사용할 수 있습니다. sprite_index 코드를 실행하는 인스턴스에 할당된 현재 스프라이트를 가져오거나 다른 스프라이트 자산을 사용할 수 있습니다. 하위 이미지도 인스턴스 변수로 설정할 수 있으므로 마찬가지입니다. image_index 하위 이미지를 현재 인스턴스 스프라이트에 대해 선택된 것으로 설정하거나(다른 스프라이트를 그릴 수 있지만 현재 인스턴스에 대한 하위 이미지 값을 계속 사용할 수 있음) 이 값을 사용하여 그릴 수 있습니다. 선택한 스프라이트의 특정 하위 이미지. 값이 하위 이미지의 수보다 크면 GameMaker 는 해당 이미지를 선택하기 위해 자동으로 숫자를 반복합니다(예를 들어, 그려지는 스프라이트에 0에서 4까지 번호가 매겨진 5개의 하위 이미지가 있고 인덱스 값을 다음으로 설정한 경우) 7, 그런 다음 함수는 0으로 번호가 매겨진 하위 이미지 3을 그립니다. 마지막으로 x 및 y 위치는 공간 내에서 스프라이트가 그려질 위치이며 스프라이트 x 오프셋 및 y 오프셋 의 중심에 있습니다.

참고 이 기능은 스켈레톤 애니메이션 스프라이트를 사용할 때 예상대로 작동하지 않을 수 있으며 이 기능은 기본 포즈의 첫 번째 프레임만 그리는 것을 알 수 있습니다. 당신은 사용하고 있어야합니다 draw_skeleton_* 대신 기능.

통사론:

draw_sprite(sprite, subimg, x, y);

논쟁 유형 설명
sprite 스프라이트 자산 The index of the sprite to draw.
subimg 진짜 The sub-image (frame) of the sprite to draw (image_index or -1 correlate to the current frame of animation in the object).
x 진짜 The x coordinate of where to draw the sprite.
y 진짜 The y coordinate of where to draw the sprite.

보고:

N/A

예시:

draw_sprite(sprite_index, image_index, x, y);
draw_sprite(spr_Halo, 0, x, y-32);

이렇게 하면 룸 내 인스턴스의 x 및 y 위치에 현재 하위 이미지가 있는 인스턴스에 할당된 스프라이트(sprite_index)가 그려집니다. 그런 다음 동일한 x 및 y 위치에 있지만 '위'에 32픽셀이 있는 "spr_Halo"에 의해 인덱싱된 스프라이트의 첫 번째 프레임을 그립니다.
