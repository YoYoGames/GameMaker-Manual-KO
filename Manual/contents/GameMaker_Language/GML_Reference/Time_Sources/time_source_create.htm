time_source_create

이 함수는 새로운 시간 소스 를 생성합니다. 개요를 보려면 시간 소스 를 읽으십시오.

새 시간 소스 를 생성하면 자동으로 시작되지 않습니다. 당신은 전화해야합니다time_source_start() 활성화합니다.

새 시간 소스 는 time_source_destroy()>반복 인수가 지정되지 않는 한 한 번만 실행됩니다(아래 참조).

다음을 사용하여 시간 소스 를 파괴해야 합니다. time_bpm_to_seconds() 더 이상 필요하지 않을 때.

인수

다음은 함수의 인수를 분석한 것입니다.

상위 시간 소스

이것은 새로운 시간 소스를 제어하는 상위 시간 소스입니다. 이것은 instance_destroy()>time_source_global, time_source_game 또는 이미 존재하는 사용자 정의 시간 소스 일 수 있습니다.

참조: 내장 시간 소스

기간

이것은 다음 인수에 지정된 단위에 따라 초 또는 프레임으로 표시될 수 있는 기간입니다.

프레임을 단위로 사용하는 경우 기간은 정수여야 합니다. 정수가 아닌 값은 1로 반올림되는 1보다 작은 값을 제외하고는 내림(내림)됩니다.

단위

이는 주기를 나타내는 단위로 time_source_units_seconds 또는 time_source_units_frames 일 수 있다.

참조: 시간 출처 단위

다음을 사용하여 분당 비트 수 값을 사용할 수 있습니다..

콜백 및 인수

시간 소스 가 만료될 때 호출되는 "콜백" method 를 지정해야 합니다.

선택적으로 메서드가 호출될 때 메서드에 전달되어야 하는 인수가 포함된 배열을 지정할 수 있습니다.

배열 자체는 메서드에 전달되지 않고 배열의 각 요소가 별도의 인수로 전달됩니다.

예를 들어, 함수가 인수 (x, y, object) 를 예상하는 경우 args 배열은 [30, 600, obj_player] 와 같을 수 있습니다.

반복

시간 소스 가 반복되어야 하는 횟수를 선택적으로 지정할 수 있습니다. 값 1 은 기본값인 한 번만 실행됨을 의미합니다.

반복해야 하는 총 횟수를 지정하거나 무한 반복하도록 -1 을 지정할 수 있습니다.

예를 들어 이것을 3으로 설정하고 시간 소스 기간이 "4초"이면 시간 소스 는 총 12초 동안 활성화되고 4초마다 콜백 메서드를 호출합니다.

만료 유형

time_source_expire_nearest 또는 time_source_expire_after 일 수 있습니다.

참조: 시간 출처 만기 유형

통사론:

time_source_create(parent, period, units, callback, [args, repetitions, expiry_type]);

논쟁 유형 설명
parent 시간 소스 ID The parent Time Source that controls the new Time Source
period 진짜 The period that the Time Source runs for, in the given units
units 시간 소스 단위 상수 The units that the given period is in
callback 메소드 또는 스크립트 기능 The method or script function to call when the Time Source expires
args 정렬 OPTIONAL An array containing the arguments to pass into the method
repetitions 진짜 OPTIONAL The number of times the Time Source should repeat, or -1 for indefinite repetition
expiry_type 시간 소스 만료 상수 OPTIONAL Whether the Time Source expires on the frame nearest to its expiry, or on the next frame

보고:

Time Source ID

예 1:

var _my_method = function()
{
    instance_destroy();
}

var _time_source = time_source_create(time_source_game, 300, time_source_units_frames, _my_method);

time_source_start(_time_source);

이 예에서는 인스턴스가 300프레임 후에 자체적으로 파괴되기를 원합니다.

코드는 먼저 단순히 호출하는 메서드를 만듭니다..

그런 다음 Game Time Source 에서 상속한 Time Source 를 만듭니다. 기간을 300프레임 으로 설정합니다.

마지막으로 Time Source 를 시작합니다.

예 2:

var _my_method = function()
{
    show_debug_message("A second has passed!");
}

global.time_per_second = time_source_create(time_source_game, 1, time_source_units_seconds, _my_method, [], -1, time_source_expire_after);

time_source_start(global.time_per_second);

이 예에서는 1초에 한 번 만료되는 전역 시간 소스 를 생성합니다. 이 코드는 script 의 루트에 위치합니다.

코드는 먼저 "1초가 지났습니다!"라는 메시지를 출력 로그에 인쇄하는 메서드를 만듭니다..

그런 다음 Game Time Source 에서 상속한 새 시간 소스 를 만듭니다. 주기를 1초로 설정합니다.

메서드는 만료될 때마다 호출될 수 있도록 시간 소스 로 전달됩니다. 인수에 대해 빈 배열이 제공됩니다.

반복 횟수는 -1로 설정되어 있으므로 시간 소스 는 멈추지 않고 계속 반복됩니다. 만료 유형이 설정되어 만료 후 첫 번째 프레임에서 콜백이 실행됩니다.

마지막으로 시간 소스 가 시작됩니다.
