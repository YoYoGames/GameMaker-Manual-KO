buffer_create

이 함수를 사용하여 게임에서 메모리의 일부를 버퍼로 할당합니다. 이 함수는 변수에 저장되어야 하고 버퍼에 대한 모든 추가 함수 호출에 사용되어야 하는 고유한 버퍼 ID 를 반환합니다. 그런 다음 버퍼를 사용하여 다양한 유형의 데이터를 저장할 수 있습니다(다음을 사용하여 버퍼에 쓸 때 지정됨).buffer_write() 함수, 버퍼 유형을 정의하는 데 사용되는 다음 상수:

버퍼 유형 상수
일정한 설명
buffer_fixed 고정 크기의 버퍼입니다.
buffer_grow 데이터가 추가될 때 동적으로 "증가"하는 버퍼
buffer_wrap 데이터가 "래핑"될 버퍼입니다. 추가되는 데이터가 버퍼 크기의 한계에 도달하면 덮어쓰기가 버퍼의 시작 부분에 다시 배치되고 그 지점에서 추가 쓰기가 계속됩니다.
buffer_fast 읽기/쓰기가 매우 빠른 특수 "제거된" 버퍼입니다. 와 함께만 사용할 수 있습니다. buffer_u8 데이터 유형 이며 1바이트로 정렬 되어야 합니다.
buffer_vbuffer 이 유형의 버퍼는 정점 버퍼로만 사용됩니다.

버퍼 유형 외에도 버퍼에 대한 바이트 정렬 도 설정해야 합니다. 이 값은 버퍼에 저장하려는 데이터에 따라 달라지며 대부분의 경우 값 1이 완벽합니다. 그러나 일부 작업의 경우 특정 정렬이 필수적 이며 잘못된 정렬로 인해 오류가 발생할 수 있습니다(정렬에 대한 자세한 내용은 버퍼 참조). 다음은 각 데이터 유형에 가장 적합한 값을 보여주는 일반적인 가이드입니다(확실하지 않은 경우 정렬 1 사용).

  • 문자열은 1바이트로 정렬되어야 합니다.
  • 부호 있는 또는 부호 없는 8비트 정수는 모든 값으로 정렬할 수 있지만 빠른 버퍼의 경우(참조buffer_write() ) 1에 맞춰야 합니다.
  • 부호 있는 또는 부호 없는 16비트 정수는 2바이트로 정렬되어야 합니다.
  • 부호 있는 또는 부호 없는 32비트 정수는 4바이트로 정렬되어야 합니다.
  • 최대 16비트의 부동 소수점은 2바이트로 정렬되어야 합니다. (현재 지원되지 않습니다!)
  • 최대 32비트의 부동 소수점은 4바이트로 정렬되어야 합니다.
  • 최대 64비트의 부동 소수점은 8바이트로 정렬되어야 합니다.

참고: 잘못된 선택은 성능에 부정적인 영향을 미칠 수 있으므로 바이트 정렬이 매우 중요할 수 있습니다.

참고: 메모리 누수를 방지하기 위해 더 이상 필요하지 않을 때 이와 같이 동적으로 생성된 리소스를 메모리에서 제거하는 것이 중요합니다. 따라서 생성한 버퍼가 끝나면 다음을 사용하여 다시 해제해야 합니다.buffer_delete().

통사론:

buffer_create(size, type, alignment)

논쟁 유형 설명
size 진짜 The size (in bytes) of the buffer.
type 버퍼 유형 상수 The type of buffer to create (see the constants list above).
alignment 진짜 The byte alignment for the buffer

보고:

Buffer ID

예시:

player_buffer = buffer_create(16384, buffer_fixed, 2);

위의 코드는 버퍼에 16384바이트의 메모리를 할당하고 나중에 사용할 수 있도록 "player_buffer" 변수에 저장된 해당 버퍼의 인덱스를 반환합니다. 버퍼는 2바이트 경계로 정렬됩니다.
