buffer_get_surface

이 기능을 사용하면 표면에서 주어진 버퍼로 정보를 쓸 수 있습니다. 버퍼는 이전에 생성된 것이어야 하며 기록하려는 표면에 대한 데이터를 저장할 수 있을 만큼 충분히 큰 1바이트 정렬 버퍼여야 합니다(확실하지 않은 경우 성장 버퍼를 사용하거나 이 페이지 하단의 예를 참조하십시오).

표면에 대한 정보는 항상 버퍼의 시작 부분에 기록되지만(검색 위치에 관계없이) 오프셋 값(바이트 단위)을 선택하여 버퍼의 시작 부분에서 쓰기 위치를 이동할 수 있습니다.

이 함수는 Windows 대상에서 RGBA 형식을 사용하여 표면의 각 픽셀을 버퍼에 기록하지만 다른 대상에서는 OS 또는 장치에 따라 다를 수 있습니다.

참고: 이 기능은 GameMaker 업데이트 2.3.1부터 변경되었습니다. 이전 버전에서는 상수를 사용했습니다. buffer_surface_copy, 이제 더 이상 사용되지 않습니다.

통사론:

buffer_get_surface(buffer, surface, offset);

논쟁 유형 설명
buffer 버퍼 ID The index of the buffer to use.
surface 표면 ID The index of the surface to use.
offset 진짜 The data offset value (in bytes).

보고:

N/A

예시:

buffer = buffer_create(surf_width * surf_height * 4, buffer_fixed, 1);
buffer_get_surface(buffer, surface, 0);

이 코드는 표면의 정확한 크기를 기준으로 픽셀당 4바이트(R, G, B 및 A 구성 요소용)를 저장할 수 있는 고정 크기 버퍼를 만든 다음 표면 데이터를 여기에 복사합니다.
