buffer_peek

와 더불어buffer_read() 기능을 사용하면 현재 "검색" 위치에 있는 주어진 버퍼에서 데이터를 읽을 수 있으며, 읽거나 쓰고 있는 바이트만큼 이 위치에서 읽고 있는 각 데이터 조각을 읽을 수 있습니다. 그러나 현재 탐색 위치를 변경하지 않고 주어진 데이터를 읽어야 할 수도 있으며 이때 이 기능을 사용합니다. 함수에 버퍼 ID를 제공한 다음 읽고자 하는 데이터 유형뿐만 아니라 읽을 해당 버퍼 내의 오프셋 위치(버퍼 시작부터)를 제공하기만 하면 됩니다.

참고: 읽고 있는 데이터에 대해 잘못된 데이터 유형을 사용하면 잘못된 값이 발생합니다!

함수가 성공하면 데이터 유형에 따라 값(Real, string, boolean 등...)이 반환되고 실패는 단순히 0이 반환됩니다.

통사론:

buffer_peek(buffer, offset, type);

논쟁 유형 설명
buffer 버퍼 ID The index of the buffer to use.
offset 진짜 The offset position (in bytes) within the buffer to read the given data from.
type 버퍼 데이터 유형 상수 The type of data that is to be read from the buffer (see the list of constants here ).

보고:

Real, Boolean, String or 0 (if it fails)

예시:

var red = buffer_peek(buff, 1, buffer_u8);
var green = buffer_peek(buff, 2, buffer_u8);
var blue = buffer_peek(buff, 3, buffer_u8);
image_blend = make_colour_rgb(red, green, blue);

위의 코드는 "buff" 변수에 인덱싱된 버퍼 내의 서로 다른 세 위치에서 세 개의 값을 가져온 다음 해당 값을 사용하여 인스턴스의 이미지 혼합을 설정합니다.
