버퍼

버퍼는 데이터가 한 장소에서 다른 장소로 이동하거나 어떤 식으로든 조작되는 동안 일시적으로 데이터를 보유하는 데 사용되는 물리적 메모리 영역입니다. 예를 들어 버퍼를 사용하여 네트워크를 통한 전송을 기다리는 패킷 데이터를 보유하거나 패킷을 저장할 수 있습니다 네트워크를 통해 받은 것입니다. GameMaker 에는 버퍼 생성, 수정 및 로드/저장과 관련된 여러 특정 기능이 있지만 사용하기 전에 다음 사항에 유의하십시오.

  • 버퍼가 생성되면 자동으로 지워지고 0으로 채워집니다.
  • 버퍼 함수의 "크기" 인수는 항상 버퍼의 크기 (바이트 )를 나타냅니다.
  • "정렬" 인수는 데이터가 버퍼 내에 저장되는 방식을 나타냅니다. 정렬이 4로 설정되어 있고 크기가 1바이트인 단일 데이터 조각을 작성하는 경우 다음을 수행하십시오. buffer_tell, 오프셋은 1이 됩니다. 그러나 크기가 1바이트인 다른 데이터 조각을 작성하는 경우 다음을 수행합니다. buffer_tell, 정렬이 해당 크기로 데이터를 "채워" 있으므로 오프셋 5를 얻게 됩니다. 정렬은 기록 되는 위치 에만 영향을 미치므로 다음을 수행하면 buffer_tell 무언가를 작성한 후 작성한 데이터 바로 다음에 오는 현재 쓰기 위치를 반환합니다. 그러나 다른 데이터 조각을 쓰는 경우 내부적으로 버퍼는 데이터 조각을 실제로 쓰기 전에 정렬 크기의 다음 배수로 쓰기 위치를 이동합니다.
  • "오프셋"을 처리할 때 지정된 버퍼 내에서 데이터를 오프셋할 바이트 단위 값입니다. 따라서 버퍼가 2바이트로 정렬되어 있고 처음 4개의 정렬된 위치를 건너뛰고 싶다면 오프셋이 2 * 4 bytes.
  • 일부 버퍼 함수는 새 버퍼도 생성합니다(예: buffer_load 예를 들어). 이러한 버퍼는 사용하지 않을 때 메모리에서 제거해야 함을 기억하십시오. buffer_delete 함수.
  • 버퍼 생성에 사용되는 메모리는 시스템 메모리 이므로 게임에 포커스가 없어도 버퍼에 저장된 모든 데이터는 안전해야 하지만 앱을 닫거나 다시 시작하면 손실됩니다.

다음 페이지에서는 GameMaker 컨텍스트 내에서 버퍼가 작동하는 방식을 설명하고 버퍼 사용 방법을 잘 모르는 사용자를 위한 몇 가지 코드 예제를 포함합니다.

  • 버퍼 사용

중요한! 버퍼를 생성할 때 버퍼를 식별하는 데 사용되는 인덱스 값은 0부터 시작하는 정수 값입니다. 이러한 인덱스는 GameMaker에서 재사용하므로 소멸된 버퍼 인덱스 값은 나중에 새로 생성된 버퍼에서 사용할 수 있으며 다음을 권장합니다. 버퍼를 파괴한 후 버퍼 인덱스를 보유하는 모든 변수를 -1로 설정합니다.
