buffer_async_group_option

이 기능을 사용하면 저장/로드되는 버퍼 그룹에 대한 일부 플랫폼 특정 옵션을 설정할 수 있습니다. 사용 가능한 옵션은 다음과 같습니다.

  • " subtitle " 또는 " slottitle " - 이 옵션의 값은 문자열이며 OS에서 저장 데이터를 관리할 때 사용자에게 표시됩니다. 이것은 데이터를 저장할 때만 중요하며 다시 로드할 때는 중요하지 않습니다.
  • " showdialog " - 이 옵션의 값은 부울입니다. true 또는 false. 로 설정하면 true, GameMaker 를 저장 하거나 로드할 때 시스템 UI가 표시됩니다. 그렇지 않으면 UI가 표시되지 않고 빠른 저장/빠른 로드가 수행됩니다. 일반적으로 사용자가 슬롯을 선택할 수 있도록 여러 개의 저장 슬롯을 지원하는 경우에만 필요하지만 사용자당 하나의 슬롯만 지원하는 경우 다음으로 설정합니다. false.
  • " savepadindex " - 이 옵션의 경우 저장 하거나 로드하는 플레이어의 패드 인덱스 를 지정하면 시스템이 이 플레이어의 저장 폴더에 데이터를 쓰고 읽습니다.
  • " saveslotsize " - 이 옵션을 사용하려면 저장할 실제 크기를 바이트 단위로 지정해야 합니다.buffer_seek() 그리고buffer_tell() 예를 들어). 모든 저장에는 일반적으로 대상 플랫폼에 따라 달라지는 최소 공간이 미리 할당되므로 이 값을 제공할 필요는 없습니다.
  • " ps_create_backup " - (PlayStation 전용) - 다음으로 설정 true 저장 디렉토리가 마운트 해제될 때(즉, buffer_async_group_end() 함수가 호출됩니다). 자세한 내용은 콘솔 설명서를 참조하십시오.

중요한! 이 기능은 현재 PSVita, PS4 및 XBox One 대상 모듈에만 유효합니다. 다른 모든 대상에서는 아무 작업도 수행하지 않습니다.

통사론:

buffer_async_group_option(option, value);

논쟁 유형 설명
option 끈 The option to set.
value 실수 또는 문자열 The value to set (can be a real or string, depending on the option).

예시:

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option("subtitle", "All your haggis are saved here!");
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

위의 코드는 버퍼 그룹을 시작한 다음 비동기식으로 저장할 4개의 파일을 설정하기 전에 그룹 옵션을 설정합니다. 그런 다음 그룹 정의가 종료됩니다(이 시점에서 저장이 시작됨).
