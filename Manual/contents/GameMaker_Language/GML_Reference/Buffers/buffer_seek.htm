buffer_seek

이 함수는 버퍼를 통해 이동하여 데이터를 읽거나 쓸 때 마지막으로 사용된 것과 관련된 시작, 끝 또는 위치를 찾는 데 사용할 수 있습니다. "오프셋" 값은 주어진 탐색 위치에 추가할 오프셋(바이트)입니다. 예를 들어 기준이 상대적이고 오프셋이 4이면 버퍼 위치는 현재 위치에서 4바이트를 따라 이동합니다. 다음 사항에 유의하십시오.

  • 오프셋에 대해 음수 값을 사용하여 양수 값뿐만 아니라 버퍼를 다시 검색할 수 있습니다.
  • 버퍼가 "랩" 유형이고 버퍼의 끝을 지나 오프셋하면 탐색 위치도 래핑됩니다.
  • 버퍼가 "wrap" 유형이 아닌 경우 오프셋이 버퍼 제한을 벗어나 탐색을 수행하더라도 탐색은 버퍼의 시작 또는 끝으로 고정됩니다.

다음 상수는 다음을 찾기 위한 "기본" 인수로 허용됩니다.

버퍼 탐색 상수
일정한 설명
buffer_seek_start 버퍼의 시작
buffer_seek_relative 현재 읽기/쓰기 위치에 상대적인 위치
buffer_seek_end 버퍼의 끝

통사론:

buffer_seek(buffer, base, offset);

논쟁 유형 설명
buffer 버퍼 ID The index of the buffer to use.
base 진짜 The base position to seek.
offset 진짜 The data offset value.

예시:

buffer_seek(buff, buffer_seek_start, 0);
buffer_write(buff, buffer_s16, 0);
buffer_write(buff, buffer_s16, x);
buffer_write(buff, buffer_s16, y);

위의 코드는 id가 변수 "buff"에 저장된 버퍼의 시작 부분을 찾아 일련의 부호 있는 16비트 정수 값을 기록합니다.
