buffer_async_group_begin

이 함수는 여러 버퍼에서 여러 파일로 저장을 시작하려는 경우(또는 여러 버퍼를 로드할 때) 호출됩니다. buffer_load()>groupname 은 문자열이며 파일이 저장되는 디렉토리 이름으로 사용되며 나중에 파일을 IDE로 다시 로드할 때 파일 경로의 일부로 사용해야 합니다(다음 중 하나를 사용하여buffer_save_async() 기능). 이 기능 은 buffer_load_async() 그리고buffer_async_group_end() 함수를 호출하여 비동기 그룹도 종료해야 합니다. 그렇지 않으면 파일이 저장/로드되지 않습니다.

콘솔 플랫폼(예: PS4)의 경우 그룹 이름 이 저장 슬롯 설명으로 사용되며 이 기능을 사용하면 저장 중인 모든 파일에 대해 UI가 표시되는 것을 방지할 수 있습니다. 또한 UWP를 사용하는 경우 ERA Xbox 그룹과 동일한 그룹을 만들어 기본 Xbox와 Microsoft 스토어 창 간에 저장 게임을 로밍할 수 있습니다. 예를 들면 다음과 같습니다.

if os_type == os_uwp
{
    buffer_async_group_begin("root/yourgroupname");
}
else
{
    buffer_async_group_begin("yourgroupname");
}

통사론:

buffer_async_group_begin(groupname);

논쟁 유형 설명
groupname 끈 The name of the group (as a string).

보고:

N/A

예시:

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end();

위의 코드는 버퍼 그룹을 시작한 다음 4개의 파일을 비동기적으로 저장하도록 설정합니다. 그런 다음 그룹 정의가 종료됩니다(이 시점에서 저장이 시작됨).
