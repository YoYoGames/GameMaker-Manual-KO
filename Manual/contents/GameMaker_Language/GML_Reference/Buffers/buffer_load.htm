buffer_load

이 함수는 이전에 사용하여 저장한 버퍼를 로드하는 데 사용됩니다.buffer_save() 기능뿐만 아니라 포함된 파일 또는 외부에서 로드된 파일. 바이트 정렬이 1인 "성장" 버퍼( 여기 참조)로 함수에 의해 생성된 새 버퍼의 ID를 반환합니다. 이 ID는 변수에 저장하고 이 버퍼에 대한 모든 추가 함수 호출에 사용해야 합니다. 어떤 이유로든 로드가 실패하면 함수는 -1을 반환하고 로드가 실패했다는 메시지가 컴파일러 출력 창에 표시됩니다.

참고 메모리 누수를 방지하기 위해 더 이상 필요하지 않을 때 이와 같이 동적으로 생성된 리소스를 메모리에서 제거하는 것이 중요합니다. 따라서 생성한 버퍼가 끝나면 다음을 사용하여 다시 해제해야 합니다.buffer_delete().

플랫폼별 참고 사항

  • HTML5에서 로컬 저장소 에서 로드된 모든 버퍼는 base64를 사용하여 디코딩되지만(base64 문자열로 저장되기 때문에), 포함된 파일이나 외부 파일에서 이 기능을 사용하면 base64 디코딩을 사용하지 않을 것으로 예상됩니다. base64 형식으로.
  • HTML5에서 모든 포함 파일은 UTF-8로 인코딩된 텍스트로 저장되며, UTF-8로 인코딩된 파일 버전의 크기가 다를 수 있으므로 원본 파일과 비교할 때 로드된 버퍼의 크기가 증가할 수 있습니다. 원래 것보다 (형식 변경으로 인해).

통사론:

buffer_load(filename);

논쟁 유형 설명
filename The name of the file to load from.

보고:

Buffer ID

예시:

player_buffer = buffer_load("Player_Save.sav");

위의 코드는 이전에 저장한 버퍼를 메모리에 로드하여 새 버퍼를 만듭니다. 이 새 버퍼의 인덱스는 "player_buffer" 변수에 저장됩니다.
