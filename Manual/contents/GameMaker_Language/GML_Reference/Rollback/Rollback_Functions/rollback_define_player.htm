rollback_define_player

이 함수는 멀티플레이어에서 플레이어에게 사용해야 하는 개체를 정의합니다. 이것은 선택 사항이며 사용 시 시스템은 연결하는 플레이어에 대한 인스턴스를 자동으로 생성하고 연결이 끊긴 플레이어에 대한 인스턴스를 파괴합니다.

기본적으로 게임 룸에 있어야 하는 "인스턴스" 레이어에 플레이어 인스턴스가 생성됩니다. 다른 레이어를 사용하려면 해당 이름을 두 번째 인수로 전달하십시오.

이 기능을 사용하거나 사용하지 않고 게임을 만드는 방법에 대한 자세한 내용은 플레이어 개체 정의 를 참조하십시오. 이 함수를 사용하는 경우 조인 / 생성 함수보다 먼저 실행해야 합니다.

이 함수로 생성된 각 인스턴스 는 이 페이지에 나열된 인스턴스 변수를 가져옵니다.

통사론:

rollback_define_player(object, [layer_name]);

논쟁 유형 설명
object 개체 자산 The object that will be used for creating player instances.
layer_name 끈 OPTIONAL The name of the layer where the player instances will be created. "Instances" by default.

보고:

N/A

예시:

rollback_define_player(obj_player);
var _joined = rollback_join_game();

if (!_joined)
{
    rollback_start_game(2, false);
}

위의 코드는 플레이어 개체를 정의한 다음 게임 참여를 시도합니다. 참여하지 않은 경우 최대 2명의 플레이어와 동기화 테스트가 비활성화된 상태에서 대신 새 게임을 시작합니다.
