rollback_sync_on_frame

이 기능은 모든 플레이어가 현재 프레임에서 동기화되었는지 확인합니다. 즉, 플레이어의 상태가 예측이 아니라 플레이어로부터 수신한 실제 데이터를 기반으로 합니다. 이 경우 rollback_confirmed_frame>true 를 반환합니다.

플레이어가 현재 프레임에서 동기화되지 않은 경우 false 를 반환하고 모든 플레이어를 동기화하려고 시도합니다.

이 기능으로 플레이어를 동기화하려고 하면 정지될 수 있습니다. 이 함수가 Step 이벤트에서 계속 호출되지 않도록 해야 합니다.

대신 토글로 사용하십시오. 페이지 하단의 예를 참조하십시오.

변수 모든 플레이어가 동기화되면 자동으로 업데이트됩니다.

통사론:

rollback_sync_on_frame();

보고:

Boolean

예시:

if (obj_game.winning_player == -1 && points >= obj_game.points_required_for_win)
{
    if (rollback_sync_on_frame())
    {
        obj_game.winning_player = player_id;
    }
}

이 코드는 플레이어 개체의 Step 이벤트에서 실행됩니다.

승리한 플레이어가 없는지, 현재 플레이어가 승리하는 데 필요한 포인트가 있는지 확인합니다.

둘 다 참이면 플레이어가 승리해야 함을 의미합니다. 그러나 그렇게 하기 전에 모든 플레이어가 동기화되었는지 확인하여 상태가 올바른지 확인합니다.

모든 플레이어가 동기화되면 win_player 변수가 설정됩니다. 이로 인해 첫 번째 조건이 false 가 되어 rollback_sync_on_frame() 이 다시 호출되지 않도록 합니다.

이는 함수가 매 프레임마다 계속 실행되는 대신 토글로 사용됨을 의미합니다.
