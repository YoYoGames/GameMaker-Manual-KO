롤백 이벤트

"기타" 범주에서 찾을 수 있는 두 가지 롤백 관련 이벤트가 있습니다.

롤백 시작

이 이벤트는 모든 플레이어가 연결되고 멀티플레이어 게임이 시작될 때 트리거됩니다. 나중에 방이 바뀔 때도 발동됩니다.

이 이벤트 내에서 이벤트에 대한 모든 관련 값을 포함하는 num_players>rollback_event_param 이라는 구조체를 얻습니다.

구조체 멤버 설명
player_id 게임의 총 플레이어 수
first_start 로컬 플레이어의 ID(0부터 시작, 처음에는 호스트 자체임)
rollback_connect_error 멀티플레이어 게임이 막 시작된 경우 rollback_event_id>true 이고, 방만 변경된 경우 rollback_chat_message>false 입니다.
rollback_chat() 다음 변수를 포함하는 구조체:

rollback_player_prefs>status 응답의 HTTP 상태 코드
rollback_connected_to_peer>error 오류 를 설명하는 문자열

rollback_synchronizing_with_peer>rollback_event_param.player_id 는 이벤트가 트리거될 수 있는 플레이어 인스턴스의 ID가 아니라 항상 기본 로컬 플레이어의 ID이므로 플레이어 개체가 아닌 게임 관리자 개체에서만 이 이벤트를 사용해야 합니다.

예를 들어 플레이어 0이고 플레이어 1의 인스턴스에서 롤백 시작 이벤트를 포착하는 경우 rollback_synchronized_with_peer>player_id 변수는 로컬 플레이어의 ID만 나타내므로 여전히 0입니다.

롤백 이벤트

롤백 이벤트 이벤트는 rollback_disconnected_from_peer>rollback_event_id 변수를 포함하는 게임 전체의 다양한 지점에서 트리거됩니다. 이 변수는 어떤 유형의 이벤트가 트리거되었는지 나타냅니다.

이벤트에 대한 모든 관련 값을 포함하는 rollback_game_interrupted>rollback_event_param 이라는 구조체가 제공됩니다.

다음 상수 중 하나가 rollback_game_resumed>rollback_event_id 에 저장될 수 있습니다.

rollback_game_full 설명
rollback_game_info 에서 메시지가 올 때 트리거됨rollback_connect_info / 메시지 보내기 가 수신되었습니다. 이 이벤트 동안 clipboard_set_text()>rollback_event_param 에는 다음 변수가 포함됩니다.

message: 수신된 메시지 문자열
from: 메시지를 보낸 플레이어의 ID
to: 메시지가 전송된 플레이어의 ID입니다. 메시지가 모든 사람에게 전송된 경우 -1이 됩니다.
플레이어가 기본 설정 을 지정할 때 트리거됩니다. 이 이벤트 동안 rollback_event_param 에는 다음 변수가 포함됩니다.

player_id: 기본 설정을 지정한 플레이어의 ID
기본 설정: 해당 플레이어에 대해 설정된 기본 설정
새 플레이어가 연결되면 트리거됩니다. 연결된 플레이어의 ID는 rollback_event_param.player_id 에 저장됩니다.
플레이어가 새 플레이어와 동기화하는 동안 트리거됩니다. 이 이벤트 동안 rollback_event_param 에는 다음 변수가 포함됩니다.

player_id: 동기화 중인 플레이어의 ID
count: 완료된 동기화 단계 수
total: 완료해야 하는 총 동기화 단계 수
플레이어와의 동기화가 완료되면 트리거됩니다. 동기화된 플레이어의 ID는 rollback_event_param.player_id 에 저장됩니다.
플레이어가 연결을 끊을 때 트리거됩니다. 연결이 끊긴 플레이어의 ID는 rollback_event_param.player_id 에 저장됩니다.
연결이 불안정한 플레이어로 인해 게임이 중단되면 트리거됩니다. 이 이벤트 동안 rollback_event_param 에는 다음 변수가 포함됩니다.

player_id: 연결 문제가 있는 플레이어의 ID
timeout: 해당 플레이어가 시간 초과되어 연결이 끊길 때까지 남은 시간(밀리초)입니다. 이 시간이 지나면 플레이어는 추방되고 나머지 플레이어와 함께 게임이 재개됩니다.
게임이 중단된 후 재개될 때 트리거됩니다(위에서 설명한 대로). 다시 접속한 플레이어의 ID는 rollback_event_param.player_id 에 저장됩니다.
연결 중인 게임이 이미 가득 찼을 때 트리거됩니다.
방에 대한 정보를 받았지만 완전히 참여하기 전에 트리거됩니다.
이 이벤트 동안 rollback_event_param 에는 다음 변수가 포함됩니다.

num_players: 게임의 총 플레이어 수
player_id: 로컬 플레이어의 ID(0부터 시작)
방이 만들어지고 다른 플레이어와 공유할 준비가 되면 트리거됩니다. 공유 가능한 URL은 rollback_event_param.share_url 에 저장됩니다.

이 URL을 호출하여 사용할 수 있게 되면 자동으로 복사할 수 있습니다..
