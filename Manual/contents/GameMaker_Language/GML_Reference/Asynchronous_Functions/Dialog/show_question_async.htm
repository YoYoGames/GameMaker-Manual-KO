show_question_async

이 함수는 창을 열고 함수에서 정의한 질문을 사용자에게 표시합니다. 이것은 비동기 기능이며 GameMaker 는 응답을 기다리는 동안 실행 중인 장치를 차단하지 않고 평소와 같이 이벤트를 계속 실행합니다. 이 기능에는 "예"와 "아니요"를 표시하는 두 개의 버튼이 있으며 사용자가 하나를 누르면 비동기 대화 상자 이벤트가 트리거되어 해당 이벤트 기간 동안에 만 DS 맵이 변수에 저장됩니다. async_load.

이 맵에는 " id " 및 " status "라는 두 개의 키가 포함됩니다. "id"는 호출될 때 함수에 의해 반환된 값이고 "status"는 다음 중 하나입니다. true 또는 false "예" 및 "아니오"에 대해 각각.

참고 이 기능은 디버깅 및 테스트 용도로만 사용되며 출시된 게임에서는 사용해서는 안 됩니다. 이를 위해서는 대화 상자의 모양과 동작을 완전히 제어할 수 있도록 플레이어로부터 입력을 수신하는 사용자 지정 사용자 인터페이스를 만들어야 합니다.

통사론:

show_question_async(string);

논쟁 유형 설명
string 끈 The question to ask to the user.

보고:

Async Request ID

확장된 예:

메시지를 표시하는 개체의 왼쪽 마우스 누르기 이벤트 (예:)에는 다음 코드가 있습니다.

msg = show_question_async("Do you want to buy some armour for " + string(armour[0, 5]) + "coins?");

위의 내용은 사용자가 "예" 또는 "아니요"를 누르도록 요청하는 주어진 문자열이 포함된 질문을 표시합니다. 함수 ID는 "msg" 변수에 저장되며 아래와 같이 비동기 대화 상자 이벤트 에서 사용됩니다.

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        coins -= armour[0,5];
        global.protection += armour[0,0];
    }
}

위의 코드는 반환된 "id" 키를 확인합니다. DS Map 변수 "msg"에 저장된 값에 대해. 그들이 같으면 두 버튼 중 하나가 눌렸는지 확인하고 반환되는지 확인합니다. true 그런 다음 변수에서 값을 빼서 전역 변수에도 값을 추가합니다.
