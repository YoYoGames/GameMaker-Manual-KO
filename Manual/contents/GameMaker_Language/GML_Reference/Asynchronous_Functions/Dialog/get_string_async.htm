get_string_async

이 기능은 창을 열고 사용자가 문자열을 입력할 수 있는 공간과 메시지를 표시합니다. 이것은 비동기 기능이며 GameMaker 는 응답을 기다리는 동안 실행 중인 장치를 차단하지 않고 평소와 같이 이벤트를 계속 실행합니다. 사용자가 문자열을 입력하고 "확인" 버튼을 누르면 해당 이벤트 기간 동안에 만 변수에 DS 맵이 저장되는 비동기식 대화 상자 이벤트 가 트리거됩니다. async_load.

이 맵에는 " id ", " status " 및 " result "의 세 가지 키가 포함됩니다. "id"는 호출될 때 함수에 의해 반환된 값이고 "status"는 다음 중 하나입니다. true "확인" 버튼을 누르거나 false 메시지가 취소된 경우(모든 대상 플랫폼에서 메시지 취소를 허용하지 않으므로 적용 가능한 경우). 마지막으로 "result"는 사용자가 입력한 문자열(또는 아무 것도 제공되지 않은 경우 빈 문자열 "")을 반환합니다.

참고 이 기능은 디버깅 및 테스트 용도로만 사용되며 출시된 게임에서는 사용해서는 안 됩니다. 이를 위해서는 대화 상자의 모양과 동작을 완전히 제어할 수 있도록 플레이어로부터 입력을 수신하는 사용자 지정 사용자 인터페이스를 만들어야 합니다.

통사론:

get_string_async(string, default);

논쟁 유형 설명
string 끈 The message to show in the dialog.
default 끈 The default string.

보고:

Async Request ID

확장된 예:

메시지를 표시하는 개체의 왼쪽 마우스 누르기 이벤트 (예:)에는 다음 코드가 있습니다.

msg = get_string_async("What's your name?","Anon");

위는 사용자가 문자열을 입력하고 "확인"을 누르도록 요청하는 메시지를 표시합니다. 함수 id는 변수 "msg"에 저장되며 아래와 같이 비동기식 Dialogs 이벤트에서 사용됩니다.

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if (ds_map_find_value(async_load, "status"))
    {
        if (ds_map_find_value(async_load, "result") != "")
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

위의 코드는 반환된 "id" 키를 확인합니다. DS Map 변수 "msg"에 저장된 값에 대해. 그들이 같으면 (창이 닫히거나 취소되지 않고) "OK"가 눌렸는지 확인하고 반환되는지 확인합니다. true 그런 다음 전역 변수를 설정하기 전에 빈 문자열이 반환되지 않았는지 문자열의 "결과"를 확인합니다.
