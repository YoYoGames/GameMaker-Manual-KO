show_message_async

이 함수는 창을 열고 함수에서 정의한 메시지를 사용자에게 표시합니다. 이것은 비동기 기능이며 GameMaker 는 응답을 기다리는 동안 실행 중인 장치를 차단하지 않고 평소와 같이 이벤트를 계속 실행합니다. 사용자가 "확인" 버튼을 누르면 해당 이벤트 기간 동안에 만 ds_map이 변수에 저장되는 비동기식 대화 상자 이벤트가 트리거됩니다. async_load.

이 맵에는 "id"와 "status"라는 두 개의 키가 포함되며, "id"는 호출될 때 함수가 반환한 값이고 "status"는 다음 중 하나입니다. true "확인" 버튼을 누르거나 false 메시지가 취소된 경우(모든 대상 플랫폼이 대화 취소를 허용하지 않기 때문에 사용 가능한 경우).

참고 이 기능은 디버깅 및 테스트 용도로만 사용되며 출시된 게임에서는 사용해서는 안 됩니다. 이를 위해서는 대화 상자의 모양과 동작을 완전히 제어할 수 있도록 플레이어로부터 입력을 수신하는 사용자 지정 사용자 인터페이스를 만들어야 합니다.

통사론:

show_message_async(string);

논쟁 유형 설명
string 끈 The message to show to the user.

보고:

Async Request ID

확장된 예:

메시지를 표시하는 개체의 왼쪽 마우스 누르기 이벤트 (예:)에는 다음 코드가 있습니다.

msg = show_message_async("You will now be taken to the store");

위의 내용은 주어진 문자열이 포함된 메시지를 표시합니다. 메시지 ID는 "msg" 변수에 저장되며 아래와 같이 비동기식 Dialogs 이벤트에서 사용됩니다.

var i_d, stat;
i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        url_open("https://play.google.com/store");
    }
}

위의 코드는 반환된 "id" 키를 확인합니다. DS Map 변수 "msg"에 저장된 값에 대해. 동일하다면 "확인" 버튼이 눌렸는지(창이 닫히거나 취소되는 것이 아니라) 확인되고 반환되는지 확인합니다. true 그것은 URL을 엽니 다.
