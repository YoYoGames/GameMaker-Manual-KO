대화

대부분의 최신 컴퓨터 및 모바일 장치는 대화 창이 표시되는 동안 게임이 중지되는 것을 좋아하지 않습니다. 그들은 이것을 게임 러너가 중지된 것으로 해석하는 경향이 있으며 오류로 간주하고 앱을 조기에 종료할 수 있습니다. 이를 방지하기 위해 GameMaker 에는 특정 형태의 사용자 피드백 또는 입력을 요청할 수 있는 비동기식 대화 기능이 있지만 언제든지 게임 실행기를 차단하지 않습니다. 즉, 호출될 때 사용자가 응답하는 동안 게임이 백그라운드에서 정상적으로 실행됩니다. 표시된 대화창으로 이동합니다. 사용자가 응답하면 결과를 처리할 수 있도록 Dialog Asynchronous Event 가 호출됩니다.

참고: 다음을 제외하고shop_leave_rating(), 이러한 기능은 디버깅 및 테스트용으로만 사용되며 출시된 게임에서는 사용하면 안 됩니다. 이를 위해서는 대화 상자의 모양과 동작을 완전히 제어할 수 있도록 플레이어로부터 입력을 수신하는 사용자 지정 사용자 인터페이스를 만들어야 합니다.

다음은 사용 가능한 모든 비동기 대화 기능 목록입니다.

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
