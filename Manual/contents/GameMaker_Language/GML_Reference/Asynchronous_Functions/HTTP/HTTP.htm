HTTP

이 섹션은 GameMaker 에서 사용할 수 있는 모든 다양한 비동기 HTTP 함수를 나열합니다. 이 함수는 비동기 HTTP 이벤트 가 있는 모든 인스턴스에서 다음을 생성합니다.

  • http_request
  • http_get
  • http_get_file
  • http_post_string

참고로 위의 http_* 도메인 간 보안 문제로 인해 기능이 예상대로 작동하지 않을 수 있습니다. 즉, 서버에 대한 요청이나 도메인 간에 리소스를 로드하려는 시도가 차단되고 빈 결과 또는 404 오류를 반환하는 것처럼 보일 수 있습니다. 이 문제를 해결할 수 있는 방법 중 하나는 특정 도메인이 서버에 액세스할 수 있도록 하는 일부 서버 측 PHP를 사용하는 것입니다(이는 PHP 허용 목록에 포함되지 않은 서버를 차단하고 리소스를 보호하는 방법이기도 합니다).

다음은 이에 사용할 수 있는 PHP 코드의 예입니다.

$http_origin = $_SERVER['HTTP_ORIGIN'];
if ($http_origin == "http://127.0.0.1:51268")
{
    header("Access-Control-Allow-Origin: *");
}

교차 출처 유형을 결정하거나 설정하는 것이 중요한 이미지 로드 요청의 경우 다음 기능이 있습니다.

  • http_get_request_crossorigin
  • http_set_request_crossorigin

대부분의 경우 이러한 기능을 사용할 필요가 없지만 게임이 보안 서버에 저장된 경우 - 특정 자산에 액세스하려면 기본 인증이 필요할 수 있고 로드할 때 보안 오류가 발생할 수 있음 - 이미지 요청에 대한 교차 출처 유형 설정 "use-credentials"가 필요할 수 있습니다(기본 "익명" 설정과 반대).
