http_request

이 기능을 사용하면 HTTP 헤더 요청을 생성하여 HTTP 트랜잭션의 작동 매개변수를 정의할 수 있습니다. 이 매개변수는 RESTful API를 사용하는 경우 HTTP 헤더를 통한 인증과 같은 많은 작업에 사용할 수 있습니다. 이 기능은 요청할 서버의 전체 IP 주소와 요청 유형(문자열로, 아래 참고 참조)을 요구합니다: "GET", "HEAD", "POST", "PUT", " 삭제", "추적", "옵션" 또는 "연결". 당신은 또한 공급해야합니다 DS map 키/값 쌍(문자열로, 여기서 키는 헤더 필드이고 값은 헤더에 필요한 데이터임), 최종 인수는 요청에 추가할 수 있는 선택적 데이터 문자열이며, 필요하지 않은 경우 0 또는 빈 문자열 ""이 될 수 있습니다. 버퍼를 보낼 수도 있습니다(자세한 내용은 버퍼 섹션 참조). 이 경우 마지막 인수는 보낼 버퍼의 인덱스가 됩니다.

참고: HTTP 헤더는 일반적으로 "key:value" 형식을 따르지만 GameMaker 는 ds_map에서 이러한 쌍을 생성하므로 맵 키 또는 값 문자열에 콜론 ":"을 포함할 필요가 없습니다.

참고: 본문 인수에 버퍼를 사용할 때 버퍼 탐색 위치가 시작(0)이면 본문이 전송되지 않고 버퍼는 http 호출의 응답으로 채워집니다. 그러나 버퍼 탐색 위치가 0이면 버퍼 문자열 내용이 본문으로 전송됩니다.

참고: 브라우저의 XSS 보호로 인해 도메인 간에 리소스를 로드하려는 요청 및 시도가 차단되고 빈 결과를 반환하는 것처럼 보일 수 있습니다. 자세한 내용은 교차 도메인 문제 부분을 참조하십시오.

이 함수는 아래 설명된 대로 콜백을 식별하는 데 사용할 수 있는 비동기 요청 ID 를 반환합니다.

비동기 콜백

이 이벤트는 모든 Async HTTP Events 에 의해 선택되는 "콜백"을 생성합니다. 이 경우 DS Map 이는 이 이벤트에만 해당되며 특수 변수 async_load 에 저장됩니다. 이 DS 맵에는 요청 기능과 관련된 다음 두 개의 키가 있습니다.

  • id: 함수에서 반환된 ID입니다. 연속으로 발사하면 http_ 그런 다음 응답을 받는 대상을 알아야 하므로 이 값을 사용하여 원래 요청을 보냈을 때 저장한 값과 비교하여 올바른 것을 찾을 수 있습니다.
  • response_headers: 이 값이 0보다 크거나 같으면 HTTP 요청에 대한 응답과 함께 반환된 HTTP 헤더가 포함된 DS 맵을 보유합니다.

통사론:

http_request(url, method, header_map, body);

논쟁 유형 설명
url 끈 The web address of the server that you wish to get information from
method 끈 The request method (normally "POST" or "GET", but all methods are supported)
header_map DS 맵 ID A ds_map with the required header fields
body 실수, 문자열 또는 버퍼 ID The data to be transmitted following the headers (can be a binary buffer handle)

보고:

Async Request ID
