http_set_request_crossorigin

이 기능을 사용하면 파일에서 이미지를 로드할 때 사용할 교차 원점 유형을 설정할 수 있습니다. 이 기능은 HTML5 플랫폼 전용이며 다음을 사용하여 파일에서 스프라이트를 로드할 때만 유용합니다. sprite_add() 함수. "use-credentials"로 설정하면 더 이상 로드되는 스프라이트에 절대 경로를 제공할 필요가 없지만 대신 상대 경로를 제공합니다(아래 예 참조). 기본적으로 교차 출처 유형은 "익명"으로 설정됩니다.

통사론:

http_set_request_crossorigin(origin_type);

논쟁 유형 설명
origin_type The cross origin type to use (a string)

보고:

예시:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

위의 코드는 먼저 현재 설정된 교차 출처 유형을 확인하고 "use-credentials"가 아닌 경우 "use-credentials"로 설정한 다음 파일에서 스프라이트를 추가합니다.
