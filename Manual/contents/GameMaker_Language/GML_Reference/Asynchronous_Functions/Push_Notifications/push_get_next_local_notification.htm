push_get_next_local_notification

이 기능은 표시될 대기열의 다음 로컬 푸시 알림에 대한 일련의 키/값 쌍으로 미리 만들어진 DS 맵 을 채웁니다. 대기열에 추가 알림이 없으면 -1을 반환하고 그렇지 않으면 알림의 ID를 나타내는 실제 값을 반환합니다. 이 ID는 다음 기능을 사용하여 알림을 취소하는 데 사용할 수 있습니다.push_cancel_local_notification(). 함수를 호출해야 합니다.push_get_first_local_notification() 대기열에서 첫 번째 알림을 받은 다음 이 기능을 사용하여 계속 진행합니다.

참고: 이 기능은 iOS 및 Android 대상 모듈로 제한됩니다.

DS 맵은 다음 키/값 쌍을 보유합니다.

  • " title " - 알림의 제목(iOS는 제목 정보를 저장하지 않으므로 Android 플랫폼에서만 사용할 수 있음)
  • " message " - 메시지 본문 텍스트
  • " data " - 데이터 패키지 문자열

통사론:

push_get_next_local_notification(map);

논쟁 유형 설명
map The DS Map ID to use.

보고:

Push Notification ID or -1

예시:

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
    {
        push_cancel_local_notification(ntf);
    }
    ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map);

위의 코드는 DS Map 그런 다음 대기열에서 사용 가능한 첫 번째 알림에 대한 데이터를 요청합니다. 알림이 있는 경우 맵이 채워지고 코드는 특정 문자열에 대한 "데이터" 키를 확인합니다. 문자열이 발견되면 알림이 취소되고 대기열의 모든 항목이 확인될 때까지 다음 알림을 확인하기 위해 이동합니다.
