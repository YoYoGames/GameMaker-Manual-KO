비동기 함수

GameMaker 컨텍스트에서 비동기 기능은 웹 서버로 전송하거나 사용자 입력을 요청하거나 데이터를 스트리밍하는 동시에 GameMaker 가 게임 실행을 차단하지 않고 계속 실행되도록 하는 기능입니다. 정보 콜백 이 수신되면 특별한 비동기 이벤트 가 발생하여 비동기 이벤트가 할당된 모든 인스턴스가 추가 기능을 실행할 수 있도록 합니다(반드시 그럴 필요는 없음). 따라서 기본적으로 응답을 기다리면서 게임이나 프로젝트를 중단하지 않고 외부 리소스(서버, 사용자 또는 장치)와 통신하는 방식입니다.

이것이 왜 중요한가? 글쎄요, GameMaker 가 데이터를 보내거나 받는 동안 계속 작동할 수 있기 때문에 다음과 같은 게임 루프를 차단하지 않고도 많은 일을 할 수 있습니다.

  • 플레이어가 로드될 때까지 기다리지 않고 장치에서 게임으로 데이터 스트리밍
  • 올바른 정보가 수신될 때만 작업을 수행하는 콜백 이벤트가 있습니다.
  • 웹 서버와 통신 및 데이터 교환

이것은 모든 것이 훨씬 더 유동적이고 눈에 거슬리지 않는 방식으로 작동하도록 합니다. 비동기 이벤트는 이벤트가 있는 모든 인스턴스에 대해 트리거되므로 한 인스턴스에서 비동기 함수를 사용할 수 있고 다른 인스턴스 또는 다양한 다른 인스턴스에서 응답을 처리하는 비동기 이벤트를 가질 수 있습니다. 비동기 이벤트 작동 방식에 대한 자세한 내용은 여기를 참조하세요. 비동기 이벤트.

다른 대상 플랫폼에서도 사용할 수 있지만 GameMaker 가 지원하는 다른 콘솔에서 작업할 때 필요한 비동기 방식으로 버퍼를 로드하고 저장하는 데 사용할 수 있는 여러 버퍼 함수가 있습니다.

  • buffer_save_async
  • buffer_load_async
