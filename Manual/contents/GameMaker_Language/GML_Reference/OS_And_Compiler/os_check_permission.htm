os_check_permission

이 기능을 사용하면 사용자가 게임에 특정 권한을 부여했는지 확인할 수 있습니다. " 형식을 사용하여 문자열로 확인할 수 있는 권한을 제공합니다.android.permission.<permission> "를 확인하기 위해 RECORD_AUDIO (예를 들어) 당신이 호출할 권한

os_check_permission("android.permission.RECORD_AUDIO");

이 함수는 아래에 표시된 세 가지 상수 중 하나를 반환하여 권한 상태를 게임에 알려줍니다. 앱 권한에 대한 자세한 내용은 Android 설명서 를 참조하십시오.

중요한! 이 기능은 Android 대상 전용입니다.

통사론:

os_check_permission(permission)

논쟁 유형 설명
permission끈 The permission to check (a string)

보고:

Permission State Constant

권한 상태 상수
일정한 설명
os_permission_granted 권한이 부여되었음을 나타냅니다.
os_permission_denied 권한이 부여되지 않았음을 나타냅니다.
os_permission_denied_dont_request 이것은 권한이 전화 설정에 의해 차단되었거나 사용자가 이전에 요청을 거부하고 "다시 묻지 않음"을 선택했음을 나타냅니다.

예시:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

위의 코드는 OS 종류를 확인하고 안드로이드라면 권한을 확인하고, 아직 "INTERNET" 권한이 부여되지 않았다면 요청한다.
