os_is_paused

모바일 장치용으로 개발할 때 장치가 여전히 게임에 집중되어 있는지 확인할 방법이 필요합니다. 특히 게임이 iOS, Android 또는 Windows Phone 장치인 경우 전화 통화로 중단될 수 있습니다. 다른 앱. 이 기능을 사용하여 장치가 일시 중지되었는지 확인할 수 있습니다.

이 함수는 장치의 코드 블록에 포함된 모든 코드를 실행하지만 장치가 게임을 백그라운드 프로세스로 캐시하기 때문에 GameMaker 가 즉시 일시 중지됩니다. 따라서 아래에 제공된 예제 코드와 같은 것이 있는 경우 생성될 인스턴스 는 게임이 캐시된 상태에서 다시 시작될 때까지 실제로 나타나지 않거나 이벤트를 실행하지 않습니다. 이는 귀하가 다음을 선호할 수 있음을 의미합니다 os_is_paused 기능은 알람을 호출한 다음 알람 이벤트(장치가 일시 중지될 때 설정되지만 일시 중지가 해제될 때까지 실행되지 않음)에서 일시 중지 메뉴를 만들고 음악을 다시 시작하는 등...

Windows, Ubuntu(Linux), MacOS 또는 HTML5 플랫폼에서 이 기능을 사용하는 경우 이 기능은 true 게임이 초점을 잃으면 다시 돌아옵니다. false 게임이 초점으로 돌아올 때까지(그리고 다시 잃을 때까지) 이는 이러한 플랫폼이 앱과 게임을 캐시하지 않고 백그라운드에서 계속 실행되기 때문입니다.

참고: 모바일 장치에서는 OS가 일시 중지되면 소리와 음악이 중지됩니다. 적절한 기능으로 특별히 시작하지 않는 한 다시 시작되지 않습니다.

통사론:

os_is_paused()

보고:

예시:

if os_is_paused()
{
    if (!instance_exists(obj_PauseMenu))
    {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
    }
}

위의 코드는 게임이 일시 중지되었는지 확인하고, 일시 중지된 경우 이미 존재하지 않는 경우 "obj_PauseMenu" 개체의 인스턴스를 만듭니다.
