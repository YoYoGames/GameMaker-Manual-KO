os_get_info

이 함수는DS Map 게임이 실행되는 OS에 대한 자세한 정보와 함께. 반환되는 정확한 정보는 OS와 장치에 따라 다르므로 타겟팅하려는 모든 플랫폼에서 이 함수를 테스트하고 다음과 같은 함수를 사용하여 반환된 값을 출력하는 것이 좋습니다. show_debug_message(). DS 맵은 메모리에서 자동으로 지워지지 않으므로 다음을 사용해야 합니다.ds_map_destroy() 더 이상 필요하지 않을 때 기능.

아래에서 다음 플랫폼과 관련된 DS 맵에서 사용할 수 있는 추가 데이터에 대한 몇 가지 세부정보를 찾을 수 있으며 HTML5를 제외한 모든 플랫폼에서 맵에는 " is64bit" 어느 쪽이든 될 것입니다 true 또는 false 러너가 64비트 모드에서 실행 중인지 여부에 따라 다릅니다.

윈도우 / UWP

Windows 데스크톱 및 Windows UWP에서 DS 맵에서 사용할 수 있는 다음 키를 사용하여 DX11에서 일부 추가 데이터를 얻을 수 있습니다(확장에 유용).

  • udid: 기계에 고유한 식별자
  • video_d3d11_device: DX11 장치에 대한 포인터
  • video_d3d11_context: DX11 컨텍스트에 대한 포인터
  • video_d3d11_spawnchain: DX11 스왑체인에 대한 포인터
  • video_adapter_vendorid: 어댑터의 공급업체 ID가 있는 문자열
  • video_adapter_deviceid: 어댑터의 장치 ID가 있는 문자열
  • video_adapter_subsysid: 어댑터의 하위 시스템 ID가 있는 문자열
  • video_adapter_revision: 어댑터의 개정 번호가 있는 문자열
  • video_adapter_description: 비디오 어댑터의 설명이 있는 문자열
  • video_adapter_dedicatedvideomemory: 비디오 메모리의 크기가 있는 문자열
  • video_adapter_dedicatedsystemmemory: 시스템 메모리 크기의 문자열(DX11 어댑터에서 사용)
  • video_adapter_sharedsystemmemory: 공유 시스템 메모리의 크기가 있는 문자열

맥 OS

이 함수는 macOS에서 일부 제한된 OpenGL 정보를 반환합니다.

  • udid: 기계에 고유한 식별자
  • gl_vendor_string: GL_VENDOR OpenGL의 예 - "Intel Inc."
  • gl_version_string: GL_VERSION OpenGL에서 예 - "2.1 INTEL-16.2.16"
  • gl_renderer_string: GL_RENDERER OpenGL의 예 - "Intel(R) Iris(TM) Graphics 6100"

기계적 인조 인간

Android 플랫폼에서는 DS 맵에서 사용할 수 있는 다음 키를 사용하여 다양한 API에서 추가 데이터를 얻을 수 있습니다.

  • android_tv: 이것은 돌아올 것이다 true 기기가 Android TV 기기로 감지되는 경우 또는 false 그렇지 않으면
  • GL_VERSION: 드라이버에서 보고한 OpenGL 버전
  • GL_VENDOR: 드라이버가 보고한 OpenGL 공급업체
  • GL_RENDERER: 드라이버가 보고한 OpenGL의 렌더러
  • GL_EXTENSIONS: 드라이버에서 보고한 대로 사용 가능한 OpenGL 확장
  • GL_SHADING_LANGUAGE_VERSION: 이 OpenGL 드라이버가 지원하는 GLSL 버전
  • GL_MAX_TEXTURE_SIZE: 이 OpenGL 드라이버가 지원하는 최대 텍스처 크기
  • SDK_INT: android.os.Build.VERSION.SDK_INT 의 값 (아래 참고 참조)
  • RELEASE: android.os.Build.VERSION.RELEASE 의 값
  • MODEL: android.os.Build.MODEL 의 값
  • DEVICE: android.os.Build.DEVICE 의 값
  • MANUFACTURER: android.os.Build.MANUFACTURER 의 값
  • CPU_ABI: android.os.Build.CPU_ABI 의 값
  • CPU_ABI2: android.os.Build.CPU_ABI2 의 값
  • BOOTLOADER: android.os.Build.BOOTLOADER 의 값
  • BOARD: android.os.Build.BOARD 의 값
  • VERSION: Android의 os.version 값 System.getProperty 방법.
  • REGION: Android의 user.region 값 System.getProperty 방법
  • VERSION_NAME: 이 패키지의 versionName 값 - 자세한 내용은 여기 를 참조하세요.
  • PHYSICAL_KEYBOARD: 물리적 키보드를 사용할 수 있다고 생각되면 문자열 "TRUE" 그렇지 않으면 "FALSE"

참고: android.os.Build 변수에 대한 정보는 Android 개발자 문서 를 참조하십시오.

  • cpuCount: CPU 수

여기에서 참조하는 기능에 대한 정보는 PS4 문서를 참조하십시오.

이 함수는 PlayStation 4 콘솔에서 다음 키를 반환합니다.

  • display_safe_area_ratio: 반환된 부동 소수점 값 sceSystemServiceGetDisplaySafeAreaInfo().
  • is_neo_mode: 이것은 ~이 될 것이다 1 게임이 Neo(PS4 Pro)에서 실행 중이고 0 일반 PS4에서 실행 중인 경우.
