OS와 컴파일러

크로스 플랫폼 게임을 만들 때 게임을 실행하는 장치가 사용하는 운영 체제의 세부 정보를 얻는 것이 매우 중요합니다. 언어, 버전 또는 네트워크 상태와 같은 사항을 사용하여 게임을 실행하는 시스템에 게임을 적용하고 최종 사용자 경험을 가능한 한 최상으로 만들 수 있으므로 GameMaker 에는 필요한 정보.

게임을 실행하는 운영 체제 또는 브라우저에 대한 특정 세부 정보를 얻는 데 사용할 수 있는 다음 함수와 변수가 있습니다.

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

특정 함수는 게임을 실행할 때 미리 컴파일 되며(예: 최종 게임이 어떻게 컴파일되는지 정의하는 데 사용됨), 일부는 런타임 환경에 대한 특정 세부 정보를 얻기 위해 게임이 컴파일되었을 때도 사용됩니다. 여기에서 이러한 기능의 전체 목록을 찾을 수 있습니다.

  • GM_build_date
  • GM_build_type
  • GM_version
  • GM_runtime_version
  • GM_project_filename
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

Windows에서 Windows 스레드 스케줄러의 해상도는 게임이 실행될 때 기본적으로 1ms로 설정됩니다. 런타임 시 Windows 스레드 스케줄러의 해상도를 검색하고 변경하는 데 사용할 수 있는 함수는 다음과 같습니다.

  • 스케줄러_해상도_get
  • 스케줄러_해상도_세트

마지막으로 확장 으로 작업할 때 런타임에 외부 함수를 정의하고 호출할 수 있어야 하므로 다음을 사용합니다.
