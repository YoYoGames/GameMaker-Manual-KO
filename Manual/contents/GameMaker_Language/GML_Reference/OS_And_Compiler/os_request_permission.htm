os_check_permission

이 기능을 사용하여 특정 권한을 요청할 수 있습니다. " 형식을 사용하여 문자열로 요청할 수 있는 권한을 제공합니다.android.permission.<permission> "를 요청하기 위해 RECORD_AUDIO (예를 들어) 당신이 호출할 권한

os_request_permission("android.permission.RECORD_AUDIO");

함수는 내장된 시스템 비동기 이벤트 를 트리거 반환합니다.async_load DS 맵에는 다음 키/값 쌍이 포함됩니다.

  • " type ": 트리거된 이벤트 유형이며 값은 "permission_request_result ".
  • " <permission> ": 다음 상수 중 하나를 포함하는 문자열로 함수가 요청한 권한입니다.
일정한 설명
os_permission_granted 권한이 부여되었음을 나타냅니다.
os_permission_denied 권한이 부여되지 않았음을 나타냅니다.

다음 권한은 기본적으로 GameMaker 에서 지원되지만 Google에서는 "위험한" 것으로 간주하므로 명시적으로 요청 해야 합니다 (일부 권한은 이 기능 없이 Android 게임 옵션 을 사용하여 요청할 수 있습니다. ):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

앱 권한에 대한 자세한 내용은 Android 설명서 를 참조하십시오.

중요한! 이 기능은 Android 대상 전용입니다.

예시:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

위의 코드는 OS 종류를 확인하고 안드로이드라면 권한을 확인하고, 아직 "INTERNET" 권한이 부여되지 않았다면 요청한다.
