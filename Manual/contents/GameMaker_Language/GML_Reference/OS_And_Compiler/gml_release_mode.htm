gml_release_mode

게임이 컴파일되면 실행 중 오류를 확인하는 특정 내부 기능도 포함됩니다. 이러한 내부 검사는 범위, 매개변수 및 내부 GML 컴파일러의 일부 일반적인 측면을 포함하여 게임 코드의 여러 측면을 다루며 많은 경우에 최종 오류 메시지를 생성합니다. 그러나 이 기능을 사용하여 이러한 내부 검사를 해제할 수 있으므로(즉, 컴파일된 게임에서 완전히 제거하여) 최종 프로젝트의 속도를 높일 수 있습니다.

즉, 게임 코드에서 잘못된 작업을 수행하면 컴파일된 게임 이 충돌 하거나 매우 예기치 않은 동작이 발생할 수 있습니다. 예를 들어 DS Grid 정의된 크기를 벗어난 좌표에서 액세스하는 경우 개발자 모드에서는 컴파일러 창 오류 메시지가 표시되지만 적절한 검사를 통해 게임을 계속 실행할 수 있습니다. 그러나 릴리스 모드에서는 이러한 검사가 없으므로 코드가 NULL 포인터를 반환하거나 잘못된 포인터를 반환하여 임의의 메모리를 가리킬 수 있습니다. 따라서 잘못된 데이터가 반환되거나 최악의 경우 메모리에 있는 데이터, 그래픽 또는 코드 자체를 덮어쓸 수 있습니다. 또 다른 예는 프리미티브를 구축할 때입니다. 색상, 위치, 질감 등... 데이터를 정점 형식을 만들 때 정의한 것과 다른 순서로 제공하면 게임이 충돌하거나 출시할 때 예기치 않은 결과가 생성될 수 있습니다. 모드가 켜져 있습니다.

통사론:

gml_release_mode(flag);

논쟁 유형 설명
flag Either true to set release mode or false for general development.

보고:

예시:

gml_release_mode(true);

위의 예제 코드는 게임을 컴파일할 때 릴리스 모드를 사용하도록 설정합니다.
