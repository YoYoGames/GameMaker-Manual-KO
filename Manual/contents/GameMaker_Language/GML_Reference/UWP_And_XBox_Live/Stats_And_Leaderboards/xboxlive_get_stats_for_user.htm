xboxlive_get_stats_for_user

이 기능은 Xbox Live 서버에서 특정 통계에 대한 데이터를 검색하는 데 사용할 수 있습니다. 함수에서 반환된 사용자 ID를 제공합니다.xboxlive_get_user(), 게임 서비스 구성 ID(XDP 콘솔에 표시됨), 마지막으로 필요한 통계입니다. 최대 14개의 통계를 요청할 수 있지만 요청의 총 길이에 제한이 있고 따라서 최대 통계 수는 이름의 길이에 따라 다르기 때문에 14개의 통계 결과를 얻는다는 보장은 없습니다. 통계 자체.

함수는 통계 요청이 전송된 경우 0을 반환하고 오류가 있는 경우 -1을 반환하고 콜백은 시스템 비동기 이벤트 를 트리거합니다. 이 이벤트에는 특별 DS 맵이 있습니다. async_load 그런 다음 다음 키에 대해 구문 분석되어야 합니다.

  • " event_type " - 이벤트가 이 함수에 의해 트리거된 경우 " stat_result " 문자열을 보유해야 합니다.

이벤트 유형이 이 기능과 관련된 경우 다음과 같은 추가 키도 있습니다.

  • " user " - 요청된 통계에 대한 사용자 ID를 보유합니다.

키 값 쌍의 추가 세트가 있을 수도 있습니다. 여기서 키는 요청된 통계 이름이고 서버의 해당 통계 값입니다. 이 사용자에 대한 통계가 생성된 적이 없는 경우(설정한 이벤트를 실행하지 않았기 때문에) 해당 통계에 대해 값이 반환되지 않을 수 있습니다(ds_map_exists() 에서 통계를 확인하는 데 사용할 수 있습니다. async_load 지도).

요청 길이가 너무 길어 요청이 실패하면 콘솔 출력에 오류 코드를 나타내는 메시지가 있어야 합니다.

xboxlive_get_stats_for_user - exception occurred getting results - 0x80190190

이 경우 비동기 이벤트 DS 맵에는 값이 "0"인 "성공" 키가 있어야 하며 이 경우 단일 호출에서 더 적은 수의 통계를 검색해야 합니다.

통사론:

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, …);

논쟁 유형 설명
user_id Xbox 사용자 ID The user ID pointer.
serviceconfig_id 끈 The service config file ID
statname1 (2, 3, etc...) 문자열 The stat names to retrieve the information for.

보고:

Real

예시:

var _uid = xboxlive_get_user(0);
var _configid = "00000000-0000-0000-0000-000000000000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode");

위의 코드는 사용자 ID를 가져온 다음 이를 사용하여 해당 사용자의 특정 통계에 대한 정보를 요청합니다.
