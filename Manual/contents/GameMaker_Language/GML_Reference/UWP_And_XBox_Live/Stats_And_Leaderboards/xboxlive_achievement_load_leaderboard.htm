xboxlive_achievement_load_leaderboard

이 함수는 주어진 리더보드에 대한 정보에 대한 요청을 서버에 보냅니다. 리더보드는 게임의 개발자 대시보드에서 이전에 생성되어야 하며 주어진 ID는 함수의 첫 번째 인수로 제공해야 하는 ID입니다. 다음을 포함하는 콜백 Social Asynchronous Event 를 트리거합니다. async_load 관련 키/값 쌍으로 채워진 맵. 이것의 아이디 키는 DS Map 올바른 콜백을 식별하는 데 사용되며(주어진 비동기 이벤트에 대해 둘 이상의 트리거 함수가 있을 수 있음) 상수 와 쌍을 이룹니다. xboxlive_achievement_leaderboard_info 뿐만 아니라 각 플레이어에 대한 기타 여러 키/값 쌍입니다. 지도의 정확한 내용은 다음과 같습니다.

  • " id " - 이 함수의 경우 다음과 같아야 합니다. xboxlive_achievement_leaderboard_info
  • " Leaderboardid " - 공급자 대시보드에 정의된 리더보드의 고유 ID입니다.
  • " numentries " - 받은 리더보드의 항목 수입니다.
  • " PlayerN " - 플레이어의 이름입니다. 여기서 "N"은 리더보드 항목 목록 내 위치에 해당하는 정수 값입니다.
  • " PlayeridN " - 플레이어의 고유 사용자 ID "N".
  • " RankN " - 리더보드 내 플레이어 "N"의 순위입니다.
  • " ScoreN " - 플레이어 "N"의 점수입니다.

최소 및 최대 인덱스 값을 제공하여 표시할 리더보드의 정확한 부분을 지정할 수 있을 뿐만 아니라 다음 필터 상수 를 사용하여 친구 또는 모든 플레이어가 게시한 점수만 검색하도록 결과를 필터링하도록 요청할 수 있습니다.

Xbox Live 업적 필터 상수
일정한 설명
xboxlive_achievement_filter_friends_only  플레이어 친구의 점수만 가져옵니다.
xboxlive_achievement_filter_all_players  모든 플레이어의 모든 점수를 가져옵니다.

통사론:

xboxlive_achievement_load_leaderboard(ident, minindex, maxindex, filter)

논쟁 유형 설명
ident 끈 The unique ID of the leaderboard as shown on the developer dashboard.
minindex 진짜 The starting index value to get the leaderboard data from.
maxindex 진짜 The maximum index value to get the leaderboard data from.
filter Xbox Live 업적 필터 상수 Whether to filter the results to those that are on your friends list or to all players (see the Description above)

확장된 예:

다음 코드는 플레이어가 게임 계정에 로그인하여 게임의 모든 친구 순위표 위치 목록을 가져온 후에 호출될 것입니다.

xboxlive_achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, xboxlive_achievement_filter_friends_only);

이것은 주어진 리더보드에 대한 정보에 대한 요청을 보내고 특별한 async_load 다음 데이터를 포함하는 DS 맵:

var ident = ds_map_find_value(async_load, "id");
if ident == xboxlive_achievement_leaderboard_info
{
    var lbid = ds_map_find_value(async_load,"leaderboardid");
    if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
    {
        global.numentries = ds_map_find_value(async_load,"numentries");
        for(var i = 0; i < numentries; i++)
        {
            global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
            global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
            global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
            global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
        }
    }
}

위의 코드는 반환된 내용을 확인합니다. DS Map Social Asynchronous Event 에서 해당 "id"가 확인 중인 상수와 일치하면 지도를 반복하고 여러 배열에 다른 모든 값을 저장하기 전에 전송되는 리더보드가 원래 함수 호출과 일치하는지 확인합니다.
