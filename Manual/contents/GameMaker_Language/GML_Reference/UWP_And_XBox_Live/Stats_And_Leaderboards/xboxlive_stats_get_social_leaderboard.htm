xboxlive_stats_get_social_leaderboard

이 기능은 주어진 통계에 대한 순위의 소셜 리더보드를 검색하는 데 사용할 수 있습니다. 사용자 ID를 제공합니다(예: 함수에 의해 반환됨).xboxlive_get_user() ), 통계 문자열(xboxlive_stats_set_stat_* 함수를 사용하여 생성할 때 정의됨), 검색할 리더보드 정보에 대한 여러 세부 정보를 지정합니다. int 또는 real stats에 대해서만 소셜 리더보드를 검색할 수 있지만 string stats에 대해서는 검색할 수 없습니다. "favorites_only" 인수를 true로 플래그 지정하면 결과에는 사용자가 표시한 친구에 대한 데이터만 포함됩니다. "즐겨찾기"로.

중요한! 소셜 리더보드에 사용된 통계는 XDP/Windows 개발자 센터에서 "추천 통계"로 등록할 필요가 없습니다.

이 함수는 소셜 비동기 이벤트 를 트리거하는 콜백을 생성합니다. 이 이벤트에는 특별 DS 맵이 있습니다. async_load 그런 다음 다음 키에 대해 구문 분석해야 합니다.

  • " id " - 상수를 유지합니다. achievement_stat_event
  • " event " - " GetLeaderboardComplete " 문자열을 보유합니다.
  • " userid " - 요청과 연결된 사용자 ID
  • " error " - 성공하면 0, 오류가 있으면 다른 값. 다음은 반환되는 가장 일반적인 오류입니다.
    • -2145844848: 잘못된 요청(통계가 유효한 리더보드 유형이 아닙니다. 즉, 문자열)
  • " errormessage " - 사용 가능한 경우 오류 메시지가 있는 문자열
  • " display_name " - 공급자 대시보드에 정의된 리더보드의 고유 ID입니다.
  • " numentries " - 받은 리더보드의 항목 수입니다.

나머지 DS 맵에는 다음 형식의 리더보드 데이터도 포함됩니다(여기서 "N"은 0에서 "숫자"까지의 리더보드 데이터 위치).

  • " 플레이어 N " - 플레이어의 이름입니다. 여기서 "N"은 리더보드 항목 목록 내 위치에 해당하는 정수 값입니다.
  • " Playerid N " - 플레이어의 고유 사용자 ID "N"입니다.
  • " Rank N " - 리더보드 내에서 플레이어 "N"의 순위입니다.
  • " Score N " - 플레이어 "N"의 점수입니다.

통사론:

xboxlive_stats_get_social_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, favourites_only);

논쟁 유형 설명
user_id Xbox 사용자 ID The user ID of the user to get the leaderboard for
stat 끈 The stat (as string) to create the global leaderboard from
num_entries 진짜 The number of entries from the global leaderboard to retrieve
start_rank 진짜 The rank in the leaderboard to start from (use 0 if the "start_at_user" argument is set to true )
start_at_user 부울 Set to true to start at the user ID rank, false otherwise (set to false if the "start_rank" argument is anything other than 0)
favourites_only 부울 Set to true to show only friends that are marked as "favourites" or false otherwise

확장된 예:

다음은 이 기능을 사용할 수 있는 확장된 예입니다. 시작하려면 Room Start 또는 Create 와 같은 이벤트에서 호출합니다.

xboxlive_stats_get_social_leaderboard(user_id, "GlobalTime", 20, 1, false, false);

위의 코드는 게임의 모든 글로벌 리더보드 위치 목록을 가져오기 위해 호출되며 다음과 같은 방식으로 처리할 Social Asynchronous Event 콜백을 생성합니다.

if (async_load[? "id"] == achievement_stat_event)
{
    if (async_load[? "event"] == "GetLeaderboardComplete")
    {
        global.numentries = async_load[? "numentries"];
        for(var i = 0; i < numentries; i++;)
        {
            global.playername[i] = async_load[? "Player" + string(i)];
            global.playerid[i] = async_load[? "Playerid" + string(i)];
            global.playerrank[i] = async_load[? "Rank" + string(i)];
            global.playerscore[i] = async_load[? "Score" + string(i]);
        }
    }
}

위의 코드는 Social Asynchronous Event에서 반환된 DS 맵을 확인하고 해당 "id"가 확인 중인 상수와 일치하면 지도를 반복하고 다른 모든 것을 저장하기 전에 반환된 리더보드 데이터에 의해 이벤트가 트리거되었는지 확인합니다. 여러 전역 배열의 값.
