xboxlive_matchmaking_find

이 기능은 XDP 대시보드에서 미리 생성했어야 하는 세션 템플릿, 매치메이킹 호퍼 및 보안 장치 연결 템플릿을 사용하여 멀티플레이어 세션을 검색합니다. 호출하여 특정 게임 패드에 대해 검색할 수 있는 사용자 IDxboxlive_user_for_pad() 함수. 이 함수는 고유한 요청 ID 값을 반환하고 이 값은 이 함수에 대한 올바른 소셜 비동기 이벤트 를 식별하는 데 사용할 수 있습니다. 이 이벤트는 세션이 생성되고 변수에 DS 맵을 포함할 때 트리거됩니다. async_load 다음 키/값 쌍으로:

  • "requestid" – 호출 함수에서 반환된 요청 ID를 포함합니다.
  • "status" – 어떤 유형의 이벤트가 트리거되었는지 알려주는 "session_created" 문자열을 포함합니다.
  • "num_results" – 세션이 발견된 경우 1 이상을 포함하고 그렇지 않은 경우 0을 포함합니다.
  • "sessionid <index> " – 세션 ID를 포함하거나 실패의 경우 -1을 포함합니다( <index/gt; 는 0에서 "num_results" - 1 사이의 값).
  • "sessionOwner <index> " - 세션이 발견된 경우 세션 호스트의 ID를 포함합니다( <index> 는 0에서 "num_results" - 1 사이의 값).
  • "오류" – 성공 시 0, 실패 시 -1 포함

통사론:

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

논쟁 유형 설명
user_id Xbox 사용자 ID The user ID pointer to use
template 끈 The name of the session template
hopper 끈 The name of the matchmaking hopper
sdatemplate 끈 The name of the secure device association template
matchattributes 진짜 OPTIONAL The match attributes

보고:

Async Request ID

예시:

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

위는 주어진 게임패드에서 사용자의 사용자 ID를 검색한 다음 매치메이킹 세션을 찾으려고 시도합니다.
