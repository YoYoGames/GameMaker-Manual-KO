충돌

모션을 계획하거나 특정 행동을 결정할 때 종종 게임 세계의 특정 위치에서 다른 개체와의 충돌이 있는지 확인하는 것이 중요하며 작업에 적합한 충돌을 선택하는 것이 가장 중요한 작업인 경우가 많습니다. GameMaker 에는 프로젝트의 요구 사항에 따라 충돌을 올바르게 처리하는 데 도움이 되는 여러 기능이 내장되어 있습니다.

참고 충돌 함수는 이전 버전의 GameMaker에서 정수 값을 사용하는 데 사용되었지만 이제는 부동 소수점 값을 사용합니다. 즉, 충돌 함수는 더 이상 정수가 아닌 인수를 반올림하지 않습니다. 자세한 내용은 충돌 호환성 모드 를 참조하십시오.

다음 함수는 인스턴스가 다른 인스턴스와의 충돌을 확인하도록 할 수 있는 다양한 방법을 다룹니다.

  • place_free
  • place_meeting
  • position_empty
  • position_meeting
  • position_change
  • position_destroy
  • instance_place
  • instance_place_list
  • instance_position
  • instance_position_list

고급 충돌 검사

이를 위해 다음 루틴을 사용할 수 있습니다( 이동 및 인스턴스 와 관련된 섹션에서 자세히 설명하는 기타 기능 포함). 이러한 충돌 함수는 모두 약간 다르게 작동하지만 여기에서 설명할 세 가지 공통 인수를 유지합니다.

  • obj: 여기에서 함수가 충돌을 확인해야 하는 객체를 지정합니다. 인스턴스 ID, 개체 유형 또는 특수 GM 키워드 all 이 될 수 있습니다. 선택한 개체 유형이 다른 개체의 지정된 상위 항목인 경우 해당 개체의 모든 하위 항목도 이 검사에 포함된다는 점에 유의할 필요가 있습니다.
  • prec: 다음 중 하나일 수 있습니다. true 또는 false 충돌 검사의 정밀도와 관련이 있습니다. 로 설정하면 true, 그런 다음 정확한 충돌이 계산되어야 합니다(그러나 검사 중인 개체가 마스크 속성에서 충돌 감지에 대해 "정확한" 표시가 있는 경우에만). 이는 CPU에 대해 약간 더 복잡한 작업이므로 주의해서 사용해야 합니다. 만약에 false 그런 다음 검사할 개체의 경계 상자를 기반으로 충돌이 계산됩니다.
  • notme: 코드를 실행하는 객체와 동일한 인덱스를 가진 객체와의 충돌을 확인해야 할 수도 있습니다. 따라서 코드가 자신과의 충돌을 찾는 것을 방지하기 위해 다음과 같이 설정할 수 있습니다. true. 코드를 실행하는 개체의 인스턴스를 특별히 찾고 있거나 중요하지 않은 경우 다음과 같이 둘 수 있습니다. false.

기본 함수는 충돌하는 것으로 발견된 인스턴스의 ID 또는 특수 키워드를 반환합니다.noone 충돌이 없을 때 목록 함수는 충돌하는 인스턴스의 수를 반환하고 미리 만들어진 DS 목록을 해당 ID로 채웁니다. 이러한 함수에 의해 정의된 영역과 주어진 객체의 인스턴스에 여러 충돌이 있는 경우 하나 의 인스턴스 ID만 반환 되며 충돌의 인스턴스 중 하나일 수 있습니다.

마스크 없이 충돌 확인

위의 모든 기능은 충돌 검사 인스턴스 와 관련되어 있으므로 인스턴스에 대해 정의된 충돌 마스크에 의존합니다. 그러나 특히 인스턴스에 할당된 스프라이트가 없거나 마우스 등으로 작업할 때 점 또는 영역과의 "충돌"을 확인해야 하는 많은 순간이 있습니다. 따라서 GameMaker 는 또한 다음을 제공합니다. 이러한 상황에서 도움이 되는 다음 기능:

  • point_in_rectangle
  • point_in_triangle
  • point_in_circle
  • rectangle_in_rectangle
  • rectangle_in_triangle
  • rectangle_in_circle

물리 충돌

내장 물리학을 사용할 때 위의 충돌 기능은 물리학이 활성화된 인스턴스에서 작동하지 않을 수 있습니다. 이는 주로 이러한 인스턴스가 더 이상 대부분의 일반 내장 변수를 사용하지 않기 때문이며(대신 물리 활성화된 인스턴스에는 자체 변수 세트가 있음) 고정 장치 를 사용하기 때문에 충돌 마스크나 경계 상자도 사용하지 않습니다. 대신에. 그러나 겹치는 고정 장치를 확인하는 물리학과의 충돌 테스트를 위한 특수 기능이 있습니다.

  • physics_test_overlap
