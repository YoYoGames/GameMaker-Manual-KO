collision_line

Collision_line 점 x1,y1에서 점 x2,y2까지의 선을 따라 "obj" 인수로 지정된 객체 인스턴스와의 충돌을 확인 합니다. 이 검사는 정확하거나 정확하지 않을 수 있지만 정확한 충돌을 활성화하려면 검사하는 개체 또는 인스턴스에도 해당 스프라이트에 대해 정확한 충돌이 활성화되어 있어야 합니다. 그렇지 않은 경우 기본 검사는 경계 상자를 기반으로 합니다. 다음 이미지는 작동 방식을 보여줍니다.

정확한 충돌을 고려하려면 오브젝트 스프라이트와 충돌 기능이 모두 켜져 있는 것으로 정확하게 표시되어 있어야 합니다. 또한 함수의 반환 값은 라인을 따라 충돌하는 것으로 간주되는 인스턴스 중 하나 의 id가 될 수 있으므로 세 개의 인스턴스가 정의된 라인과 겹치면 해당 id 중 하나가 반환 값이 될 수 있습니다. 기능의.

개체 인덱스 대신 인스턴스를 제공할 수 있습니다. id 특정 인스턴스 또는 인스턴스 키워드 를 확인하려면 all, 또는 other (이벤트 및 현재 코드 범위에 따라 다름).

통사론:

collision_line(x1, y1, x2, y2, obj, prec, notme);

논쟁 유형 설명
x1 진짜 The x coordinate of the start of the line.
y1 진짜 The y coordinate of the start of the line.
x2 진짜 The x coordinate of the end of the line.
y2 진짜 The y coordinate of the end of the line.
obj 개체 자산 또는 인스턴스 ID The object to check for instance collisions.
prec 부울 Whether the check is based on precise collisions (true, which is slower) or its bounding box in general (false, faster).
notme 부울 Whether the calling instance, if relevant, should be excluded (true ) or not (false ).

보고:

Instance ID or noone

예시:

with (obj_Enemy)
{
    if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
}

이 코드는 "obj_Enemy"의 모든 인스턴스를 가져와 100x, 400y에서 100x, 600y까지 라인을 따라 자신과의 충돌을 확인하고 충돌이 있는 경우 파괴됩니다.
