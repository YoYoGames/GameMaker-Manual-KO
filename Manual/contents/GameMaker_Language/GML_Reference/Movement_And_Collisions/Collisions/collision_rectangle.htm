collision_rectangle

Collision_rectangle 처음 4개의 인수(x1,y1,x2,y2)를 사용하여 현재 방 내의 영역을 정의한 다음 "obj" 인수로 정의 된 개체가 해당 영역과 충돌하는지 확인합니다. 이 충돌은 정확한지 여부를 확인할 수 있으며 코드 자체를 실행하는 인스턴스를 확인하도록 선택할 수도 있습니다. 다음 이미지를 고려하십시오.

여기에서 중간에 있는 인스턴스는 충돌 사각형을 사용하여 공 개체를 확인하고 있습니다. 이제 파란색 항목에는 정확한 경계 상자가 없으며 보시다시피 스프라이트가 실제로 사각형에 닿지 않더라도 충돌은 여전히 발생합니다(함수에서 정확한 옵션을 true로 설정하더라도). 해당 스프라이트의 경계 상자는 collision_rectangle과 겹칩니다. 반면에 녹색 공은 실제 스프라이트가 직사각형과 겹치는 경우에만 충돌로 간주됩니다. 정확한 충돌을 고려하려면 오브젝트 스프라이트와 충돌 기능이 모두 켜져 있는 것으로 정확하게 표시되어 있어야 합니다. 또한 함수의 반환 값은 충돌하는 것으로 간주되는 인스턴스 중 하나 의 id일 수 있습니다.

개체 인덱스 대신 인스턴스를 제공할 수 있습니다. id 특정 인스턴스 또는 인스턴스 키워드 를 확인하려면 all, 또는 other (이벤트 및 현재 코드 범위에 따라 다름).

통사론:

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

논쟁 유형 설명
x1 진짜 The x coordinate of the left side of the rectangle to check.
y1 진짜 The y coordinate of the top side of the rectangle to check.
x2 진짜 The x coordinate of the right side of the rectangle to check.
y2 진짜 The y coordinate of the bottom side of the rectangle to check.
obj 개체 자산 또는 인스턴스 ID The object to check for instance collisions.
prec 부울 Whether the check is based on precise collisions (true, which is slower) or its bounding box in general (false, faster).
notme 부울 Whether the calling instance, if relevant, should be excluded (true ) or not (false ).

보고:

Instance ID or noone

예시:

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if (inst != noone)
{
    with (inst) instance_destroy();
}

이 짧은 코드는 "obj_ball"이라는 개체의 인스턴스에 대해 50x, 50y(사각형의 왼쪽 상단)에서 200x, 200y(사각형의 오른쪽 하단)까지 방의 영역을 확인하는 collision_rectangle을 사용합니다. 반환 값을 임시 변수에 저장한 다음 해당 값이 인스턴스 ID인지 또는 키워드 noone 인지 확인합니다. 아무도 아닌 경우 저장된 인스턴스 ID를 사용하여 개체를 파괴합니다.
