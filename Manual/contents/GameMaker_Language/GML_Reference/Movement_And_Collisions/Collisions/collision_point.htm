collision_point

Collision_point 인수 x1,y1에 의해 지정된 지점에서 "obj" 인수로 지정된 객체 인스턴스와의 충돌이 있는지 확인합니다. 이 검사는 정확하거나 정확하지 않을 수 있지만 정확한 충돌을 활성화하려면 검사하는 개체 또는 인스턴스에도 해당 스프라이트에 대해 정확한 충돌이 활성화되어 있어야 합니다. 그렇지 않은 경우 기본 검사는 경계 상자를 기반으로 합니다. 다음 이미지는 작동 방식을 보여줍니다.

정확한 충돌을 고려하려면 오브젝트 스프라이트와 충돌 기능이 모두 켜져 있는 것으로 정확하게 표시되어 있어야 합니다. 또한 함수의 반환 값은 충돌하는 것으로 간주되는 인스턴스 중 하나 의 id일 수 있으므로 해당 지점에서 세 개의 인스턴스가 겹치면 해당 id 중 하나가 함수의 반환 값이 될 수 있습니다..

개체 인덱스 대신 인스턴스를 제공할 수 있습니다. id 특정 인스턴스 또는 인스턴스 키워드 를 확인하려면 all, 또는 other (이벤트 및 현재 코드 범위에 따라 다름).

통사론:

collision_point(x, y, obj, prec, notme);

논쟁 유형 설명
x 진짜 The x coordinate of the point to check.
y 진짜 The y coordinate of the point to check.
obj 개체 자산 또는 인스턴스 ID The object to check for instance collisions.
prec 부울 Whether the check is based on precise collisions (true, which is slower) or its bounding box in general (false, faster).
notme 부울 Whether the calling instance, if relevant, should be excluded (true ) or not (false ).

보고:

Instance ID or noone

예시:

if collision_point(x, y, obj_Cursor, false, true)
{
    score += 10S;
}

여기서 우리는 "obj_Cursor" 개체에 대한 코드가 있는 개체의 위치에서 점을 확인하고 있습니다. 하나가 있으면 점수 변수에 10을 추가합니다.
