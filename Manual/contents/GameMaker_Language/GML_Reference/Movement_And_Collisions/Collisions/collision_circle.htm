collision_circle

Collision_circle 점 x1,y1 주위에 사용자가 지정한 반지름의 원형 영역을 만든 다음 해당 영역에서 "obj" 인수로 지정된 개체 인스턴스와의 충돌이 있는지 확인합니다. 이 검사는 정확하거나 정확하지 않을 수 있지만 정확한 충돌을 활성화하려면 검사하는 개체 또는 인스턴스에 해당 스프라이트에 대한 정확한 충돌도 활성화되어 있어야 합니다. 그렇지 않은 경우 기본 검사는 경계 상자를 기반으로 합니다. 다음 이미지는 작동 방식을 보여줍니다.

여기에서 중간에 있는 인스턴스는 충돌 원을 사용하여 공 개체를 확인하고 있습니다. 이제 파란색 항목에는 정확한 경계 상자가 없으며 보시다시피 스프라이트가 실제로 원에 닿지 않더라도 충돌은 여전히 발생할 수 있습니다(함수에서 정확한 옵션을 true로 설정하더라도). 해당 스프라이트의 경계 상자는 collision_circle에 의해 정의된 원형 영역과 겹칩니다. 반면에 녹색 공은 실제 스프라이트가 정의된 원과 겹치는 경우에만 충돌로 간주됩니다. 정확한 충돌을 고려하려면 오브젝트 스프라이트와 충돌 기능이 모두 켜져 있는 것으로 정확하게 표시되어 있어야 합니다. 또한 함수의 반환 값은 충돌하는 것으로 간주되는 인스턴스 중 하나 의 id일 수 있습니다.

개체 인덱스 대신 인스턴스를 제공할 수 있습니다. id 특정 인스턴스 또는 인스턴스 키워드 를 확인하려면 all, 또는 other (이벤트 및 현재 코드 범위에 따라 다름).

통사론:

collision_circle(x1, y1, rad, obj, prec, notme);

논쟁 유형 설명
x1 진짜 The x coordinate of the center of the circle to check.
y1 진짜 The y coordinate of the center of the circle to check.
rad 진짜 The radius (distance in pixels from its center to its edge).
obj 개체 자산 또는 인스턴스 ID The object to check for instance collisions.
prec 부울 Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).
notme 부울 Whether the calling instance, if relevant, should be excluded (true ) or not (false ).

보고:

Instance ID or noone

예시:

if collision_circle(x, y, 20, obj_Cursor, false, true)
{
    image_index = 1;
}
else image_index = 0;

위의 코드는 "obj_Cursor"와의 충돌에 대해 반경이 20픽셀인 원형을 확인하고 하나가 있으면 코드를 실행하는 객체의 image_index를 1로 설정하지만, 그렇지 않은 경우에는 image_index를 설정합니다. 0에 반대합니다.
