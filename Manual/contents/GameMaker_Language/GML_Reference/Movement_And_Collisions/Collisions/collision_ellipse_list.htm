collision_ellipse_list

이 기능은 다음과 같습니다.collision_ellipse() 기능을 사용하면 한 번에 하나의 충돌 인스턴스만 감지하는 대신 여러 인스턴스를 감지합니다. 확인할 객체와 함께 확인할 타원형 영역의 왼쪽 상단 및 오른쪽 하단의 x/y 위치를 제공하고 확인을 정밀하게 설정할 수 있습니다(이 경우 확인 중인 모든 인스턴스는 정확한 충돌 마스크) 및 검사에 호출 인스턴스가 포함되어야 하는지 여부.

DS 목록 도 제공하므로 id 호출하는 인스턴스와 충돌하는 모든 인스턴스의 값은 지정된 목록의 끝에 추가됩니다. 이 함수 호출의 결과만 포함하도록 이 함수를 호출하기 전에 목록을 지울 수 있습니다. 타원 영역의 원점에서 원점까지의 거리를 기준으로 인스턴스를 정렬할 수도 있습니다. 이 함수는 찾은 인스턴스의 수를 반환하거나 찾지 못한 경우 0을 반환합니다.

객체 인덱스 대신 instance 키워드 를 제공할 수 있습니다.all, 코드를 실행하는 인스턴스를 포함할 수 있는 현재 방의 모든 인스턴스를 확인하려면(에 따라 notme 논쟁).

통사론:

collision_ellipse_list(x1, y1, x2, y2, obj, prec, notme, list, ordered);

논쟁 유형 설명
x1 진짜 The x coordinate of the left side of the ellipse to check.
y1 진짜 The y coordinate of the top side of the ellipse to check.
x2 진짜 The x coordinate of the right side of the ellipse to check.
y2 진짜 The y coordinate of the bottom side of the ellipse to check.
obj 개체 자산 또는 인스턴스 ID The object to check for instance collisions.
prec 부울 Whether the check is based on precise collisions (true, which is slower) or its bounding box in general (false, faster).
notme 부울 Whether the calling instance, if relevant, should be excluded (true ) or not (false ).
list DS 목록 ID The DS list to use to store the IDs of colliding instances.
ordered 부울 Whether the list should be ordered by distance (true ) or not (false ).

보고:

Real (The number of instances found to be in collision)

예시:

var _list = ds_list_create();
var _num = collision_ellipse_list(x - 100, y - 100, x + 100, y + 100, obj_Enemy, false, true, _list, false);
if _num > 0
{
    for (var i = 0; i < _num; ++i;)
    {
        instance_destroy(_list[| i]);
    }
}
ds_list_destroy(_list);

위의 코드는 "obj_Enemy" 인스턴스와의 충돌에 대해 호출 인스턴스 위치 주변의 타원형 영역 100픽셀을 확인합니다. 충돌이 있는 경우 미리 생성된 목록이 반복되고 충돌에 있었던 각 인스턴스가 소멸됩니다.
