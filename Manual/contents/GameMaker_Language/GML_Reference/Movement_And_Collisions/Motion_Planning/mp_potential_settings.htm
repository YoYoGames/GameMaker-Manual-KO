mp_potential_settings

그만큼 mp_potential_ 함수는 이 함수를 사용하여 변경할 수 있는 여러 매개변수를 사용하여 작업을 수행합니다. 전역적으로 이 방법은 다음과 같이 작동합니다. 먼저 목표를 향해 직선으로 이동하려고 시도하고 "앞으로" 매개변수로 설정할 수 있는 여러 단계를 찾습니다(기본값은 3). 이 값을 줄이면 인스턴스가 멀리 "보지" 않기 때문에 나중에 방향 변경을 시작한다는 의미이고, 이 값을 늘리면 더 빨리 방향 변경을 시작한다는 의미입니다. 이 검사가 충돌로 이어지면 이상적인 직선 방향의 왼쪽과 오른쪽 방향을 더 보기 시작합니다. "rotstep"(기본값은 10) 크기의 단계로 이 작업을 수행하며, 이를 줄이면 인스턴스에 더 많은 이동 가능성이 제공되지만 더 많은 처리 능력이 필요하므로 속도가 느려집니다.

"maxrot" 매개변수는 설명하기가 조금 더 어렵습니다! 인스턴스에는 현재 이동 방향이 있으며 maxrot(기본값은 30)은 한 단계에서 현재 방향의 양쪽 방향을 변경할 수 있는 정도를 나타냅니다. 따라서 예를 들어 목표로 똑바로 이동할 수 있더라도 이 최대 방향 변경을 위반하지 않는 경우에만 그렇게 할 것입니다. maxrot을 큰 값으로 설정하면 인스턴스가 각 단계에서 방향을 많이 변경할 수 있으며, 이렇게 하면 짧은 경로를 더 쉽게 찾을 수 있지만 경로가 더 추하고 덜 자연스러워집니다. 값을 더 작게 만들면 경로가 더 부드러워지지만 우회하는 데 더 오래 걸릴 수 있습니다(때로는 목표를 찾지 못할 수도 있음). 마지막으로 단계를 수행할 수 없는 경우 "onspot" 매개변수의 값에 따라 동작이 달라집니다. onspot이 true(기본값)이면 인스턴스는 "maxrot"으로 표시된 양만큼 제자리에서 회전하고 false이면 전혀 이동하지 않습니다. false로 설정하면 예를 들어 자동차에 유용하지만 인스턴스가 경로를 찾을 가능성이 줄어듭니다.

통사론:

mp_potential_settings(maxrot, rotstep, ahead, onspot)

논쟁 유형 설명
maxrot 진짜 The number of degrees either side of the current direction that the instance can rotate in a step
rotstep 진짜 The number of degrees either side of the current direction that the instance can check for a collision
ahead 진짜 The number of steps ahead that the instance can check for a collision. Larger values are slower than lower ones.
onspot 부울 Use this to allow the instance to rotate on the spot when no path is found (true) or not (false)

보고:

N/A

예시:

mp_potential_settings(45, 5, 5, 0);

위의 코드는 mp_potential_ 다음 설정을 사용하는 기능: 인스턴스는 5단계 앞을 내다보고, 각 단계마다 방향을 45도 변경하고, 장애물이 있는 경우 현재 방향의 양쪽 5도를 보고 충돌이 있는 자리에서 회전하지 않을 수 있습니다.
