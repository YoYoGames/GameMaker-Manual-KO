mp_potential_path

이 함수는 현재 위치 와 방향 에서 xgoal,ygoal 인수로 지정된 위치까지 인스턴스의 경로를 계산합니다. 모든 인스턴스 또는 솔리드로 플래그가 지정된 인스턴스와만 충돌을 방지하는 동안 표시된 단계 크기를 사용합니다. 기능에서처럼mp_potential_step() 이를 수행하기 위해 잠재적인 필드 단계를 사용하며 이를 수행하는 방식은 함수를 통해 수정할 수 있습니다.mp_potential_settings(). 표시된 경로는 이미 존재해야 하며 새 경로로 덮어씁니다. 함수는 경로를 찾았는지(true) 또는 찾지 않았는지(false) 반환합니다.

함수가 영원히 계속 계산되는 것을 방지하려면 1보다 큰 인수를 제공해야 합니다. 이 인수에 시작과 목표 사이의 거리를 곱한 것보다 짧은 경로를 찾을 수 없으면 함수가 중지되고 실패를 보고합니다. 일반적으로 요소 4로 충분하지만 긴 우회를 예상하면 더 길게 만들 수도 있습니다. 실패하면 목표의 대략적인 방향으로 실행되는 경로가 계속 생성되지만 도달하지는 않습니다.

참고: 이 기능은 인스턴스를 이동하지 않습니다. 경로만 설정하며 이동을 위해서는 Path 기능을 사용해야 합니다.

통사론:

mp_potential_path(path, xgoal, ygoal, stepsize, factor, checkall)

논쟁 유형 설명
path index of the path that is to be used by the function
xgoal The target x position.
ygoal The target y position.
stepsize The speed the instance moves in pixels per step.
factor This number is used to prevent an infinite loop. It should be larger than 1. See the function description for more information.
checkall Whether to check all instances (true) or just solid ones (false).

보고:

예시:

path = path_add();
mp_potential_path(path, obj_Player.x, obj_Player.y, 3, 4, 0);
path_start(path, 3, 0, 0);

위의 코드는 새 경로 리소스를 만들고 해당 인덱스를 "path" 변수에 저장합니다. 그런 다음 다음을 사용하여 경로를 생성합니다. mp_potential_path 이 코드가 있는 개체와 "obj_Player" 개체 사이에서 방의 모든 인스턴스와의 충돌을 확인합니다. 마지막으로 함수가 "obj_Player"에 대한 전체 경로를 반환하는 데 실패한 경우에도 해당 경로를 따라 개체를 시작합니다.
