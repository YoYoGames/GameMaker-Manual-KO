mp_grid_create

이 기능을 사용하여 모션 계획 기능을 위한 mp_grid를 생성할 수 있습니다. 다른 모든 MP 그리드 함수 호출에 사용하기 위해 변수에 저장해야 하는 인덱스를 반환합니다. x 및 y 좌표는 그리드의 왼쪽 상단 모서리 위치를 나타내고 hcells 및 vcells는 그리드 내에서 생성할 가로 및 세로 셀 수를 나타내며 셀 너비와 셀 높이는 셀 크기(픽셀)를 나타냅니다.. 서로 다른 용도로 필요한 경우 여러 그리드 구조를 동시에 만들고 유지할 수 있지만 각 그리드에는 메모리가 필요하고 셀이 많을수록 더 많은 메모리가 필요하다는 점에 유의하십시오. 또한 MP 기능은 사용하기에 프로세서가 매우 무거우므로 자주 호출하고 셀 해상도가 높을수록 성능이 저하될 가능성이 더 높다는 점에 유의하십시오.

통사론:

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight);

논쟁 유형 설명
left 진짜 Starting x coordinate of the mp_grid in the room
top 진짜 Starting y coordinate of the mp_grid in the room
hcells 진짜 Number of horizontal cells that the mp_grid will contain
vcells 진짜 Number of vertical cells that the mp_grid will contain
cellwidth 진짜 The width (in pixels) of each individual cell of the mp_grid
cellheight 진짜 The height (in pixels) of each individual cell of the mp_grid

보고:

MP Grid ID

예시:

global.grid = mp_grid_create(0, 0, room_width / 32, room_height / 32, 32, 32);

위의 코드는 전역 변수 "grid"를 생성한 다음 mp_grid를 생성하고 모든 추가 mp_grid 함수 호출에서 사용하기 위해 해당 변수에 인덱스(id)를 할당합니다. 격자는 32x32 픽셀의 셀 해상도로 방을 덮습니다. 예를 들어, 640x480 크기의 방에서 그리드에는 300개의 셀이 포함됩니다. 20개의 수평 셀(640/32)에 15개의 수직 셀(480/32)을 곱하면 300개의 셀이 생성됩니다.
