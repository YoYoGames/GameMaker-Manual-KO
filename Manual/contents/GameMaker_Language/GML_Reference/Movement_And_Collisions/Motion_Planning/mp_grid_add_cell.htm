mp_grid_add_cell

이 기능을 사용하여 주어진 MP 그리드 내의 개별 셀을 "금지된" 것으로 표시할 수 있습니다. 즉, 경로 찾기 기능이 해당 셀을 가로지르지 않을 것입니다. 이제 일반적으로 MP 그리드 셀 좌표가 아닌 x/y 방 좌표에서 작업합니다. 따라서 둘의 상관 관계를 보여주는 다음 이미지를 살펴보겠습니다.

보시다시피 mp_grid는 일련의 셀로 번호가 매겨져 있습니다(왼쪽 상단 모서리의 0,0에서 시작). 이 그리드를 방 위에 놓으면 방 좌표 사이에 직접적인 상관 관계가 있습니다(0에서 시작하기도 합니다.,0 왼쪽 상단) 및 그리드 내의 셀. 이미지에서 위치 260,130의 인스턴스가 그리드 셀 8,4에 있는 것을 볼 수 있습니다. 숫자를 그리드 해상도로 나눈 다음 내림 하거나, 2 그리드 해상도(예: 8, 16, 32 등...)를 사용하는 경우 위치가 정확히 맞는 셀을 수학적으로 쉽게 계산할 수 있습니다. 비트 연산자 를 사용할 수 있습니다. 비트 연산이 더 빠르며 아래 예는 두 가지 방법을 모두 보여줍니다.

통사론:

mp_grid_add_cell(id, h, v);

논쟁 유형 설명
id MP 그리드 ID Index of the mp_grid that is to be used
h 진짜 Horizontal position of the cell in the grid
v 진짜 Vertical position of the cell in the grid

보고:

N/A

예시:

with (obj_Wall)
{
    mp_grid_add_cell(grid, floor(x / 32), floor(y / 32));
}
with (obj_Door)
{
    mp_grid_add_cell(grid, x>>5, y>>5);
}

위의 코드는 모든 "obj_Wall"과 모든 obj_Door가 변수 "grid"에 색인된 mp_grid에 동등한 셀 위치를 추가하도록 합니다. 예제의 첫 번째 부분에서 이것은 객체의 x/y 좌표를 가져와 그리드의 해상도로 나누어 수행합니다(사용 floor 값을 정수로 유지하기 위해), 그리고 두 번째 부분에서(그리드 해상도는 2의 거듭제곱이기 때문에) x, y 좌표를 오른쪽으로 5자리 비트 시프트하여 수행됩니다.
