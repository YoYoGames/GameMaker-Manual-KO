mp_grid_add_instances

이 함수는 인스턴스 충돌 마스크(호출 인스턴스의 마스크 인덱스 또는 스프라이트 속성에 의해 결정됨)를 사용하여 금지된 셀을 경로 찾기 기능으로 교차할 수 없는 MP 그리드에서 금지된 셀을 표시하거나 금지합니다. 정확한 충돌을 고려할지 여부를 기능에서 지정할 수 있으며 이 설정에 따라 표시된 금지 셀이 변경됩니다. 이 이미지는 다음 동작을 보여줍니다.

위의 두 인스턴스는 다음을 사용하여 MP 그리드에 추가되었습니다. mp_grid_add_instances() 정확한 값이 true로 설정되어 있는 함수입니다. 보시다시피, 녹색 인스턴스는 스프라이트 마스크도 정밀하게 설정되어 있기 때문에 "만지는" 그리드 사각형만 금지된 것으로 표시했습니다. 그러나 두 번째 인스턴스는 접촉하지 않는 것으로 보이는 다른 셀도 표시했습니다. 이는 스프라이트 마스크가 정밀하게 설정되어 있지 않기 때문입니다. 즉, 함수에서 정확한 설정을 사용하더라도 스프라이트의 경계 상자만 고려됩니다. 이것은 또한 함수의 정확한 인수를 false로 설정하면 발생합니다. 정확한 스프라이트 마스크가 있는 인스턴스라도 경계 상자를 기반으로 그리드에 추가됩니다.

통사론:

mp_grid_add_instances(id, obj, prec);

논쟁 유형 설명
id MP 그리드 ID Index of the mp_grid that is to be used
obj 개체 자산 또는 인스턴스 ID Object index, or instance id, of the instances to be added into the mp_grid
prec 부울 Whether the check is based on pixel-perfect collisions (true = slow) or its bounding box in general (false = fast).

보고:

N/A

예시:

mp_grid_add_instances(grid, obj_Wall, true);

위의 코드는 경계 상자가 아닌 정확한 충돌 마스크를 사용하여 변수 "grid"에 색인된 mp_grid에 "obj_Wall"의 모든 인스턴스를 추가합니다.
