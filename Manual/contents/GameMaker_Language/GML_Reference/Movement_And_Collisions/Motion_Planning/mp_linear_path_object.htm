mp_linear_path_object

이 함수는 현재 인스턴스 위치에서 "xgoal" 및 "ygoal" 값으로 지정된 지점까지의 직선 경로를 계산합니다. 함수에서와 같이 표시된 단계 크기를 사용합니다.mp_linear_step(). 표시된 경로는 이미 존재해야 하며 새 경로로 덮어쓰여지며 완전한 경로를 찾았는지(true) 또는 찾지 못한 경우(false) 함수가 반환됩니다. 전체 경로는 지정된 개체 또는 인스턴스와의 충돌이 없는 경우에만 발견되며 false가 반환되면 경로는 계속 생성되지만 경로가 차단된 위치까지만 실행됩니다.

참고: 이 기능은 인스턴스를 이동하지 않습니다. 경로만 설정하며 이동을 위해서는 Path 기능을 사용해야 합니다.

통사론:

mp_linear_path_object(path, xgoal, ygoal, stepsize, obj);

논쟁 유형 설명
path 경로 자산 The index of the path to be used
xgoal 진짜 The target x position.
ygoal 진짜 The target y position.
stepsize 진짜 The speed the instance moves in pixels per step.
obj 개체 자산 또는 인스턴스 ID The object that is to block the path. Can be an object index, an instance id or the special keyword, all

보고:

Boolean

예시:

if mp_linear_path_object(path, mouse_x, mouse_y, 4, obj_Wall)
{
    path_start(path, 4, 0, 0);
}

위의 코드는 코드를 실행하는 개체와 마우스의 x/y 위치 사이의 경로를 따라 "obj_Wall"과의 충돌을 확인합니다. 충돌이 없고 완전한 경로가 생성되면 생성된 경로를 따라 개체를 시작합니다.
