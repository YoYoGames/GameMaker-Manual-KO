DS 대기열

DS 대기열은 DS 스택 과 다소 유사하지만 FIFO(선입 선출) 기반으로 작동합니다. 먼저 대기열에 넣은 값은 대기열에서 가장 먼저 제거됩니다. 계산을 하기 위해 가게에 줄을 서 있는 것처럼 생각할 수 있습니다. 대기열에 먼저 있는 사람이 먼저 돈을 지불한 다음 가게를 나가는 것입니다. 대기열은 일반적으로 인스턴스에서 수행해야 하지만 다른 용도로 많이 사용되는 동작이나 동작(예: 스크립트 형태)을 저장하는 데 사용됩니다.

참고: 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 필요하지 않을 때 항상 파괴되어 속도가 느려지고 결국 게임이 중단되는 메모리 누수를 방지해야 합니다.

참고: 데이터 구조가 존재하는지 확인해야 하는 경우 다음을 사용할 수 있습니다.ds_exists() 함수.

DS 대기열을 처리하는 다음과 같은 기능이 있습니다.

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
