ds_queue_read

이 기능을 사용하여 저장된 DS 대기열(이전에ds_queue_write() ). 문자열을 읽을 새 DS 대기열을 먼저 생성해야 합니다. DS 대기열이 이미 존재하고 정보가 저장된 경우 읽기 전에 지워집니다. 이 기능은 게임에 대한 저장/로드 메커니즘을 만들 때 매우 중요합니다.

지정된 DS 문자열이 GameMaker: Studio 1.2.x 런타임(또는 이전 버전)에 의해 작성된 경우 선택적 인수 "legacy"를 지정하고 다음으로 설정해야 합니다. true 그 이후에 문자열 형식이 변경되었기 때문입니다.

통사론:

ds_queue_read(id, str [, legacy]);

논쟁 유형 설명
id DS 대기열 ID The id of the data structure to read into.
str 끈 The string to read from.
legacy 부울 OPTIONAL Can be either true or false or omitted completely.

보고:

N/A

예시:

queue = ds_queue_create();
ini_open("save.ini");
var str = ini_read_string("Queues", "0", "");
if (str != "")
{
    ds_queue_read(queue, str);
}
ini_close();

위의 코드는 큐를 생성하고 "queue" 변수에 인덱스를 저장합니다. 그런 다음 열립니다. ini 파일에서 문자열을 읽고 문자열이 먼저 비어 있는 것으로 반환되지 않았는지 확인합니다. 그런 다음 이 문자열을 새로 생성된 DS 대기열로 읽습니다.
