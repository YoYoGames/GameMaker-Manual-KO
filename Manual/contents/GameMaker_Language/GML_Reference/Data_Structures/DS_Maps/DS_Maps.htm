DS 맵

DS 맵 데이터 구조는 키 와 값 쌍을 저장할 수 있으므로 매우 유용합니다. 예를 들어 게임의 캐릭터는 다양한 항목( 키 )을 가질 수 있고 각 개별 항목에 대해 여러 가지( 값 )를 가질 수 있습니다. 예를 들어 RPG에서 10개의 건강 물약, 5개의 마나 물약 및 100골드. 지도는 이러한 쌍을 한 곳에서 모두 함께 유지하고 지도에 쌍을 추가하고 몇 가지 간단한 기능을 사용하여 특정 키에 해당하는 값을 검색할 수 있습니다. 그러나 지도를 사용하기 전에 지도에 대해 알아야 할 몇 가지 사항이 있습니다! 맵은 (인식할 수 있는) 어떤 방식으로도 정렬 되지 않습니다. 즉, 특정 키를 찾으려면 전체를 반복해야 할 수도 있습니다( 매우 느림). 또한 동일한 두 개의 키를 보유할 방법이 없으며 하나의 키에 두 개의 값을 할당할 수도 없습니다.

참고: 유사한 기능을 갖고 사용하기 쉽고 자동으로 가비지 수집되기 때문에 DS 맵보다 구조체 를 사용하는 것이 좋습니다.

참고 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 필요하지 않을 때 항상 삭제하여 속도를 늦추고 결국 게임을 중단시키는 메모리 누수를 방지해야 합니다.

아래 나열된 특정 기능 외에도 모두 표현식 ( 접근 자 라고 함)을 사용하여 DS 맵의 내용을 추가하거나 수정할 수 있습니다. 이 접근자는 다음 구문을 사용하는 1D 배열과 유사합니다.

map_index[? key]

접근 자 의 GML 개요 페이지에서 추가 정보와 예를 찾을 수 있습니다.

DS 맵을 처리하는 다음 기능이 있습니다.

  • ds_map_create
  • ds_map_add
  • ds_map_clear
  • ds_map_copy
  • ds_map_replace
  • ds_map_delete
  • ds_map_empty
  • ds_map_size
  • ds_map_find_first
  • ds_map_find_last
  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_keys_to_array
  • ds_map_values_to_array
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

DS 맵을 저장하고 로드하는 데 사용할 수 있는 4가지 추가 기능이 있습니다. 이러한 기능은 맵을 난독화하고 파일 형식을 사용하여 대상 플랫폼의 안전한 위치에 저장합니다. 즉, 최종 파일을 장치 간에 전송할 수 없습니다.

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

DS 맵의 용도 중 하나는 JSON으로 작업할 때이므로 이에 관련된 몇 가지 특수 기능이 있습니다.

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

참고 이러한 기능을 사용하면 맵 내에 목록과 맵을 추가할 수 있지만 JSON 이외의 다른 용도로는 쓸모가 없으며 디스크에 쓰거나 다른 방식으로 액세스하는 경우 중첩 맵과 목록을 올바르게 읽을 수 없습니다.
