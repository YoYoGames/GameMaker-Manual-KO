ds_map_destroy

DS 맵은 생성될 때 할당되는 메모리 공간을 차지합니다. 즉, 게임을 실행할 때 오류, 메모리 누수 및 성능 손실을 방지하기 위해 DS 맵이 필요하지 않은 경우에도 이 메모리를 해제해야 합니다. 이 기능은 바로 그 일을 합니다. 지도를 파괴하면 지도에 저장된 모든 데이터 구조가 역참조되어 메모리 누수가 발생하므로 이를 방지하려면 지도를 파괴하기 전에 수동으로 지도를 살펴보고 모든 데이터 구조 항목을 파괴해야 합니다. 이것이 필요하지 않은 유일한 경우는 지도의 항목에 DS 목록 또는 다른 DS map 으로 플래그를 지정했을 때입니다. 이 경우 이러한 항목이 삭제되고 해당 메모리도 자동으로 정리됩니다.

중요한! 데이터 구조를 생성할 때 식별하는 인덱스 값은 0부터 시작하는 정수 값입니다. 이는 다른 유형의 데이터 구조가 동일한 인덱스 값을 가질 수 있음을 의미하므로 의심스러운 경우 다음을 사용해야 합니다.ds_exists() 액세스하기 전에 기능. 또한 인덱스는 재사용되므로 파괴된 데이터 구조 인덱스 값은 나중에 새로 생성된 값에서 사용될 수 있으므로 파괴 후 DS 인덱스를 보유하는 변수를 항상 -1로 설정하는 것이 좋습니다.

통사론:

ds_map_destroy(id);

논쟁 유형 설명
id The id of the map to destroy.

보고:

예시:

ds_map_destroy(inventory);
inventory = -1;

위의 코드는 "inventory" 변수에 인덱싱된 ID를 가진 DS 맵을 파괴합니다.
