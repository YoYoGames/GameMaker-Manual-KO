ds_set_precision

예를 들어 지도에서 검색하거나 목록을 정렬할 때 값을 비교할 때 GameMaker 는 두 값이 같은 경우를 결정해야 합니다. 문자열과 정수 값의 경우 이는 분명하지만 실수의 경우 부동 소수점 반올림 오류로 인해 같아 보이는 숫자가 쉽게 같지 않을 수 있습니다. 예를 들어, (5 / 3) * 3 은 5와 같지 않을 수 있습니다! 이를 방지하기 위해 모든 실수 함수에 정밀도 값이 사용 되며 두 숫자의 차이가 이 정밀도보다 작 으면 동일한 것으로 간주됩니다. 이 함수에 의해 변경되지 않는 한 기본 정밀도 0.0000001은 모든 데이터 구조 함수에 사용됩니다.

참고: 이 정밀도는 모든 데이터 구조에서 사용되지만 GML의 다른 비교에서는 사용되지 않습니다!

통사론:

ds_set_precision(prec);

논쟁 유형 설명
prec 진짜 The precision value (default 0.0000001)

보고:

N/A

예시:

ds_set_precision(0.0001);

위의 코드는 모든 데이터 구조 함수에 대한 기본 정밀도 설정을 변경합니다.
