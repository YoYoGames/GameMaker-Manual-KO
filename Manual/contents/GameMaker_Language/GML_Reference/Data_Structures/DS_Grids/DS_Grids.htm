DS 그리드

DS 그리드는 기본적으로 2차원 배열 유형입니다. 정수 너비와 높이를 설정하여 그리드를 정의한 다음 그리드 내 "셀"의 총 수(너비와 높이 곱)를 설정하고 이러한 셀은 다른 데이터 값(실수 또는 문자열)을 유지하는 데 사용됩니다.

이 유형의 구조를 사용하면 그리드의 x 및 y 인덱스를 제공하여 그리드의 셀 값을 설정하고 검색할 수 있습니다(x 및 y 축 모두 0에서 시작하여 너비 - 1 및 높이까지 -1). 그러나 특정 그리드 영역에서 값을 설정하고 단일 값을 추가하고 주어진 영역에서 합계, 최대값, 최소값 및 평균값을 검색하여 이 데이터 구조를 사용 가능한 모든 데이터 구조 중에서 가장 다용도로 만들 수도 있습니다.

DS 그리드 데이터 구조에 접근할 때 항상 셀 위치에 정수 값 을 사용하려고 시도해야 하며 정수가 아닌 모든 인덱스는 그렇지 않은 경우 GameMaker 에 의해 바닥이 됩니다. 이것이 필요하지 않은 경우 확인하려는 인덱스를 전달하기 전에 미리 반올림을 수행해야 합니다.

참고 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 필요하지 않을 때 항상 삭제하여 속도를 늦추고 결국 게임을 중단시키는 메모리 누수를 방지해야 합니다.

참고 데이터 구조가 존재하는지 확인해야 하는 경우 다음을 사용할 수 있습니다.ds_exists() 함수.

아래 나열된 특정 기능 외에도 모두 표현식 ( 접근 자 라고 함)을 사용하여 DS 그리드의 내용을 추가하거나 수정할 수 있습니다. 이 접근자는 다음 구문을 사용하는 2D 배열과 유사합니다.

grid_index[# xpos, ypos]

접근 자 의 GML 개요 페이지에서 추가 정보와 예를 찾을 수 있습니다.
