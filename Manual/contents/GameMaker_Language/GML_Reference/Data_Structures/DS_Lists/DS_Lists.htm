DS 목록

DS 목록은 추가될 때 정보를 순차적으로 저장하는 데이터 구조입니다( 배열 과 유사). DS 목록은 끝에 값을 추가하거나 목록 중간에 삽입할 수 있을 뿐만 아니라 순서를 섞거나(값을 무작위로 만들기 위해) 오름차순으로 정렬하는 기능을 제공하는 매우 유연한 데이터 구조입니다. 또는 내림차순.

참고: 유사한 기능을 갖고 사용하기 쉽고 자동으로 가비지 수집되므로 DS 목록보다 배열 을 사용하는 것이 좋습니다.

DS 목록 데이터 구조에 접근할 때 항상 목록 위치에 정수 값 을 사용하려고 시도해야 하며 정수가 아닌 모든 인덱스는 사용하지 않는 경우 GameMaker 에 의해 바닥이 됩니다. 이것이 필요하지 않은 경우 확인하려는 인덱스를 전달하기 전에 미리 반올림을 수행해야 합니다.

참고: 모든 동적 리소스와 마찬가지로 데이터 구조는 메모리를 차지하므로 더 이상 필요하지 않을 때 항상 파괴되어 속도가 느려지고 결국 게임이 중단되는 메모리 누수를 방지해야 합니다.

아래 나열된 특정 기능 외에도 모두 표현식 ( 접근 자 라고 함)을 사용하여 DS 목록의 내용을 추가하거나 수정할 수 있습니다. 이 접근자는 다음 구문을 사용하는 1D 배열과 유사합니다.

list_index[| index]

접근 자 의 GML 개요 페이지에서 추가 정보와 예를 찾을 수 있습니다.

DS 목록을 처리하는 다음 함수가 있습니다.

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

Ds 목록은 DS 맵 및 기타 DS 목록도 보유하는 데 사용할 수 있으므로 목록 내의 위치를 이러한 데이터 구조를 보유하는 것으로 "플래그" 또는 "표시"하는 데 사용되는 몇 가지 특수 기능이 있습니다. 일반적으로 이것은 필요하지 않지만 부모 목록이 파괴될 때 표시된 데이터 구조를 자동으로 가비지 수집(파기)하는 이점이 있으며, 다음과 함께 사용할 때도 필요합니다.json_encode() 기능을 사용하여 목록과 해당 데이터가 올바르게 인코딩되도록 합니다. 이를 위한 기능은 다음과 같습니다.

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
