ds_list_mark_as_list

이 기능은 이전에 생성된 DS 목록 내의 주어진 위치를 다른 DS 목록을 보유하는 것으로 "표시"(또는 "플래그")합니다. 이 기능은 JSON 문자열(다음을 사용하여 생성할 수 있음)을 인코딩할 때 필요합니다.json_encode() ), 그러나 이러한 방식으로 표시된 항목은 상위 DS 목록이 파괴될 때 자동으로 가비지 수집(파기)되기 때문에 데이터 구조를 중첩할 때도 유용할 수 있습니다. 즉, "상위" 목록을 삭제하기 전에 수동으로 목록 내용을 살펴보고 표시된 데이터 구조를 개별적으로 삭제할 필요가 없습니다. 그러나 목록 위치를 개별적으로 삭제하는 경우 포함된 DS 목록은 가비지 수집 되지 않으므로 상위 목록 위치에서 항목을 삭제하기 전에 적절한 DS 목록 삭제 기능을 호출해야 합니다. 또한 함수를 호출하면ds_list_clear() 목록에서 목록으로 플래그가 지정된 모든 항목은 목록이 지워지면 삭제됩니다.

표시된 위치가 목록 내에서 변경되면 "표시"도 함께 이동하므로 목록에서 위치 3을 표시한 경우(예:) 그 아래에 2개의 항목을 더 삽입하여 위치 5로 이동 하지만 여전히 목록으로 표시됩니다.

통사론:

ds_list_mark_as_list(id, pos);

논쟁 유형 설명
id The id of the list to mark.
pos The position within the list to mark.

보고:

DS List ID or -1 (if it fails)

예시:

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

위의 코드는 두 개의 DS 목록을 만든 다음 전역 변수의 다양한 값으로 하나를 채웁니다. 그런 다음 이 목록은 두 번째 목록에 추가되고 나중에 올바르게 인코딩될 수 있도록 위치가 "표시"됩니다.
