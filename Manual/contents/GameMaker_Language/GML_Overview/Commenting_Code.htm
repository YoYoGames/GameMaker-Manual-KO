코드 주석 달기

다른 사람들과 공동 작업을 할 때, 대규모 프로젝트를 다룰 때, 또는 자신의 참조 및 디버깅을 위해서라도 이벤트 코드와 스크립트에 주석을 남기는 것은 매우 중요합니다. GameMaker 에는 현재 편집 중인 개체의 이벤트 목록에서 볼 수 있는 코드 섹션 및 전체 코드 블록에 대한 메모와 설명을 남길 수 있는 다양한 메커니즘이 있습니다. 이런 식으로 자신과 동료에게 메모를 남기거나 특히 까다로운 코드를 설명하거나, 어떤 작업이 수행되는지 또는 앞으로 하고 싶은 작업에 대한 알림을 남길 수 있습니다.

간단한 한 줄 댓글을 남기려면 가장 먼저 할 수 있는 일은 다음을 사용하는 것입니다.// 텍스트 전에. 예를 들어:

//initialize variables
sr = 10;
hp = 100;

여러 줄 주석을 남길 수도 있고, 크레딧을 주기 위해, 디버깅을 위해 코드의 전체 섹션을 생략하거나 미래에 코드를 변경하는 방법에 대한 메모를 남길 수도 있습니다. 그것을 위해 당신은 사용할 수 있습니다/* ... */ 이와 같이:

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

주석 외에도 특수 기능을 사용할 수도 있습니다. #region 코드 접기 에 대한 코드 섹션을 표시하는 태그이며 각 영역에는 주석이 포함될 수 있습니다. 이것이 작동하는 방식은 #region 태그 - 주석 텍스트와 함께 - 코드 섹션 상단에 추가한 다음 해당 #endregion 태그를 섹션 맨 아래에 붙인 다음 코드를 "접을" 수 있습니다. 즉, 편집기에서 한 번에 표시되는 코드 줄 수를 줄일 수 있도록 코드를 숨길 수 있습니다.

실제 코드는 다음과 같습니다.

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

마지막으로 Object Editor 에서 이벤트로 작업할 때 특별한 "description " 이벤트 시작 부분에 주석을 달아 이벤트의 일반적인 내용이 무엇인지 말하세요. 그러면 개체 편집기 이벤트 창에 표시됩니다. 이것은 코드 편집기의 맨 처음 줄에 있어야 하며 다음과 같은 형식입니다.

/// @description <This Is An Event Description>

이제 이벤트 창에 다음과 같이 표시됩니다.

Object Editor Preferences 에서 모든 이벤트에 대해 생성될 기본 주석을 정의할 수 있습니다.

마지막으로, 스크립트 함수 를 작성 중이고 IDE에서 처리하는 방식을 제어하려면(즉, 자동 완성 기능을 사용하거나 스크립트 매개변수를 표시하는 경우) 여기 에 설명된 대로 JSDoc 표기법을 사용하여 주석을 작성해야 합니다.
