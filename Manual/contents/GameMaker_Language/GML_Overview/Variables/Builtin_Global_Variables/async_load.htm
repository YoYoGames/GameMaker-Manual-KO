async_load

이 변수는 범위가 전역적 이며 비동기 이벤트 에서 사용될 때 DS 맵 을 유지하는 데 사용되며 다른 모든 경우에는 -1입니다. DS 맵의 실제 콘텐츠는 이벤트를 트리거하는 데 사용된 함수뿐만 아니라 비동기 이벤트 콜백 의 유형에 따라 달라지므로 가능한 모든 DS 맵 콘텐츠에 대한 자세한 내용은 해당 이벤트의 개별 페이지를 참조하세요.

통사론:

async_load;

보고:

DS Map ID

확장된 예:

sprite =  sprite_add("site.com/path/image.png", 16, true, true, 0, 0);

위의 코드는 외부 URL에서 스프라이트를 로드하는 이벤트에서 호출됩니다. 그러면 Image Loaded Asynchronous Event가 트리거되어 async_load 지도:

if ds_map_find_value(async_load, "id") == sprite
{
    if (ds_map_find_value(async_load, "status") >= 0)
    {
        sprite_index = sprite;
    }
}

위의 코드는 먼저 ID를 확인합니다. async_load map으로 이동한 다음 콜백 상태를 확인합니다. 값이 0보다 크거나 같으면(성공 신호) 콜백의 결과가 인스턴스의 스프라이트 인덱스를 새로 로드된 이미지로 설정하는 데 사용됩니다.
