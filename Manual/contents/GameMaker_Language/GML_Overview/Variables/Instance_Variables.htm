인스턴스 변수

인스턴스 변수는 개체의 인스턴스 내에서 생성되고 해당 인스턴스에 고유한 것으로 간주됩니다. 즉, 동일한 개체의 많은 인스턴스가 동일한 변수를 가질 수 있지만 각 변수는 각 인스턴스에 고유 하므로 다른 값을 가질 수 있습니다. 그러나 인스턴스 변수는 어떻게 생성됩니까? 글쎄, 당신은 이 작은 예와 같이 값을 할당함으로써 새로운 변수를 생성할 수 있습니다(이것을 변수 선언이라고 합니다):

potions = 12;
life = 100;
name = "Jock MacSweeney";
strength = 5.5;
armour = -2;

보시다시피 이름을 지정한 다음 값(값은 모든 데이터 유형 이 될 수 있음)을 지정하여 해당 변수를 설정하고 코딩 중인 객체의 인스턴스 내에서 사용할 준비가 되도록 해야 합니다(값은 함수의 반환 값이나 다른 변수 간의 연산 결과 등...). 이러한 변수는 인스턴스 내에서 다양한 방식으로 사용 및 수정할 수 있습니다. 예를 들어 이 코드는 충돌 이벤트에 있을 수 있으며 변수 "life ":

life -= 5 + armour;

만약에 "life "는 100에 있으며 이제 97(100 - (5 + -2) = 97)의 값 을 갖게 됩니다. 이제 이것은 간단한 예이며 "armour " 실제 값이 -2인 경우 해당 값이 여러 위치에 하드 코딩 되어 있고 변경하기로 결정했다면 어떻게 됩니까? 모든 코드를 살펴보고 모든 -2를 새 값으로 변경해야 합니다., 이는 시간이 많이 걸리고 오류가 발생하기 쉽습니다! 하지만 변수를 사용하는 경우 새 값을 다시 할당하기만 하면 코드가 자동으로 새 값을 사용하므로 훨씬 더 유연하고 쉽게 작업을 수행할 수 있습니다. 문제가 있는 경우 수정합니다. 값이 변경되지 않더라도 "변수"라는 변수를 기억하는 것이 훨씬 쉽습니다.life "는 단순히 숫자를 보는 것이 아니라 의미합니다.

GameMaker 에는 "내장된" 인스턴스 변수 모음도 있으므로 고유한 인스턴스 변수 중 하나의 이름을 동일하게 지정하거나 동일한 이름을 가진 고유한 전역 변수를 원할 수 있으며 왜 이러한 변수가 표시되는지 궁금할 수 있으므로 이를 알고 있어야 합니다. 오류. 그러나 코드 편집기에서 다른 색상으로 표시되고 자동 완성으로도 나타나며 코드 편집기 하단의 막대에 표시되므로 쉽게 식별할 수 있습니다.
