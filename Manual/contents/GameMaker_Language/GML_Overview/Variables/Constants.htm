상수

상수는 게임 시작 시 한 번 설정되고 절대 변경되지 않는 변수 유형입니다. 실제로 상수 값 은 선언된 후에 변경할 수 없습니다. 따라서 게임 전체에서 특수 데이터를 식별하는 데 사용되는 값을 유지하는 데 이상적입니다.

GameMaker 언어에는 두 가지 유형의 사용자 정의 상수가 있습니다: 매크로 및 열거, 둘 다 아래에 설명되어 있습니다. 또한 데이터 유형 (예: 문자열 또는 숫자 3)에 관계없이 항상 동일한 값은 상수로 분류됩니다.

참고 GameMaker 언어에는 특정 항목을 식별하는 데 사용되는 여러 내장 상수 값도 있습니다. 이는 GML 참조 섹션에서 이를 필요로 하는 런타임 기능에 대한 해당 페이지에 설명되어 있습니다.

매크로

정확히 변수는 아니지만 매크로는 사용 방식에서 매크로와 유사합니다. 즉, 코드 전체에서 하드 코딩된 값을 대체하는 데 사용할 수 있는 명명된 값입니다. 기본적으로 매크로는 표현식을 보유하는 명명된 변수입니다. 스크립트 편집기 를 사용하여 고유한 매크로를 정의한 다음 마치 일반 변수인 것처럼 코드와 작업에서 매크로를 사용할 수 있지만 한 가지 차이점 은 게임에서 변경할 수 없다는 점 입니다.

매크로의 구문 구조는 다음과 같습니다.

#macro <variable> <expression>

예를 들어 다음 매크로 "total_weapons ":

#macro total_weapons 10

매크로 구문...

매크로를 정의하려면 위에 표시된 구문을 올바르게 사용해야 합니다. 등호를 사용하거나(변수에서처럼) 끝에 세미콜론을 넣을 수 없습니다(일반 명령문에서처럼). 그렇게 하면 매크로 정의가 유효하지 않게 됩니다.

예를 들어, 다음은 매크로를 정의하는 올바른 방법이 아닙니다.

\>#macro total_weapons = 10;

등호와 콜론을 제거하면 유효한 매크로 정의가 됩니다.

그런 다음 코드에서 다음과 같이 사용할 수 있습니다.

pos++;

if (pos >= total_weapons)
{
    pos = 0;
}

상수 값을 변경할 수 없으므로 다음과 같은 코드로 인해 게임이 중단됩니다.

total_weapons = 11;

코드나 스크립트의 어느 곳에서나 매크로를 정의할 수 있으며 마치 처음부터 있었던 것처럼 미리 컴파일 되어 게임에 포함되지만 전용 스크립트 자산을 만들고 모든 매크로를 다음에서 정의하는 것이 좋습니다. 거기. 나중에 구성하고 디버그하는 것이 더 쉬울 것입니다!

런타임에 매크로 값을 변경해야 하는 경우 매크로를 런타임 function 로 설정하지 않는 한 게임 중에 코드에서 변경할 수 있으므로 이를 전역 변수 로 만들어야 합니다. 매크로를 함수로 설정하면 매크로를 사용할 때마다 이 함수가 호출됩니다. 예를 들어:

#macro col make_colour_hsv(irandom(255), 255, 255)

그런 다음 이 매크로를 다음과 같이 호출합니다.

image_blend = col;

이 코드를 사용하면 매크로를 사용할 때마다 이미지가 다른 색상으로 혼합됩니다. 다음을 사용하여 여러 줄에 매크로를 분할할 수도 있습니다. : 줄 바꿈 위치를 나타내는 문자. 예를 들면 다음과 같습니다.

#macro hello show_debug_message("Hello" + \
string(player_name) + \
", how are you today?");

이와 같이 매크로를 분할해도 최종 매크로를 사용할 때 결과에 영향을 미치지 않으며 단순히 코드 줄이 더 긴 매크로에서 여러 줄 텍스트를 지원하기 위한 것입니다.

매크로의 매우 중요한 기능 중 하나는 특정 구성 (구성)과 함께 사용하도록 정의할 수 있다는 것입니다. 즉, 동일한 매크로 이름을 가질 수 있지만 현재 선택한 구성에 따라 다른 값을 지정할 수 있습니다. 예를 들어 Android 광고에 대한 구성과 iOS 광고에 대한 구성이 있다고 가정하면 필수 앱 ID 값을 보유하는 단일 매크로를 정의할 수 있습니다.

#macro ad_id "";
#macro Android:ad_id "com.yoyogames.googlegame"
#macro iOS:ad_id "com.yoyogames.appstoregame"

보시다시피 구성 이름을 먼저 지정한 다음 콜론을 지정합니다. : 매크로 이름과 값. 콜론 사이에는 공백이 없어야 합니다. red = 0 구성 이름이나 매크로 이름이 없으면 오류가 발생합니다.

열거

열거형은 "열거자"이며 본질적으로 상수 값 목록을 사용하여 제한된 데이터 유형을 생성할 수 있도록 하며 구조는 다음과 같습니다.

enum <variable>
{
    <constant> [= <value>],
    <constant> [= <value>],
    // etc...
}

다음 예에서는 무지개 색상에 대한 열거형을 만들고 다양한 상수와 기본값을 할당합니다.

enum rainbowcolours
{
    red,
    orange,
    yellow,
    green,
    blue,
    indigo,
    violet
}

열거형 항목은 정수 또는 정수 로 평가되는 이전 열거형이 있는 표현식 s 일 수 있으며 기본적으로 0부터 번호가 지정되므로 위에 제공된 예제는 기본적으로 다음과 같이 지정됩니다. orange = 1, yellow = 2, ".", 등...

생성 시 열거형 변수에 값을 할당할 수도 있습니다.

enum enum_test
{
    val = 10;
}

enum rainbowcolours
{
    red = 5,
    orange = 5 * 2,
    yellow = 15,
    green = 20,
    blue = 25,
    indigo = 30,
    violet = 35 * enum_test.val
}

위의 예에서 다른 열거형을 사용하여 "보라색"에 대한 식을 만듭니다. 이것은 참조되는 열거형이 표현식에서 사용하는 열거형 보다 먼저 생성된 경우에만 작동하지만 프로젝트가 Compiling 중일 때 열거형 값을 상수로 평가할 수 있어야 하기 때문에 변수나 함수에 대해서는 작동하지 않습니다. 또한 모든 열거형 값은 정수 값으로 평가되며 직접 만들 때 열거형이 작동하려면 정수 값만 허용 된다는 점에 유의해야 합니다. 이 값은 음수 값을 포함하여 부동 소수점 배정밀도 숫자가 나타낼 수 있는 모든 정수일 수 있습니다.

지정된 열거형 유형 내의 값에 나중에 액세스하려면 포인트를 사용할 수 있습니다.rainbowcolours 다음과 같은 방법:

variable = <enum_name>.<enum_variable>;

예를 들어 " colour_value " 위 코드에서 생성한 enum:

colour_value = rainbowcolours.green * rainbowcolours.red;

그만큼 is_real() 변수는 이제 값 100(20 * 5)을 유지합니다.

생성된 후에는 매크로를 수정할 수 없는 것과 마찬가지로 생성된 열거형 상수의 값을 수정할 수 없습니다.

참고: 열거형 값은 int64로 저장되므로 실행 false 그들에게 돌아올 것이다pointer_null.
