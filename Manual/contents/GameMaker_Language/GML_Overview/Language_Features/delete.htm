delete

그만큼 delete operator 는 struct 를 역참조하는 데 사용되며 구문은 다음과 같습니다.

delete <variable>;

이것이 의미하는 바는 구조체를 보유하는 변수와 함께 삭제 연산자를 호출하면 주어진 변수에 저장된 구조체에 대한 특정 참조 가 제거된다는 것입니다(참조는 단순히 저장하고 있는 메모리 영역을 가리키는 값입니다. 구조체와 그 내용). 이 참조가 게임의 구조체에 대한 마지막 참조인 경우 가비지 수집기 는 다음 반복 에서 일반적으로 다음 단계의 맨 처음에 메모리에서 구조체를 제거합니다.

참고:delete 변수와 함께만 사용할 수 있으며 표현식은 사용할 수 없습니다.

기본적으로 코드에 더 이상 참조가 없는 구조체는 사용 후 단계에서 자동으로 가비지 수집되지만 다음 시작 부분에서 가비지 수집기가 메모리에서 제거하도록 이 연산자를 사용하여 명시적으로 플래그를 지정하는 것이 좋습니다. 단계, 메모리를 더 빠르고 효율적으로 해제합니다.

다음 예제는 생성 중인 구조체를 보여줍니다(예: 인스턴스의 이벤트 생성):

mystruct =
{
    a : 0,
    b : 100,
    c : "Hello World"
}

이 구조체는 다음과 같이 Clean Up 이벤트에서 최종적으로 가비지 수집 플래그가 지정되기 전에 평소와 같이 인스턴스 이벤트에서 사용됩니다.

delete mystruct;
