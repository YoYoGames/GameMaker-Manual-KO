기본 코드 구조

코드는 블록 으로 작성되며 일반적인 코드 블록은 명령문 s 라고 하는 명령 세트로 구성되며, 이 명령은 GameMaker 에서 해석되어 게임 내에서 어떤 일이 일어나도록 하는 데 사용됩니다. 그 "무언가"는 2와 2를 더하여 4를 만드는 것처럼 간단할 수도 있고, 적의 건강이 특정 값 아래로 떨어지면 도망가게 만드는 것처럼 복잡할 수도 있습니다. 프로그램의 실제 구조는 사용하는 기능에 따라 크게 다를 수 있지만 기본적으로 분류하면 다음과 같습니다.

<statement>;
<statement>;
...

문은 ';'로 구분해야 합니다. 변수 선언의 오류를 방지하고 코드를 깨끗하고 깔끔하게 유지하기 위한 기호이며 변수 선언, 표현식 및 특정 함수 호출로 구성될 수 있습니다. 중괄호를 사용하여 문을 블록으로 함께 "그룹화"할 수도 있습니다. {} 다음 조건부 예제와 같이 함께 실행되도록 합니다.

if (<conditional>)
{
    <statement>;
    <statement>;
    ...
}

참고 GameMaker 언어는 begin 그리고 end  중괄호 대신 {}, 비록 이것이 일반적으로 가장 일반적인 방법은 아니지만:

if (<conditional>)
begin
    <statement>;
    <statement>;
    ...
end

다음은 코드 블록이 어떻게 보이는지 더 시각적으로 표현한 것입니다. 이번에는 GameMaker 스크립트 편집기 에서 스크립트 로 생성되었습니다.

다양한 유형의 명령문, 표현식, 조건부 및 함수가 있으며, 이들 모두는 매뉴얼의 후속 섹션에서 자세히 설명합니다.

참고 프로그래밍이 처음이라면 계속하기 전에 빠른 시작 안내서 를 확인하는 것이 좋습니다.
