런타임 함수

함수의 일반적인 정의는 다음과 같습니다.

함수에는 입력과 출력이 있으며 출력은 입력과 관련됩니다.

GameMaker 에서 이것은 두 가지 다른 것으로 해석되지만 둘 다 같은 방식으로 작동합니다.

  • 런타임 기능 - 런타임 기능은 GML( GameMaker Language )의 일부로 제공되는 기능입니다. 이것들은 언어에 내장되어 있으며 게임에서 일을 수행하는 데 사용할 수 있으며 많은 수의 GML 런타임 기능을 사용할 수 있습니다. 이 기능은 모두 매뉴얼의 언어 참조 섹션에 설명되어 있습니다.
  • 스크립트 함수 및 메서드 - 스크립트 함수 및 메서드 변수 는 GameMaker Language (GML)의 기본 빌딩 블록을 사용하여 직접 생성한 함수이며 코드의 일부로 런타임 함수를 포함할 수 있습니다.

위의 두 경우 모두에서 함수는 함수 이름 의 형식을 가지며 대괄호 사이에 입력 인수 s 가 뒤따릅니다. () 쉼표로 구분합니다(함수에 입력 인수가 없으면 대괄호만 사용됨). 이 페이지는 런타임 기능 (예: GameMaker 언어를 형성하기 위해 GameMaker에 내장된 기능)에 집중하지만 모든 기능 유형에 대해 아래에 표시된 일반 규칙을 따릅니다.

다음은 함수 구조의 개요입니다.

<function>(<arg0>, <arg1> ,... <argN>);

일부 함수는 값을 반환할 수 있고 표현식 s 에서 사용할 수 있지만 다른 함수는 다음 두 런타임 함수 예제와 같이 단순히 명령을 실행합니다.

// Destroy the calling instance
// This requires no arguments and returns nothing
instance_destroy();

// Get the distance from the current instance position to the mouse position
// This takes four arguments and returns a real value
dist = point_distance(x, y, mouse_x, mouse_y);

참고 설명서에 함수에 대한 반환 값으로 "해당 없음"이 있는 경우 해당 기능이 작업의 일부로 값을 반환하지 않는다는 의미입니다. 이러한 함수에서 반환 값을 확인하려고 하면 값을 얻게 됩니다. undefined.

때때로 함수의 반환 값은 할당 에서 사용하려는 값일 수 있지만 할당의 왼쪽으로 직접 함수를 사용하는 것은 불가능하다는 점에 유의해야 합니다. 예를 들어 인스턴스의 속도를 설정하는 다음 코드는 오류를 제공합니다.

instance_nearest(x, y, obj).speed = 0;

해당 코드 예제에서 표현식의 반환 값은 정수(가장 가까운 인스턴스의 고유 ID 값)이므로 이러한 방식으로 사용하고 필요한 인스턴스를 올바르게 처리하려면 대괄호로 묶어야 합니다( 기타에서 변수 주소 지정 참조). 자세한 내용은 인스턴스 ). 위의 코드는 다음과 같이 올바르게 작성됩니다.

(instance_nearest(x, y, obj)).speed = 0;

//or more correctly still

var inst = instance_nearest(x, y, obj);
inst.speed = 0;
