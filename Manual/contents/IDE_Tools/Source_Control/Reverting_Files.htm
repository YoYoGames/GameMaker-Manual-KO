파일 되돌리기

이전 섹션 에서 저장소를 만들고 커밋 하는 방법을 보았지만 실수를 하여 이전 커밋으로 "롤백"하려는 경우 어떻게 됩니까? 고의적인 오류를 범하고 어떻게 처리할 수 있는지 봅시다...

우선 새 리소스를 만듭니다. 이 예에서는 스크립트 리소스를 만듭니다. 그러면 소스 제어에 관한 한 아직 존재하지 않기 때문에 처음에는 상태 아이콘이 표시되지 않습니다. 스크립트 리소스에서 다음과 같은 간단한 코드 줄을 추가합니다.

show_debug_message("Hello World");

이제 코드 편집기를 닫으면 스크립트가 저장되고 수정된 파일 아이콘이 자산 브라우저 에서 옆에 나타납니다. 이제 커밋 창을 열면 새 리소스가 자동으로 추가됨에 따라 4가지 단계적 변경 사항이 있음을 알 수 있습니다.

단계적 변경 사항은 다음과 같습니다.

  • 그만큼 .yyp 프로젝트 파일
  • 파일 보기
  • GML 스크립트 자체
  • 그만큼 .yy 그것을 참조하는 파일

이전에 설명한 대로 커밋 메시지를 입력한 다음 커밋 버튼을 클릭해야 합니다. 이제 스크립트로 돌아가서 코드 줄을 (예를 들어) 다음과 같이 변경합니다.

show_debug_message("Hello World, how are you?");

코드 편집기를 닫으면 다시 빨간색 아이콘 이 스크립트 옆에 표시되지만 코드에 실수가 있음을 깨달았고 수정하기 위해 이전 버전으로 롤백하고 싶습니다(분명히 스크립트를 열고 쉽게 수정할 수 있습니다. 매우 간단하지만 일반적으로 실현 가능한 접근 방식이 아닌 대규모 프로젝트에서). 우리가 해야 할 일은 변경 사항을 되돌리는 것입니다.

변경 사항을 되돌리려면 먼저 커밋 창을 다시 열어야 하므로 소스 제어 > 변경 사항 커밋 으로 한 번 더 이동합니다. 변경된 파일은 다시 커밋할 준비가 되지만 이것이 우리가 원하는 것이 아니므로 Unstage All 버튼을 클릭하여 스테이징 영역에서 파일을 가져옵니다.

이제 마우스 오른쪽 버튼을 클릭하면 변경한 파일에서 Revert Path 옵션이 있는 컨텍스트 메뉴가 표시됩니다.

이 옵션을 선택하면 파일이 이전 상태로 되돌아갑니다.

중요한! 되돌리면 파일 검사기가 디스크의 변경 사항을 감지하기 때문에 현재 프로젝트를 다시 로드하거나 저장할 것인지 묻는 메시지가 표시됩니다.
저장하면 되돌린 파일을 다시 로드하는 대신 메모리를 저장하므로 저장하지 말고 다시 로드를 눌러야 합니다.

이것은 무언가를 편집하고 이미 커밋을 수행하지 않고 다시 변경하려는 경우에 모두 유용하지만 하나 이상의 변경 사항을 커밋하고 이전 커밋으로 롤백하려는 경우는 어떻습니까? 자, 스크립트를 다시 편집하여 다음과 같이 말합시다.

show_debug_message("This is a bad idea");

이제 소스 제어 하에 가져오기 위해 이전과 같이 커밋합니다. 따라서 이것은 잘못된 생각이었고 롤백하고 싶습니다. 즉, 시작하려면 소스 제어 > 기록 보기 창을 열어야 합니다. 여기에서 초기 커밋, 스크립트 커밋, 그리고 "나쁜 아이디어"로 편집한 스크립트에 대한 커밋으로 시작하여 이 창에서 모든 이전 커밋 목록을 볼 수 있습니다.

이제 두 가지 선택이 있습니다. 우리는 전체 개정을 되돌릴 수 있습니다(변경된 모든 파일을 선택한 커밋의 파일로 되돌립니다). 또는 경로를 되돌리도록 선택할 수 있습니다. 이는 단순히 단일 파일을 되돌리는 것입니다. 다음과 같이 시작하는 첫 번째 옵션을 수행해 보겠습니다.

  • 마우스 오른쪽 버튼으로 클릭 커밋 기록 창의 두 번째 커밋("Hello World" 커밋).
  • 이 개정으로 되돌리기를 선택한 다음 경고 메시지에서 다시 로드를 선택합니다.

이제 스크립트에 빨간색 상태 아이콘이 있는 것을 볼 수 있습니다., 열면 다음이 표시됩니다.

그런 다음 프로젝트에 대한 커밋을 수행할 수 있으며 다시 원래 위치로 돌아갑니다. 그러나 다음을 수행하여 파일에 대한 해당 작업을 실행 취소하고 "나쁜 아이디어" 커밋을 다시 가져올 수도 있습니다.

  • 변경 사항 커밋 창을 엽니다.
  • "Hello World" 코드가 포함된 스크립트를 언스테이징합니다.
  • 마우스 오른쪽 버튼으로 클릭 및 경로 되돌리기.

이제 우리는 다시 프로젝트에 잘못된 파일을 가지고 있습니다! 글쎄, 우리는 이 기회에 모든 것을 이전 버전으로 되돌리는 대신 해당 파일 경로만 되돌릴 수 있습니다. 이를 위해 다음을 수행해야 합니다.

  • 기록 보기 창을 엽니다.
  • 두 번째 커밋("Hello World" 커밋)을 클릭합니다.
  • 마우스 오른쪽 버튼으로 클릭 파일 창의 스크립트 GML에서 경로 되돌리기 를 선택합니다.

"Hello World" 코드로 돌아가야 하고 수정된 파일로 소스 제어를 다시 업데이트하기 위해 새로운 커밋을 수행할 수 있습니다.
