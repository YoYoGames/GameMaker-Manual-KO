리포지토리 복제

프로젝트가 외부 저장소에 저장되면 변경 사항을 푸시하고 새 변경 사항을 가져와야 한다는 것을 기억해야 합니다. 그 이유는 커밋할 때 변경 사항을 로컬에 저장하여 인터넷 연결 없이 작업하고 준비가 되면 동기화할 수 있으며 동기화를 위해 변경 사항을 원격 리포지토리에 푸시하기 때문입니다. 변경 사항 가져오기는 다른 사람이 원격 저장소에서 수행한 모든 변경 사항을 로컬 저장소로 가져오려는 것을 의미합니다. 그러나 이는 사람들이 동일한 파일을 편집할 수 있는 경우가 있을 수 있음을 의미하며 결과적으로 병합 충돌 이 발생합니다. 이러한 충돌을 처리하는 방법을 알아야 하므로 지금 의도적으로 충돌을 만든 다음 해결 방법을 보여드리겠습니다.

참고: 일반적으로 커밋이 직접 기록되므로 충돌 방지 기능이 거의 없기 때문에 로컬 프로젝트 리포지토리(Dropbox와 같은 파일 동기화 서비스를 통해서도)를 공유하는 것은 권장되지 않습니다.

충돌을 일으키려면 먼저 마스터 리포지토리와 복사본에 대한 2개의 로컬 리포지토리가 필요합니다. 이것을 생성하려면 다음 단계에 따라 리포지토리를 복제해야 합니다.

  • 소스 제어 > 복제 리포지토리 로 이동합니다.
  • URL 필드에 위의 파일 되돌리기 가이드에서 설정한 프로젝트의 경로를 입력합니다(기본적으로 <DRIVE>:\Users\<Username>\Documents\GameMaker\<ProjectName>).
  • 그런 다음 복제할 경로를 다음으로 설정해야 합니다.
    "확인" 버튼을 클릭하면 "비 베어" 리포지토리를 복제하려는 시도에 대해 다음과 같은 경고가 표시됩니다.
    여기에서 "확인"을 다시 클릭할 수 있습니다.
  • 이제 "베어" 프록시 리포지토리(예: Dropbox에 안전하게 저장할 수 있는 리포지토리)를 만들기 위한 디렉터리 탐색기가 표시됩니다. 이 프록시 리포지토리를 생성할 위치로 이동하고 사용할 폴더를 선택(또는 생성)합니다.

일단 완료하면 GameMaker 는 원본 프로젝트를 프록시로 푸시하고 Clone Repository 창에서 설정한 경로로 새 프로젝트를 가져온 다음 자동으로 파일 브라우저를 열어 저장된 프로젝트를 열 수 있습니다. 이제 소스 제어 > 기록 보기 로 이동하면 원본 커밋과 새로운 "Push For Clone" 커밋이 표시됩니다.

이제 리포지토리를 복제했으며 프로젝트의 로컬 버전에서 계속 작업할 수 있습니다.
