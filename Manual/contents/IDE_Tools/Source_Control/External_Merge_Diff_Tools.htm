병합 도구

다른 사람들과 함께 프로젝트를 진행하면 여러 사람이 동일한 파일을 편집하고 대부분의 소스 제어 시스템이 충돌하는 파일을 병합하는 데 좋은 시도를 할 수 있기 때문에 항상 충돌이 있을 수 있습니다. 같은 장소에서 무엇을 해야 하는지 알기 위해서는 약간의 도움이 필요합니다. 여기에서 병합 도구 가 변경 사항을 표시하고 사용자가 결정할 수 있도록 합니다.

각 병합 도구가 다르기 때문에 파일을 지정하는 데 사용할 수 있는 4개의 매크로가 있습니다. 이 매크로는 병합 도구 선택 버튼을 클릭하여 충돌 창에서 설정할 수 있습니다.:

해당 버튼을 클릭하면 Git 플러그인에서 기본 설정 창이 열립니다.

여기에서 선택한 병합 도구에 대한 경로를 제공한 다음 수행할 작업을 결정할 때 사용할 병합 도구 옵션 매크로(또는 매크로)를 제공하며 다음 옵션을 사용할 수 있습니다.

  • ${scm_mine}: 버전 경로
  • ${scm_theirs}: 버전 경로
  • ${scm_base}: 충돌하는 변경 사항이 발생하기 전 버전의 경로
  • ${scm_merged}: 병합 도구가 저장해야 하는 경로

설정에 도움이 되도록 특정 병합 도구에 대한 경로를 제공하면 GameMaker 는 선택한 도구에 따라 시작하기 위해 몇 가지 기본값을 채웁니다.

  • KDiff3: "${scm_theirs} "${scm_mine}" -o "${scm_merged}"
  • 융합: "${scm_mine}" "${scm_merged}" "${scm_theirs}"
  • 메모장++: ${scm_merged}
  • 거북이 병합: /base:"${scm_base}" /theirs:"${scm_theirs}" /mine:"${scm_mine}" /merged:"${scm_merged}"
  • DiffMerge: "${scm_mine}" "${scm_merged}" "${scm_theirs}"

병합 도구가 없거나 사용하지 않으려는 경우 IDE에서 스크립트를 열고 거기에서 수정을 시도할 수 있습니다.

따라서 충돌 섹션에 설명된 충돌 예제로 돌아가서... 마스터에서 가져오기를 수행했으며 파일 중 하나가 충돌했습니다. 그러면 소스 제어 충돌 창이 열립니다. 여기에서 충돌하는 스크립트를 클릭하고 병합 도구를 구성했으므로 이제 병합 을 클릭합니다. 이제 병합을 클릭하면 충돌을 처리할 수 있는 도구가 열리고 제공된 매크로를 사용하여 필요한 변경을 수행할 수 있습니다. 병합 도구를 완료하고 닫으면 GameMaker IDE에 다음 확인 창이 표시됩니다.

병합 도구를 사용하여 직접 수행한 것처럼 이 프롬프트에 "예"라고 대답하기만 하면 병합 작업의 끝을 나타내기 위해 이 변경 사항을 커밋해야 합니다( 소스 제어 > 변경 사항 커밋 ). 이제 커밋 창에 병합이 진행 중이고 일부 준비되지 않은 파일이 표시됩니다.

이러한 준비되지 않은 파일은 다음이 추가된 스크립트 파일입니다. .base, .ours 그리고 .theirs. 이 모든 것을 안전하게 삭제할 수 있으므로 클릭하십시오. 상단에 그리고 다음 + 그들을 모두 선택하려면 하단에. 다음으로 선택한 파일을 마우스 오른쪽 버튼으로 클릭하고 경로 삭제 를 선택하여 제거합니다. 우리를 위해 미리 채워진 커밋 메시지도 있는데, 이것은 병합이었고 충돌이 있었다는 것입니다(여기에 추가하거나 원하는 대로 편집할 수 있음).

지금 커밋 을 클릭할 수 있습니다.

다른 파일에 문제가 있을 수 있기 때문에 원격 또는 공유 리포지토리를 사용할 때의 워크플로는 다음과 같아야 합니다.

Make Changes > Save > Commit > Pull > Merge > Push

차이 도구

또한 저장소에 커밋할 때 사용할 Diff 도구 를 설정해야 합니다. 이것은 병합 도구와 마찬가지로 Preferences 에서 설정됩니다.

Diff 도구는 Merge 도구와 거의 동일한 방식으로 구성되지만 ${scm_base} 그리고 ${scm_theirs} 옵션 - 이 경우, ${scm_base} 수정되지 않은 파일을 나타내는 반면 ${scm_theirs} 파일의 현재 상태를 나타냅니다. 도구 옵션은 병합 도구와 마찬가지로 Meld, KDiff3 및 TortoiseMerge 에 대한 기본 옵션으로 미리 채워져 있어야 합니다. GameMaker 는 단일 파일 diff 패치의 출력을 지원하지 않으므로 Notepad++ 에 대한 기본값이 없습니다.

Diff 도구를 사용하여 현재 버전과 이전 버전 간의 변경 사항을 보려면 Commit 창을 열고 파일을 두 번 클릭하기만 하면 됩니다. 도구가 올바르게 설정되면 도구가 실행되고 파일 간의 변경 사항을 볼 수 있습니다.
