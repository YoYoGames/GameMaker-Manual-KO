갈등

이 섹션에서는 복제된 로컬 리포지토리 와 마스터 리포지토리 간의 충돌을 해결하는 방법을 살펴보겠습니다. 이제 로컬 저장소를 사용하고 있다고 가정하면 파일을 편집한 다음 일종의 충돌이 발생하므로 사용하고 있는 테스트 스크립트를 열고 "Hello World" 메시지를 다음과 같이 변경해야 합니다.

show_debug_message("Hello World, How Are You?");

이제 프로젝트를 저장하고 로컬 저장소에 커밋해야 합니다. 이제 원격 마스터 리포지토리에 연결되어 있으므로(같은 디스크에 복제한 경우에도) 변경 사항을 마스터에 적용하려면 푸시를 수행해야 합니다. 이를 위해 Source Control > Push Changes 로 이동하여 Source Control Output 탭 에서 상태 업데이트를 제공합니다.

이제 원본 프로젝트(복제된 프로젝트가 아님)를 열어야 하므로 디스크에서 저장한 위치를 찾아보거나 현재 프로젝트 바로 아래에 있어야 하므로 파일 메뉴 에서 최근 프로젝트 를 확인하고 로드합니다. 게임메이커 로. 올바른 프로젝트가 있는지 확인하는 쉬운 방법은 단순히 스크립트를 확인하고 메시지가 "Hello World"이면 올바른 것입니다. 또는 View History 창을 열고 그런 식으로 확인할 수 있습니다.

이제 스크립트를 다시 편집해야 하므로 다음과 같이 읽힙니다.

show_debug_message("This will cause a conflict.");

이것은 디스크에 저장할 수 있고 변경 사항이 커밋되지만 이러한 변경 사항을 푸시하려고 하면 다음 메시지가 표시됩니다.

당신이 아직 가지고 있지 않은 변경 사항을 누군가가 이미 푸시했습니다! 즉, 저장소에서 pull을 수행하고 문제가 정확히 무엇인지 확인해야 하므로 Source Control > Pull Changes 로 이동합니다. 이렇게 하면 두 개의 창이 표시됩니다. 첫 번째는 변경된 파일에 대한 경고(이전 작업과 마찬가지로 여기에서 다시 로드 를 선택하고 저장 을 선택 하지 않으려 는 경우)이고 다른 하나는 충돌 창입니다.

이 창은 왼쪽의 목록에 충돌하는 파일을 표시하며 마우스 왼쪽 버튼을 사용하여 해당 목록에서 파일을 선택할 수 있습니다. (또는 다음을 사용하여 여러 파일 선택 / 또는 와 함께 ). 오른쪽에 있는 버튼에서 사용할 수 있는 충돌 파일을 처리하기 위한 네 가지 옵션이 있습니다.

  • 사용자 사용 - 원격 리포지토리의 변경 사항을 덮어씁니다.
  • Use Mine - 원격 저장소의 변경 사항을 방금 만든 변경 사항으로 무시합니다.
  • 병합 - 충돌을 처리하기 위해 외부 병합 도구를 열려고 시도합니다.
  • 병합 도구 선택 - 병합 도구를 구성할 수 있습니다(자세한 내용은 외부 병합/차이 도구 섹션 참조).

이제 버튼을 사용하여 충돌을 해결할 수 있지만, 그 전에 충돌 파일을 열어 문제가 정확히 무엇인지 확인하겠습니다. 충돌하는 스크립트를 열면 이제 다음과 같이 표시됩니다.

이것은 "This Will Cause A Conflict"가 HEAD에 있고(이것이 우리가 로컬 저장소로 푸시한 것입니다) "Hello World, How Are You?" 원격 저장소인 마스터에서 가져오고 있습니다. 이제 충돌 창으로 돌아가서 사용 을 클릭하여 변경된 파일을 가져오고 현재 HEAD 파일을 덮어쓸 수 있습니다. 스크립트는 이제 변경된 것으로 플래그가 지정되며 계속하기 전에 새 커밋과 푸시를 수행해야 합니다.

GameMaker IDE에서 실제로 이 문제를 해결할 수 있습니다. 1, 2, 3, 5행을 제거하면 show_debug_message("Hello World, How Are You?") Git은 여러분이 자신의 파일로 무엇을 하고 있는지 알고 있다고 가정하므로 충돌을 해결합니다. 그러나 이것은 오류가 발생하기 쉬우므로 병합 도구를 설치한 다음 수동으로 병합을 시도하는 대신 구성하고 사용하는 것이 좋습니다.
