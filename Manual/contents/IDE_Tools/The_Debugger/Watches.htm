시계

디버거의 전체 아이디어는 게임의 모든 세부 사항을 "관찰"할 수 있도록 하는 것입니다. 이것은 기본적으로 IDE 하단에 도킹된 출력 창 에서 열리는 "감시 창"을 사용하여 수행됩니다. 각각의 다른 탭을 클릭하여 포함된 시계를 볼 수 있으며, 그 중 일부는 실시간으로 업데이트할 수 있습니다(실시간 디버깅 시 디버거 도구 모음에서 활성화됨) 반면 다른 것들은 디버거가 게임을 일시 중지해야 합니다.

창에서 변수 를 볼 때 여러 데이터 유형 중 하나일 수 있습니다. 그러나 GameMaker 가 특정 항목을 저장하는 방식으로 인해 디버거는 항상 변수가 무엇을 보유하는지 정확히 알지 못합니다. 이는 데이터 구조 또는 인스턴스의 내부 ID 값이 정수 이므로 디버거가 변수에 저장되는 정수 값이 데이터 구조 또는 인스턴스에 대한 것인지 아니면 단순히 정수 값인지 항상 알지 못하기 때문입니다. 따라서 마우스 오른쪽 버튼을 클릭하면 모든 변수 값에 대해 이 변수가 될 수 있는 사용 가능한 데이터 유형을 나열하는 팝업 창을 가져옵니다.

문자열과 배열은 이 옵션을 허용하지 않으며 데이터 구조가 인스턴스 변수에 저장되더라도 해당 ID 번호는 전역이므로 4개의 인스턴스가 있고 각각이 ds_list를 생성하는 경우(예를 들어), 각 목록에는 0에서 3까지의 ID가 있으며 모두 0은 아닙니다.

데이터 유형을 선택하면 변수는 작은 버튼을 클릭하여 구조의 항목을 표시합니다. 이름 옆에. 이렇게 하면 구조 내용이 확장되고 단순히 두 번 클릭하여 표시되는 변수의 일부 값을 변경할 수도 있습니다. 가치에 새로운 가치를 부여합니다.

참고: 이러한 방식으로 배열이나 데이터 구조를 재할당할 수는 없지만 그 내용 중 일부는 변경할 수 있습니다(맵의 "키" 값이나 우선 순위 대기열의 우선 순위 값을 변경할 수 없음).

아래에서 각 감시 창의 목록과 그 용도 및 사용 방법에 대한 설명을 찾을 수 있습니다.

현지인

Locals 창은 실행 중인 현재 이벤트(또는 스크립트)의 이름, 현재 스크립트에 대한 가상 머신(VM) 코드의 오프셋을 포함하는 현재 단계 속성을 보여줍니다(약어 "PC"는 "프로그램 카운터"를 나타냄) ), 코드 블록을 실행하는 "self" 인스턴스 및 이벤트의 "other" 인스턴스. 이 마지막 부분은 이벤트가 충돌 이벤트이거나 "with " 코드에서 사용됩니다. 인스턴스가 충돌하는 경우 "other"는 충돌 시 다른 인스턴스(및 해당 변수)를 표시하지만 충돌이 발생한 경우 with "Self" 인스턴스는 코드를 실행하는 인스턴스가 되고 "Other"는 실제로 코드를 포함하는 인스턴스가 됩니다. with. 다른 모든 경우에 "기타"는 "자신"과 동일한 데이터를 단순히 표시합니다.

"Self" 및 "Other" 인스턴스는 다음을 클릭하여 확장할 수 있습니다. 단계별 실행 중인 코드 블록에 대해 현재 선언된 모든 로컬 및 인스턴스 변수 목록을 보려면 마우스 오른쪽 버튼을 클릭하면 값에 대한 유형 을 설정하여 데이터 구조의 세부 정보를 볼 수 있습니다.


글로벌

Globals 창에는 현재 선언된 모든 전역 변수와 해당 값의 목록이 표시됩니다. 마우스 오른쪽 버튼을 클릭하면 값에 대한 유형 을 설정하여 데이터 구조의 세부 정보를 볼 수 있습니다.


시계

이 창에서는 추적하려는 특정 변수를 추가할 수 있습니다. 이것은 전역 또는 인스턴스 범위 변수, 내장 변수 또는 배열 또는 데이터 구조 일 수 있습니다. 단계별 실행 중인 인스턴스 코드에 감시된 변수가 있는 경우 여기에 표시됩니다. 즉, Locals 또는 Globals 창에서 값을 검색하지 않고도 값을 쉽게 추적할 수 있습니다. 마우스 오른쪽 버튼을 클릭하면 값에 대해 유형 을 설정하거나 삭제하거나 창에서 모든 시계를 지울 수 있습니다.


사례

이름에서 알 수 있듯이 이 창은 모든 인스턴스 변수, 인스턴스 이벤트에 대한 로컬 변수, 기본 제공 변수 및 소스 창에서 현재 단계적으로 진행되는 인스턴스와 관련된 구조체 를 표시합니다(인스턴스가 "기타" 키워드를 사용하여 참조되는 경우 " 아니면 그 with 문을 입력하면 이 창에 해당 인스턴스 속성이 대신 표시됩니다.) 마우스 오른쪽 버튼을 클릭하면 값에 대해 유형 을 설정하거나 삭제하거나 창에서 모든 시계를 지울 수 있습니다.


모든 인스턴스

현재 방에 있는 모든 인스턴스를 확인해야 하는 경우 이 창을 선택하면 나열됩니다. 또한 내장된 변수, 인스턴스 변수 또는 구조체 의 상태를 볼 수 있으며 마우스 오른쪽 버튼을 클릭할 수 있습니다. 값에 대한 유형 을 설정합니다.


선택한 인스턴스

이 창은 게임에서 인스턴스를 선택할 때 사용합니다. 게임을 일시 중지한 다음 게임 창에서 보이는 인스턴스를 클릭하면 됩니다. 그러면 마우스 커서 아래에 있는 인스턴스의 정보가 선택된 인스턴스 창에 표시되며, 여기서 인스턴스 창에서와 동일한 내장 변수, 인스턴스 변수 및 구조체를 볼 수 있습니다.


렌더링 상태

이 시계는 그리기 알파 또는 채우기 색상과 같이 렌더링 되는 현재 프레임의 상태에 대해 사용할 수 있는 모든 정보를 보여줍니다. 이것은 실시간으로 업데이트할 수 없으므로 업데이트하고 프레임 세부 정보를 얻으려면 게임을 일시 중지해야 합니다.


표면 / 질감

이 시계를 사용하면 게임에 있는 모든 표면 (응용 프로그램 표면 포함)에 무엇이 그려지고 있는지 정확히 볼 수 있을 뿐만 아니라 메모리에 있는 텍스처 페이지 도 볼 수 있습니다. 시계 하단의 메뉴에서 텍스처 또는 표면 중에서 선택한 다음 텍스처 또는 표면 ID 위로 마우스를 가져가 팝업 창에 표시되도록 할 수 있습니다.

업데이트하기 전에 디버거를 일시 중지 해야 합니다.


프로필

이 창은 테스트 중인 게임의 현재 시스템 프로필 을 볼 수 있는 곳이며 전체 이벤트에서 개별 함수 호출에 이르기까지 게임의 모든 것을 확인하여 성능을 확인하고 문제 영역이나 병목 현상을 찾아낼 수 있는 곳입니다. 프로파일링에 대한 자세한 내용은 다음 페이지를 참조하십시오.

  • 프로파일러


완충기

게임에서 버퍼를 초기화했다면 이 창에 버퍼 데이터 가 표시됩니다. 버퍼 ID 값을 설정하여 생성된 버퍼를 선택할 수 있습니다. 이 값은 게임에서 생성된 첫 번째 버퍼에 대해 0에서 시작하고 각 연속 버퍼에 대해 1씩 증가합니다.

마우스 오른쪽 버튼을 클릭하면 데이터 창에서 데이터 표시 버튼을 클릭하거나 데이터 표시 방법을 선택하고 정렬 및 유형 을 설정할 수 있으며 열 선택 상자에서 데이터 레이아웃을 설정할 수도 있습니다. 여기에는 기본 레이아웃 목록에서 선택할 수 있는 드롭다운 창이 있습니다.
