로컬 자산 패키지

도구 메뉴 아래에는 로컬 자산 패키지 를 만들고 가져오기 위한 두 가지 옵션이 있습니다. 로컬 자산 패키지는 다음을 사용하여 저장되는 리소스 그룹입니다. *.yymp 기존 프로젝트에 쉽게 설치할 수 있는 파일 형식입니다. 이것이 의미하는 바는 GameMaker M 마켓플레이스를 사용하지 않고도 스크립트 라이브러리, 스플래시 화면 인트로, 메뉴 시스템 등과 같이 일반적으로 사용되는 리소스에 대한 패키지 라이브러리를 생성할 수 있다는 것입니다.

자산 패키지 생성

자산 패키지를 생성하려면 먼저 프로젝트에 저장하려는 리소스를 생성한 다음 도구 메뉴에서 로컬 패키지 생성 옵션을 선택해야 합니다. 그러면 다음 창이 열립니다.

이 창에서 선택적 게시자 이름 과 필수 표시 이름 및 패키지 ID 를 제공할 수 있습니다. 패키지 ID는 패키지가 저장될 파일 이름을 생성하는 데 사용되며 자동으로 추가됩니다. *.yymp 생성 시 파일 확장자. 패키지의 버전 번호 를 설정하고 서명 여부를 선택할 수도 있습니다(기본적으로 꺼져 있음). 패키지에 서명하려면 서명 인증서 를 생성해야 하며( Marketplace 환경 설정 에서 수행할 수 있음) 서명된 자산이 생성 후 변조된 경우 가져올 수 없음을 의미합니다.

패키지 세부 정보를 입력했으면 오른쪽의 " 프로젝트 " 섹션에서 포함할 자산을 선택할 수 있습니다(왼쪽 마우스 버튼 사용 또는 다음의 조합 / 그리고 마우스 왼쪽 버튼과 함께). 선택한 리소스는 모두 추가 또는 모두 추가 버튼을 사용하여 " 자산 패키지 " 보기로 이동할 수 있으며 제거 및 모두 제거 버튼을 사용하여 다시 제거할 수 있습니다.

리소스 선택에 만족하고 필요한 세부 정보를 입력한 후 확인 버튼을 클릭하면 패키지를 저장할 위치를 선택할 수 있는 파일 탐색기가 열립니다. 선택하면 패키지가 생성되고 OS 파일 탐색기에서 패키지 위치를 여는 옵션과 함께 확인 창이 표시됩니다.

자산 패키지 가져오기

로컬 자산 패키지를 가져오는 방법에는 두 가지가 있습니다.

  • 첫 번째는 단순히 드래그하는 것입니다 *.yymp 탐색기 창에서 GameMaker IDE로 파일
  • 두 번째는 도구 메뉴 에서 " 로컬 패키지 가져오기 " 옵션을 선택하는 것입니다.

도구 메뉴를 사용할 때 탐색기 창이 열리고 패키지 위치를 찾아 가져올 수 있습니다. 어느 쪽이든 GameMaker 에서 다음 창이 열립니다.

왼쪽에는 패키지의 내용이 있고 오른쪽에는 가져올 부분이 있습니다. 패키지에서 하나 이상의 리소스를 선택할 수 있습니다(마우스 왼쪽 버튼을 사용하여 또는 다음의 조합 / 그리고 왼쪽 마우스 버튼과 함께) 그런 다음 추가 버튼을 사용하여 가져올 항목을 추가하거나 모두 추가 버튼을 사용하여 가져올 모든 항목을 추가합니다.

모든 것이 준비되면 가져오기 버튼을 클릭하여 선택한 리소스를 프로젝트에 추가할 수 있습니다. 모든 리소스를 새 프로젝트로 가져 오는 옵션도 있습니다. 그러면 새 프로젝트가 생성되고(저장할 위치를 지정하라는 메시지가 표시됨) 여기에 패키지 리소스를 추가합니다.

GameMaker.3 이전 버전을 사용하여 생성된 자산 패키지는 2.3 업데이트로 프로젝트 파일 형식이 변경되었으므로 변환이 필요합니다. 즉, 자산을 다른 프로젝트로 자주 가져오는 경우 전체 프로젝트를 가져오고 모든 것을 변환한 다음 새 패키지로 새 위치로 내보내는 것이 좋습니다.
