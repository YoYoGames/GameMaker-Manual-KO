소스 제어

소스 제어 관리 (SCM)는 저장소 및 로컬 소스를 통해 프로젝트의 하위 버전 또는 백업으로 작업하는 방법에 부여된 이름입니다. 기본적으로 SCM 솔루션은 작업할 때 프로젝트 버전을 유지 관리, 변경 및 비교하는 모든 측면을 제어하는 독립 소프트웨어 패키지입니다. 이것은 팀에서 작업하고 누가 무엇을 하는지 제어할 수 있어야 하고 데이터 손실이나 나중에 취소해야 할 수 있는 변경에 대해 걱정할 필요가 없는 사람들에게 특히 유용하지만 개인은 이 강력하면서도 유연한 이점을 누릴 수 있습니다. 시스템도.

GameMaker 는 SCM에 Git 을 사용합니다. SCM은 GameMaker 내에서 SCM을 사용하기 전에 별도로 설치 및 설정해야 합니다. Git에 대한 자세한 정보는 Git 문서 를 참조하십시오.

아래에서 이 플러그인을 설정하고 프로젝트에서 SCM 도구를 사용하는 방법에 대한 작은 자습서 링크를 찾을 수 있습니다.

중요한! GameMaker 에서 소스 제어를 사용하기 전에 소스 제어가 작동하는 방식에 대한 작업 지식이 있는 것이 좋습니다.

  • Git 플러그인 설정
  • 리포지토리 옵션
  • 표준 워크플로
  • 파일 되돌리기
  • 리포지토리 복제
  • 갈등
  • 외부 병합/차이 도구 설정

GameMaker 는 소스 제어용 메뉴 표시줄에 드롭다운 메뉴를 표시하며, 프로젝트에 대해 소스 제어가 활성화되면 다음 옵션으로 채워집니다(자세한 내용은 위의 Git 플러그인 설정 섹션 참조).

여기에서 사용 가능한 각 옵션에 대해 간략히 설명합니다(대부분은 위의 자습서 섹션에서 더 자세히 설명됨).

  • 프로젝트 리포지토리 생성: 이 옵션을 사용하면 프로젝트 파일과 동일한 디렉토리에 로컬 리포지토리를 생성할 수 있습니다.
  • 리포지토리 복제: 이 옵션을 사용하면 소스에서 새 대상으로 리포지토리를 복제할 수 있습니다.
  • 변경 사항 커밋: 이 옵션을 사용하면 프로젝트에서 변경된 파일을 준비한 다음 저장소에 커밋할 수 있습니다.
  • 변경 사항 푸시: 커밋 또는 병합을 수행한 후 이 옵션을 사용하여 변경 사항을 마스터 리포지토리로 푸시합니다.
  • 변경 사항 가져오기: 이 옵션을 사용하면 마스터 리포지토리에서 변경된 파일을 가져와서 로컬 리포지토리를 업데이트할 수 있습니다.
  • 기록 보기: 프로젝트의 모든 버전 기록을 보고 특정 파일 경로 또는 전체 개정판을 롤백하도록 선택할 수 있는 기록 창을 엽니다.
  • 충돌 표시: 충돌 창이 열리고 프로젝트에 존재할 수 있는 충돌 파일이 나열되어 GameMaker IDE 또는 특정 병합 도구를 사용하여 충돌을 처리할 수 있습니다.
