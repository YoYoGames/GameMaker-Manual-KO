디버거

디버거 는 게임을 확인하기 위한 강력한 도구이며 코드의 오류 및 버그를 추적하고 스크립트가 실행되어야 할 때 실행되고 변수와 배열에 예상한 값 등이 포함되어 있는지 확인하는 데 특히 유용합니다... 디버그 버튼을 사용하여 GameMaker IDE 에서 게임을 실행하여 디버거를 시작할 수 있습니다. IDE 상단에서 또는 빌드 메뉴 로 이동하여 " 디버그 "를 선택합니다.

참고: Windows 방화벽(또는 실행 중인 다른 방화벽 프로그램)에서 이 모듈에 대한 보안 예외를 생성하라는 메시지가 표시될 수 있습니다. 이렇게 하지 않으면 올바르게 작동하지 않습니다. 이는 디버거가 작동하는 방식 때문이며 모바일 플랫폼 디버깅에 필수적입니다.

디버그 모드에서 게임을 실행하면 IDE에 디버거에 사용할 수 있는 다양한 도구가 표시됩니다.

위에 표시된 이미지는 디버거의 기본 레이아웃을 보여주지만 IDE의 다른 측면과 마찬가지로 이를 필요에 맞게 사용자 정의하거나 필요하지 않은 창을 닫거나 필요한 문서 및 창의 크기를 변경할 수 있습니다. IDE 상단의 디버거 컨텍스트 메뉴에서 언제든지 디버거 레이아웃을 재설정하거나 닫힌 창을 다시 열 수 있습니다.

아래에서 디버거의 각 섹션이 무엇을 위한 것인지 설명합니다.

그래프 보기

그래프 보기는 시간 경과에 따른 프로젝트의 메모리 사용 방식과 프레임 속도를 그래픽으로 보여줍니다.

그래프는 현재 및 평균 메모리 사용량은 물론 시간 경과에 따른 현재 및 평균 FPS를 보여주며 다음을 사용하여 확대 또는 축소할 수 있습니다. / 마우스 휠과 함께, 그래프 요소 위에 마우스 커서를 놓으면 선택한 지점에서 해당 요소의 값이 표시됩니다.

그래프 보기는 또한 특정 디버그 이벤트를 그래프 길이를 따라 점으로 표시합니다. 이러한 이벤트는 특정 중요한 이벤트를 표시하기 위해 게임 실행의 특정 지점에서 GameMaker 에 의해 자동으로 생성되지만 함수를 사용하여 자신의 그래프 요소를 추가할 수도 있습니다.show_debug_message() 그리고debug_event(). 시스템 이벤트는 코드를 통해 트리거되는 이벤트와 다른 색상으로 플래그가 지정됩니다.

디버그에서 게임을 실행하고 이러한 이벤트 위에 마우스를 올려 놓으면 게임의 해당 지점에서 트리거된 모든 이벤트의 목록이 표시되며, " 새 창에서 열려면 클릭하십시오 " 텍스트:

리소스 보기

리소스 보기에는 코드 또는 GML 시각적 작업을 포함할 수 있는 프로젝트에서 사용 중인 모든 리소스 목록이 표시됩니다. 이 섹션은 자산 브라우저 와 매우 유사하게 작동하며 그 안의 항목을 확장하여 특정 개체에 대한 다양한 이벤트와 같은 추가 정보를 표시할 수 있습니다. 그런 다음 두 번 클릭할 수 있습니다. 값을 확인하고 중단점 을 추가 또는 제거할 수 있는 코드 보기 창에서 해당 코드를 열 수 있습니다.

호출 스택

이 창은 이벤트의 현재 호출 스택과 함께 단계별로 실행되는 코드의 행 번호를 표시합니다. 호출 스택은 기본적으로 GameMaker 러너에서 모든 게임 코드의 실행 순서를 보여주는 "후입선출" 데이터 구조입니다. 디버깅은 가상 머신(VM) 을 사용하여 수행되기 때문입니다. 게임을 위해. 정보가 여기에 표시되기 전에 디버거가 일시 중지 되어야 합니다.

처음 세 개의 버튼은 게임을 제어하는 데 사용되며 언제든지 시작, 일시 중지, 다시 시작 또는 중지 할 수 있습니다. 게임이 일시 중지되면(일시 중지 버튼을 클릭하거나 중단점 에서) 3 단계 버튼을 사용하여 코드를 단계별로 실행할 수 있습니다.

코드를 통해 "단계별 실행"에 대해 이야기할 때 우리는 단순히 현재 코드 줄을 실행하도록 게임에 지시한 다음 다음 코드로 "단계별"하여 전체 게임에서 한 줄의 코드를 실행할 수 있음을 의미합니다. 원하는 시간. 일반적으로 단계 인 버튼을 사용하여 이 작업을 수행하지만 코드를 단계별 로 실행하기 위한 버튼과 코드 에서 벗어나는 버튼도 있습니다. 한 단계씩 건너뛰는 것은 실행할 큰 스크립트가 있을 때 유용합니다. 이를 클릭하면 전체 스크립트 또는 코드 블록이 단일 단계로 실행되고 이후 다음 줄에서 중지되는 반면, 한 단계씩 나가면 스크립트에서 빠져나가기 때문입니다., 그러나 모두 한 번에) 범위가 변경되면 나중에 다음 단계에서 중지합니다.

실시간 업데이트 의 마지막 아이콘을 사용하면 디버그 창이 실시간으로 데이터를 업데이트할 수 있으므로 변수가 어떻게 변경되거나 인스턴스가 생성 및 소멸되는지 확인할 수 있습니다. 이는 게임 성능에 영향을 줄 수 있으며 모든 정보가 실시간으로 제공되는 것은 아닙니다. 예를 들어 Render State watch( Watches 섹션 참조)는 실시간으로 볼 수 없습니다.

코드 보기

코드 보기는 프로젝트가 실행될 때 프로젝트의 다양한 스크립트 또는 이벤트 코드를 보고 검토할 수 있는 곳입니다. 기본적으로 비어 있는 상태로 시작하지만 두 번 클릭할 수 있습니다. 리소스 보기 의 모든 자산에서 여기에서 엽니다. 여러 개의 다른 이벤트나 스크립트도 열 수 있으며 상단에 탭으로 코드 보기 작업 공간에 추가됩니다.

코드 보기에서 코드 창이 열려 있는 동안 여백(줄 번호가 있는 곳)에서 중단점을 추가하거나 제거합니다( 중단점 에 대한 정보는 이 페이지 아래를 참조하십시오).

마우스 오른쪽 버튼을 사용할 수도 있습니다. 컨텍스트 메뉴 를 열려면 중단점을 추가하거나 제거하고 선택한 코드를 클립보드에 복사할 수도 있습니다.

시계

이 섹션은 더 심도 있고 상세한 디버깅을 허용하기 위해 게임의 다양한 측면을 "관찰"하는 데 전념합니다. 이 내용은 다음 페이지에서 자세히 설명합니다.

  • 시계


게임이 실행되는 동안 기본적으로 변수 값 및 기타 세부 정보로 디버그 창을 업데이트하지 않으므로 먼저 일시 중지 버튼을 사용해야 합니다. 지역, 전역 및 기타 데이터를 사용할 수 있기 전에 디버거에서 게임을 일시 중지하거나 상단의 아이콘에서 실시간 업데이트를 활성화합니다.. 이 규칙의 유일한 예외는 프로필 보기입니다. 프로필 보기는 자체 켜기/끄기 버튼이 있으므로 실시간 스위치와 별도로 활성화될 때 실시간으로 업데이트됩니다. 다음 페이지에서 이 보기에 대한 전체 정보를 찾을 수 있습니다.

  • 프로파일러

게임 디버깅의 또 다른 중요한 부분은 Breakpoints 사용입니다. 중단점은 진행 상황을 볼 수 있도록 중지하려는 프로젝트 코드의 위치입니다. 디버거가 실행 중인 상태에서 마우스 왼쪽 버튼을 클릭하여 코드 또는 스크립트 블록에서 언제든지 중단점을 추가하거나 제거할 수 있습니다. 여백(줄 번호가 있는 곳)에서 또는 마우스 오른쪽 버튼을 사용할 수 있습니다. 중단점을 추가하거나 제거할 수 있는 컨텍스트 메뉴 를 열려면 다음을 수행하십시오.

중단점은 GameMaker IDE에서 설정할 수도 있습니다. 프로젝트를 작성할 때 다음을 사용하여 언제든지 추가하거나 제거할 수 있습니다. f9 (또는 마우스 오른쪽 버튼을 사용하여 개체 창의 작업(GML Visual 사용자의 경우) 또는 코드 편집기 창의 줄에서 " 중단점 전환 "을 선택합니다. 이제 디버그 모드에서 게임을 실행하면 정의한 지점에서 일시 중지됩니다.

IDE에서 설정한 중단점은 프로젝트와 함께 저장되며 다시 제거될 때까지 다양한 실행 동안 지속되지만 디버거에서 코드에 중단점을 추가하면 모듈이 닫히거나 새 테스트 실행이 시작된 후 손실됩니다.. 모든 중단점은 활성화 또는 비활성화 또는 제거할 수 있는 출력 창 에 나타납니다.
