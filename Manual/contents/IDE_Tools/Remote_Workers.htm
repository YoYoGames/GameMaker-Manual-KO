원격 작업자

게임을 컴파일할 때 일반적으로 GameMaker 가 설치된 컴퓨터인 "로컬" 작업자를 사용합니다. 그러나 일부 상황에서는 다른 컴퓨터를 사용하여 프로젝트를 빌드해야 할 수도 있습니다. 예를 들어 Mac에서 작업 중이고 Windows 프로젝트를 빌드하려는 경우 원격 작업자 를 구성하고 Windows PC에서 빌드를 수행하도록 하고 싶을 것입니다. 기본적으로 원격 작업자를 사용할 때 GameMaker 는 실행 중인 (로컬) 컴퓨터에서 필요한 모든 파일을 원격 작업자 컴퓨터로 보낸 다음 해당 컴퓨터를 사용하여 프로젝트의 최종 실행 파일을 빌드합니다(저장 원격 시스템, 또는 나중에 로컬 시스템으로 반환). 그러나 이 기능을 사용하려면 빌드 시스템에 특정 도구를 설치하고 일부 설정을 수행해야 합니다. 이에 대해 여기에서 설명합니다.

중요한! 로컬 Windows PC와 원격 Windows PC를 사용하거나 로컬 Mac PC와 원격 Windows PC 간에 원격 작업자를 설정할 수 있지만 현재는 로컬 Windows(또는 Mac) PC와 원격 Mac PC를 설정할 수 없습니다.: Mac 원격 작업자는 가질 수 없으며 Windows 작업자만 가질 수 있습니다.

초기 설정

원격 작업자를 설정하려면 먼저 사용할 PC에 원격 작업자 빌드 도구 애플리케이션 을 설치해야 합니다. 이 도구는 로컬 컴퓨터의 GameMaker 에서 빌드 메뉴 로 이동하고 " 원격 작업자 설치 프로그램 표시 " 옵션을 선택하여 찾을 수 있습니다.

또는 대상 관리자 에서 "눈" 아이콘을 클릭하고 원격 작업자 설치 프로그램을 다음과 같이 가져올 수 있습니다.

설치 프로그램은 "GameMakerStudio-Remote-Installer-X.X.X.XXX.exe " 그리고 원격 작업자로 사용하려는 PC에 복사해야 합니다(원격 작업자에 GameMaker 를 설치할 필요가 없으며 이 프로그램만 있으면 됩니다).

빌드 PC에 복사한 후 화면 프롬프트에 따라 설치하고 설치 프로그램이 완료되면 OS에서 방화벽을 통해 네트워크에 액세스하도록 요청할 수 있습니다. 액세스 권한을 부여하면 설치 프로그램이 웹 브라우저에서 페이지를 열고 다음 단계에 필요하므로 이 페이지에 표시된 IP 주소 와 포트 번호 를 기록해 두어야 합니다.

이제 GameMaker 2가 설치된 컴퓨터로 돌아가서 대상 관리자 버튼을 클릭해야 합니다. Target Manager 창을 열고 작업자 섹션으로 이동하여 작업자 추가 버튼을 클릭합니다.. 그러면 다음 창이 열립니다.

여기에서 원격 작업자에게 이름을 지정해야 합니다(대상 관리자의 작업자 목록 에 표시됨). 그런 다음 호스트 이름 섹션에서 이전에 표시되고 기록해 둔 포트와 함께 주소를 추가합니다. 작업자 기계.

모든 것이 올바르게 진행되면 이제 원격 작업자 시스템에 성공적으로 연결할 수 있다는 메시지가 표시됩니다. 원격 작업자를 설치하고 기본 IDE 컴퓨터에 연결했으면 프로젝트를 컴파일하기 전에 각 플랫폼에 대한 로컬 빌드 설정 을 구성해야 합니다(자세한 내용은 아래 섹션 참조).

원격 설정

설정 페이지에는 원격 작업자 애플리케이션에 대한 다양한 경로와 빌드 설정이 표시됩니다. 설정 상단에는 원격 작업자 응용 프로그램에서 사용할 다른 경로가 있으며 다른 작업을 수행하기 전에 이러한 경로를 수정하고 필요한 경우 대체 경로로 설정해야 합니다(일반적으로 기본 경로는 괜찮음). 다음 방법 중 하나를 사용하여 원격 작업자 설정 페이지에 액세스할 수 있습니다.

  • 원격 컴퓨터의 브라우저에서 열리는 원격 작업자 페이지 상단의 "설정" 링크를 클릭합니다.
  • 원격 작업자 트레이 아이콘의 RMB를 사용하여 원격 컴퓨터의 브라우저에서 설정을 엽니다.
  • 로컬 컴퓨터에서 대상 관리자를 열고 "톱니" 아이콘을 클릭하여 로컬 컴퓨터의 창에서 작업자 설정을 엽니다.GameMaker IDE"/>

그러면 프로젝트 빌드를 위한 경로를 설정할 수 있는 다양한 옵션을 볼 수 있는(설정할 수 있는) 설정 페이지가 표시됩니다.


로컬 빌드 설정

원격 작업자 애플리케이션이 사용할 경로를 확인한 후 필요한 경우 다른 로컬 빌드 설정 을 설정할 수 있습니다. 플랫폼마다 빌드 PC에 다른 빌드 도구를 설치해야 하며 원격 작업자가 설정 페이지를 통해 게임을 빌드하는 데 사용할 수 있어야 합니다. 첫 번째 설정은 일반 설정입니다.

여기에서 도움말 서버 의 포트와 디버거 가 사용할 포트 를 설정할 수 있습니다. 디버그 모듈을 사용할 때 연결 문제가 있는 경우 변경할 수 있지만 일반적으로 기본값을 그대로 두어야 합니다.

이제 원격 작업자가 빌드할 수 있는 플랫폼에 대해 다양한 옵션을 설정할 수 있습니다.

Windows 대상 플랫폼에 액세스할 수 있는 경우 가상 머신(VM) 옵션을 사용하여 Windows PC에서 즉시 원격 작업자를 사용하여 빌드할 수 있어야 합니다. 그러나 YoYo 컴파일러(YYC) 를 사용하려면 빌드 PC에서 Visual Studio 경로도 설정해야 합니다. 프로젝트를 컴파일하기 위해 Windows PC를 설정하는 방법에 대한 자세한 내용은 다음 헬프 데스크 문서를 참조하십시오.

컴파일

원격 작업자를 설정하고 로컬 빌드 설정을 완료하면(위 섹션 참조) 게임 컴파일에 사용할 수 있습니다. 이를 위해 Target Manager 목록에서 Remote Worker를 선택하고 테스트/컴파일할 플랫폼 및 기타 옵션(예: 컴파일 유형 또는 구성)을 선택한 다음 Play를 누르기만 하면 됩니다. 또는 실행 파일 생성.

참고: 대상 관리자에서 사용 가능한 플랫폼은 원격 작업자 컴퓨터의 기능에 따라 변경되며 모든 대상을 사용할 수 있는 것은 아닙니다. 또한 원격 작업자에 대한 "테스트" 대상이 없다는 점에 유의하십시오.

단순히 프로젝트를 재생하는 경우 원격 작업자 시스템이 이를 컴파일하고 실행하지만 실행 파일을 생성하는 경우 원격 작업자가 프로젝트를 컴파일한 다음 반환합니다. *.zip 파일을 로컬 시스템에 저장하여 저장합니다.

디버그 버튼도 청소 버튼도 원격 작업자를 사용할 때 사용할 수 있습니다. 또한 현재 한 번에 하나의 원격 빌드만 수행할 수 있습니다.


원격 시스템에 원격 작업자를 설치하고 실행하면 작업 표시줄에 아이콘으로 표시되며 마우스 오른쪽 버튼을 클릭하여 다음 옵션을 표시할 수 있습니다.

  • 열기: 기본 브라우저에서 원격 작업자 애플리케이션을 엽니다.
  • 로그 표시: 브라우저에서 원격 작업자에 대한 로그 파일을 엽니다.
  • 네트워크 설정: 여기에서 원격 작업자 컴퓨터에 빌드를 보낼 수 있는 컴퓨터를 필터링할 수 있습니다. 각 필터는 CIDR 스타일 필터 또는 허용할 시스템의 호스트 이름일 수 있습니다. 여러 대의 컴퓨터를 한 명의 원격 작업자에게 연결하려는 경우 또는 연결 유형(예: 유선 및 Wi-Fi)을 혼합하는 경우 여기에서 IP 허용 범위를 편집해야 할 수 있습니다.
  • 설정 편집: 원격 작업자 설정 메뉴를 엽니다(위의 설정 섹션에서 설명).
  • 종료: 원격 작업자 앱을 종료합니다. 즉, 원격 작업자 앱을 다시 시작할 때까지 로컬 컴퓨터에서 앱과 더 이상 연결(또는 컴파일)할 수 없습니다.
