마이크로 웹 서버

이 작은 도구의 주요 기능은 GameMaker 용으로 사용 가능한 일부 대상 플랫폼(예: HTML5 대상)에 연결하고 파일을 제공하는 것입니다. 또한 오프라인에서 사용할 때 GameMaker 설명서에 대한 파일을 제공합니다.

Micro Web Server가 필요한 대상에서 게임을 실행하면 선택한 장치에 게임이 제공될 때 새 창이 열리는 것을 볼 수 있습니다.

참고 Micro Web Server가 필요한 대상에서 게임을 처음 실행할 때 방화벽 경고를 받을 수 있습니다. 이 경우 웹 서버가 방화벽을 통해 통신하도록 허용해야 합니다.

Micro Web Server가 실행 중일 때 모든 로컬 장치에서 게임에 액세스할 수 있습니다(포트 포워딩이 올바르게 설정된 경우 원격으로도 가능). 이 창에 표시된 포트를 통해 컴퓨터의 로컬 IP 주소에 액세스하기만 하면 휴대폰이나 다른 컴퓨터와 같은 다른 장치에서 게임을 할 수 있습니다.

예를 들어, 컴퓨터의 로컬 IP 주소가 192.168.0.110 이고(찾는 방법을 알아보려면 이 주소를 읽으 십시오) Micro Web Server 창에 표시된 포트가 51264라고 가정해 보겠습니다. 컴퓨터에서 게임을 실행한 후 모바일 브라우저에서 192.168.0.110:51264 에 연결하여 게임을 하게 됩니다.

웹 서버는 제공된 파일과 관련된 일정량의 디버그 정보를 표시하며 하단에 이를 지우는 버튼이 있습니다. "지우기" 버튼 옆에는 프로젝트의 루트 JS 파일 경로가 있고 그 아래에는 네트워크 속도 옵션이 있습니다. 기본적으로 "제한 없음"으로 설정되어 있지만 클릭하면 다양한 네트워크 옵션 목록이 열립니다.

이 설정을 변경하면 패키지 크기 옵션도 오른쪽으로 변경됩니다. 네트워크 속도가 "사용자 지정"으로 설정되는 경우 수동으로 편집할 수도 있습니다. 최대값 이외의 값으로 설정하면 네트워크 전송 속도가 효과적으로 조절되고 낮은 대역폭 환경에서 게임을 테스트할 수 있습니다.

또한 사용 중인 포트 번호와 함께 허용된 IP 목록을 볼 수 있지만 마이크로 웹 서버에서 편집할 수 없습니다. 대신 지정된 장치 대상에 대한 장치 관리자 설정에서 설정하거나 "기본" 장치를 사용할 때HTML5 기본 설정 에서 설정해야 합니다.
