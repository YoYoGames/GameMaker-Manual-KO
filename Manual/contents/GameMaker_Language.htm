게임메이커 언어

GameMaker Language를 사용하여 두 가지 다른 방법으로 게임을 프로그래밍할 수 있습니다. 블록을 배치하여 코드를 시각적으로 "작성"하거나 수동으로 코드를 작성하는 것입니다.

첫 번째 옵션인 GML Visual 은 자신의 게임을 빠르게 만들기 시작하려는 초보자와 애호가에게 가장 적합합니다.

두 번째 옵션인 GML 코드 를 사용하면 코드를 수동으로 작성하여 GameMaker Language의 모든 기능을 활용할 수 있습니다.

GML Visual 은 시작하기가 더 쉽지만 GML 코드 는 여전히 강력한 기능( 함수, 메소드, 구조체 및 생성자 등)을 갖춘 사용하기 쉬운 프로그래밍 언어입니다.

비주얼/코드 스크립트 생성

새 개체 이벤트, 새 스크립트 자산 또는 스크립트를 만드는 결과를 생성할 때 GML 코드 와 GML Visual 중에서 선택하라는 메시지가 표시됩니다.

GML 코드 와 GML Visual 중에서 선택하는 것 외에도 이 창에는 두 개의 확인란이 있습니다.

  • 이 프로젝트에 대해 다시 묻지 않음: 이 옵션이 활성화되면 "기본 스크립팅 언어" 게임 옵션 이 새 선택으로 설정되므로 현재 프로젝트에 대해 다시 선택하라는 메시지가 표시되지 않습니다.
  • 다시 묻지 않음: 이 옵션이 활성화되면 "기본 스크립팅 언어" 기본 설정 을 새 선택으로 설정하므로 모든 프로젝트에 대해 다시 선택하라는 메시지가 표시되지 않습니다.
