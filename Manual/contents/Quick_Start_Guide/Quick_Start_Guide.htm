빠른 시작 가이드

빠른 시작 가이드 에 오신 것을 환영합니다! 설명서의 이 섹션은 GameMaker 의 가장 중요한 측면에 익숙해지도록 설계되었으며 최대한 빨리 게임을 만드는 데 필요한 기본 사항을 보여줍니다. 이 가이드는 필수 IDE 기능을 다루고 프로그래밍에 대한 간략한 소개도 제공하므로 GML 코드 를 사용하든 GML Visual 을 사용하든 제품을 최대한 빠르게 사용할 수 있습니다.

GameMaker 워크플로를 설명하고 바로 만들 수 있도록 GameMaker 가 무엇인지, 프로그램이 무엇인지에 대한 기초부터 일반 자산 생성, 다음과 같은 기본적인 코딩 방법까지 이 가이드를 준비했습니다. 움직임과 점수.

GameMaker 로 게임을 만드는 것은 다음과 같은 일반적인 단계를 따르면 개념적으로 상당히 간단하고 간단한 작업입니다.

  • 이미지 자산( sprites, 타일 세트, 글꼴 )을 만들고 자산 브라우저에 추가 합니다.
  • 게임에서 사용하려는 항목을 나타내는 개체 자산을 만들고 필요한 이미지 자산을 할당합니다.
  • 특정 이벤트 에 대한 응답으로 작업을 수행하도록 이러한 개체를 프로그래밍합니다.
  • 이러한 개체의 인스턴스 를 프로그램을 수행할 방 에 추가합니다.

다음 페이지에서 위에 나열된 각 항목의 세부 사항을 살펴보겠지만 이것이 게임을 만드는 데 필요한 일반적인 작업 흐름입니다. 여기서 주목해야 할 중요한 점은 게임의 기본 "구성 요소"가 항상 스프라이트, 개체 및 방이라는 것입니다. 이러한 항목에 추가 기능이나 동작을 추가하는 데 사용할 수 있는 많은 보조 자산이 있습니다(예: 셰이더 또는 타임라인 ).

이제 이러한 각 항목에 대해 더 자세히 설명하기 전에 GameMaker 가 위의 구성 요소를 게임으로 바꾸는 방법에 대해 간략하게 설명하겠습니다. 기본적으로 모든 것은 초당 프레임 수로 측정되는 속도를 갖는 게임 루프 라고 하는 것에서 발생하며 하나의 "프레임"은 GameMaker에서 단계 라고 합니다. 기본 초당 프레임(또는 FPS )은 60입니다. 즉, GameMaker 를 통과하는 1초마다 게임 루프가 60번 실행되고 각 단계(루프/프레임)에는 여러 이벤트가 발생하고 게임이 진행됩니다.

이것을 이해하는 데 도움이 되도록 게임룸에서 움직이고 싶은 캐릭터가 있다고 가정해 보겠습니다. 해당 캐릭터에 대한 스프라이트 를 만든 다음 개체 를 만들어 스프라이트에 할당하고 개체에 단계 이벤트 라는 이벤트 를 추가합니다. 이 경우 오른쪽으로 2픽셀 이동하도록 지시합니다. 그런 다음 해당 개체의 인스턴스 를 게임 룸에 배치하고 게임을 실행할 때 인스턴스가 매 단계마다 오른쪽으로 2픽셀씩 움직이기 시작하므로(단계 이벤트는 게임 단계마다 한 번 실행됨) 1초 안에 인스턴스가 오른쪽으로 120픽셀 이동했습니다.

위의 예에서 우리는 객체에게 오른쪽으로 이동하도록 지시하는 것에 대해 이야기했지만 어떻게 합니까? 글쎄, 그것은 우리가 GML Visual 또는 GML Code 를 사용하여 작성하는 프로그램 을 사용하여 수행됩니다. 어떤 것을 사용할지는 당신과 당신이 원하는 작업 방식에 달려 있지만, 그것들을 사용함으로써 당신은 당신의 게임을 프로그래밍 하는 것입니다. 다음 섹션에서는 이에 대해 더 자세히 다룹니다.
