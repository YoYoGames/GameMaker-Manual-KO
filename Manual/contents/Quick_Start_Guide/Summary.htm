요약

이 퀵 스타트 가이드를 마치며 GameMaker 를 사용하여 자신만의 프로젝트를 만들기에 충분한 지식과 자신감을 얻으셨기를 바랍니다! 이 가이드의 모든 페이지를 살펴보았다면 이제 다음 사항을 알고 있을 것입니다.

  • 이미지 편집기 에서 스프라이트 및 타일 세트 를 만들고 편집하는 방법
  • 개체 및 방 을 만드는 방법
  • 객체 와 인스턴스 의 차이점
  • GameMaker 가 사물을 그리는 방법과 그리기 이벤트 를 사용하는 방법
  • Draw 이벤트와 Draw GUI 이벤트의 차이점
  • 방 에서 인스턴스를 이동하는 방법
  • 변수, 함수 및 표현식 과 같은 기본 프로그래밍 개념

당신은 짧은 시간에 먼 길을 왔지만 그것이 여정의 끝이 아닙니다. 이 매뉴얼에는 IDE의 모든 측면과 GML Visual 또는 GML을 사용하여 프로그래밍하는 방법을 다루는 두 개의 광범위한 참조 섹션이 있으며, 확실하지 않거나 새로운 것을 시도하고 싶을 때마다 해당 섹션으로 다시 돌아올 것을 적극 권장합니다.

  • IDE 참조
  • GML 시각적 참조
  • GML 코드 참조
  • 추가 정보

시작 페이지 에서 Space Rocks 와 같은 작고 재미있는 게임을 만들기 위해 여기서 배운 핵심 개념을 사용하는 GameMaker 자습서를 얻을 수도 있습니다.

다양한 대상 플랫폼 설정과 같은 추가 기술 지원을 받으려면 GameMaker 헬프데스크 를 참조하세요. GameMaker는 또한 가장 기본적인 개념에서 가장 고급 기술에 이르기까지 GameMaker 사용의 다양한 측면을 다루는 일련의 기술 블로그 를 운영합니다. 그래서 모두를 위한 무언가가 있습니다. 특히 다음 기사를 살펴보십시오.

  • 전문가에게 GameMaker로 코딩하는 방법을 가르쳐준 6가지 리소스

또한 GameMaker 에는 커뮤니티 가 운영하는 GMC ( 게임 메이커 커뮤니티)라는 자체 포럼이 있으며 프로그래밍 이나 기술 지원을 위해 시간을 기꺼이 내줄 의향이 있는 유용한 회원들로 가득합니다. 당신이 가질 수있는 문제.

마지막으로, 인터넷에는 엄청난 양의 GameMaker 리소스가 있으므로 막히거나 튜토리얼을 찾을 때 항상 브라우저에서 정보를 검색해 보십시오. 또한 Reddit 또는 Discord 와 같은 사이트에는 등록하고 등록할 수 있는 활성 커뮤니티가 많이 있습니다. 참가하다.

무엇을 하든지 GameMaker 를 사용하여 즐거운 시간을 보내시기 바랍니다. (언제!) 게임을 만들고 게시할 경우 GameMaker 쇼케이스 에서 이에 대해 알려주는 것을 잊지 마십시오!
