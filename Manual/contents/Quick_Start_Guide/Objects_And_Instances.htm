객체 및 인스턴스

지금까지 본 자산으로 게임 자산 브라우저에 멋진 이미지와 사운드를 추가할 수 있지만 아직 게임 자체에서는 아무 것도 하지 않습니다! 이를 위해서는 GameMaker 에서 가장 중요한 자산인 Object 자산이 필요합니다. 개체는 게임의 측면을 제어하고 특정 작업을 수행하는 데 사용하는 특별한 자산입니다. 대부분의 경우 게임 룸에서 볼 수 있도록 연결된 스프라이트가 있지만 때로는 통계 추적 또는 컷씬의 타이밍 설정과 같은 작업을 수행하기 위해 "장면 뒤에서" 컨트롤러 개체로 사용됩니다., etc... 객체는 행동을 줄 수 있고 특정 이벤트에 반응할 수도 있고 서로 반응할 수도 있습니다. 게임에서 볼 수 있는 대부분의 사물은 객체와 사용자 및 서로 간의 상호 작용을 기반으로 합니다.

이제 Instances 없이 Objects에 대해 이야기할 수 없습니다. GameMaker 로 만든 게임에서는 게임에서 보는 캐릭터, 몬스터, 공, 벽 등... 실제로 모두 인스턴스 로 간주됩니다. 인스턴스는 기본적으로 자산 브라우저에서 찾은 다음 방에 배치되는 개체의 "사본"입니다(객체를 인스턴스의 템플릿 으로 생각하십시오). 따라서 인스턴스에 영향을 미치거나 변경하는 것에 대해 이야기할 때 방에 있는 개체의 특정 복사본 하나는 영향을 받고 나머지는 모두 영향을 받지 않는다는 것을 의미합니다. 그러나 개체에 영향을 미치거나 변경하는 것에 대해 말할 때는 우리가 하는 모든 것을 의미합니다. 해당 시점부터 생성된 모든 인스턴스에도 반영됩니다. 따라서 개체는 인스턴스의 템플릿이고 인스턴스는 게임을 만들기 위해 방에 배치하는 것 입니다.

새 개체 자산을 생성하려면 다른 모든 자산과 동일하며 마우스 오른쪽 버튼을 클릭하기만 하면 됩니다. 개체 자산 폴더에서 만들기 를 선택합니다(또는 자산 브라우저 에서 아무 곳이나 마우스 오른쪽 버튼으로 클릭하고 개체 만들기 선택).

이것은 개체 편집기 창이며 현재 개체 속성을 보여줍니다. 가장 먼저 "와 같은 접두사를 사용하여 개체의 이름을 지정해야 합니다. obj_ " 또는 다른 식별자. 일반적으로 객체가 무엇이든 그리고 다른 객체의 인스턴스와 충돌을 등록하려면 해당 객체에 할당된 스프라이트가 필요하며, 이는 다음을 사용하여 추가됩니다. 단추. 이렇게 하면 사용할 스프라이트를 선택할 수 있는 자산 탐색기 가 열립니다(자산 브라우저에서 스프라이트를 끌어 개체 편집기에 놓아 할당할 수도 있습니다).

아직 적절한 스프라이트가 없으면 다음을 클릭할 수 있습니다. 새 스프라이트 버튼에서 새 스프라이트 자산을 생성하고 원하는 대로 편집하려면 표준 스프라이트 생성 옵션을 통해 자산을 생성한 것처럼 이제 생성을 시작할 때 열려 있던 객체에 자동으로 할당됩니다. 스프라이트를 선택하면 다음을 클릭할 수도 있습니다. 편집 스프라이트 에서 또는 이미지 편집 버튼을 사용하여 스프라이트 자체 또는 스프라이트 자산의 이미지를 편집할 수 있습니다. 이는 애셋 브라우저에서 먼저 스프라이트를 찾은 다음 편집하고 싶다고 표시하는 것보다 빠를 수 있습니다.

스프라이트 를 추가하거나 생성한 후에는 표시 여부와 같은 몇 가지 다른 객체 속성을 변경할 수 있습니다. 간단한 게임의 경우 여기에서 아무 것도 변경할 필요가 없으며 이제 개체에 이벤트 를 추가할 수 있습니다.

이벤트는 게임 루프에서 사용자가 프로그래밍한 내용을 기반으로 일이 발생하는 신중한 순간입니다. GameMaker 는 이러한 이벤트의 주기로 작동합니다. 방이 시작되는 순간부터 완료되는 순간까지 모든 단계에서 일련의 이벤트가 실행되는 게임 루프가 실행되고 있으며 GML 코드 또는 GML Visual 작업을 배치하도록 선택할 수 있습니다. 이러한 이벤트에 응답하는 인스턴스( 단계 는 게임 속도 설정에 의해 제어되는 게임 시간의 한 순간이며 "프레임"이라고도 함).

클릭한 이벤트를 추가하려면 이벤트 추가 버튼에서:

위에서 모든 다른 이벤트 카테고리 를 볼 수 있으며 그 중 일부는 다른 하위 카테고리를 포함합니다. 여기에서 모든 이벤트를 다루지는 않겠지만 가장 중요한 5가지 이벤트를 간략하게 다룰 것입니다.

이벤트 생성 이 이벤트는 객체의 인스턴스가 처음 생성될 때 발생하며, 방에 입장할 때 방 편집기를 통해 방에 배치된 인스턴스 내에서 가장 먼저 일어나는 일입니다. 이것은 이 이벤트가 변수를 초기화하고, 타임라인을 시작하고, 경로를 설정하는 등의 이상적인 장소이며 일반적으로 한 번만 또는 인스턴스가 방에 처음 나타날 때만 수행되어야 하는 다른 모든 작업을 수행하는 이상적인 장소임을 의미합니다.
단계 이벤트 GameMaker 는 게임 속도를 사용하여 초당 이러한 단계 수를 정의하는 단계로 게임 시간을 나눕니다. 단계는 기본적으로 게임이 실행되는 동안 필요에 따라 모든 이벤트를 확인하고 트리거하면서 지속적으로 실행되는 루프이므로 상상할 수 있듯이 단계 이벤트는 인스턴스가 존재하는 동안 게임의 모든 단계를 확인하는 이벤트입니다.. 단계 이벤트 범주에는 인스턴스 내에서 타이밍을 "미세 조정"하는 세 가지 하위 범주가 있지만 일반적으로 기본 단계 이벤트를 가장 많이 사용합니다.
충돌 이벤트 분명히 게임을 만들 때 객체의 두 개(또는 그 이상) 인스턴스가 충돌한 때를 아는 것이 매우 중요하며 이를 위해 충돌 이벤트가 있습니다. 이것은 개체에 배치한 다음 충돌을 확인해야 하는 다른 개체를 지정하는 이벤트입니다. 이벤트를 추가하고 확인할 개체를 선택하면 GML 코드 또는 GML Visual을 추가하여 충돌을 해결할 수 있습니다.
추첨 이벤트 그리기 이벤트 범주에는 여러 가지 다른 그리기 이벤트가 있지만 가장 많이 사용하는 것은 상단에 있는 단순히 "그리기"라고 하는 이벤트입니다. 이 일반 그리기 이벤트는 모든 게임 프레임에서 단계 이벤트와 같이 호출되지만 두 가지 방식으로 작동합니다. 첫 번째는 이 이벤트를 개체에 추가하지 않는 경우입니다... 지정된 그리기 이벤트가 없으면 GameMaker 는 기본적으로 개체의 다른 곳에 적용된 변형을 사용하여 개체에 할당된 스프라이트를 그리는 것입니다( 크기 또는 알파 블렌딩을 변경하는 것과 같습니다. 따라서 이벤트 목록에 그리기 이벤트가 나열되지 않은 경우에도 여전히 실행 중이며 인스턴스가 방에 배치될 때 여전히 화면에 그려집니다. 작동하는 두 번째 방법은 이벤트 자체를 추가하고 GML 또는 GML Visual을 포함할 때입니다. 이벤트에 무언가를 추가하면 GameMaker 에 " 이 개체의 인스턴스에 대해 그리는 것을 제어하고 싶습니다 "라고 알려주고 기본 그리기를 완전히 무시합니다. 즉, 스프라이트가 할당된 개체를 가질 수 있으며 텍스트를 그리도록 그리기 이벤트를 설정하면 GameMaker 에 그리도록 지시하지 않았으므로 스프라이트가 그려지지 않습니다.
이벤트 파괴 이 이벤트는 인스턴스가 소멸될 때 실행되는 가장 마지막 이벤트입니다. 객체에 동작을 추가할 때 종종 간과되지만 매우 유용할 수 있습니다. 예를 들어 적이 죽었을 때 폭발 또는 입자 효과를 생성하거나 방의 다른 부분에서 객체의 새 인스턴스를 다시 생성하는 경우, 또는 점수에 점수를 추가하는 경우에도 마찬가지입니다.

다양한 이벤트에 대한 자세한 내용은 여기 를 참조하십시오.
