러너 오류

코드 편집기에서 구문 검사를 수행한 다음 컴파일러에서 코드에서 컴파일러 오류 를 검사하도록 한 후에도 여전히 문제가 발생할 수 있습니다. 대부분의 경우 다음과 같은 가상 머신(VM) 실행기 오류( 런타임 예외 라고도 함)가 발생합니다.

러너 오류는 일반적으로 컴파일 오류보다 더 심각합니다. 이는 코드 편집기나 컴파일러가 감지할 수 없는 심각한 문제가 코드에 있음을 의미하므로 그러한 모든 오류에 주의를 기울여야 합니다. 오류가 발생하면 팝업의 복사 버튼을 사용하여 오류를 클립보드에 복사한 다음 나중에 참조할 수 있도록 텍스트 파일(또는 어디에든)에 붙여넣을 수 있습니다.

이 오류의 구조는 다음과 같습니다.


  • 이는 오류가 발생한 이벤트와 개체(해당되는 경우)를 보여줍니다.

  • 이것은 실제 오류를 보여줍니다. 따라서 오류의 범위 와 오류 유형을 보여줍니다. 괄호 안의 첫 번째 값은 코드를 실행 중인 인스턴스의 인스턴스 ID이거나 아래 표에 나열된 특수 값 중 하나일 수 있으며 두 번째 값은 GameMaker 가 버그를 식별하는 데 사용하는 내부 값입니다. 무시된다. 위 이미지의 경우 변수가 _image 객체에 설정되지 않았습니다(즉, 액세스하기 전에 존재하지 않음). obj_EA_DEMO_Control.

  • 이것은 오류가 발생한 줄 번호와 위치를 보여주는 코드 조각을 보여줍니다. 게임이 컴파일되는 방식으로 인해 이것이 항상 100% 정확하지는 않지만 문제가 있는 곳을 정확히 찾는 데 도움이 되기에는 충분해야 합니다.

위에서 언급했듯이 특정 오류 메시지는 인스턴스 ID 값이 아닌 음수 값으로 범위를 식별합니다. 이러한 값을 사용하여 오류의 정확한 특성과 가능한 다음 값으로 오류가 참조하는 내용을 찾아낼 수 있습니다.

접두사 범위
-1 본인
-2 다른
-삼 모두
-4 아무도
-5 글로벌
-6 명시되지 않은
-7 현지

VM 실행기에서 발생할 수 있는 오류는 다음과 같습니다.

오류 메시지 작업 설명
DoSet 잘못된 비교 유형 데이터 유형 이는 러너가 실수와 문자열과 같이 호환되지 않는 두 데이터 유형을 비교하려고 시도했음을 나타냅니다.
DoConv 실행 오류 데이터 유형 이것은 한 데이터 유형을 다른 데이터 유형으로 변환하는 데 오류가 있음을 나타냅니다.
DoAdd 실행 오류 수학 덧셈을 사용할 때 문제가 발생했습니다(+ ) 표현.
DoMul 실행 오류 수학 곱셈을 사용할 때 문제가 발생했습니다(* ) 표현.
DoSub 실행 오류 수학 빼기를 사용할 때 문제가 발생했습니다(- ) 표현.
DoSub 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 수학 잘못된 유형의 변수를 빼려고 합니다(예: 실수에서 문자열 빼기).
DoDiv 실행 오류 수학 나눗셈을 사용할 때 문제가 발생했습니다(/ 또는 div ) 표현.
DoDiv 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 수학 잘못된 유형의 변수를 나누려고 합니다(예: 문자열을 실수로 나누기).
DoDiv 0으로 나누기 수학 0으로 나누려고 합니다(이는 정수 나누기를 사용할 때만 발생합니다. 0이 아닌 실수를 0으로 나누면 무한대가 답이 되고 0을 0으로 나누면 답이 NaN이 됩니다). (is_infinity ) 그리고 (is_nan )
DoMod 실행 오류 수학 모듈로(mod ) 표현.
DoMod 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 수학 모듈로(mod ) 잘못된 유형의 변수(예: mod 실수에 의한 문자열).
DoAnd 실행 오류 비트 단위 비트 단위 "and"(& ) 표현.
DoAnd 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 비트 단위 비트 단위 "and"(& ) 잘못된 유형의 변수(예: 실수로 문자열 "및" 시도).
DoOr 실행 오류 비트 단위 비트 단위 "또는"(| ) 표현.
DoOr 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 비트 단위 "또는"(| ) 잘못된 유형의 변수(예: 문자열을 실수로 "또는" 시도).
DoXor 실행 오류 비트 단위 비트 단위 "xor"(^ ) 표현.
DoXor 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 비트 단위 "xor"(^ ) 잘못된 유형의 변수(예: 문자열을 실수로 "xor"하려고 시도).
DoShl 실행 오류 비트 단위 왼쪽으로 비트 시프팅할 때 문제가 발생했습니다(<< ) 가치.
DoShl 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 비트 단위 비트시프트(<< ) 잘못된 유형의 변수(예: 문자열을 비트 시프트하려는 경우).
DoShr 실행 오류 비트 단위 오른쪽으로 비트 시프팅할 때 문제가 발생했습니다(>> ) 가치.
DoShr 실행 엔진 - 문자열 유형에서 작동할 수 없습니다. 비트 단위 비트 시프트(>> ) 잘못된 유형의 변수(예: 문자열을 비트 시프트하려는 경우).
DoNeg 실행 오류 부정 이 유형이 그러한 작업을 허용하지 않을 때 변수 유형을 음수로 바꾸려고 합니다.
DoNot 실행 오류 부정 이 유형이 그러한 작업을 허용하지 않을 때 변수 유형을 "금지"하려고 합니다.
Push 실행 오류 - 변수 인덱스가 범위를 벗어남(var) 스택 액세스 중인 변수가 러너에 대해 설정된 범위를 벗어났습니다.
Push 변수 가져오기(var) 스택 주어진 변수가 정의되지 않았거나 알 수 없습니다.
Pop 변수 인덱스가 범위를 벗어남(var) 스택 액세스 중인 변수가 러너에 대해 설정된 범위를 벗어났습니다.
Pop 변수 가져오기(var) 스택 주어진 변수가 정의되지 않았거나 알 수 없습니다.
With with 문에서 전역을 사용할 수 없습니다. 와 함께 "를 사용하려고 시도했습니다.global " 내의 변수로 "with " 문, 즉:
with (global)
   {
   //do something;
   }
With with 문에서 로컬을 사용할 수 없습니다. 와 함께 "를 사용하려고 시도했습니다.local " 내의 변수로 "with " 문, 즉:
with (local)
   {
   //do something;
   }
DoCall 실행 엔진 유형 오류 엔진 이것은 가상 머신 내에서 정의되지 않은 오류입니다. 이런 일이 발생하면 버그 보고서를 제출해야 합니다(이 작업을 수행하는 방법에 대한 자세한 내용은 도움말 메뉴 참조).
Stack Overflow - 엔진 스택 오버플로는 호출 스택에서 너무 많은 메모리가 사용되고 게임이 호출 스택에서 사용 가능한 것보다 더 많은 공간을 사용하려고 할 때 발생합니다(즉, 기본적으로 호출 스택의 경계를 넘어 메모리에 액세스하려고 시도할 때). 버퍼 오버플로), 스택이 오버플로되어 프로그램 충돌이 발생한다고 합니다. 컴퓨터와 GameMaker 를 다시 시작하고 오류가 지속되면 지원팀에 연락하거나 버그를 신고하세요(위에 설명된 대로).
